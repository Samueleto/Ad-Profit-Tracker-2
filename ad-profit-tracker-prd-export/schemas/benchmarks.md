# benchmarks

## Description

PURPOSE: Stores per-user custom benchmark target configuration and cached performance scores for the Performance Benchmarking feature

FIELDS:
id: string
├── Purpose: Auto-generated Firestore document ID
│   ├── Constraints: required
│   └── Mutable: no

userId: string
├── Purpose: UID of the authenticated user who owns this benchmark config. Used as the primary partition key and enforced by Firestore Security Rules.
│   ├── Constraints: required
│   └── Mutable: no

metricTargets: map
├── Purpose: Map of metric keys to custom target values. Keys: roi, ctr, cpm, revenue, cost, impressions, clicks. Each value is an object: { customTarget: number|null, useDefault: boolean, updatedAt: timestamp }. Null customTarget means use system default.
│   ├── Constraints: required
│   └── Mutable: yes

systemDefaults: map
├── Purpose: Snapshot of system default benchmark values at time of last save. Keys match metricTargets. Stored for display purposes — source of truth is server-side config.
│   └── Mutable: yes

historicalWindowDays: number
├── Purpose: Number of days used for historical average computation. Default 90. User-configurable 7–365.
│   └── Mutable: yes

performanceScore: number
├── Purpose: Cached 0–100 performance score computed by GET /api/benchmarks/score. Updated on each score computation. Null if never computed.
│   └── Mutable: yes

performanceScoreComputedAt: timestamp
├── Purpose: Server timestamp of when performanceScore was last computed. Used to determine staleness.
│   └── Mutable: yes

createdAt: timestamp
├── Purpose: Server timestamp written at document creation. Immutable.
│   ├── Constraints: required
│   └── Mutable: no

updatedAt: timestamp
└── Purpose: Server timestamp updated on every PATCH /api/benchmarks/settings call.
    ├── Constraints: required
    └── Mutable: yes

FIRESTORE SECURITY RULES:
- Create: [AUTH_USER_ID] != null
- Read: [user_id] == [AUTH_USER_ID]
- Update: [user_id] == [AUTH_USER_ID]
- Delete: Soft delete (mark deleted, retain data)
- Immutable: createdAt, userId

LIFECYCLE:
- Created: set [SERVER_TIMESTAMP] for createdAt
- Updated: update [UPDATED_AT]
