# Feature In App Notifications Endpoint List

GET /api/notifications

## Description

REQUEST: GET with optional query params: limit? (integer 1–50, default 20), cursor? (notification id for pagination), filter? ('all'|'unread', default 'all'). Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate limit (1–50) and filter ('all'|'unread') — return 400 if invalid.
3. Read users Firestore document for uid via Admin SDK. Extract users.notifications array and users.unreadNotificationCount.
4. If users.notifications is null or undefined, return empty array with unreadCount=0.
5. Filter out isDismissed=true items. If filter=='unread', also filter to isRead=false items.
6. Sort by createdAt descending (newest first).
7. Apply cursor-based pagination: if cursor provided, find the index of the notification with matching id and slice from that index+1. Fetch limit+1 to detect hasMore.
8. Return paginated notifications with unreadCount.
SUCCESS (200): { notifications: [{ id, type, title, body, networkId, resourceType, resourceId, isRead, severity, actionUrl, metadata, createdAt }], unreadCount: number, hasMore: boolean, nextCursor: string|null }
ERRORS:
- 400: invalid limit or filter value
- 401: invalid/expired Firebase ID token
- 404: users document not found
- 500: Firestore read failure
SECURITY: userId from verified token. Reads only authenticated user's users document. No cross-user access possible.
