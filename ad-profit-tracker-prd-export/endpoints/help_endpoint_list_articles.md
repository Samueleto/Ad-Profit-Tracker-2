# Help Endpoint List Articles

## Description

REQUEST: GET with optional query params: category? (one of 'faq'|'api_guide'|'troubleshooting'|'video_tutorial'|'network_setup'|'account'), role? (one of 'owner'|'admin'|'member'|'viewer'|'analyst'), limit? (integer 1–50, default 20), cursor? (Firestore document ID for pagination). PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only. 2. Validate category if provided — return 400 if not in allowed enum. 3. Read users Firestore document for uid to get workspaceRole and effectivePermissions. 4. Build Firestore query on helpArticles collection: filter isPublished==true. If category provided, add filter category==category. Use getDocs (one-time read). 5. Filter results in memory: for each article, if requiredPermission is set, check that the user's effectivePermissions[requiredPermission] === true — exclude articles the user lacks permission for. If requiredRole is set, check that users.workspaceRole matches or is higher in hierarchy. 6. Sort by updatedAt descending. Apply cursor-based pagination. 7. Group articles by category and compute per-category counts for the category grid. 8. Return paginated article list with category counts. SUCCESS (200): { articles: [{ id, title, slug, category, summary, tags, readTimeMinutes, videoUrl, viewCount, isNew: boolean, authorName, updatedAt }], categoryGroups: [{ category: string, count: number }], hasMore: boolean, nextCursor: string|null } ERRORS: 400 (invalid category), 401 (unauthenticated), 500 (Firestore read failure). SECURITY: uid from verified token. Articles filtered by user's effectivePermissions — no gated content leaks. isPublished=false articles never returned to non-admin users.
