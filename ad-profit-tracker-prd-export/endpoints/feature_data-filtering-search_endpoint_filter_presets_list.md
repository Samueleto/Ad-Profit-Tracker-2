# Feature Data Filtering Search Endpoint Filter Presets List

GET /api/filters/presets

## Description

REQUEST: GET — no query params. Bearer token in Authorization header. Returns all saved filter presets for the authenticated user.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Read the users Firestore document for uid. Extract users.savedFilterPresets array (embedded in the users document — not a separate collection, since presets are simple 1:1 user data with no independent query needs).
3. If users.savedFilterPresets is null or undefined, return an empty array (not 404 — absence of presets is a valid state).
4. Sort presets by createdAt descending.
5. Return the presets array.
SUCCESS (200): { presets: [{ id: string, name: string, networks: string[], countries: string[], metric: string, dataQuality: string, createdAt: timestamp }] }
ERRORS:
- 401: invalid/expired Firebase ID token
- 404: user document not found
- 500: Firestore read failure
SECURITY: userId from verified token. Reads only the authenticated user's users document. Presets are embedded in users collection — no cross-user access possible.
