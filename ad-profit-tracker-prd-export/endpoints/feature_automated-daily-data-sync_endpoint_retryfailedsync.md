# Feature Automated Daily Data Sync Endpoint RetryFailedSync

POST /api/scheduled/retry-failed

## Description

REQUEST: POST { networkId: string } — retries the last failed sync for a specific network for the authenticated user. Bearer token in Authorization header.

PROCESS:
1. Verify Firebase ID token — return 401 if invalid.
2. Validate networkId is one of the 4 supported networks — return 400 if not.
3. Check rate limit: max 5 retries/hour per user per network — return 429 if exceeded.
4. Fetch networkConfigs document for userId+networkId. If lastSyncStatus != 'failed', return 400 with message 'No failed sync to retry for this network'.
5. Fetch and decrypt the API key from apiKeys collection. Return 404 if no key saved.
6. Determine the date to retry: use the date from the last failed sync attempt (stored in metadata of the most recent failed auditLogs entry for this network) or default to yesterday.
7. Dispatch the sync by calling the appropriate network-specific sync logic (same as the individual network sync endpoints): decrypt key → call network API → batch upsert adStats → update networkConfigs → write audit log.
8. On success: update networkConfigs lastSyncStatus='success', lastSyncedAt=serverTimestamp(), lastSyncError=null. Write audit log with status='success'.
9. On failure: update networkConfigs lastSyncStatus='failed', lastSyncError=errorMessage. Write audit log with status='failure'. Return 502.

SUCCESS (200): { networkId: string, status: 'success', rowsFetched: number, date: string, latencyMs: number }

ERRORS:
- 400: invalid networkId, no failed sync to retry
- 401: unauthenticated
- 404: no API key saved for this network
- 429: rate limit exceeded (5/hour per user per network)
- 502: network API unreachable or returned error
- 500: decryption or Firestore failure

SECURITY: API key decrypted server-side only. userId from verified token. Rate limited to prevent retry abuse.
