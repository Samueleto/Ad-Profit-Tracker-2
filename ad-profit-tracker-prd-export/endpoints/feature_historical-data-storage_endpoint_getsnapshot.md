# Feature Historical Data Storage Endpoint GetSnapshot

GET /api/stats/snapshot

## Description

REQUEST: GET with query params: date (YYYY-MM-DD, required), networkId? (one of the 4 networks, optional — omit for all networks), groupBy? ('country'|'total', default 'total').
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 immediately if invalid. uid from verified token only.
2. Validate date: required, valid YYYY-MM-DD. Validate networkId if provided. Return 400 if invalid.
3. Build Firestore query on adStats collection: filter userId==uid AND date==date. If networkId provided, also filter networkId==networkId. Use getDocs (one-time read, not onSnapshot).
4. If no documents found, return 404.
5. If groupBy='country': group rows by networkId+country, returning per-country breakdown with cost, revenue, impressions, clicks, ctr, cpm.
6. If groupBy='total': aggregate all rows into per-network totals (sum cost, revenue, impressions, clicks; compute average ctr and cpm).
7. Compute cross-network summary: totalRevenue (sum of all revenue-network rows), totalCost (sum of exoclick cost rows), netProfit (totalRevenue - totalCost), roi (netProfit / totalCost * 100 if totalCost > 0 else null).
8. Return structured snapshot.
SUCCESS (200): { date: string, networks: [{ networkId: string, dataRole: string, cost: number|null, revenue: number|null, impressions: number, clicks: number, ctr: number, cpm: number, countryCount: number }], summary: { totalRevenue: number, totalCost: number, netProfit: number, roi: number|null }, groupBy: string }
ERRORS:
- 400: missing/invalid date, invalid networkId
- 401: unauthenticated
- 404: no data found for this date
- 500: Firestore read failure
SECURITY: userId always from verified token. Query scoped to authenticated user's documents only.
