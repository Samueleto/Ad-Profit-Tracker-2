# Feature Email Alerts Notifications Endpoint Get Email Log

GET /api/email/log

## Description

REQUEST: GET with query params: limit? (integer 1–50, default 20), cursor? (auditLog document ID for pagination), type? (one of the 9 notification types or 'summary' or 'test'). Bearer token in Authorization header. Returns the authenticated user's email send history from auditLogs.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate limit (1–50) and type if provided. Return 400 if invalid.
3. Query auditLogs Firestore collection: filter userId==uid AND action=='email_sent'. If type provided, filter resourceId==type. Order by createdAt DESC. Apply cursor-based pagination using startAfter if cursor provided. Fetch limit+1 to detect hasMore.
4. Map documents to response shape: id, emailType (from resourceId), recipientEmail (from metadata.recipientEmail), subject (from metadata.subject), status (from status field), sentAt (from createdAt), triggeredBy (from metadata.triggeredBy).
5. Return paginated email log.
SUCCESS (200): { logs: [{ id: string, emailType: string, recipientEmail: string, subject: string|null, status: 'success'|'failure', sentAt: timestamp, triggeredBy: 'user'|'system'|'user_test' }], hasMore: boolean, nextCursor: string|null }
ERRORS:
- 400: invalid limit or type value
- 401: unauthenticated
- 500: Firestore read failure
SECURITY: userId from verified token. Query scoped to authenticated user's auditLogs only. Read-only.
