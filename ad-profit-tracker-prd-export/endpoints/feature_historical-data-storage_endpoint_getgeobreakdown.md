# Feature Historical Data Storage Endpoint GetGeoBreakdown

GET /api/stats/geo-breakdown

## Description

REQUEST: GET with query params: dateFrom (YYYY-MM-DD, required), dateTo (YYYY-MM-DD, required), networkId? (optional — omit for cross-network aggregate), metric? ('revenue'|'cost'|'profit', default 'profit'), limit? (integer 1–100, default 20).
PROCESS:
1. Verify Firebase ID token — return 401 if invalid. uid from verified token only.
2. Validate all params. Return 400 if invalid.
3. Query adStats Firestore collection: filter userId==uid AND date>=dateFrom AND date<=dateTo AND country != null. If networkId provided, also filter networkId==networkId. Use getDocs.
4. Group results by country in memory. For each country, aggregate: sum cost (exoclick rows), sum revenue (rollerads+zeydoo+propush rows), compute netProfit=revenue-cost, sum impressions and clicks.
5. Compute the requested metric value per country (revenue, cost, or profit). Sort countries by that metric descending. Slice to limit.
6. Compute percentage share for each country (country metric / total metric * 100).
7. Enrich country codes with full country names using a static ISO 3166-1 alpha-2 lookup map bundled server-side.
8. Return the ranked country list with cross-network aggregated metrics.
SUCCESS (200): { dateFrom: string, dateTo: string, metric: string, networkId: string|null, countries: [{ country: string, countryName: string, revenue: number, cost: number, netProfit: number, impressions: number, clicks: number, metricValue: number, metricShare: number }], totals: { revenue: number, cost: number, netProfit: number } }
ERRORS:
- 400: missing/invalid params, range exceeds 90 days, invalid metric or networkId
- 401: unauthenticated
- 500: Firestore read failure
SECURITY: userId always from verified token. Query scoped to authenticated user only.
