# Feature Real Time Sync Status Monitoring Endpoint Livesyncstate

GET /api/sync/live-state

## Description

REQUEST: GET — no body. Optional query param: networkId? (one of 4 networks — omit for all). Bearer token in Authorization header.

PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate networkId if provided — return 400 if not one of ['exoclick','rollerads','zeydoo','propush'].
3. Query networkConfigs Firestore collection: filter userId==uid. If networkId provided, also filter networkId==networkId. Use getDocs (one-time read — not onSnapshot, as this is a polling endpoint).
4. For each of the 4 networks (or the requested one), extract: lastSyncedAt, lastSyncStatus, lastSyncError, isActive, syncSchedule, retryCount, circuitBreakerOpen, circuitBreakerOpenedAt, lastErrorCode, totalFailureCount.
5. Query adStats collection for each network: filter userId==uid AND networkId==network, order by syncedAt DESC, limit 1 — to get the most recent syncedAt timestamp and rowCount for that network. Use getDocs.
6. Query auditLogs collection: filter userId==uid AND resourceType=='sync' AND action=='manual_sync_triggered', order by createdAt DESC, limit 1 per network — to get the most recent sync event metadata including latencyMs and rowsFetched. Use getDocs with networkId filter per iteration.
7. Compute derived fields per network: isCurrentlySyncing (lastSyncStatus=='in_progress' — set by sync helpers when a sync starts), syncPhase (derived from lastSyncStatus: 'idle'|'fetching'|'writing'|'complete'|'failed'), timeUntilNextScheduledSync (based on syncSchedule + lastSyncedAt).
8. Compute overall health: 'healthy' if all active networks have lastSyncStatus in ['success','never'] and no circuitBreakerOpen; 'degraded' if any network has lastSyncStatus=='failed' but no circuit open; 'critical' if any circuitBreakerOpen==true.
9. Return consolidated live state.

SUCCESS (200): { overallHealth: 'healthy'|'degraded'|'critical', networks: [{ networkId: string, isActive: boolean, syncPhase: string, lastSyncStatus: string, lastSyncedAt: timestamp|null, lastSyncError: string|null, latestSyncedAt: timestamp|null, latestDataDate: string|null, lastRowsFetched: number|null, lastLatencyMs: number|null, circuitBreakerOpen: boolean, circuitBreakerOpenedAt: timestamp|null, lastErrorCode: string|null, retryCount: number, totalFailureCount: number, timeUntilNextScheduledSync: number|null }], polledAt: string }

ERRORS:
- 400: invalid networkId
- 401: invalid/expired Firebase ID token
- 500: Firestore read failure

SECURITY: userId always from verified token. All Firestore queries scoped to authenticated user's documents only. Read-only endpoint — no writes. Designed for polling — no rate limit but responses are lightweight (no large payloads).
