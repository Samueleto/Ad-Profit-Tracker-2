# Feature Network Configuration Management Endpoint ConfigList

GET /api/networks/config/list

## Description

REQUEST: GET — no body. Bearer token in Authorization header. PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. 2. Query networkConfigs collection where userId == uid, ordered by displayOrder ascending. 3. If fewer than 4 documents exist (first-time user), initialize missing network configs with defaults (isActive=true, syncSchedule='daily', timeoutSeconds=30, retryAttempts=3, displayOrder per network index) and batch-write them to Firestore. 4. Also fetch apiKeys subcollection status for each network (reuse GET /api/keys/status logic) to enrich response with connection status. 5. Return array of 4 config objects merged with connection status. SUCCESS (200): { networks: [{ networkId, isActive, syncSchedule, dataRole, endpointOverride, timeoutSeconds, retryAttempts, lastSyncedAt, lastSyncStatus, lastSyncError, displayOrder, connectionStatus: 'connected'|'not_connected', updatedAt }] } ERRORS: 401 (invalid token), 500 (Firestore read failure). SECURITY: UID always from verified token. Users only see their own configs.
