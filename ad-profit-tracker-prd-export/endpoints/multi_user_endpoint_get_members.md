# Multi User Endpoint Get Members

GET /api/team/members

## Description

REQUEST: GET — no query params. Bearer token in Authorization header. Returns all members of the authenticated user's workspace.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Read users Firestore document for uid. Extract users.workspaceId. If workspaceId is null/undefined, the user has no workspace — return 404 with 'No workspace found for this user'.
3. Query users Firestore collection: getDocs with filter where workspaceId == users.workspaceId. This returns all members of the workspace in a single query.
4. For each member document, return safe fields only: uid, displayName, email, workspaceRole, workspaceJoinedAt, photoURL. Never return encrypted API keys or sensitive fields.
5. Sort by workspaceJoinedAt ascending (oldest members first, owner at top).
6. Return members array with total count.
SUCCESS (200): { members: [{ uid, displayName, email, workspaceRole, workspaceJoinedAt, photoURL }], total: number, workspaceId: string, workspaceName: string }
ERRORS:
- 401: invalid/expired Firebase ID token
- 404: user has no workspace
- 500: Firestore read failure
SECURITY: userId from verified token. Query scoped to authenticated user's workspaceId. Members can only see their own workspace's members — no cross-workspace data leakage.
