# Audit Endpoint ExportLogs

GET /api/audit/logs/export

## Description

REQUEST: GET with same query params as /api/audit/logs (action?, resourceType?, startDate?, endDate?, status?) but no pagination — exports all matching records up to a hard cap of 10,000 rows.
PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. 2. Validate query params same as /api/audit/logs. 3. Build Firestore query on auditLogs collection filtered by userId == uid (from token) plus any provided filters, ordered by createdAt DESC, limited to 10,000 documents. 4. Execute getDocs. 5. Map documents to flat CSV rows: id, action, resourceType, resourceId, status, errorMessage, metadata (JSON stringified), ipAddress, userAgent, createdAt (ISO 8601 string). 6. Build CSV string with header row. 7. Return response with Content-Type: text/csv, Content-Disposition: attachment; filename='activity-log-{date}.csv', and the CSV body.
SUCCESS (200): CSV file download (Content-Type: text/csv)
ERRORS: 400 (invalid query params), 401 (unauthenticated), 429 (rate limited — 5/hour, exports are expensive), 500 (Firestore read or CSV generation failure)
SECURITY: userId always from verified token. Hard cap of 10,000 rows prevents runaway Firestore reads. Rate limited to 5/hour per user to control read costs.
