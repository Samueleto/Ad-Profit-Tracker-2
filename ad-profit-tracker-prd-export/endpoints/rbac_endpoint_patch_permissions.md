# Rbac Endpoint Patch Permissions

PATCH /api/rbac/permissions

## Description

REQUEST: PATCH { role: 'admin'|'member'|'viewer'|'analyst' (required), permissions: { [permKey]: boolean } (required, partial update — only keys provided are updated), resetToDefaults?: boolean (optional — if true, ignores permissions map and resets the specified role to system defaults) }. Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Read users Firestore document for uid. Check users.workspaceRole — must be 'owner' to edit admin/member/viewer/analyst permissions. 'admin' role can only edit 'member' and 'viewer' permissions. Return 403 if insufficient.
3. Validate role: must be one of ['admin','member','viewer','analyst'] — 'owner' is immutable, return 400 if owner is specified.
4. Validate permissions map: each key must be one of the 9 defined permission keys (canViewDashboard, canExportData, canManageApiKeys, canSyncNetworks, canViewReports, canCreateReports, canManageTeam, canViewAuditLogs, canManageBenchmarks). Values must be boolean. Return 400 if invalid.
5. Check rate limit: max 20 permission updates/hour per user — return 429 if exceeded.
6. Use Firestore Admin SDK set() with merge:true on workspacePermissions document for workspaceId==uid (upsert). Update only the specified role's permission keys using dot-notation (e.g., rolePermissions.admin.canExportData). If resetToDefaults=true, overwrite the entire role's map with system defaults. Update customizedRoles array: add role if not present and not resetting; remove role if resetting to defaults. Set updatedAt=serverTimestamp(), updatedByUid=uid.
7. Trigger recomputation of effectivePermissions for all workspace members with the affected role: query users collection where workspaceId==workspaceId AND workspaceRole==role, batch-update their effectivePermissions maps and permissionsUpdatedAt.
8. Write audit log entry to auditLogs via Admin SDK: action='network_config_updated', resourceType='workspace', resourceId=workspaceId, metadata={ changedRole: role, changedPermissions: Object.keys(permissions), resetToDefaults: resetToDefaults||false }, status='success'.
9. Return the full updated permission matrix.
SUCCESS (200): { workspaceId: string, roles: { [role]: { [permKey]: boolean } }, customizedRoles: string[], updatedAt: timestamp }
ERRORS:
- 400: invalid role, invalid permission key, owner role specified, empty permissions map
- 401: invalid/expired Firebase ID token
- 403: insufficient role to edit target role's permissions
- 429: rate limit exceeded (20/hour)
- 500: Firestore write failure
SECURITY: Role hierarchy enforced server-side. Owner can edit any non-owner role. Admin can only edit member/viewer. Batch effectivePermissions update ensures all members see changes immediately.
