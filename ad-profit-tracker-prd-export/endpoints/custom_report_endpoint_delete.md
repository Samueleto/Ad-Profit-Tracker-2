# Custom Report Endpoint Delete

DELETE /api/reports/:id

## Description

REQUEST: DELETE /api/reports/:id — :id is the report UUID in the URL path. Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate :id: required, non-empty string — return 400 if missing.
3. Check rate limit: max 20 report deletes/hour per user — return 429 if exceeded.
4. Read users Firestore document for uid. Find the report with matching id in users.savedReports array. Return 404 if not found.
5. Use Firestore Admin SDK arrayRemove to remove the matching report object from users.savedReports. Set users.updatedAt=serverTimestamp().
6. Write audit log entry to auditLogs Firestore collection via Admin SDK: action='preferences_updated', resourceType='report', resourceId=id, metadata={ reportName: deletedReport.name, action: 'deleted' }, status='success'.
7. Return deletion confirmation.
SUCCESS (200): { deleted: true, reportId: string }
ERRORS:
- 400: missing reportId
- 401: invalid/expired Firebase ID token
- 404: report not found in user's savedReports
- 429: rate limit exceeded (20/hour)
- 500: Firestore write failure
SECURITY: userId from verified token. Report ownership verified by reading users document scoped to uid before deletion. Audit logged for accountability.
