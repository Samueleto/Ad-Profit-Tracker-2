# Api Key Management Storage Endpoint GetStatus

GET /api/keys/status

## Description

REQUEST: GET. PROCESS: 1. Verify the user's Firebase ID token to get their UID. 2. Query the `users/{uid}/apiKeys` subcollection in Firestore to get all documents. 3. Map over the results to create an array of status objects. For each of the four supported networks, determine if a corresponding document exists. 4. Return an array containing an object for each supported network, indicating its connection status and last update time. SUCCESS (200): [{ networkId, status: 'connected' | 'not_connected', updatedAt }]. ERRORS: 401 (unauthenticated), 500 (Firestore read error). SECURITY: This endpoint is critical for security as it avoids sending sensitive keys to the client. It only confirms the existence of a key.
