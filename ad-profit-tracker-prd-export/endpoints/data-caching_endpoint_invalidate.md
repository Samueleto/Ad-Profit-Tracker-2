# Data Caching Endpoint Invalidate

POST /api/cache/invalidate

## Description

REQUEST: POST { scope: 'all' | 'network' | 'endpoint', keys?: string[] }. Manually invalidates entries in the server-side in-memory cache. PROCESS: 1. Verify Firebase ID token. 2. Validate request body. 3. Check rate limit (15/hour per user). 4. Based on the scope, construct the cache keys to delete (e.g., for scope 'network' and keys ['exoclick'], delete all cache entries starting with 'exoclick_stats_...'). 5. Use the server-side cache provider (e.g., node-cache) to delete the specified keys. 6. Write an audit log entry to `auditLogs` with action='cache_invalidated'. SUCCESS (200): { invalidatedCount: number, scope: string }. ERRORS: 400 (invalid scope/keys), 401 (unauthenticated), 429 (rate limit exceeded). SECURITY: User can only invalidate their own cached data as cache keys are namespaced by UID. Rate-limited to prevent abuse.
