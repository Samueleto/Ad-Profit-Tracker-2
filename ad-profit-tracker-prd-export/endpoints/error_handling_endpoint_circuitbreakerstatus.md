# Error Handling Endpoint CircuitBreakerStatus

GET /api/errors/circuit-breaker/status

## Description

REQUEST: GET — no query params. Bearer token in Authorization header. Returns circuit breaker status for all four networks for the authenticated user, including whether each circuit is open, when it opened, and when it will auto-reset.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid.
2. Query networkConfigs Firestore collection: filter userId==uid. Use getDocs.
3. For each of the 4 networks, extract circuitBreakerOpen, circuitBreakerOpenedAt, retryCount, retryAttempts, lastErrorCode.
4. Compute auto-reset time: if circuitBreakerOpen==true and circuitBreakerOpenedAt is set, autoResetAt = circuitBreakerOpenedAt + 3600 seconds. If current time > autoResetAt, mark as 'auto-reset-pending' (will be reset on next sync attempt).
5. Compute health summary: networksHealthy (circuitBreakerOpen==false count), networksDegraded (circuitBreakerOpen==true count).
6. Return consolidated circuit breaker status.
SUCCESS (200): { summary: { networksHealthy: number, networksDegraded: number }, circuits: [{ networkId: string, isOpen: boolean, openedAt: timestamp|null, autoResetAt: timestamp|null, isAutoResetPending: boolean, retryCount: number, retryAttempts: number, lastErrorCode: string|null }] }
ERRORS:
- 401: unauthenticated
- 500: Firestore read failure
SECURITY: userId from verified token. Query scoped to authenticated user only.
