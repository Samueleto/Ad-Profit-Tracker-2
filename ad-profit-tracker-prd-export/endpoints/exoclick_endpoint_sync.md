# Exoclick Endpoint Sync

POST /api/networks/exoclick/sync

## Description

REQUEST: POST { dateFrom?: string (YYYY-MM-DD, defaults to yesterday), dateTo?: string (YYYY-MM-DD, defaults to yesterday) } — date range is optional; defaults to yesterday for daily pulls.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 immediately if invalid. uid from verified token only.
2. Validate dateFrom/dateTo if provided: must be valid YYYY-MM-DD strings, dateFrom <= dateTo, range max 90 days. Return 400 if invalid.
3. Check rate limit: max 10 manual syncs/hour per user. Return 429 if exceeded.
4. Fetch networkConfigs document for userId+networkId='exoclick'. If isActive=false, return 400 with message 'ExoClick network is disabled'.
5. Fetch encrypted API key from apiKeys collection where userId==uid AND networkId=='exoclick'. Return 404 if not found.
6. Decrypt API key using server-side AES/cryptr helper (same secret as /api/keys/save). Never log or return the decrypted key.
7. Retrieve endpointOverride and timeoutSeconds from networkConfigs (defaults: ExoClick Stats API base URL, 30s timeout).
8. Call ExoClick Global Statistics API using the exoclick-api client library: initialize client with decrypted API key, call getGlobalStatistics() with groupByCountry(), setTimezone(user timezone from users.preferences.timezone or 'UTC'), applyFilter({ date: { from: dateFrom, to: dateTo } }), enableDetailedResults(). Record start time for latency measurement.
9. On API success: parse response rows. For each row (date + country combination), upsert a document into adStats collection: set userId, networkId='exoclick', date, country, cost (from ExoClick spend field), impressions, clicks, ctr, cpm, rawResponse (sanitized — strip any key/token/secret fields), syncedAt=serverTimestamp(). Use Firestore batch writes in chunks of 500 for efficiency.
10. Update networkConfigs document: lastSyncedAt=serverTimestamp(), lastSyncStatus='success', lastSyncError=null.
11. Call POST /api/audit/log internally (via Admin SDK direct write to avoid HTTP overhead) with action='manual_sync_triggered', resourceType='sync', resourceId='exoclick', metadata={ triggered: ['exoclick'], latencyMs, rowsFetched: parsedRows.length, dateFrom, dateTo }, status='success'.
12. On ExoClick API error: update networkConfigs lastSyncStatus='failed', lastSyncError=errorMessage. Write audit log with status='failure', errorMessage. Return 502.
SUCCESS (200): { networkId: 'exoclick', rowsFetched: number, dateFrom: string, dateTo: string, latencyMs: number, syncedAt: timestamp }
ERRORS:
- 400: invalid date range, range exceeds 90 days, network disabled
- 401: invalid/expired Firebase ID token
- 404: no ExoClick API key saved
- 429: rate limit exceeded (10/hour per user)
- 502: ExoClick API unreachable or returned error
- 500: decryption failure, Firestore write failure
SECURITY: API key decrypted server-side only, never returned or logged. userId always from verified token. Metadata sanitized before audit log write. Rate limited to prevent runaway ExoClick API calls and Firestore write costs.
