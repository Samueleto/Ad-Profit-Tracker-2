GET /api/filters/search

## Description

REQUEST: GET with query params: q (string, required, min 1 char, max 100 chars), dateFrom (YYYY-MM-DD, required), dateTo (YYYY-MM-DD, required), scope? ('country'|'network'|'all', default 'all'), limit? (integer 1–50, default 20).
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate q: required, 1–100 chars, strip leading/trailing whitespace — return 400 if empty after trim. Validate dateFrom/dateTo and limit.
3. Check node-cache for key '{uid}_search_{q}_{dateFrom}_{dateTo}_{scope}_{limit}'. If cache hit, return with X-Cache: HIT header.
4. Query adStats Firestore collection: filter userId==uid AND date>=dateFrom AND date<=dateTo. Use getDocs (one-time read — Firestore does not support full-text search, so we fetch the bounded set and filter in memory).
5. Apply in-memory search: normalize q to lowercase. For scope=='country' or scope=='all': match against country field (ISO code) and enriched country name (from static ISO 3166-1 lookup). For scope=='network' or scope=='all': match against networkId field. Collect all matching documents.
6. Group matching documents by their primary key (country or networkId depending on scope). For each group, aggregate: sum revenue, sum cost, netProfit, roi (null-safe), sum impressions, sum clicks, unique dates count.
7. Sort results by netProfit descending. Slice to limit.
8. Enrich country results with full country names from static ISO lookup.
9. Store in node-cache with 3-minute TTL (shorter than stats endpoints since search results are more query-specific).
10. Return search results with match context.
SUCCESS (200): { q: string, dateFrom: string, dateTo: string, scope: string, results: [{ type: 'country'|'network', key: string, label: string, revenue: number, cost: number, netProfit: number, roi: number|null, impressions: number, clicks: number, daysWithData: number }], totalResults: number, cachedAt: string|null }
ERRORS:
- 400: missing/empty q, invalid date range, range exceeds 90 days, invalid scope
- 401: invalid/expired Firebase ID token
- 404: no results found for query
- 500: Firestore read failure
SECURITY: userId always from verified token. Firestore query scoped to authenticated user's adStats only. In-memory search never exposes other users' data. Cache keys namespaced by uid. Query string sanitized before use.
