# Rollerads Endpoint Scheduled Sync

POST /api/networks/rollerads/scheduled-sync

## Description

REQUEST: POST — no body. Secured with a service-auth header (x-internal-secret matching INTERNAL_SYNC_SECRET environment variable), NOT a user Firebase ID token. Invoked by Cloud Scheduler on a daily schedule.
PROCESS:
1. Validate x-internal-secret header matches the INTERNAL_SYNC_SECRET environment variable. Return 401 immediately if missing or mismatched — this endpoint is never called by end users.
2. Query networkConfigs Firestore collection (admin SDK collection-group or top-level query) for all documents where networkId=='rollerads' AND isActive==true AND syncSchedule=='daily'. This is a cross-user admin query using the Firebase Admin SDK.
3. For each matching networkConfig document, retrieve the userId. Fetch the encrypted API key from apiKeys where userId==userId AND networkId=='rollerads'. Skip users without a saved key — add to skipped list.
4. For each user with a valid key: decrypt the API key using the server-side cryptr/AES helper. Using axios, call GET /sites/stat on the RollerAds API (https://api.rollerads.com/sites/stat) with Authorization: Bearer {decryptedKey}, params: { date_from: yesterday, date_to: yesterday, group_by: 'country' }. Parse response.data rows.
5. For each row, upsert into adStats collection: userId, networkId='rollerads', date, country, revenue, impressions, clicks, ctr, cpm, rawResponse (sanitized), syncedAt=serverTimestamp(). Batch writes in chunks of 500.
6. Update networkConfigs for this user+rollerads: lastSyncedAt=serverTimestamp(), lastSyncStatus='success'|'failed', lastSyncError=null|errorMessage.
7. Write audit log entry per user directly to auditLogs Firestore collection via Admin SDK: action='manual_sync_triggered', resourceType='sync', resourceId='rollerads', metadata={ triggered: ['rollerads'], scheduledSync: true, latencyMs, rowsFetched }, status='success'|'failure'.
8. Return a summary of all users processed.
SUCCESS (200): { processed: number, succeeded: number, failed: number, skipped: number, date: string }
ERRORS:
- 401: missing or invalid internal secret
- 500: Firestore query failure or unhandled exception
SECURITY: Internal endpoint only — protected by x-internal-secret header, not user JWT. Never exposed to client. API keys decrypted server-side only. Rate limit not applicable (called by scheduler, not users).
