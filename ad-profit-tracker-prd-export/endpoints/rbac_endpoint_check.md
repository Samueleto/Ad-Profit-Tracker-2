# Rbac Endpoint Check

POST /api/rbac/check

## Description

REQUEST: POST { permission: string (required — one of the 9 permission keys), targetUid?: string (optional — check another user's permission, Owner/Admin only) }. Bearer token in Authorization header. Server-side permission gate used by other endpoints and Cloud Functions to verify access before executing privileged operations.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate permission: must be one of the 9 defined keys — return 400 if not.
3. Determine checkUid: if targetUid provided, verify actor is Owner or Admin (read actor's users.workspaceRole) — return 403 if not. Use targetUid as checkUid. Otherwise use uid.
4. Read users Firestore document for checkUid. Extract workspaceRole and effectivePermissions.
5. If workspaceRole == 'owner', return { granted: true, reason: 'owner_all_access' }.
6. If effectivePermissions is fresh (permissionsUpdatedAt within 5 minutes), use cached value.
7. On stale/missing cache: read workspacePermissions for workspaceId, resolve permission for the role.
8. Return granted boolean with role context.
SUCCESS (200): { granted: boolean, permission: string, workspaceRole: string, uid: string, reason: 'owner_all_access'|'role_permission'|'permission_denied' }
ERRORS:
- 400: invalid permission key
- 401: invalid/expired Firebase ID token
- 403: actor lacks permission to check another user's permissions
- 404: target user not found
- 500: Firestore read failure
SECURITY: Used as an internal gate by other API routes. uid always from verified token. targetUid check requires elevated role.
