# Feature Historical Data Storage Endpoint ListDates

GET /api/stats/dates

## Description

REQUEST: GET with query params: networkId? (optional — omit for cross-network), dateFrom? (YYYY-MM-DD, optional — defaults to 90 days ago), dateTo? (YYYY-MM-DD, optional — defaults to today).
PROCESS:
1. Verify Firebase ID token — return 401 if invalid. uid from verified token only.
2. Validate params if provided. Return 400 if invalid.
3. Default dateFrom to 90 days ago and dateTo to today if not provided.
4. Query adStats Firestore collection: filter userId==uid AND date>=dateFrom AND date<=dateTo. If networkId provided, also filter networkId==networkId. Use getDocs.
5. Extract unique dates from the result set. For each unique date, compute: which networks have data (array of networkIds), total row count across all networks for that date, whether the date is 'complete' (all 4 active networks have data) or 'partial' (some missing).
6. Sort dates descending (most recent first).
7. Return the list of dates with data availability metadata — this powers calendar/date-picker components in the dashboard to show which dates have data available for querying.
SUCCESS (200): { dateFrom: string, dateTo: string, networkId: string|null, dates: [{ date: string, networks: string[], rowCount: number, isComplete: boolean }], totalDates: number }
ERRORS:
- 400: invalid date params
- 401: unauthenticated
- 500: Firestore read failure
SECURITY: userId always from verified token. Query scoped to authenticated user only.
