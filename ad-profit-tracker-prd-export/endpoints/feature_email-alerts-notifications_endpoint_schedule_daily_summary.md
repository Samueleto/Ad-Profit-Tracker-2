# Feature Email Alerts Notifications Endpoint Schedule Daily Summary

POST /api/email/schedule-summary

## Description

REQUEST: POST — no body. Internal-only endpoint triggered by Cloud Scheduler (e.g., Google Cloud Scheduler) once daily at a configured time (e.g., 08:00 UTC). Fans out to all users who have email summary enabled and dispatches POST /api/email/send-summary for each.
PROCESS:
1. Validate x-internal-secret header against INTERNAL_SYNC_SECRET env var. Return 401 if invalid.
2. Determine today's date and compute dateFrom/dateTo for 'yesterday' (single day summary).
3. Query users Firestore collection using Admin SDK: getDocs with filter where notificationPreferences.schedule_delivered.emailEnabled == true (or fall back to fetching all users and filtering in memory if Firestore index not available — note: add composite index on notificationPreferences for cost efficiency). Limit to 500 users per invocation to stay within Cloud Function memory limits.
4. For each qualifying user, check users.preferences.emailSummaryFrequency — if set to 'daily' or not set (default), include in this batch. Skip users where emailSummaryFrequency is 'weekly' or 'monthly' (those are handled by separate scheduler jobs).
5. For each qualifying user, invoke POST /api/email/send-summary internally (direct function call, not HTTP) with { userId, dateFrom, dateTo, summaryType: 'daily' }. Collect results.
6. Write a single audit log entry to auditLogs: action='email_sent', resourceType='notification', resourceId='daily_summary_batch', metadata={ totalUsers: number, sent: number, skipped: number, failed: number, date: dateFrom }, status='success'.
7. Return batch summary.
SUCCESS (200): { processed: number, sent: number, skipped: number, failed: number, date: string }
ERRORS:
- 401: invalid internal secret
- 500: Firestore read failure or batch processing error
SECURITY: Internal-only. x-internal-secret required. Never callable by end-users. Designed for Cloud Scheduler invocation.
