# Feature In App Notifications Endpoint Create

POST /api/notifications/create

## Description

REQUEST: POST { userId: string (required — internal use only, from verified token or x-internal-secret), type: string (required — one of 'sync_success'|'sync_failure'|'export_complete'|'export_failure'|'schedule_delivered'|'schedule_failed'|'reconciliation_anomaly'|'circuit_breaker_opened'|'rate_limit_exceeded'), title: string (required, max 100 chars), body: string (required, max 300 chars), networkId?: string|null, resourceType?: string, resourceId?: string|null, severity: 'info'|'success'|'warning'|'error' (required), actionUrl?: string|null, metadata?: map }. Dual-auth: Bearer token (user-initiated test) or x-internal-secret header (server-side triggers from sync/export/schedule Cloud Functions).
PROCESS:
1. Check auth: if Authorization header present, verify Firebase ID token and derive uid. If x-internal-secret header present, validate against INTERNAL_SYNC_SECRET env var and use userId from request body. Return 401 if neither is valid.
2. Validate all required fields. type must be one of the 9 allowed values. severity must be one of 4 values. title max 100 chars, body max 300 chars. Return 400 if invalid.
3. Check user's notificationPreferences for the given type — if preferences[type].enabled === false, return 200 with { created: false, reason: 'disabled_by_user' } without writing.
4. Generate UUID v4 for notification id. Build notification object: { id, type, title, body, networkId: networkId||null, resourceType: resourceType||null, resourceId: resourceId||null, isRead: false, isDismissed: false, severity, actionUrl: actionUrl||null, metadata: metadata||{}, createdAt: serverTimestamp() }.
5. Read users document for uid. Check current notifications array length. If >= 100, identify the oldest 10 dismissed or read notifications and remove them via arrayRemove before appending (pruning strategy). If still >= 100 after pruning, remove the oldest 10 regardless.
6. Use Firestore Admin SDK transaction: arrayUnion new notification object into users.notifications. Increment users.unreadNotificationCount by 1. Set users.notificationsUpdatedAt=serverTimestamp().
7. Return the created notification.
SUCCESS (201): { created: true, notification: { id, type, title, body, severity, createdAt } }
ERRORS:
- 400: invalid type, severity, or field lengths
- 401: neither valid Firebase ID token nor valid internal secret
- 404: users document not found for userId
- 500: Firestore write failure
SECURITY: Dual-auth pattern. Internal calls use x-internal-secret. User calls use Firebase ID token. userId always from verified token for user calls, from body only for internal-secret calls. Never exposes API keys or encrypted values.
