# Auth Endpoint Sync User

POST /api/auth/sync-user

## Description

REQUEST: POST — no body required. Firebase ID token passed as Bearer token in Authorization header.
PROCESS: 1. Extract Bearer token from Authorization header. 2. Verify token using Firebase Admin SDK (admin.auth().verifyIdToken(token)). 3. Check if users/{uid} document exists in Firestore. 4. If not exists: create document with uid, email, displayName, photoURL, authProvider (derived from token.firebase.sign_in_provider), role='user', createdAt=serverTimestamp(), lastLoginAt=serverTimestamp(). 5. If exists: update only lastLoginAt=serverTimestamp(). 6. Return user document data.
SUCCESS (200): { uid, email, displayName, photoURL, authProvider, role, createdAt, lastLoginAt }
ERRORS: 401 (invalid or expired Firebase ID token), 500 (Firestore write failure)
SECURITY: Firebase Admin SDK verifyIdToken is the only auth check — no session cookies. Called immediately after client-side Firebase Auth success. Idempotent — safe to call on every login.
