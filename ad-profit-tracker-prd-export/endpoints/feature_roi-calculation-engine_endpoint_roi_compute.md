# Feature Roi Calculation Engine Endpoint Roi Compute

GET /api/roi/compute

## Description

REQUEST: GET with query params: dateFrom (YYYY-MM-DD, required), dateTo (YYYY-MM-DD, required), networkId? (optional — omit for cross-network aggregate), groupBy? ('total'|'daily'|'country', default 'total').
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 immediately if invalid. uid from verified token only.
2. Validate dateFrom and dateTo: required, valid YYYY-MM-DD, dateFrom <= dateTo, max 90-day range — return 400 if invalid.
3. Validate networkId if provided: must be one of ['exoclick','rollerads','zeydoo','propush'] — return 400 if not.
4. Check server-side node-cache for key '{uid}_roi_{dateFrom}_{dateTo}_{networkId||all}_{groupBy}'. If cache hit, return cached result with X-Cache: HIT header.
5. On cache miss, query adStats Firestore collection: filter userId==uid AND date>=dateFrom AND date<=dateTo. If networkId provided, also filter networkId==networkId. Use getDocs (one-time read).
6. If no documents found, return 404.
7. Apply the null-safe ROI formula: totalRevenue = sum of revenue from rollerads+zeydoo+propush rows; totalCost = sum of cost from exoclick rows; netProfit = totalRevenue - totalCost; roi = (totalCost > 0) ? (netProfit / totalCost * 100) : null. Never return Infinity or NaN — always null when totalCost === 0.
8. Compute roiIndicator: roi > 0 → 'positive', roi < 0 → 'negative', roi === 0 → 'breakeven', roi === null → 'no_cost_data'.
9. Compute colorCode: roi > 0 → 'green', roi < 0 → 'red', roi === 0 || roi === null → 'amber'.
10. If groupBy='daily': group rows by date, compute per-day roi using same formula. Return time-series array.
11. If groupBy='country': group rows by country, compute per-country roi. Sort by netProfit descending.
12. Compute prior-period ROI for roiChange: shift dateFrom/dateTo back by the same number of days, filter already-fetched adStats data in memory (no second Firestore query). roiChange = (currentRoi !== null && priorRoi !== null) ? currentRoi - priorRoi : null.
13. Store result in node-cache with TTL of 5 minutes.
14. Return structured ROI payload.
SUCCESS (200): { dateFrom, dateTo, roi: number|null, roiIndicator: 'positive'|'negative'|'breakeven'|'no_cost_data', colorCode: 'green'|'red'|'amber', totalRevenue: number, totalCost: number, netProfit: number, roiChange: number|null, roiChangeDirection: 'up'|'down'|'flat'|null, groupBy: string, series?: [{ date|country: string, roi: number|null, revenue: number, cost: number, netProfit: number, colorCode: string }], cachedAt: string|null }
ERRORS:
- 400: missing/invalid dates, range exceeds 90 days, invalid networkId or groupBy
- 401: invalid/expired Firebase ID token
- 404: no data found for date range
- 500: Firestore read failure
SECURITY: userId always from verified token. All Firestore queries scoped to authenticated user's documents only. Cache keys namespaced by uid. No API keys or sensitive data in response.
