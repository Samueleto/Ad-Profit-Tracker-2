# Feature Roi Calculation Engine Endpoint Roi Thresholds Get

GET /api/roi/thresholds

## Description

REQUEST: GET — no query params. Bearer token in Authorization header. Returns the authenticated user's custom ROI threshold configuration used for color-coded indicator logic and alert triggers.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Read the users Firestore document for uid. Extract users.preferences.roiThresholds if present.
3. Merge with system defaults for any null/missing fields: { positiveThreshold: 0 (roi > 0 = green), warningThreshold: -10 (roi between -10 and 0 = amber), criticalThreshold: -10 (roi < -10 = red), targetRoi: null (optional user-defined target), alertOnNegative: false, alertOnTargetMiss: false }.
4. Return the effective thresholds with a flag indicating which fields are using defaults vs custom values.
SUCCESS (200): { thresholds: { positiveThreshold: number, warningThreshold: number, criticalThreshold: number, targetRoi: number|null, alertOnNegative: boolean, alertOnTargetMiss: boolean }, usingDefaults: string[] }
ERRORS:
- 401: unauthenticated
- 404: user document not found
- 500: Firestore read failure
SECURITY: Read-only. userId from verified token. Reads from users collection scoped to authenticated uid only.
