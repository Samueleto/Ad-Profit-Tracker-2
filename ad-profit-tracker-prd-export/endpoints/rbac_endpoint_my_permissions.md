# Rbac Endpoint My Permissions

GET /api/rbac/my-permissions

## Description

REQUEST: GET — no query params. Bearer token in Authorization header. Returns the authenticated user's own resolved effective permission set.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Read users Firestore document for uid. Extract users.workspaceRole, users.workspaceId, users.effectivePermissions, users.permissionsUpdatedAt.
3. If users.workspaceRole == 'owner', return all permissions as true without reading workspacePermissions — owner is always all-access.
4. If users.effectivePermissions is set and permissionsUpdatedAt is within the last 5 minutes, return cached effectivePermissions directly (fast path).
5. On cache miss or stale cache: read workspacePermissions document for workspaceId. Merge with system defaults. Resolve the user's role-specific permission map.
6. Update users.effectivePermissions and users.permissionsUpdatedAt via Admin SDK (background write — do not block response).
7. Return the resolved permission set with role context.
SUCCESS (200): { uid: string, workspaceRole: string, permissions: { canViewDashboard: boolean, canExportData: boolean, canManageApiKeys: boolean, canSyncNetworks: boolean, canViewReports: boolean, canCreateReports: boolean, canManageTeam: boolean, canViewAuditLogs: boolean, canManageBenchmarks: boolean }, isOwner: boolean, workspaceId: string|null, cachedAt: timestamp|null }
ERRORS:
- 401: invalid/expired Firebase ID token
- 404: user document not found
- 500: Firestore read failure
SECURITY: Users can only read their own permissions. uid always from verified token. Fast-path cache avoids unnecessary Firestore reads.
