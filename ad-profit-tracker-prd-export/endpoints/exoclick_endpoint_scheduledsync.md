# Exoclick Endpoint ScheduledSync

POST /api/networks/exoclick/scheduled-sync

## Description

REQUEST: POST — no body. Secured with a service-auth header (x-internal-secret matching an environment variable), NOT a user Firebase ID token. This endpoint is invoked by Cloud Scheduler (or a cron job) on a daily schedule.
PROCESS:
1. Validate x-internal-secret header matches the INTERNAL_SYNC_SECRET environment variable. Return 401 immediately if missing or mismatched — this endpoint is never called by end users.
2. Query networkConfigs Firestore collection (collection-group query or top-level collection) for all documents where networkId=='exoclick' AND isActive==true AND syncSchedule=='daily' (or 'hourly' if triggered hourly). This is a cross-user admin query using the Firebase Admin SDK.
3. For each matching networkConfig document, retrieve the userId. Fetch the encrypted API key from apiKeys where userId==userId AND networkId=='exoclick'. Skip users without a saved key — add to skipped list.
4. For each user with a valid key: decrypt the API key, call ExoClick Global Statistics API for yesterday's date with groupByCountry() and enableDetailedResults(). Parse and upsert results into adStats collection (same upsert logic as /api/networks/exoclick/sync). Update networkConfigs lastSyncedAt, lastSyncStatus.
5. Write an audit log entry per user via Admin SDK direct Firestore write (not HTTP call): action='manual_sync_triggered', resourceType='sync', resourceId='exoclick', metadata={ triggered: ['exoclick'], scheduledSync: true, latencyMs, rowsFetched }, status='success'|'failure'.
6. Return a summary of all users processed.
SUCCESS (200): { processed: number, succeeded: number, failed: number, skipped: number, date: string }
ERRORS:
- 401: missing or invalid internal secret
- 500: Firestore query failure or unhandled exception
SECURITY: Internal endpoint only — protected by x-internal-secret header, not user JWT. Never exposed to client. API keys decrypted server-side only. Rate limit not applicable (called by scheduler, not users).
