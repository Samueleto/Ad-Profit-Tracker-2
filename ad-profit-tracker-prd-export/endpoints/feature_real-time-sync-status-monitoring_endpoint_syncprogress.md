# Feature Real Time Sync Status Monitoring Endpoint Syncprogress

PATCH /api/sync/progress

## Description

REQUEST: PATCH { networkId: string (required), phase: string (required — one of 'fetching'|'writing'|'complete'|'idle'), rowsFetched?: number, totalRows?: number, errorMessage?: string }. This is an INTERNAL server-side endpoint called only by sync helper functions (lib/sync/syncNetworkForUser) to update the in-progress sync phase on the networkConfigs document. Bearer token in Authorization header for user-initiated syncs; x-internal-secret header for scheduled syncs.

PROCESS:
1. Check auth: if Authorization header present, verify Firebase ID token and derive uid. If x-internal-secret header present, validate against INTERNAL_SYNC_SECRET env var. Return 401 if neither is valid.
2. Validate networkId: must be one of ['exoclick','rollerads','zeydoo','propush'] — return 400 if not.
3. Validate phase: must be one of ['fetching','writing','complete','idle'] — return 400 if not.
4. If user-authenticated: verify the networkConfigs document for userId+networkId exists and belongs to the authenticated user.
5. If internal-secret authenticated: accept any userId passed in the request body (used by scheduled sync to update any user's config).
6. Update networkConfigs document: set currentSyncPhase=phase, currentSyncRowsFetched=rowsFetched||null, currentSyncTotalRows=totalRows||null, updatedAt=serverTimestamp(). If phase=='complete' or phase=='idle', also clear currentSyncPhase to null after a 5-second delay (handled by the caller, not this endpoint).
7. Do NOT write an audit log for phase updates — these are high-frequency internal state transitions, not user actions.
8. Return confirmation.

SUCCESS (200): { networkId: string, phase: string, updatedAt: string }

ERRORS:
- 400: invalid networkId or phase
- 401: neither valid Firebase ID token nor valid internal secret
- 404: networkConfigs document not found for this user+network
- 500: Firestore write failure

SECURITY: Dual auth — user JWT for manual syncs, internal secret for scheduled syncs. userId always from verified token (never from body) for user-authenticated calls. Rate limit not applicable — called by sync helpers, not directly by users.
