# Feature Per Network Analytics Tabs Endpoint Network Stats

GET /api/networks/stats

## Description

REQUEST: GET with query params: networkId (required — one of 'exoclick'|'rollerads'|'zeydoo'|'propush'), dateFrom (YYYY-MM-DD, required), dateTo (YYYY-MM-DD, required), groupBy? ('total'|'daily'|'country', default 'total').
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 immediately if invalid. uid from verified token only.
2. Validate networkId: required, must be one of the 4 supported networks — return 400 if not.
3. Validate dateFrom and dateTo: required, valid YYYY-MM-DD, dateFrom <= dateTo, max 90-day range — return 400 if invalid.
4. Check node-cache for key '{uid}_network_stats_{networkId}_{dateFrom}_{dateTo}_{groupBy}'. If cache hit, return with X-Cache: HIT header.
5. Query adStats Firestore collection: filter userId==uid AND networkId==networkId AND date>=dateFrom AND date<=dateTo. Use getDocs (one-time read).
6. If no documents found, return 404.
7. Determine dataRole: 'cost' for exoclick, 'revenue' for rollerads/zeydoo/propush.
8. If groupBy='total': aggregate all rows — sum cost (exoclick) or revenue (others), sum impressions, sum clicks, compute averageCtr (weighted), averageCpm (weighted), countryCount (unique countries), daysWithData (unique dates).
9. If groupBy='daily': group by date, compute per-day primary metric, impressions, clicks, ctr, cpm. Return time-series array sorted by date ASC.
10. If groupBy='country': group by country, sum primary metric, impressions, clicks, compute per-country ctr and cpm. Sort by primary metric descending. Enrich with ISO 3166-1 alpha-2 country name lookup. Compute metricShare (country value / total * 100).
11. Fetch networkConfigs document for userId+networkId to include lastSyncedAt, lastSyncStatus, circuitBreakerOpen in response.
12. Store result in node-cache with 5-minute TTL.
13. Return structured network stats payload.
SUCCESS (200): { networkId: string, dataRole: 'cost'|'revenue', dateFrom: string, dateTo: string, groupBy: string, summary: { primaryMetric: number, impressions: number, clicks: number, averageCtr: number, averageCpm: number, countryCount: number, daysWithData: number }, series?: [{ date: string, primaryMetric: number, impressions: number, clicks: number, ctr: number, cpm: number }], countries?: [{ country: string, countryName: string, primaryMetric: number, impressions: number, clicks: number, ctr: number, cpm: number, metricShare: number }], networkStatus: { lastSyncedAt: timestamp|null, lastSyncStatus: string, circuitBreakerOpen: boolean }, cachedAt: string|null }
ERRORS:
- 400: missing/invalid networkId, invalid date range, range exceeds 90 days
- 401: invalid/expired Firebase ID token
- 404: no data found for this network and date range
- 500: Firestore read failure
SECURITY: userId always from verified token. All Firestore queries scoped to authenticated user's documents only. Cache keys namespaced by uid. No API keys or sensitive data in response.
