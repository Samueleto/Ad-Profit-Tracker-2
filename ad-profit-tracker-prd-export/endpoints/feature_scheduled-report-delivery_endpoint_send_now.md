# Feature Scheduled Report Delivery Endpoint Send Now

POST /api/schedules/:id/send-now

## Description

REQUEST: POST /api/schedules/:id/send-now — :id is the schedule UUID in the URL path. No request body. Bearer token in Authorization header. PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only. 2. Validate :id: required, non-empty string — return 400 if missing. 3. Check rate limit: max 5 manual sends/hour per user — return 429 if exceeded (prevents email abuse). 4. Read users Firestore document for uid. Find the schedule with matching id in users.scheduledReports array. Return 404 if not found. 5. Resolve the dateFrom/dateTo for the schedule's dateRangePreset: 'yesterday' → yesterday's date, 'last_7' → last 7 days, 'last_30' → last 30 days, 'last_90' → last 90 days, 'this_month' → first day of current month to today. Compute using date-fns-tz in the schedule's timezone. 6. Invoke the core report generation logic: if format=='excel', call the same internal logic as POST /api/export/excel with the resolved dateFrom/dateTo and all sheets enabled; if format=='pdf', call the same internal logic as POST /api/export/pdf with all sections enabled. Generate the file buffer in memory. 7. Use a transactional email provider SDK (e.g., Nodemailer with SMTP, Resend, SendGrid, Postmark) configured via environment variables (EMAIL_PROVIDER_API_KEY, EMAIL_FROM_ADDRESS). Send email to schedule.deliveryEmail with subject 'Ad Profit Tracker Report — [reportName] — [dateFrom] to [dateTo]', body text explaining the attached report, and the generated file attached as [reportName]-[dateFrom]-[dateTo].[xlsx|pdf]. 8. Write audit log entry to auditLogs via Admin SDK: action='scheduled_report_delivered', resourceType='scheduledReport', resourceId=id, metadata={ reportName: schedule.reportName, deliveryEmail: schedule.deliveryEmail, format: schedule.format, dateFrom, dateTo, triggeredBy: 'user', fileSizeBytes: buffer.length }, status='success'. 9. Return delivery confirmation. SUCCESS (200): { sent: true, deliveryEmail: string, format: string, dateFrom: string, dateTo: string }. ERRORS: 400 (missing id), 401 (unauthenticated), 404 (schedule not found), 429 (rate limit exceeded — 5/hour), 502 (email provider delivery failure), 500 (report generation failure). SECURITY: userId from verified token. Schedule ownership verified before execution. deliveryEmail from stored schedule config — never from request body. Rate limited to prevent email abuse.
