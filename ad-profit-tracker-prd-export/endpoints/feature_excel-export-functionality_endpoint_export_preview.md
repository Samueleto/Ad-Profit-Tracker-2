# Feature Excel Export Functionality Endpoint Export Preview

GET /api/export/preview

## Description

REQUEST: GET with query params: dateFrom (YYYY-MM-DD, required), dateTo (YYYY-MM-DD, required). Bearer token in Authorization header. Returns estimated row counts per sheet for the export configuration modal — used to populate the row count badges on each sheet checkbox before the user initiates the export. PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only. 2. Validate dateFrom/dateTo: required, valid YYYY-MM-DD, dateFrom <= dateTo, max 90-day range — return 400 if invalid. 3. Check node-cache for key '{uid}_export_preview_{dateFrom}_{dateTo}'. If cache hit, return with X-Cache: HIT header. 4. Query adStats Firestore collection: filter userId==uid AND date>=dateFrom AND date<=dateTo. Use getDocs (one-time read). 5. Query auditLogs Firestore collection: filter userId==uid AND createdAt>=dateFrom AND createdAt<=dateTo. Use getDocs. 6. Compute row counts in memory: summary=1 (always one aggregate row), daily_trend=count of unique dates in adStats, geo_breakdown=count of unique countries in adStats, exoclick=count of adStats docs where networkId=='exoclick', rollerads=count where networkId=='rollerads', zeydoo=count where networkId=='zeydoo', propush=count where networkId=='propush', activity_log=count of auditLogs docs. 7. Compute totalRows (sum of all sheet row counts). 8. Store in node-cache with 3-minute TTL. 9. Return row counts per sheet. SUCCESS (200): { dateFrom, dateTo, sheets: { summary: number, daily_trend: number, geo_breakdown: number, exoclick: number, rollerads: number, zeydoo: number, propush: number, activity_log: number }, totalRows: number, hasData: boolean, cachedAt: string|null }. ERRORS: 400 (invalid params), 401 (unauthenticated), 500 (Firestore read failure). SECURITY: userId from verified token. Read-only. All queries scoped to authenticated user's data only.
