# Feature Automated Daily Data Sync Endpoint SchedulerHealth

GET /api/scheduled/health

## Description

REQUEST: GET — no body. Secured with x-internal-secret header. Used by Cloud Scheduler or monitoring systems to verify the sync infrastructure is healthy before triggering a sync.

PROCESS:
1. Validate x-internal-secret header against INTERNAL_SYNC_SECRET env var — return 401 if missing or mismatched.
2. Perform a lightweight Firestore Admin SDK read: fetch a single document from networkConfigs collection to verify database connectivity.
3. Check that all required environment variables are set: INTERNAL_SYNC_SECRET, ENCRYPTION_SECRET (for cryptr), and any network-specific API base URLs.
4. Return a health status object with component-level checks.

SUCCESS (200): { status: 'healthy', checks: { firestore: 'ok'|'error', encryptionSecret: 'set'|'missing', internalSecret: 'set'|'missing' }, timestamp: string }

ERRORS:
- 401: missing or invalid internal secret
- 503: one or more health checks failed (Firestore unreachable, missing env vars)

SECURITY: Internal endpoint only — x-internal-secret header required. Never exposed to client. Used by Cloud Scheduler health checks and monitoring dashboards.
