# Reconciliation Endpoint Rules Get

GET /api/reconciliation/rules

## Description

REQUEST: GET with optional query param: networkId? (one of 4 networks — omit for all).
PROCESS:
1. Verify Firebase ID token — return 401 if invalid. uid from verified token only.
2. Validate networkId if provided — return 400 if invalid.
3. Query networkConfigs Firestore collection: filter userId==uid. If networkId provided, also filter networkId==networkId. Use getDocs.
4. For each network document, extract validationRules map. Merge with system defaults for any null/missing fields: { maxDailyRevenueUSD: 50000, minDailyRevenueUSD: 0, maxDailyCostUSD: 50000, maxCtrPercent: 50, minImpressions: 0, allowNullCountry: true }.
5. Return per-network rules with effective values (merged defaults) and a flag indicating which fields are using defaults vs custom values.
SUCCESS (200): { networks: [{ networkId: string, dataRole: string, rules: { maxDailyRevenueUSD: number, minDailyRevenueUSD: number, maxDailyCostUSD: number, maxCtrPercent: number, minImpressions: number, allowNullCountry: boolean }, usingDefaults: string[] }], systemDefaults: object }
ERRORS:
- 400: invalid networkId
- 401: unauthenticated
- 500: Firestore read failure
SECURITY: Read-only. userId from verified token. Query scoped to authenticated user only.
