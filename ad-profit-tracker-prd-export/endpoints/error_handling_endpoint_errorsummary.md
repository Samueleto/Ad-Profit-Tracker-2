# Error Handling Endpoint ErrorSummary

GET /api/errors/summary

## Description

REQUEST: GET with optional query params: dateFrom? (YYYY-MM-DD, defaults to 7 days ago), dateTo? (YYYY-MM-DD, defaults to today). Bearer token in Authorization header. Returns an aggregated error summary across all networks for the authenticated user — used for dashboard health widgets and error monitoring panels.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid.
2. Validate dateFrom and dateTo if provided. Default dateFrom to 7 days ago, dateTo to today.
3. Query auditLogs Firestore collection: filter userId==uid AND resourceType=='sync' AND createdAt>=dateFrom AND createdAt<=dateTo. Use getDocs.
4. Separate into success and failure records. Group failures by resourceId (networkId).
5. For each network, compute: totalAttempts, successCount, failureCount, successRate (%), mostRecentError (errorMessage + createdAt), mostCommonErrorCode (from metadata.errorCode), averageLatencyMs (from metadata.latencyMs across successes).
6. Also compute cross-network totals: overallSuccessRate, totalFailures, networksWithOpenCircuitBreaker (query networkConfigs for circuitBreakerOpen==true).
7. Identify the most problematic network (highest failureCount in range).
8. Return the aggregated summary.
SUCCESS (200): { dateFrom: string, dateTo: string, overall: { totalAttempts: number, totalSuccesses: number, totalFailures: number, overallSuccessRate: number, networksWithOpenCircuit: string[] }, perNetwork: [{ networkId: string, totalAttempts: number, successCount: number, failureCount: number, successRate: number, mostRecentError: string|null, mostCommonErrorCode: string|null, averageLatencyMs: number|null }], mostProblematicNetwork: string|null }
ERRORS:
- 400: invalid date params
- 401: unauthenticated
- 500: Firestore read failure
SECURITY: userId from verified token. All queries scoped to authenticated user only.
