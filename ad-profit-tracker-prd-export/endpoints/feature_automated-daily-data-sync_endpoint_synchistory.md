# Feature Automated Daily Data Sync Endpoint SyncHistory

GET /api/scheduled/sync-history

## Description

REQUEST: GET with query params: networkId? (one of the 4 networks, optional — omit for all networks), limit? (integer 1–50, default 10), cursor? (Firestore document ID for pagination). Bearer token in Authorization header.

PROCESS:
1. Verify Firebase ID token — return 401 if invalid.
2. Validate query params: networkId must be one of ['exoclick','rollerads','zeydoo','propush'] if provided; limit must be integer 1–50.
3. Query auditLogs Firestore collection: filter userId==uid AND action=='manual_sync_triggered' AND resourceType=='sync'. If networkId provided, also filter resourceId==networkId. Order by createdAt DESC. Apply cursor-based pagination using startAfter if cursor provided. Limit to requested page size + 1 to detect hasMore.
4. Map documents to response shape: id, networkId (from resourceId), status, metadata (rowsFetched, latencyMs, scheduledSync flag, dateFrom, dateTo), createdAt.
5. Return paginated sync history.

SUCCESS (200): { history: [{ id: string, networkId: string, status: 'success'|'failure', rowsFetched: number|null, latencyMs: number|null, scheduledSync: boolean, date: string|null, createdAt: timestamp }], hasMore: boolean, nextCursor: string|null }

ERRORS:
- 400: invalid networkId or limit param
- 401: unauthenticated
- 500: Firestore read failure

SECURITY: userId always from verified token. Query scoped to authenticated user's audit logs only.
