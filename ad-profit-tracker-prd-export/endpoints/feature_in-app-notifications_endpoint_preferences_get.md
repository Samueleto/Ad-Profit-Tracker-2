# Feature In App Notifications Endpoint Preferences Get

GET /api/notifications/preferences

## Description

REQUEST: GET — no query params. Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Read users Firestore document for uid. Extract users.notificationPreferences.
3. Merge with system defaults for any missing type keys: all 9 notification types default to { enabled: true }.
4. Return the effective preferences with a flag indicating which are using defaults.
SUCCESS (200): { preferences: [{ type: string, enabled: boolean, isDefault: boolean }] }
ERRORS:
- 401: unauthenticated
- 404: users document not found
- 500: Firestore read failure
SECURITY: Read-only. userId from verified token. Reads only authenticated user's users document.
