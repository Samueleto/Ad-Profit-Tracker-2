# Error Handling Endpoint RetryConfig

GET /api/errors/retry-config

## Description

REQUEST: GET — no query params. Bearer token in Authorization header. Returns the current retry configuration for all four networks for the authenticated user, including retryAttempts, timeoutSeconds, and computed backoff schedule.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid.
2. Query networkConfigs Firestore collection: filter userId==uid. Use getDocs.
3. For each network document, extract retryAttempts, timeoutSeconds, retryCount, nextRetryAt, circuitBreakerOpen.
4. Compute the full backoff schedule for each network: for attempt 1 through retryAttempts, compute delay = min(baseDelayMs * 2^(attempt-1), maxDelayMs) where baseDelayMs=1000 and maxDelayMs=30000. Add jitter range (±20% of computed delay). Return as an array of { attempt: number, delayMs: number, jitterRangeMs: [min, max] }.
5. Return the retry configuration with computed backoff schedules.
SUCCESS (200): { networks: [{ networkId: string, retryAttempts: number, timeoutSeconds: number, currentRetryCount: number, nextRetryAt: timestamp|null, circuitBreakerOpen: boolean, backoffSchedule: [{ attempt: number, delayMs: number, jitterRangeMs: [number, number] }] }] }
ERRORS:
- 401: unauthenticated
- 500: Firestore read failure
SECURITY: userId from verified token. Read-only endpoint — no writes.
