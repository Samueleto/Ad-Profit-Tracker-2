# Feature Automated Daily Data Sync Endpoint SyncStatus

GET /api/scheduled/sync-status

## Description

REQUEST: GET — no body. Bearer token in Authorization header. Returns the current sync status for all four networks for the authenticated user, including last sync time, status, error messages, and whether a sync is currently in progress.

PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid.
2. Query networkConfigs Firestore collection for all documents where userId==uid. Use getDocs (one-time read).
3. For each of the four networks, extract lastSyncedAt, lastSyncStatus, lastSyncError, isActive, syncSchedule from the corresponding networkConfigs document. If a document doesn't exist for a network, return default values (lastSyncedAt=null, lastSyncStatus='never', lastSyncError=null).
4. Also query adStats collection for each network: filter userId==uid AND networkId==network, order by date DESC, limit 1 — to get the most recent data date available.
5. Compute a 'nextScheduledSync' estimate based on syncSchedule field and lastSyncedAt (e.g., if daily and lastSyncedAt was today at 02:00 UTC, next is tomorrow at 02:00 UTC).
6. Return a consolidated status object for all four networks.

SUCCESS (200): { networks: [{ networkId: string, isActive: boolean, syncSchedule: string, lastSyncedAt: timestamp|null, lastSyncStatus: 'success'|'failed'|'partial'|'never', lastSyncError: string|null, latestDataDate: string|null, nextScheduledSync: string|null }], lastMasterSyncAt: timestamp|null }

ERRORS:
- 401: invalid/expired Firebase ID token
- 500: Firestore read failure

SECURITY: userId always from verified token. Query scoped to authenticated user's documents only.
