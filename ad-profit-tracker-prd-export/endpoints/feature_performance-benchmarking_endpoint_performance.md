# Feature Performance Benchmarking Endpoint Performance

GET /api/benchmarks/performance

## Description

REQUEST: GET with query params: dateFrom (YYYY-MM-DD, required), dateTo (YYYY-MM-DD, required), metric? ('roi'|'ctr'|'cpm'|'revenue'|'cost'|'impressions'|'clicks', default 'roi'), networkId? (one of 4 networks, optional — omit for cross-network aggregate). PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only. 2. Validate dateFrom/dateTo: required, valid YYYY-MM-DD, dateFrom <= dateTo, max 90-day range. Validate metric and networkId if provided. Return 400 if invalid. 3. Check node-cache for key '{uid}_benchmarks_performance_{dateFrom}_{dateTo}_{metric}_{networkId||all}'. If cache hit, return with X-Cache: HIT header. 4. On cache miss, execute two parallel Firestore queries via Promise.all(): (a) Query adStats collection: filter userId==uid AND date>=dateFrom AND date<=dateTo. If networkId provided, also filter networkId==networkId. Use getDocs. (b) Query adStats collection for historical window: filter userId==uid AND date>=(dateFrom minus 90 days) AND date<dateTo. Use getDocs. (c) Read benchmarks document for userId==uid to get customTarget for the requested metric. 5. From current-period adStats: compute actual metric value using null-safe aggregation — totalRevenue (rollerads+zeydoo+propush), totalCost (exoclick), netProfit, roi=(netProfit/totalCost*100 if totalCost>0 else null), averageCtr (weighted), averageCpm (weighted), totalImpressions, totalClicks. 6. From historical adStats: compute historical average for the same metric over the 90-day window prior to dateFrom. Group by date, compute per-day metric, then average across all days with data. Return null if fewer than 3 days of historical data exist (insufficient baseline). 7. Load industry benchmark target: if benchmarks.metricTargets[metric].useDefault==false and customTarget is set, use customTarget. Otherwise use server-side system defaults: { roi: 50, ctr: 2.5, cpm: 1.5, revenue: 1000, cost: 500, impressions: 100000, clicks: 2500 }. 8. Compute benchmark gap: actual - industryTarget. Compute historical gap: actual - historicalAverage. Compute performanceRatio: (actual / industryTarget * 100) if industryTarget > 0 else null. Compute trend direction: 'above'|'below'|'at' benchmark. 9. Store in node-cache with 5-minute TTL. 10. Return structured benchmark comparison payload. SUCCESS (200): { dateFrom, dateTo, metric, networkId: string|null, actual: { value: number|null, unit: string }, historicalAverage: { value: number|null, daysUsed: number, windowDays: number }, industryBenchmark: { value: number, isCustom: boolean, source: 'custom'|'system_default' }, gaps: { vsHistorical: number|null, vsIndustry: number|null }, performanceRatio: number|null, trend: 'above'|'below'|'at'|'no_data', cachedAt: string|null } ERRORS: - 400: missing/invalid dates, range exceeds 90 days, invalid metric or networkId - 401: unauthenticated - 404: no adStats data found for date range - 500: Firestore read failure SECURITY: userId always from verified token. All Firestore queries scoped to authenticated user's documents only. Cache keys namespaced by uid. No API keys or sensitive data in response.
