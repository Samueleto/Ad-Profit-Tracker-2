# Feature User Settings Preferences Endpoint Get Preferences

GET /api/settings/preferences

## Description

REQUEST: GET — no body. Bearer token in Authorization header.
PROCESS: 1. Extract Bearer token from Authorization header. 2. Verify Firebase ID token via Firebase Admin SDK (admin.auth().verifyIdToken(token)) — return 401 immediately if invalid. 3. Fetch users/{uid} document from Firestore. 4. Extract the preferences sub-object from the document. 5. If preferences is undefined/null (first-time user), return default values object. 6. Return the preferences object.
SUCCESS (200): { timezone: string, currency: string, defaultDateRange: string, notifications: { dailySummaryEmail: boolean, weeklyReportEmail: boolean }, updatedAt: timestamp | null }
ERRORS: 401 (invalid or expired Firebase ID token), 404 (users document not found — should not happen in normal flow, trigger sync-user), 500 (Firestore read failure)
SECURITY: Firebase Admin SDK token verification as step 1. Only returns the authenticated user's own preferences. Never returns other users' data — UID is always derived from the verified token, never from a request parameter.
