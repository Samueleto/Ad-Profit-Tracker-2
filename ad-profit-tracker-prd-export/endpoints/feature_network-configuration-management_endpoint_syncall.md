# Feature Network Configuration Management Endpoint SyncAll

POST /api/networks/sync-all

## Description

REQUEST: POST — no body required. Bearer token in Authorization header. PROCESS: 1. Verify Firebase ID token — return 401 if invalid. 2. Query networkConfigs collection for this user where isActive == true. 3. For each active network, verify an API key exists in users/{uid}/apiKeys/{networkId} — skip networks without keys and include them in the 'skipped' response array. 4. Enqueue a Cloud Function task (or invoke directly if within timeout) for each active network with a key: decrypt key, call network API for yesterday's date range, write results to adStats collection (used by dashboard feature). 5. Update lastSyncedAt and lastSyncStatus on each networkConfigs document after each network completes. 6. Return a summary of triggered, skipped, and failed networks. SUCCESS (200): { triggered: string[], skipped: string[], failed: string[], triggeredAt: timestamp } ERRORS: 401 (unauthenticated), 429 (rate limited — prevent accidental spam), 500 (Firestore or task queue error). SECURITY: Must be server-side Cloud Function. API keys decrypted server-side only. Rate limited to 3/hour per user to prevent excessive API calls to ad networks.
