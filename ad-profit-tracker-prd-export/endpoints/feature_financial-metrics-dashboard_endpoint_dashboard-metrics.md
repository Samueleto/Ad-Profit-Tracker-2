# Feature Financial Metrics Dashboard Endpoint Dashboard Metrics

GET /api/dashboard/metrics

## Description

REQUEST: GET with query params: dateFrom (YYYY-MM-DD, required), dateTo (YYYY-MM-DD, required). A convenience aggregation endpoint that fans out to the core stats logic and returns all data needed for the Financial Metrics Dashboard in a single round-trip, reducing client-side waterfall fetches.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 immediately if invalid. uid from verified token only.
2. Validate dateFrom and dateTo: required, valid YYYY-MM-DD, dateFrom <= dateTo, max 90-day range — return 400 if invalid.
3. Check server-side in-memory cache (node-cache) for key '{uid}_dashboard_metrics_{dateFrom}_{dateTo}'. If cache hit, return cached result immediately with a 'X-Cache: HIT' response header.
4. On cache miss, execute three parallel Firestore queries using Promise.all():
   a. Query adStats collection: filter userId==uid AND date>=dateFrom AND date<=dateTo. Use getDocs.
   b. Query networkConfigs collection: filter userId==uid. Use getDocs.
5. From adStats results, compute in memory:
   - totalRevenue: sum of revenue from rollerads+zeydoo+propush rows
   - totalCost: sum of cost from exoclick rows
   - netProfit: totalRevenue - totalCost
   - roi: (netProfit / totalCost * 100) if totalCost > 0 else null
   - dailySeries: group by date, compute per-day profit for trend chart
   - topCountries: group by country, sum netProfit, sort desc, slice top 10
   - perNetwork: per-network primary metric subtotals
6. From networkConfigs results, extract lastSyncedAt and lastSyncStatus per network.
7. Compute period-over-period change: if dateRange >= 7 days, compute the same metrics for the prior equivalent period (shift dateFrom/dateTo back by the same number of days) using the already-fetched adStats data filtered by date. Return percentChange for each KPI.
8. Store result in cache with TTL of 5 minutes.
9. Return consolidated dashboard payload.
SUCCESS (200): { dateFrom, dateTo, kpis: { totalRevenue, totalCost, netProfit, roi, revenueChange, costChange, profitChange, roiChange }, dailySeries: [{ date, revenue, cost, netProfit, roi }], topCountries: [{ country, countryName, revenue, cost, netProfit, metricShare }], perNetwork: [{ networkId, dataRole, primaryMetric, lastSyncedAt, lastSyncStatus }], cachedAt: string|null }
ERRORS:
- 400: missing/invalid dates, range exceeds 90 days
- 401: invalid/expired Firebase ID token
- 404: no data found for date range
- 500: Firestore read failure
SECURITY: userId always from verified token. All Firestore queries scoped to authenticated user's documents only. Cache keys namespaced by uid. No API keys or sensitive data in response.
