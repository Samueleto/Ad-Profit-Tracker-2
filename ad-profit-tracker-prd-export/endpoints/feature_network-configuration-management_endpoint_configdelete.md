# Feature Network Configuration Management Endpoint ConfigDelete

DELETE /api/networks/config/reset

## Description

REQUEST: DELETE { networkId: string } — resets a single network config back to defaults. Bearer token in Authorization header. PROCESS: 1. Verify Firebase ID token — return 401 if invalid. 2. Validate networkId — return 400 if not one of the 4 supported networks. 3. Overwrite the networkConfigs document for this user+networkId with all default values (isActive=true, syncSchedule='daily', timeoutSeconds=30, retryAttempts=3, endpointOverride=null, displayOrder=default index). Preserve lastSyncedAt, lastSyncStatus, lastSyncError (these are written by Cloud Functions, not user-configurable). Set updatedAt=serverTimestamp(). 4. Return the reset config. SUCCESS (200): { networkId, status: 'reset', config: { isActive, syncSchedule, timeoutSeconds, retryAttempts, endpointOverride, displayOrder } } ERRORS: 400 (invalid networkId), 401 (unauthenticated), 404 (config document not found — nothing to reset), 500 (Firestore write failure). SECURITY: UID from verified token. Only resets user-configurable fields — sync status fields written by Cloud Functions are preserved.
