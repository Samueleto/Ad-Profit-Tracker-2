# Rate Limit Endpoint Violations

GET /api/rate-limits/violations

## Description

REQUEST: GET with query params: networkId? (optional), endpoint? (optional — filter by endpoint path), limit? (integer 1–50, default 20), cursor? (Firestore document ID for pagination). Bearer token in Authorization header. Returns paginated rate limit violation history from auditLogs filtered to rate_limit events.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate networkId if provided — return 400 if not one of the 4 supported networks. Validate limit (1–50).
3. Query auditLogs Firestore collection: filter userId==uid AND resourceType=='rate_limit' AND action=='rate_limit_exceeded'. If networkId provided, filter resourceId==networkId. Order by createdAt DESC. Apply cursor-based pagination using startAfter if cursor provided. Fetch limit+1 to detect hasMore.
4. If endpoint filter provided, apply in-memory filtering on metadata.endpoint field.
5. Map documents to response shape: id, networkId (from resourceId), endpoint (from metadata.endpoint), limitType (from metadata.limitType: 'outbound'|'user-quota'), limitValue (from metadata.limitValue), windowSeconds (from metadata.windowSeconds), retryAfterSeconds (from metadata.retryAfterSeconds), createdAt.
6. Return paginated violation log.
SUCCESS (200): { violations: [{ id: string, networkId: string|null, endpoint: string, limitType: string, limitValue: number, windowSeconds: number, retryAfterSeconds: number|null, createdAt: timestamp }], hasMore: boolean, nextCursor: string|null }
ERRORS:
- 400: invalid networkId or limit
- 401: unauthenticated
- 500: Firestore read failure
SECURITY: userId from verified token. Query scoped to authenticated user's audit logs only. No API keys or sensitive data in response.
