# Feature In App Notifications Endpoint Dismiss

PATCH /api/notifications/dismiss/:id

## Description

REQUEST: PATCH /api/notifications/dismiss/:id — notification id in URL path. No request body. Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate :id: required, non-empty string — return 400 if missing.
3. Check rate limit: max 60 dismissals/hour per user — return 429 if exceeded.
4. Read users Firestore document for uid. Find notification with matching id in users.notifications array. Return 404 if not found or already dismissed.
5. Build updated notification object with isDismissed=true, isRead=true.
6. Use Firestore Admin SDK transaction: arrayRemove old notification object, arrayUnion updated notification object. If the notification was previously unread (isRead=false), decrement users.unreadNotificationCount by 1 (floor at 0). Set users.notificationsUpdatedAt=serverTimestamp().
7. Return confirmation.
SUCCESS (200): { dismissed: true, notificationId: string, unreadCount: number }
ERRORS:
- 400: missing notification id
- 401: unauthenticated
- 404: notification not found or already dismissed
- 429: rate limit exceeded (60/hour)
- 500: Firestore transaction failure
SECURITY: userId from verified token. Notification ownership verified by reading users document scoped to uid before mutation.
