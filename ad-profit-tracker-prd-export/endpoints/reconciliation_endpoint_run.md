# Reconciliation Endpoint Run

POST /api/reconciliation/run

## Description

REQUEST: POST { networkId?: string (one of 4 networks, optional — omit for all active), dateFrom?: string (YYYY-MM-DD, defaults to yesterday), dateTo?: string (YYYY-MM-DD, defaults to yesterday) }
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate networkId if provided (must be one of ['exoclick','rollerads','zeydoo','propush']). Validate dateFrom/dateTo (max 90-day range). Return 400 if invalid.
3. Check rate limit: max 10 reconciliation runs/hour per user. Return 429 if exceeded.
4. Determine target networks: if networkId provided, use [networkId]; otherwise query networkConfigs for userId==uid AND isActive==true.
5. For each target network, query adStats Firestore collection: filter userId==uid AND networkId==network AND date>=dateFrom AND date<=dateTo. Use getDocs.
6. Load per-network validation rules from networkConfigs.validationRules (fall back to system defaults if null): maxDailyRevenueUSD (default 50000), minDailyRevenueUSD (default 0), maxDailyCostUSD (default 50000), maxCtrPercent (default 50), minImpressions (default 0), allowNullCountry (default true).
7. For each adStats document, run validation checks: (a) Revenue/cost range check — flag if revenue or cost exceeds maxDailyRevenue/maxDailyCost or is negative. Severity: critical if >10x threshold, warning otherwise. (b) CTR sanity check — flag if ctr > maxCtrPercent/100. (c) Null/missing field check — flag if required fields (date, networkId, userId) are null. (d) Duplicate detection — within the fetched set, flag if two documents share the same userId+networkId+date+country combination. (e) Cross-network consistency — for dates where all 4 networks have data, verify revenue networks have non-null revenue and cost network has non-null cost. Flag mismatches as 'missing' severity warning. (f) Stale data check — if syncedAt is older than 48 hours for a 'daily' schedule network, flag as 'stale'.
8. For each document, build anomalyFlags array from all triggered checks. Set validationStatus: 'valid' if no flags, 'anomaly' if any flags, 'stale' if only stale flag. Write validationStatus, anomalyFlags, validationCheckedAt=serverTimestamp() back to each adStats document using Firestore Admin SDK batch writes in chunks of 500.
9. Update networkConfigs document for each network: lastValidationStatus='clean'|'anomalies_detected'|'missing_data', lastValidationAt=serverTimestamp(), anomalyCount=count of documents with validationStatus!='valid'.
10. If any anomalies detected, write audit log entry to auditLogs via Admin SDK: action='manual_sync_triggered', resourceType='reconciliation', resourceId=networkId, metadata={ anomaliesFound: number, dateFrom, dateTo, anomalyTypes: string[], triggeredBy: 'user' }, status='success'.
11. Map any validation failure types to existing semantic error codes: schema/parse failures → lastErrorCode='PARSE_ERROR' on networkConfigs.
12. Return reconciliation summary.
SUCCESS (200): { networksChecked: string[], dateFrom: string, dateTo: string, totalDocumentsChecked: number, totalAnomaliesFound: number, results: [{ networkId: string, documentsChecked: number, anomaliesFound: number, anomalyTypes: string[], lastValidationStatus: string }], durationMs: number }
ERRORS:
- 400: invalid networkId, invalid date range, range exceeds 90 days
- 401: invalid/expired Firebase ID token
- 429: rate limit exceeded (10/hour per user)
- 500: Firestore read/write failure
SECURITY: userId always from verified token. All Firestore reads and writes scoped to authenticated uid. Anomaly flags never contain raw API keys or encrypted values.
