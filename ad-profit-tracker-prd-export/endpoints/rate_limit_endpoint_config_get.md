# Rate Limit Endpoint Config Get

GET /api/rate-limits/config

## Description

REQUEST: GET — no query params. Bearer token in Authorization header. Returns the current rate limit configuration for all four ad network outbound API limiters and all user-facing endpoint quotas. This is a read-only view of the server-side Bottleneck configuration and sliding window settings.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid.
2. Read the static Bottleneck configuration from lib/rate-limit/limiterConfig.ts for each of the 4 networks: maxConcurrent, minTime (ms between requests), reservoir (max requests per window), reservoirRefreshInterval (window size in ms), reservoirRefreshAmount (tokens added per refresh).
3. Read the user-facing endpoint quota configuration from lib/rate-limit/userLimiterConfig.ts: endpoint path, limit, windowSeconds for each rate-limited endpoint.
4. Compute human-readable descriptions: e.g., 'ExoClick: max 10 requests/minute, 1 concurrent' from the Bottleneck config values.
5. Return the full configuration object. This is static config — no Firestore reads needed.
SUCCESS (200): { networkLimiters: [{ networkId: string, maxConcurrent: number, minTimeMs: number, reservoir: number, reservoirRefreshIntervalMs: number, reservoirRefreshAmount: number, description: string }], userQuotaConfig: [{ endpoint: string, limit: number, windowSeconds: number, description: string }] }
ERRORS:
- 401: unauthenticated
- 500: config read failure
SECURITY: Read-only. No sensitive data — config values are not secrets. userId from verified token (auth required to prevent config enumeration by unauthenticated clients).
