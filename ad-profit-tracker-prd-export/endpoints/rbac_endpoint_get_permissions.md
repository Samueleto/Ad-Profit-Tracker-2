# Rbac Endpoint Get Permissions

GET /api/rbac/permissions

## Description

REQUEST: GET — no query params. Bearer token in Authorization header. Returns the full role permission matrix for the authenticated user's workspace, merged with system defaults for any unconfigured roles.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Read users Firestore document for uid. Extract users.workspaceId. Return 404 if no workspace.
3. Query workspacePermissions Firestore collection: getDocs with filter workspaceId==users.workspaceId. One-time read.
4. If no document found, return system defaults for all roles with isCustomized=false — do NOT return 404.
5. Merge found document's rolePermissions with system defaults: for each role (admin/member/viewer/analyst), if a permission key is missing, use system default value.
6. System defaults: admin gets all permissions true; member gets canViewDashboard, canViewReports, canSyncNetworks true, others false; viewer gets canViewDashboard, canViewReports true only; analyst gets canViewDashboard, canViewReports, canCreateReports, canManageBenchmarks true.
7. Owner permissions are always all-true — computed inline, not stored.
8. Return full matrix with per-role permission maps and customizedRoles array.
SUCCESS (200): { workspaceId: string, roles: { owner: { [permKey]: boolean }, admin: { [permKey]: boolean }, member: { [permKey]: boolean }, viewer: { [permKey]: boolean }, analyst: { [permKey]: boolean } }, customizedRoles: string[], systemDefaults: { [role]: { [permKey]: boolean } }, updatedAt: timestamp|null }
ERRORS:
- 401: invalid/expired Firebase ID token
- 404: user has no workspace
- 500: Firestore read failure
SECURITY: userId from verified token. Read access for all authenticated workspace members — any member can view the permission matrix. workspaceId derived from users document, never from request.
