# Multi User Endpoint Update Workspace

PATCH /api/team/workspace

## Description

REQUEST: PATCH { workspaceName?: string (1-100 chars, required) }. Bearer token in Authorization header. Updates workspace metadata — currently only the workspace name.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Read users Firestore document for uid. Check users.workspaceRole — must be 'owner' to rename workspace. Return 403 if not owner.
3. Validate workspaceName: 1-100 chars, non-empty after trim — return 400 if invalid.
4. Check rate limit: max 10 workspace updates/hour per user — return 429 if exceeded.
5. Query users collection where workspaceId == uid (all workspace members). Use a Firestore batch write to update workspaceName on all member documents (denormalized field). Also update users.workspaceName on the owner's document.
6. Write audit log to auditLogs: action='workspace_updated', resourceType='workspace', resourceId=uid, metadata={ changedFields: { workspaceName: { from: oldName, to: newName } } }, status='success'.
7. Return updated workspace.
SUCCESS (200): { workspaceId: string, workspaceName: string, updatedAt: timestamp }
ERRORS:
- 400: invalid workspaceName
- 401: unauthenticated
- 403: not workspace owner
- 429: rate limit exceeded
- 500: Firestore write failure
SECURITY: Only workspace owner can rename. Batch write updates all member documents atomically.
