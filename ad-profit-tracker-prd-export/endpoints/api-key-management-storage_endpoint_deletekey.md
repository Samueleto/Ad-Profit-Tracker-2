# Api Key Management Storage Endpoint DeleteKey

DELETE /api/keys/delete

## Description

REQUEST: DELETE { networkId: string }. PROCESS: 1. Verify the user's Firebase ID token to get their UID. 2. Validate the `networkId` from the request body. 3. Delete the Firestore document at `users/{uid}/apiKeys/{networkId}`. 4. Return a success response indicating the key was disconnected. SUCCESS (200): { networkId, status: 'disconnected' }. ERRORS: 400 (invalid networkId), 401 (unauthenticated), 404 (key not found), 500 (Firestore delete error). SECURITY: User can only delete keys from their own subcollection, enforced by using the UID from the verified token.
