# Feature Performance Benchmarking Endpoint Settings Get

GET /api/benchmarks/settings

## Description

REQUEST: GET — no query params. Bearer token in Authorization header. Returns the authenticated user's custom benchmark target configuration, merged with system defaults for any unset metrics. PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only. 2. Query benchmarks Firestore collection: filter userId==uid. Use getDocs (one-time read). 3. If no document found, return system defaults for all metrics with isCustom=false for each — do NOT return 404, as the absence of a document is a valid state (user has never saved custom settings). 4. Merge found document's metricTargets with system defaults: for each of the 7 metrics, if metricTargets[metric].useDefault==true or customTarget is null, use system default value. 5. Compute effectiveTarget per metric (the value that will actually be used in benchmark comparisons). 6. Include historicalWindowDays (default 90 if not set). 7. Return the full settings object with effective values and per-metric isCustom flags. SUCCESS (200): { historicalWindowDays: number, metricTargets: [{ metric: string, customTarget: number|null, useDefault: boolean, effectiveTarget: number, systemDefault: number, unit: string, isCustom: boolean, updatedAt: timestamp|null }], lastUpdatedAt: timestamp|null } ERRORS: - 401: unauthenticated - 500: Firestore read failure SECURITY: Read-only. userId from verified token. Query scoped to authenticated user's benchmarks document only.
