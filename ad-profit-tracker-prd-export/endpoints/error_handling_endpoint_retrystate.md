# Error Handling Endpoint RetryState

GET /api/errors/retry-state

## Description

REQUEST: GET with optional query param: networkId? (one of 'exoclick'|'rollerads'|'zeydoo'|'propush' — omit for all networks). Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate networkId if provided — return 400 if not one of the 4 supported networks.
3. Query networkConfigs Firestore collection: filter userId==uid. If networkId provided, also filter networkId==networkId. Use getDocs (one-time read).
4. For each document, extract retry state fields: retryCount, nextRetryAt, circuitBreakerOpen, circuitBreakerOpenedAt, lastErrorCode, lastSyncStatus, lastSyncError, retryAttempts, totalFailureCount.
5. Compute derived fields: isRetryable (lastErrorCode in ['NETWORK_TIMEOUT','RATE_LIMITED','PARSE_ERROR'] and circuitBreakerOpen==false), timeUntilNextRetry (nextRetryAt - now in seconds, null if no pending retry), circuitBreakerAutoResetAt (circuitBreakerOpenedAt + 3600s if open).
6. Return per-network retry state array.
SUCCESS (200): { networks: [{ networkId: string, retryCount: number, nextRetryAt: timestamp|null, timeUntilNextRetrySeconds: number|null, circuitBreakerOpen: boolean, circuitBreakerOpenedAt: timestamp|null, circuitBreakerAutoResetAt: timestamp|null, lastErrorCode: string|null, lastSyncStatus: string, lastSyncError: string|null, retryAttempts: number, totalFailureCount: number, isRetryable: boolean }] }
ERRORS:
- 400: invalid networkId
- 401: unauthenticated
- 500: Firestore read failure
SECURITY: userId always from verified token. Query scoped to authenticated user's networkConfigs only.
