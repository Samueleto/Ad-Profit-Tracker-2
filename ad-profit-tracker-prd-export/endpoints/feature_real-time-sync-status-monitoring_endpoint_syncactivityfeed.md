# Feature Real Time Sync Status Monitoring Endpoint Syncactivityfeed

GET /api/sync/activity-feed

## Description

REQUEST: GET — query params: limit? (integer 1–20, default 10), networkId? (optional filter), cursor? (Firestore document ID for pagination). Bearer token in Authorization header.

PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate limit (1–20) and networkId if provided. Return 400 if invalid.
3. Query auditLogs Firestore collection: filter userId==uid AND resourceType=='sync' AND action=='manual_sync_triggered'. If networkId provided, also filter resourceId==networkId. Order by createdAt DESC. Apply cursor-based pagination using startAfter if cursor provided. Fetch limit+1 to detect hasMore.
4. For each audit log document, extract: id, resourceId (networkId), status, metadata.rowsFetched, metadata.latencyMs, metadata.scheduledSync, metadata.dateFrom, metadata.dateTo, metadata.triggeredBy, createdAt.
5. Enrich each record with a human-readable eventLabel: 'Scheduled Sync' if metadata.scheduledSync==true, 'Manual Sync' if triggeredBy=='user', 'Backfill' if triggeredBy=='backfill', 'Retry' if triggeredBy=='retry'.
6. Return paginated activity feed.

SUCCESS (200): { feed: [{ id: string, networkId: string, status: 'success'|'failure', eventLabel: string, rowsFetched: number|null, latencyMs: number|null, dateFrom: string|null, dateTo: string|null, createdAt: timestamp }], hasMore: boolean, nextCursor: string|null }

ERRORS:
- 400: invalid networkId or limit
- 401: unauthenticated
- 500: Firestore read failure

SECURITY: userId from verified token. Query scoped to authenticated user's audit logs only. Read-only.
