# Feature Historical Data Storage Endpoint DeleteSnapshot

DELETE /api/stats/snapshot

## Description

REQUEST: DELETE with query params: date (YYYY-MM-DD, required), networkId? (optional — omit to delete all networks for this date). Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 immediately if invalid. uid from verified token only.
2. Validate date: required, valid YYYY-MM-DD. Validate networkId if provided. Return 400 if invalid.
3. Check rate limit: max 5 deletes/hour per user — return 429 if exceeded. Deletions are destructive and irreversible.
4. Query adStats Firestore collection: filter userId==uid AND date==date. If networkId provided, also filter networkId==networkId. Use getDocs to find matching documents.
5. If no documents found, return 404.
6. Use Firestore Admin SDK batch delete in chunks of 500 to delete all matching documents.
7. Write an audit log entry directly to auditLogs Firestore collection via Admin SDK: action='manual_sync_triggered' (closest existing enum — note: consider adding 'snapshot_deleted' to the action enum in a future iteration), resourceType='sync', resourceId=networkId||'all', metadata={ deletedCount, date, networkId: networkId||'all', triggeredBy: 'user' }, status='success'.
8. Return deletion summary.
SUCCESS (200): { deleted: true, deletedCount: number, date: string, networkId: string|null }
ERRORS:
- 400: missing/invalid date, invalid networkId
- 401: unauthenticated
- 404: no data found for this date/network combination
- 429: rate limit exceeded (5/hour — destructive operation)
- 500: batch delete failure
SECURITY: userId always from verified token. Batch delete scoped to authenticated user's documents only — userId==uid filter is mandatory. Rate limited to prevent accidental repeated purges.
