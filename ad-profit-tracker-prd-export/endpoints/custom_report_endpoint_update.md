# Custom Report Endpoint Update

PATCH /api/reports/:id

## Description

REQUEST: PATCH { name?: string (1-100 chars), metrics?: string[] (non-empty subset of 8 allowed), networks?: string[] (non-empty subset of 4 networks), countries?: string[] (max 50 ISO codes), groupBy?: string ('daily'|'country'|'network'), dataQuality?: string ('all'|'anomalies'|'clean'), dateRangePreset?: string (one of 6 allowed), dateFrom?: string|null (YYYY-MM-DD), dateTo?: string|null (YYYY-MM-DD) }. At least one field required. Bearer token in Authorization header. :id is the report UUID in the URL path.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate :id: required, non-empty string — return 400 if missing.
3. Validate all provided fields using same rules as POST /api/reports/save — return 400 if any field is invalid.
4. At least one field must be provided — return 400 if body is empty.
5. Check rate limit: max 30 report updates/hour per user — return 429 if exceeded.
6. Read users Firestore document for uid. Find the report with matching id in users.savedReports array. Return 404 if not found.
7. Build updated report object by merging provided fields into the existing report object. Set updatedAt=serverTimestamp().
8. Use Firestore Admin SDK to update the users document: use arrayRemove to remove the old report object, then arrayUnion to add the updated report object. Set users.updatedAt=serverTimestamp(). Execute as a Firestore transaction to prevent race conditions.
9. Write audit log entry to auditLogs Firestore collection via Admin SDK: action='preferences_updated', resourceType='report', resourceId=id, metadata={ reportName: updatedName, changedFields: Object.keys(body) }, status='success'.
10. Return the full updated report object.
SUCCESS (200): { report: { id, name, metrics, networks, countries, groupBy, dataQuality, dateRangePreset, dateFrom, dateTo, createdAt, updatedAt } }
ERRORS:
- 400: missing id, invalid field values, empty body
- 401: invalid/expired Firebase ID token
- 404: report not found in user's savedReports
- 429: rate limit exceeded (30/hour)
- 500: Firestore transaction failure
SECURITY: userId from verified token. Report ownership verified by reading users document scoped to uid before update. Audit logged.
