# Cloudrun Endpoint Deployment Status

GET /api/deployment/status

## Description

REQUEST: GET — no query params. Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Check users Firestore document for uid — verify workspaceRole is 'owner' or 'admin'. Return 403 if not.
3. Read deployment metadata from environment variables: CLOUD_RUN_SERVICE_NAME, CLOUD_RUN_REGION, K_REVISION (injected by Cloud Run at runtime), K_SERVICE, K_CONFIGURATION.
4. Compute uptime from process.uptime(). Return current revision, service name, region, and environment label (derived from NODE_ENV or DEPLOYMENT_ENV env var).
5. Return deployment context for display in admin tooling.
SUCCESS (200): { service: string, revision: string, region: string, environment: string, uptimeSeconds: number, nodeVersion: string, deployedAt: string|null }
ERRORS:
- 401: invalid/expired Firebase ID token
- 403: insufficient role (owner/admin only)
- 500: env var read failure
SECURITY: Firebase ID token required. Owner/admin only — deployment metadata is sensitive infrastructure context. uid always from verified token.
