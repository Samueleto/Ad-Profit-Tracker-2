# Audit Endpoint Log

POST /api/audit/log

## Description

REQUEST: POST { action: string, resourceType: string, resourceId?: string, metadata?: object, status: 'success'|'failure', errorMessage?: string }
PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 immediately if invalid. 2. Validate action is one of the allowed enum values (api_key_saved, api_key_deleted, network_config_updated, network_config_reordered, network_config_reset, network_connection_tested, manual_sync_triggered, preferences_updated, profile_updated, account_deleted) — return 400 if not. 3. Validate resourceType is one of (api_key, network_config, preferences, profile, sync). 4. Validate status is 'success' or 'failure'. 5. Sanitize metadata — strip any fields containing 'key', 'token', 'secret', 'password' to prevent accidental credential logging. 6. Build auditLogs document: userId from verified token UID (never from request body), action, resourceType, resourceId, metadata (sanitized), status, errorMessage, ipAddress from request headers (x-forwarded-for or remoteAddress), userAgent from User-Agent header, createdAt: admin.firestore.FieldValue.serverTimestamp(). 7. Write document to auditLogs collection via Admin SDK. 8. Return { logged: true, logId }.
SUCCESS (200): { logged: true, logId: string }
ERRORS: 400 (invalid action enum, invalid resourceType, invalid status), 401 (invalid/expired Firebase ID token), 429 (rate limited — 60/min per user, prevents log flooding), 500 (Firestore write failure)
SECURITY: userId always from verified token — client cannot forge the actor identity. Metadata sanitized server-side to prevent credential leakage. This endpoint is called internally by other endpoints after mutations — not directly by UI components. Rate limited at 60/min per user to bound Firestore write costs.
