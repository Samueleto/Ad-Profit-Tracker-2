# Reconciliation Endpoint Status

GET /api/reconciliation/status

## Description

REQUEST: GET with optional query param: networkId? (one of 4 networks — omit for all).
PROCESS:
1. Verify Firebase ID token — return 401 if invalid. uid from verified token only.
2. Validate networkId if provided — return 400 if not one of the 4 supported networks.
3. Query networkConfigs Firestore collection: filter userId==uid. If networkId provided, also filter networkId==networkId. Use getDocs (one-time read).
4. For each network document, extract lastValidationStatus, lastValidationAt, anomalyCount, lastSyncStatus, circuitBreakerOpen.
5. For each network, also query adStats collection: filter userId==uid AND networkId==network AND validationStatus IN ['anomaly','stale']. Order by date DESC. Limit 5 — to surface the most recent anomalous records for quick preview.
6. Compute an overall health score: 'healthy' (all networks clean, no open circuits), 'degraded' (some anomalies but no critical failures), 'critical' (any network has critical anomalies or open circuit breaker).
7. Return consolidated validation status across all networks.
SUCCESS (200): { overallHealth: 'healthy'|'degraded'|'critical', networks: [{ networkId: string, lastValidationStatus: string, lastValidationAt: timestamp|null, anomalyCount: number, recentAnomalies: [{ date: string, country: string, anomalyFlags: array, validationStatus: string }], lastSyncStatus: string, circuitBreakerOpen: boolean }] }
ERRORS:
- 400: invalid networkId
- 401: unauthenticated
- 500: Firestore read failure
SECURITY: userId from verified token. All queries scoped to authenticated user only. Read-only endpoint.
