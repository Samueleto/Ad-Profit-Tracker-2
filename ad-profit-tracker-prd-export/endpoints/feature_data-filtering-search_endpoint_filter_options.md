# Feature Data Filtering Search Endpoint Filter Options

GET /api/filters/options

## Description

REQUEST: GET with query params: type (required — one of 'country'|'network'|'metric'|'all'), dateFrom (YYYY-MM-DD, required), dateTo (YYYY-MM-DD, required), networkId? (optional — scope country list to a specific network).
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate type: must be one of ['country','network','metric','all'] — return 400 if not.
3. Validate dateFrom and dateTo: required, valid YYYY-MM-DD, dateFrom <= dateTo, max 90-day range — return 400 if invalid.
4. Check node-cache for key '{uid}_filter_options_{type}_{dateFrom}_{dateTo}_{networkId||all}'. If cache hit, return with X-Cache: HIT header.
5. On cache miss, query adStats Firestore collection: filter userId==uid AND date>=dateFrom AND date<=dateTo. If networkId provided, also filter networkId==networkId. Use getDocs (one-time read).
6. If type=='country' or type=='all': extract unique non-null country codes from result set. Enrich with ISO 3166-1 alpha-2 country name lookup (same static map used by /api/stats/geo-breakdown). Sort alphabetically by country name. Return array of { country: string, countryName: string, hasData: true }.
7. If type=='network' or type=='all': return the four supported networks with their dataRole and whether the user has any adStats data for each in the date range: { networkId: string, label: string, dataRole: 'cost'|'revenue', hasData: boolean }.
8. If type=='metric' or type=='all': return static metric options (no Firestore query needed): [{ metric: string, label: string, unit: string }] for revenue, cost, profit, roi, impressions, clicks, ctr, cpm.
9. Store result in node-cache with TTL of 5 minutes (same as other stats endpoints).
10. Return consolidated options object.
SUCCESS (200): { dateFrom: string, dateTo: string, countries?: [{ country: string, countryName: string }], networks?: [{ networkId: string, label: string, dataRole: string, hasData: boolean }], metrics?: [{ metric: string, label: string, unit: string }], cachedAt: string|null }
ERRORS:
- 400: missing/invalid type, invalid date range, range exceeds 90 days
- 401: invalid/expired Firebase ID token
- 404: no adStats data found for date range (empty countries/networks arrays, not a hard 404)
- 500: Firestore read failure
SECURITY: userId always from verified token. All Firestore queries scoped to authenticated user's documents only. Cache keys namespaced by uid. No API keys or sensitive data in response. Country and network lists derived only from the user's own data — no cross-user data leakage.
