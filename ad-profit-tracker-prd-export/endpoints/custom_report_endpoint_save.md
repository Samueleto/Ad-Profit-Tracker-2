# Custom Report Endpoint Save

POST /api/reports/save

## Description

REQUEST: POST { name: string (required, 1-100 chars), metrics: string[] (required, non-empty subset of ['revenue','cost','netProfit','roi','impressions','clicks','ctr','cpm']), networks: string[] (required, non-empty subset of ['exoclick','rollerads','zeydoo','propush']), countries: string[] (optional, array of ISO alpha-2 codes, max 50, default []), groupBy: string (required, one of 'daily'|'country'|'network'), dataQuality: string (required, one of 'all'|'anomalies'|'clean'), dateRangePreset: string (required, one of 'last_7'|'last_14'|'last_30'|'last_90'|'this_month'|'custom'), dateFrom: string|null (YYYY-MM-DD, required if dateRangePreset=='custom'), dateTo: string|null (YYYY-MM-DD, required if dateRangePreset=='custom') }. Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate name: 1-100 chars — return 400 if invalid.
3. Validate metrics: non-empty array, each item must be one of the 8 allowed metric keys — return 400 if invalid.
4. Validate networks: non-empty array, each item must be one of the 4 supported networks — return 400 if invalid.
5. Validate countries: max 50 ISO alpha-2 codes if provided — return 400 if exceeded.
6. Validate groupBy: must be one of ['daily','country','network'] — return 400 if not.
7. Validate dataQuality: must be one of ['all','anomalies','clean'] — return 400 if not.
8. Validate dateRangePreset: must be one of the 6 allowed values — return 400 if not.
9. If dateRangePreset=='custom': validate dateFrom and dateTo are valid YYYY-MM-DD, dateFrom <= dateTo, max 90-day range — return 400 if invalid.
10. Check rate limit: max 20 report saves/hour per user — return 429 if exceeded.
11. Read users Firestore document for uid. Check current savedReports array length — enforce max 20 reports per user (return 400 with 'Maximum 20 saved reports allowed' if at limit).
12. Generate a new report ID (UUID v4). Build report object: { id, name, metrics, networks, countries, groupBy, dataQuality, dateRangePreset, dateFrom: dateFrom||null, dateTo: dateTo||null, createdAt: serverTimestamp(), updatedAt: serverTimestamp() }.
13. Use Firestore Admin SDK arrayUnion to append the new report to users.savedReports array. Set users.updatedAt=serverTimestamp().
14. Write audit log entry to auditLogs Firestore collection via Admin SDK: action='preferences_updated', resourceType='report', resourceId=id, metadata={ reportName: name, metrics, networks, groupBy }, status='success'.
15. Return the created report object.
SUCCESS (201): { report: { id, name, metrics, networks, countries, groupBy, dataQuality, dateRangePreset, dateFrom, dateTo, createdAt } }
ERRORS:
- 400: invalid fields, name too long, invalid metric/network/groupBy/dataQuality values, max 20 reports reached, invalid custom date range
- 401: invalid/expired Firebase ID token
- 429: rate limit exceeded (20/hour)
- 500: Firestore write failure
SECURITY: userId from verified token. Writes to authenticated user's users document only. Report count capped at 20 to prevent document size abuse. Audit logged.
