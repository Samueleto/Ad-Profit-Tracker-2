# Cloudrun Endpoint Health Check

GET /api/health

## Description

REQUEST: GET — no auth required. Used by Cloud Run health checks, load balancers, and uptime monitors.
PROCESS:
1. No auth check — this is a public liveness probe endpoint required by Cloud Run.
2. Verify Firestore connectivity via a lightweight admin.firestore().collection('users').limit(1).get() call with a 2-second timeout. Record latencyMs.
3. Check memory usage via process.memoryUsage(). Flag if heapUsed > 400MB.
4. Return overall status: 'healthy' if Firestore responds within 2s and memory is within bounds; 'degraded' if Firestore is slow (>1s) but responsive; 'unhealthy' if Firestore times out or throws.
5. Cloud Run uses HTTP 200 for healthy, 503 for unhealthy to determine instance health.
SUCCESS (200): { status: 'healthy'|'degraded', firestore: 'ok'|'slow', firestoreLatencyMs: number, memoryMB: number, uptime: number, version: string }
UNHEALTHY (503): { status: 'unhealthy', error: string }
ERRORS:
- 503: Firestore unreachable or timeout
SECURITY: Public endpoint — no auth. Returns no sensitive data. Firestore query is read-only and bounded (limit 1).
