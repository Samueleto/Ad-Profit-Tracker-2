# Feature Data Filtering Search Endpoint Filtered Stats

GET /api/filters/stats

## Description

REQUEST: GET with query params: dateFrom (YYYY-MM-DD, required), dateTo (YYYY-MM-DD, required), networks? (comma-separated networkIds, e.g. 'exoclick,rollerads'), countries? (comma-separated ISO alpha-2 codes, e.g. 'US,DE,FR'), metric? ('revenue'|'cost'|'profit'|'roi'|'impressions'|'clicks'|'ctr'|'cpm', default 'profit'), dataQuality? ('all'|'anomalies'|'clean', default 'all'), searchQuery? (string, max 100 chars), groupBy? ('daily'|'country'|'network', default 'daily'), limit? (integer 1–100, default 50), cursor? (Firestore document ID for pagination).
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate all params: dateFrom/dateTo required, valid YYYY-MM-DD, dateFrom <= dateTo, max 90-day range. networks must be subset of ['exoclick','rollerads','zeydoo','propush'] if provided. metric must be one of allowed values. dataQuality must be one of ['all','anomalies','clean']. limit 1–100. Return 400 if any invalid.
3. Check node-cache for key '{uid}_filtered_stats_{hash_of_all_params}'. If cache hit, return with X-Cache: HIT header.
4. Build Firestore query on adStats collection: filter userId==uid AND date>=dateFrom AND date<=dateTo. If networks provided, filter networkId IN networks array (Firestore 'in' operator, max 10 values — safe since we have 4 networks). If countries provided, filter country IN countries array (Firestore 'in' operator, max 10 values — if more than 10 countries requested, apply in-memory after fetching). Use getDocs.
5. Apply in-memory filters after Firestore fetch: (a) dataQuality=='anomalies': keep only documents where validationStatus IN ['anomaly','stale']; (b) dataQuality=='clean': keep only documents where validationStatus=='valid' or validationStatus is null; (c) searchQuery: case-insensitive match against country field or networkId field.
6. Group and aggregate filtered results by groupBy dimension: 'daily' groups by date, 'country' groups by country, 'network' groups by networkId. For each group, compute: sum revenue (rollerads+zeydoo+propush rows), sum cost (exoclick rows), netProfit=revenue-cost, roi=(netProfit/cost*100 if cost>0 else null), sum impressions, sum clicks, averageCtr, averageCpm. Apply null-safe ROI formula throughout.
7. Sort by the requested metric descending. Apply cursor-based pagination (offset by cursor document position). Slice to limit+1 to detect hasMore.
8. Compute summary totals across all filtered results (not just current page): totalRevenue, totalCost, netProfit, overallRoi, totalRows.
9. Store result in node-cache with 5-minute TTL.
10. Return paginated filtered results with summary.
SUCCESS (200): { dateFrom: string, dateTo: string, filters: { networks: string[], countries: string[], metric: string, dataQuality: string, searchQuery: string|null }, groupBy: string, rows: [{ key: string, label: string, revenue: number, cost: number, netProfit: number, roi: number|null, impressions: number, clicks: number, ctr: number, cpm: number, rowCount: number }], summary: { totalRevenue: number, totalCost: number, netProfit: number, overallRoi: number|null, totalRows: number }, hasMore: boolean, nextCursor: string|null, cachedAt: string|null }
ERRORS:
- 400: invalid params, invalid network IDs, range exceeds 90 days, invalid metric/dataQuality/groupBy
- 401: invalid/expired Firebase ID token
- 404: no data found matching the applied filters
- 500: Firestore read failure
SECURITY: userId always from verified token. All Firestore queries scoped to authenticated user's documents only. In-memory filtering applied after ownership-verified Firestore fetch. Cache keys namespaced by uid. No API keys or sensitive data in response.
