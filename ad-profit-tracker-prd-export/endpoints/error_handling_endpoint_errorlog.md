# Error Handling Endpoint ErrorLog

GET /api/errors/log

## Description

REQUEST: GET with query params: networkId? (optional), errorCode? (optional — filter by semantic error code), startDate? (YYYY-MM-DD), endDate? (YYYY-MM-DD), limit? (integer 1–50, default 20), cursor? (Firestore document ID for pagination). Bearer token in Authorization header. Returns paginated error history from auditLogs filtered to failed sync events.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid.
2. Validate query params: networkId must be one of the 4 networks if provided; limit must be 1–50; dates must be valid YYYY-MM-DD.
3. Query auditLogs Firestore collection: filter userId==uid AND status=='failure' AND resourceType=='sync'. If networkId provided, filter resourceId==networkId. If startDate/endDate provided, filter createdAt within range. Order by createdAt DESC. Apply cursor-based pagination using startAfter if cursor provided. Limit to requested page size + 1 to detect hasMore.
4. Map documents to response shape: id, networkId (from resourceId), errorMessage, metadata (attemptNumber, backoffMs, triggeredBy, latencyMs, dateFrom, dateTo), createdAt.
5. Enrich each record with a human-readable errorCategory derived from metadata.errorCode: 'transient' (NETWORK_TIMEOUT, RATE_LIMITED), 'auth' (AUTH_FAILURE), 'data' (PARSE_ERROR), 'infrastructure' (FIRESTORE_WRITE_FAILURE, DECRYPTION_FAILURE).
6. Return paginated error log.
SUCCESS (200): { errors: [{ id: string, networkId: string, errorMessage: string, errorCode: string|null, errorCategory: string, attemptNumber: number|null, backoffMs: number|null, triggeredBy: string|null, createdAt: timestamp }], hasMore: boolean, nextCursor: string|null, totalInPage: number }
ERRORS:
- 400: invalid networkId, limit, or date params
- 401: unauthenticated
- 500: Firestore read failure
SECURITY: userId from verified token. Query scoped to authenticated user's audit logs only. No API keys or encrypted data in response.
