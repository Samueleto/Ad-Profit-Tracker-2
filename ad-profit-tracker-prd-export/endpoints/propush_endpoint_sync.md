# Propush Endpoint Sync

POST /api/networks/propush/sync

## Description

REQUEST: POST { dateFrom?: string (YYYY-MM-DD, defaults to yesterday), dateTo?: string (YYYY-MM-DD, defaults to yesterday) }. PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) to get uid. 2. Validate dateFrom/dateTo (max 90-day range). 3. Check rate limit (10/hour per user). 4. Fetch networkConfigs for userId+networkId='propush'; return 400 if inactive. 5. Fetch and decrypt the API key from apiKeys collection using the server-side cryptr/AES helper. 6. Using axios, call the Propush API v5 at /v5/pub-statistics with the decrypted key, grouping by geo for the specified date range. 7. On success, parse the response. For each row, upsert a document into the adStats collection with networkId='propush', cost=null, and revenue populated from the API response. Use Firestore batch writes in chunks of 500. 8. Update the networkConfigs document with lastSyncedAt=serverTimestamp() and lastSyncStatus='success'. 9. Write an audit log entry to the auditLogs collection via Admin SDK. 10. On API failure, update networkConfigs with lastSyncStatus='failed' and return 502. SECURITY: API key is decrypted and used entirely server-side. User can only trigger syncs for their own account.
