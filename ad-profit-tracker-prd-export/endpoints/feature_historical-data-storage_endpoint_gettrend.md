# Feature Historical Data Storage Endpoint GetTrend

GET /api/stats/trend

## Description

REQUEST: GET with query params: dateFrom (YYYY-MM-DD, required), dateTo (YYYY-MM-DD, required), networkId? (optional — omit for cross-network aggregate), metric? ('revenue'|'cost'|'profit'|'roi'|'impressions'|'clicks', default 'profit').
PROCESS:
1. Verify Firebase ID token — return 401 if invalid. uid from verified token only.
2. Validate dateFrom and dateTo: required, valid YYYY-MM-DD, dateFrom <= dateTo, max 90-day range. Validate networkId and metric if provided. Return 400 if invalid.
3. Query adStats Firestore collection: filter userId==uid AND date>=dateFrom AND date<=dateTo. If networkId provided, also filter networkId==networkId. Order by date ASC. Use getDocs.
4. Group rows by date in memory. For each date, aggregate across all matching networks: sum cost (exoclick rows), sum revenue (rollerads+zeydoo+propush rows), compute netProfit=revenue-cost, roi=(netProfit/cost*100 if cost>0 else null), sum impressions and clicks.
5. Build a time-series array of { date, value } where value is the requested metric for that day.
6. Compute trend metadata: min, max, average, total, percentChange (last day vs first day), movingAverage7d (7-day rolling average for each point).
7. Return the time-series data and metadata.
SUCCESS (200): { dateFrom: string, dateTo: string, metric: string, networkId: string|null, series: [{ date: string, value: number|null, revenue: number, cost: number, netProfit: number, roi: number|null, impressions: number, clicks: number }], metadata: { min: number, max: number, average: number, total: number, percentChange: number|null, movingAverage7d: number[] } }
ERRORS:
- 400: missing/invalid dates, range exceeds 90 days, invalid metric or networkId
- 401: unauthenticated
- 404: no data found for the date range
- 500: Firestore read failure
SECURITY: userId always from verified token. Query scoped to authenticated user only.
