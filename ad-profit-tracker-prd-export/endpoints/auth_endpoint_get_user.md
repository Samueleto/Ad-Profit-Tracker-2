# Auth Endpoint Get User

GET /api/auth/get-user

## Description

REQUEST: GET — no body. Bearer token in Authorization header.
PROCESS: 1. Verify Firebase ID token via Admin SDK. 2. Fetch users/{uid} document from Firestore. 3. Return user profile data (excluding any sensitive internal fields).
SUCCESS (200): { uid, email, displayName, photoURL, authProvider, role, createdAt, lastLoginAt, preferences }
ERRORS: 401 (invalid token), 404 (user document not found — should trigger sync-user), 500 (Firestore read failure)
SECURITY: Only returns the authenticated user's own document. Used by AuthContext on app load to hydrate user state server-side.
