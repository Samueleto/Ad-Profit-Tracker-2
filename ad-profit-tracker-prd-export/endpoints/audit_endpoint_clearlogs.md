# Audit Endpoint ClearLogs

DELETE /api/audit/logs/clear

## Description

REQUEST: DELETE — no body required. Bearer token in Authorization header. Optional query param: olderThan? (ISO 8601 date — if provided, only deletes logs older than this date; if omitted, deletes all logs for the user).
PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. 2. If olderThan provided, validate it is a valid ISO 8601 date string. 3. Query auditLogs collection where userId == uid (from token), optionally filtered by createdAt < olderThan. 4. Use Firestore batch delete in chunks of 500 (Firestore batch limit) to delete all matching documents. For large collections, use multiple sequential batches. 5. Write a final audit log entry (via the same Firestore Admin SDK write, not via /api/audit/log to avoid recursion) recording action='audit_log_cleared', resourceType='audit_log', metadata={ deletedCount, olderThan: olderThan || 'all' }, status='success'. 6. Return { deleted: true, deletedCount }.
SUCCESS (200): { deleted: true, deletedCount: number }
ERRORS: 400 (invalid olderThan date), 401 (unauthenticated), 429 (rate limited — 3/day, destructive operation), 500 (batch delete failure — partial state possible, log for manual cleanup)
SECURITY: userId always from verified token — users can only delete their own logs. Batch delete scoped to authenticated user's documents. Rate limited to 3/day to prevent accidental repeated purges. The clear action itself is logged as a final audit entry before deletion completes.
