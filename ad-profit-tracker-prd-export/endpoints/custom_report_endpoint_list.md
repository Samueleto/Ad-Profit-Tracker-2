# Custom Report Endpoint List

GET /api/reports

## Description

REQUEST: GET — no query params. Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Read users Firestore document for uid using Admin SDK getDocs. Extract users.savedReports array.
3. If users.savedReports is null, undefined, or empty, return an empty array (not 404 — absence of saved reports is a valid state for new users).
4. Sort reports by createdAt descending (most recently saved first).
5. Return the full reports array.
SUCCESS (200): { reports: [{ id, name, metrics, networks, countries, groupBy, dataQuality, dateRangePreset, dateFrom, dateTo, createdAt, updatedAt }], total: number }
ERRORS:
- 401: invalid/expired Firebase ID token
- 404: users document not found
- 500: Firestore read failure
SECURITY: userId from verified token. Reads only the authenticated user's users document. No cross-user data access possible since reports are embedded in the users document scoped to uid.
