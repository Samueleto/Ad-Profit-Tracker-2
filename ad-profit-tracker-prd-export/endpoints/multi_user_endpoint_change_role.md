# Multi User Endpoint Change Role

PATCH /api/team/members/:memberId/role

## Description

REQUEST: PATCH { role: 'admin'|'member' (required) }. :memberId is the target user's uid. Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate role: must be 'admin' or 'member' — 'owner' is not assignable via this endpoint. Return 400 if invalid.
3. Read users Firestore document for uid (actor). Check users.workspaceRole — must be 'owner' to change any role, or 'admin' to change 'member' roles only. Return 403 if insufficient permissions.
4. Read users Firestore document for memberId (target). Verify target's workspaceId matches actor's workspaceId — return 403 if not in same workspace. Return 404 if target not found.
5. Prevent role change on workspace owner (target.workspaceRole === 'owner') — return 403 with 'Cannot change the workspace owner's role'.
6. Prevent admin from changing another admin's role — return 403 if actor is 'admin' and target is 'admin'.
7. Check rate limit: max 20 role changes/hour per user — return 429 if exceeded.
8. Update target's users.workspaceRole to the new role. Set users.updatedAt=serverTimestamp().
9. Write audit log to auditLogs: action='role_changed', resourceType='workspace', resourceId=uid (workspaceId), metadata={ actorUid: uid, targetUid: memberId, oldRole: previousRole, newRole: role }, status='success'.
10. Return updated member.
SUCCESS (200): { uid: string, displayName: string, workspaceRole: string, updatedAt: timestamp }
ERRORS:
- 400: invalid role value
- 401: unauthenticated
- 403: insufficient permissions, cross-workspace attempt, or attempting to change owner role
- 404: target member not found
- 429: rate limit exceeded
- 500: Firestore write failure
SECURITY: Role hierarchy enforced server-side. Owner can change any non-owner role. Admin can only change member roles. Cross-workspace changes blocked by workspaceId comparison.
