# Feature Automated Daily Data Sync Endpoint MasterScheduledSync

POST /api/scheduled/daily-sync

## Description

REQUEST: POST — no body. Secured with x-internal-secret header matching INTERNAL_SYNC_SECRET environment variable. This is the single master scheduled sync endpoint invoked by Cloud Scheduler (or equivalent cron trigger) once per day. It orchestrates all four network syncs in parallel.

PROCESS:
1. Validate x-internal-secret header against INTERNAL_SYNC_SECRET env var — return 401 immediately if missing or mismatched. This endpoint is NEVER called by end users.
2. Determine yesterday's date in UTC (YYYY-MM-DD format) as the target sync date.
3. Query the networkConfigs Firestore collection (collection-group query via Admin SDK) for all documents where isActive==true AND syncSchedule=='daily'. Group results by userId to build a per-user work list.
4. For each unique userId found, build a task list of active networks. For each network in ['exoclick','rollerads','zeydoo','propush'], check if a networkConfig document exists with isActive==true for that user.
5. For each user+network combination, verify an API key exists in the apiKeys collection. Skip combinations without a key — add to skipped list.
6. Use Promise.all() to fan out sync tasks across all users and networks in parallel. For each task: decrypt the API key using the server-side cryptr/AES helper, call the appropriate network API (ExoClick Global Statistics with groupByCountry for exoclick; GET /sites/stat with group_by=country for rollerads via axios; Zeydoo CPA statistics endpoint with country grouping via axios; Propush /v5/pub-statistics with geo grouping via axios), parse the response rows, batch-upsert into adStats collection in chunks of 500 using Firestore Admin SDK batch writes.
7. After each network sync attempt (success or failure), update the corresponding networkConfigs document: lastSyncedAt=serverTimestamp(), lastSyncStatus='success'|'failed', lastSyncError=null|errorMessage.
8. Write one audit log entry per user per network directly to auditLogs Firestore collection via Admin SDK (not HTTP): action='manual_sync_triggered', resourceType='sync', resourceId=networkId, metadata={ scheduledSync: true, latencyMs, rowsFetched, date: yesterday }, status='success'|'failure'.
9. Implement exponential backoff retry logic (up to retryAttempts from networkConfigs, default 3) for transient API failures — wait 1s, 2s, 4s between retries before marking as failed.
10. Collect a full summary of all operations and return it.

SUCCESS (200): { date: string, usersProcessed: number, networksTriggered: number, networksFailed: number, networksSkipped: number, totalRowsFetched: number, durationMs: number, summary: [{ userId: string, networkId: string, status: 'success'|'failed'|'skipped', rowsFetched: number, errorMessage: string|null }] }

ERRORS:
- 401: missing or invalid x-internal-secret header
- 500: Firestore collection-group query failure or unhandled exception

SECURITY: Internal endpoint only — protected by x-internal-secret header, never user JWT. API keys decrypted server-side only, never logged or returned. All Firestore writes use Admin SDK. Rate limit not applicable — called by scheduler, not users.
