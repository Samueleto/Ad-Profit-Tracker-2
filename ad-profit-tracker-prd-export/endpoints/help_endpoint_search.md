# Help Endpoint Search

GET /api/help/search

## Description

REQUEST: GET with query params: q (string, required, min 1 char, max 100 chars), category? (optional filter), limit? (integer 1–20, default 10). Bearer token in Authorization header. PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only. 2. Validate q: required, 1–100 chars, strip leading/trailing whitespace — return 400 if empty after trim. 3. Check node-cache for key '{uid}_help_search_{q}_{category}_{limit}'. If cache hit, return with X-Cache: HIT header. 4. Read users Firestore document for uid to get effectivePermissions and workspaceRole. 5. Query helpArticles Firestore collection: filter isPublished==true. Use getDocs (one-time read — Firestore does not support full-text search, so fetch the bounded published set and filter in memory). 6. Apply in-memory search: normalize q to lowercase. Match against title, summary, tags array, and category fields. Score matches: title match = 3 points, summary match = 2 points, tags match = 1 point. 7. Filter results by user permissions: exclude articles where requiredPermission is set and user lacks that permission. 8. Sort by score descending, then by viewCount descending as tiebreaker. Slice to limit. 9. Store in node-cache with 2-minute TTL. 10. Return search results with match context (which field matched). SUCCESS (200): { q: string, results: [{ id, title, slug, category, summary, readTimeMinutes, matchedIn: string[], score: number, updatedAt }], totalResults: number, cachedAt: string|null } ERRORS: 400 (empty query), 401 (unauthenticated), 500 (Firestore read failure). SECURITY: uid from verified token. Permission-gated articles excluded from results. Cache keys namespaced by uid.
