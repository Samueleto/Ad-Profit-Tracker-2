# Feature Scheduled Report Delivery Endpoint List

GET /api/schedules

## Description

REQUEST: GET — no query params. Optional query param: reportId? (string — filter to a specific saved report's schedule). Bearer token in Authorization header. PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only. 2. Validate reportId if provided — non-empty string, return 400 if empty. 3. Read users Firestore document for uid via Admin SDK getDocs. Extract users.scheduledReports array. 4. If users.scheduledReports is null, undefined, or empty, return empty array (not 404 — absence of schedules is valid for new users). 5. If reportId provided, filter array to items where scheduledReports[].reportId == reportId. 6. Sort by createdAt descending. 7. For each schedule, compute a human-readable nextRunLabel (e.g., 'Tomorrow at 8:00 AM EST', 'Next Monday at 8:00 AM') using date-fns-tz and the schedule's timezone. 8. Return the schedules array with enriched nextRunLabel. SUCCESS (200): { schedules: [{ id, reportId, reportName, frequency, dayOfWeek, dayOfMonth, deliveryHour, timezone, deliveryEmail, dateRangePreset, format, isActive, lastRunAt, lastRunStatus, lastRunError, nextRunAt, nextRunLabel, createdAt, updatedAt }], total: number }. ERRORS: 400 (invalid reportId param), 401 (unauthenticated), 404 (users document not found), 500 (Firestore read failure). SECURITY: userId from verified token. Reads only the authenticated user's users document. No cross-user data access possible.
