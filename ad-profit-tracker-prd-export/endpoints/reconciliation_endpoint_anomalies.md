# Reconciliation Endpoint Anomalies

GET /api/reconciliation/anomalies

## Description

REQUEST: GET with query params: networkId? (optional), dateFrom? (YYYY-MM-DD, defaults to 30 days ago), dateTo? (YYYY-MM-DD, defaults to today), severity? ('warning'|'critical', optional — omit for all), anomalyType? (string, optional filter), limit? (integer 1–100, default 20), cursor? (Firestore document ID for pagination).
PROCESS:
1. Verify Firebase ID token — return 401 if invalid. uid from verified token only.
2. Validate all query params. Return 400 if invalid.
3. Query adStats Firestore collection: filter userId==uid AND validationStatus IN ['anomaly','stale'] AND date>=dateFrom AND date<=dateTo. If networkId provided, also filter networkId==networkId. Order by date DESC, then by networkId. Use getDocs with cursor-based pagination (startAfter if cursor provided). Fetch limit+1 to detect hasMore.
4. If severity or anomalyType filter provided, apply in-memory filtering on the anomalyFlags array of each document.
5. Map each document to a response shape: id, networkId, date, country, validationStatus, anomalyFlags (full array), cost, revenue, syncedAt, validationCheckedAt.
6. Compute a summary: totalAnomalies in page, criticalCount, warningCount, affectedNetworks (unique networkIds in result).
7. Return paginated anomaly list.
SUCCESS (200): { anomalies: [{ id: string, networkId: string, date: string, country: string|null, validationStatus: string, anomalyFlags: array, cost: number|null, revenue: number|null, syncedAt: timestamp, validationCheckedAt: timestamp }], summary: { totalInPage: number, criticalCount: number, warningCount: number, affectedNetworks: string[] }, hasMore: boolean, nextCursor: string|null }
ERRORS:
- 400: invalid params
- 401: unauthenticated
- 500: Firestore read failure
SECURITY: userId from verified token. Query scoped to authenticated user only. No API keys or sensitive data in response.
