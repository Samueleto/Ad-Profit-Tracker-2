# Help Endpoint Get Article

GET /api/help/articles/:articleId

## Description

REQUEST: GET /api/help/articles/:articleId — articleId in URL path. Bearer token in Authorization header. PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only. 2. Validate articleId: required, non-empty string — return 400 if missing. 3. Read helpArticles Firestore document by id. Return 404 if not found or isPublished==false (treat drafts as not found for non-admin users). 4. Read users Firestore document for uid to get effectivePermissions and workspaceRole. 5. If article.requiredPermission is set, verify effectivePermissions[article.requiredPermission] === true — return 403 if not. If article.requiredRole is set, verify workspaceRole matches — return 403 if not. 6. Return full article including body markdown. SUCCESS (200): { id, title, slug, category, body, summary, tags, readTimeMinutes, videoUrl, viewCount, helpfulCount, notHelpfulCount, authorName, createdAt, updatedAt } ERRORS: 400 (missing articleId), 401 (unauthenticated), 403 (insufficient permissions for gated article), 404 (article not found or draft), 500 (Firestore read failure). SECURITY: uid from verified token. Permission check enforced server-side — client cannot bypass by guessing articleId. Draft articles return 404 to prevent content leakage.
