# Help Endpoint Feedback

POST /api/help/feedback

## Description

REQUEST: POST { articleId: string (required), rating: 'helpful'|'not_helpful' (required), comment?: string (optional, max 500 chars) }. Bearer token in Authorization header. PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only. 2. Validate articleId: required, non-empty string. Validate rating: must be 'helpful' or 'not_helpful'. Validate comment if provided: max 500 chars. Return 400 if invalid. 3. Check rate limit: max 1 feedback submission per user per article — prevents duplicate votes. Key: '{uid}_feedback_{articleId}'. Return 429 if already submitted. 4. Read helpArticles Firestore document by articleId — return 404 if not found or isPublished==false. 5. Use Firestore Admin SDK to increment helpArticles.helpfulCount by 1 if rating=='helpful', or helpArticles.notHelpfulCount by 1 if rating=='not_helpful', using FieldValue.increment(1). 6. Write audit log entry to auditLogs Firestore collection via Admin SDK: action='preferences_updated', resourceType='help_article', resourceId=articleId, metadata={ rating, hasComment: !!comment }, status='success'. 7. Return confirmation. SUCCESS (200): { submitted: true, articleId: string, rating: string } ERRORS: 400 (invalid fields), 401 (unauthenticated), 404 (article not found), 429 (already submitted feedback for this article), 500 (Firestore write failure). SECURITY: uid from verified token. One feedback per user per article enforced via rate limit key. Comment content stored in audit log metadata only — not in helpArticles document.
