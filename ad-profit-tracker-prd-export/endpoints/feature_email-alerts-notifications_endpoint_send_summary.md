# Feature Email Alerts Notifications Endpoint Send Summary

POST /api/email/send-summary

## Description

REQUEST: POST { userId: string (required — from x-internal-secret only), dateFrom: string (YYYY-MM-DD, required), dateTo: string (YYYY-MM-DD, required), summaryType: 'daily'|'weekly'|'monthly' (required) }. Internal-only endpoint triggered by Cloud Scheduler.
PROCESS:
1. Validate x-internal-secret header against INTERNAL_SYNC_SECRET env var. Return 401 if invalid. This endpoint is NOT callable by end-users — no Firebase ID token path.
2. Validate userId non-empty, dateFrom/dateTo valid YYYY-MM-DD, summaryType one of 3 values. Return 400 if invalid.
3. Read users Firestore document for userId via Admin SDK. Check users.notificationPreferences['schedule_delivered'].emailEnabled — if false, return 200 with { sent: false, reason: 'disabled_by_user' }. Also check users.preferences.emailSummaryFrequency matches summaryType — if mismatch, return 200 with { sent: false, reason: 'frequency_mismatch' }.
4. Extract delivery email from users.preferences.alertDeliveryEmail or Firebase Auth user record. Return 400 if no email found.
5. Check rate limit: max 1 summary email per summaryType per day per user — return 429 if exceeded. Key: '{userId}_email_summary_{summaryType}_{dateFrom}'.
6. Query adStats Firestore collection: filter userId==userId AND date>=dateFrom AND date<=dateTo. Use getDocs (one-time read). Compute summary KPIs in memory: totalRevenue (rollerads+zeydoo+propush), totalCost (exoclick), netProfit, roi (null-safe), topCountry (by netProfit), bestNetwork (by primary metric), daysWithData.
7. Render HTML email template: subject 'Ad Profit Tracker — [Daily/Weekly/Monthly] Summary [dateFrom] to [dateTo]'. Include KPI summary table (Revenue, Cost, Net Profit, ROI%), top country row, best network row, data freshness note, and a 'View Full Dashboard' CTA button linking to /dashboard. Footer with 'Manage email preferences' link to /settings.
8. Send via transactional email provider SDK (e.g., Nodemailer/Resend/SendGrid/Postmark) using EMAIL_PROVIDER_API_KEY and EMAIL_FROM_ADDRESS env vars.
9. Write audit log to auditLogs: action='email_sent', resourceType='notification', resourceId='summary_report', metadata={ summaryType, dateFrom, dateTo, recipientEmail, kpis: { totalRevenue, totalCost, netProfit, roi } }, status='success'|'failure'.
10. Return delivery confirmation.
SUCCESS (200): { sent: true, recipientEmail: string, summaryType: string, dateFrom: string, dateTo: string }
ERRORS:
- 400: invalid params, no delivery email
- 401: invalid internal secret
- 404: user document not found
- 429: rate limit exceeded (1/day per summaryType)
- 502: email provider failure
- 500: Firestore read failure
SECURITY: Internal-only — x-internal-secret required. Not callable by end-users. userId from request body only after secret validation. Rate limited to 1/day per summaryType to prevent duplicate sends.
