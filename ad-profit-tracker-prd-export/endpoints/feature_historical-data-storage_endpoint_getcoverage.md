# Feature Historical Data Storage Endpoint GetCoverage

GET /api/stats/coverage

## Description

REQUEST: GET with query params: dateFrom (YYYY-MM-DD, required), dateTo (YYYY-MM-DD, required), networkId? (optional).
PROCESS:
1. Verify Firebase ID token — return 401 if invalid. uid from verified token only.
2. Validate dateFrom and dateTo: required, valid YYYY-MM-DD, dateFrom <= dateTo, max 90-day range. Return 400 if invalid.
3. Query adStats Firestore collection: filter userId==uid AND date>=dateFrom AND date<=dateTo. If networkId provided, also filter networkId==networkId. Use getDocs.
4. Build a coverage map: for each date in the requested range (enumerate all calendar days from dateFrom to dateTo), check which networks have at least one adStats document. Mark each date+network combination as 'present', 'missing', or 'partial' (has some country rows but not all expected).
5. Query auditLogs Firestore collection: filter userId==uid AND resourceType=='sync' AND action=='manual_sync_triggered' AND createdAt>=dateFrom AND createdAt<=dateTo. Use getDocs. Map sync events to their target dates (from metadata.dateFrom/dateTo) to show what was synced and when.
6. Compute coverage statistics: totalDays (calendar days in range), daysFullyCovered (all 4 active networks have data), daysMissingAny (at least one network missing), perNetworkCoverage (percentage of days with data per network).
7. Identify gaps: dates where one or more networks have no data. Flag these as reconciliation candidates.
8. Return the coverage map and gap list.
SUCCESS (200): { dateFrom: string, dateTo: string, totalDays: number, daysFullyCovered: number, daysMissingAny: number, perNetworkCoverage: [{ networkId: string, daysWithData: number, coveragePercent: number }], gaps: [{ date: string, missingNetworks: string[] }], syncHistory: [{ date: string, networkId: string, syncedAt: timestamp, rowsFetched: number, status: string }] }
ERRORS:
- 400: missing/invalid dates, range exceeds 90 days
- 401: unauthenticated
- 500: Firestore read failure
SECURITY: userId always from verified token. All queries scoped to authenticated user only. auditLogs query also scoped to userId==uid.
