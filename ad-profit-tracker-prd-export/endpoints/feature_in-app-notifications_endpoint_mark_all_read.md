# Feature In App Notifications Endpoint Mark All Read

PATCH /api/notifications/mark-all-read

## Description

REQUEST: PATCH — no body. Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Check rate limit: max 30 mark-all-read actions/hour per user — return 429 if exceeded.
3. Read users Firestore document for uid. Extract users.notifications array.
4. If array is null/empty or all items already have isRead=true, return early with { updated: 0, unreadCount: 0 }.
5. Build updated notifications array where every item with isRead=false is set to isRead=true (isDismissed items remain as-is).
6. Use Firestore Admin SDK to write the full updated notifications array back to users.notifications. Set users.unreadNotificationCount=0. Set users.notificationsUpdatedAt=serverTimestamp().
7. Return count of items marked as read.
SUCCESS (200): { updated: number, unreadCount: 0 }
ERRORS:
- 401: unauthenticated
- 404: users document not found
- 429: rate limit exceeded (30/hour)
- 500: Firestore write failure
SECURITY: userId from verified token. Writes only to authenticated user's users document.
