# Feature Network Configuration Management Endpoint TestConnection

POST /api/networks/config/test-connection

## Description

REQUEST: POST { networkId: string }. PROCESS: 1. Verify Firebase ID token — return 401 if invalid. 2. Validate networkId is one of the 4 supported networks — return 400 if not. 3. Fetch the encrypted API key for this network from users/{uid}/apiKeys/{networkId} — return 404 if no key saved. 4. Decrypt the API key using the server-side encryption secret (same cryptr/AES approach as /api/keys/save). 5. Fetch the networkConfig for this user+network to get endpointOverride and timeoutSeconds. 6. Make a lightweight probe request to the network's API (e.g., a minimal date-range stats request for yesterday) using the decrypted key and configured timeout. Record start/end time for latency measurement. 7. On success: write lastSyncStatus='success' and lastSyncedAt to networkConfigs. Return latency and sample record count. 8. On API failure: write lastSyncStatus='failed' and lastSyncError to networkConfigs. Return error details. SUCCESS (200): { networkId, status: 'ok'|'error', latencyMs: number, sampleRecordCount: number|null, errorMessage: string|null } ERRORS: 400 (invalid networkId), 401 (unauthenticated), 404 (no API key saved for this network), 502 (network API unreachable or returned error), 500 (decryption or Firestore error). SECURITY: API key decrypted server-side only, never returned to client. Probe request uses server-side HTTP client (node-fetch/axios). Rate limited to prevent abuse.
