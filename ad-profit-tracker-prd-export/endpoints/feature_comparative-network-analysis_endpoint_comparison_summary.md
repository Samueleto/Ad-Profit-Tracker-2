# Feature Comparative Network Analysis Endpoint Comparison Summary

GET /api/networks/comparison

## Description

REQUEST: GET with query params: dateFrom (YYYY-MM-DD, required), dateTo (YYYY-MM-DD, required), metric? ('revenue'|'cost'|'roi'|'impressions'|'clicks'|'ctr'|'cpm', default 'roi').
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 immediately if invalid. uid from verified token only.
2. Validate dateFrom and dateTo: required, valid YYYY-MM-DD, dateFrom <= dateTo, max 90-day range — return 400 if invalid.
3. Validate metric if provided — return 400 if not one of the allowed values.
4. Check node-cache for key '{uid}_network_comparison_{dateFrom}_{dateTo}_{metric}'. If cache hit, return with X-Cache: HIT header.
5. On cache miss, execute a single Firestore query on adStats collection: filter userId==uid AND date>=dateFrom AND date<=dateTo. Use getDocs (one-time read — covers all four networks in one query to minimize Firestore reads).
6. If no documents found, return 404.
7. Group documents by networkId in memory. For each of the four networks, compute: primaryMetric (sum cost for exoclick, sum revenue for others), impressions (sum), clicks (sum), averageCtr (weighted), averageCpm (weighted), daysWithData (unique dates), dataRole ('cost'|'revenue').
8. Compute cross-network derived metrics: totalRevenue (sum of rollerads+zeydoo+propush revenue), totalCost (exoclick cost), netProfit (totalRevenue - totalCost), overallRoi (netProfit/totalCost*100 if totalCost>0 else null), revenuePerImpression (totalRevenue / totalImpressions if totalImpressions>0 else null), costPerClick (totalCost / exoclickClicks if exoclickClicks>0 else null).
9. Compute per-network metricShare: each network's primaryMetric / sum(all primaryMetrics) * 100.
10. Rank networks by the requested metric descending. Assign ordinal ranks (1–4). Handle ties by assigning same rank.
11. Fetch networkConfigs documents for userId==uid (getDocs) to include lastSyncedAt, lastSyncStatus, circuitBreakerOpen per network.
12. Store result in node-cache with 5-minute TTL.
13. Return consolidated comparison payload.
SUCCESS (200): { dateFrom, dateTo, metric, networks: [{ networkId, dataRole, primaryMetric, impressions, clicks, averageCtr, averageCpm, daysWithData, metricShare, rank, networkStatus: { lastSyncedAt, lastSyncStatus, circuitBreakerOpen } }], crossNetwork: { totalRevenue, totalCost, netProfit, overallRoi, revenuePerImpression, costPerClick }, rankings: [{ networkId, rank, metricValue, metricLabel }], cachedAt: string|null }
ERRORS:
- 400: missing/invalid dates, range exceeds 90 days, invalid metric
- 401: invalid/expired Firebase ID token
- 404: no data found for date range
- 500: Firestore read failure
SECURITY: userId always from verified token. Single Firestore query covers all networks — cost-efficient. Cache keys namespaced by uid. No API keys or sensitive data in response.
