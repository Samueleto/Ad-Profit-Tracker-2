# Feature Roi Calculation Engine Endpoint Roi Breakdown

GET /api/roi/breakdown

## Description

REQUEST: GET with query params: dateFrom (YYYY-MM-DD, required), dateTo (YYYY-MM-DD, required), dimension ('network'|'country'|'daily', required).
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate all params: dateFrom/dateTo required, valid YYYY-MM-DD, dateFrom <= dateTo, max 90-day range. dimension must be one of ['network','country','daily']. Return 400 if invalid.
3. Check node-cache for key '{uid}_roi_breakdown_{dateFrom}_{dateTo}_{dimension}'. Return cached result with X-Cache: HIT header if found.
4. Query adStats Firestore collection: filter userId==uid AND date>=dateFrom AND date<=dateTo. Use getDocs.
5. If no documents found, return 404.
6. Apply null-safe ROI formula per dimension group:
   - dimension='network': group by networkId. For each group, compute revenue (rollerads/zeydoo/propush) or cost (exoclick), then compute cross-network roi=(totalRevenue-totalCost)/totalCost*100 if totalCost>0 else null.
   - dimension='country': group by country code. For each country, sum revenue from revenue networks and cost from exoclick. Compute per-country roi. Enrich with ISO 3166-1 alpha-2 country name lookup. Sort by netProfit descending.
   - dimension='daily': group by date. For each date, compute daily roi. Include roiIndicator and colorCode per day.
7. For each breakdown item, compute: roi (null-safe), roiIndicator ('positive'|'negative'|'breakeven'|'no_cost_data'), colorCode ('green'|'red'|'amber'), contribution % (item netProfit / total netProfit * 100).
8. Compute summary totals: totalRevenue, totalCost, netProfit, overallRoi.
9. Store in node-cache with 5-minute TTL.
10. Return breakdown array with summary.
SUCCESS (200): { dateFrom, dateTo, dimension, breakdown: [{ key: string, label: string, revenue: number, cost: number, netProfit: number, roi: number|null, roiIndicator: string, colorCode: string, contributionPercent: number }], summary: { totalRevenue: number, totalCost: number, netProfit: number, overallRoi: number|null }, cachedAt: string|null }
ERRORS:
- 400: missing/invalid params, range exceeds 90 days, invalid dimension
- 401: unauthenticated
- 404: no data found
- 500: Firestore read failure
SECURITY: userId from verified token. Query scoped to authenticated user's adStats only. Cache keys namespaced by uid.
