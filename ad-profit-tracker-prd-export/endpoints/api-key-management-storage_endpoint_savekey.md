# Api Key Management Storage Endpoint SaveKey

POST /api/keys/save

## Description

REQUEST: POST { networkId: string, key: string }. PROCESS: 1. Verify the user's Firebase ID token to get their UID. 2. Validate the request body to ensure `networkId` is one of the allowed networks and `key` is not empty. 3. Use a server-side encryption library (like `cryptr`) with a master secret from environment variables to encrypt the provided `key`. 4. Perform an upsert operation on the `users/{uid}/apiKeys/{networkId}` Firestore document, storing the `encryptedKey` and updating the `updatedAt` timestamp. 5. Return a success response. SUCCESS (200): { networkId, status: 'connected' }. ERRORS: 400 (invalid networkId or empty key), 401 (unauthenticated), 403 (forbidden), 429 (rate limited), 500 (encryption or Firestore error). SECURITY: Must be a Cloud Function. The encryption secret must only exist in a secure server environment (e.g., Cloud Secret Manager, environment variables). Never expose the raw key in logs.
