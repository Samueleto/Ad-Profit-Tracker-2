# Feature In App Notifications Endpoint Preferences Patch

PATCH /api/notifications/preferences

## Description

REQUEST: PATCH { preferences: { [type: string]: { enabled: boolean } } } — partial update, only provided types are changed. Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Validate preferences: each key must be one of the 9 allowed notification types. Each value must have enabled as boolean. At least one type must be provided — return 400 if empty.
3. Check rate limit: max 20 preference updates/hour per user — return 429 if exceeded.
4. Fetch users Firestore document for uid — return 404 if not found.
5. Merge provided preferences into users.notificationPreferences using Firestore Admin SDK dot-notation partial update (e.g., 'notificationPreferences.sync_failure.enabled'). Set users.notificationsUpdatedAt=serverTimestamp().
6. Write audit log entry to auditLogs Firestore collection via Admin SDK: action='preferences_updated', resourceType='notification', resourceId='preferences', metadata={ changedTypes: Object.keys(preferences) }, status='success'.
7. Return the full updated preferences merged with defaults.
SUCCESS (200): { preferences: [{ type: string, enabled: boolean, isDefault: boolean }], updatedAt: timestamp }
ERRORS:
- 400: invalid type key, non-boolean enabled value, empty body
- 401: unauthenticated
- 404: users document not found
- 429: rate limit exceeded (20/hour)
- 500: Firestore write failure
SECURITY: userId from verified token. Users can only update their own preferences. Audit logged.
