# Feature Performance Benchmarking Endpoint Settings Patch

PATCH /api/benchmarks/settings

## Description

REQUEST: PATCH { metricTargets?: { [metric: string]: { customTarget?: number|null, useDefault?: boolean } }, historicalWindowDays?: number (7–365) }. Bearer token in Authorization header. Upserts the user's custom benchmark target configuration. PROCESS: 1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only. 2. Validate metricTargets if provided: each key must be one of ['roi','ctr','cpm','revenue','cost','impressions','clicks']. customTarget must be non-negative number or null. ROI% and CTR% capped at 10000. CPM capped at 1000. Revenue and cost capped at 10000000. Return 400 with specific field errors if any value is out of range. 3. Validate historicalWindowDays if provided: must be integer 7–365. Return 400 if out of range. 4. At least one of metricTargets or historicalWindowDays must be provided — return 400 if body is empty. 5. Check rate limit: max 20 updates/hour per user — return 429 if exceeded. 6. Use Firestore Admin SDK set() with merge: true on the benchmarks document for userId==uid (upsert pattern — creates document if it doesn't exist). Set updatedAt=serverTimestamp(), createdAt=serverTimestamp() only if document is new (use set with merge). 7. Write audit log entry to auditLogs Firestore collection via Admin SDK: action='preferences_updated', resourceType='preferences', resourceId='benchmark_settings', metadata={ changedMetrics: string[], historicalWindowDays: { from: oldValue, to: newValue } }, status='success'. 8. Return the full updated settings object merged with system defaults. SUCCESS (200): { historicalWindowDays: number, metricTargets: [{ metric: string, customTarget: number|null, useDefault: boolean, effectiveTarget: number, systemDefault: number, unit: string, isCustom: boolean }], updatedAt: timestamp } ERRORS: - 400: invalid metric key, value out of range, empty body, historicalWindowDays out of range - 401: unauthenticated - 429: rate limit exceeded (20/hour) - 500: Firestore write failure SECURITY: userId from verified token. Users can only update their own benchmarks document. Audit logged for accountability.
