# Rate Limit Endpoint Reset

POST /api/rate-limits/reset

## Description

REQUEST: POST { networkId: string (required), scope: 'outbound'|'user-quota'|'both' (required) }. Dual-auth: Bearer token (user-initiated) or x-internal-secret header (scheduled/admin). Resets the Bottleneck limiter reservoir and/or user-facing quota counters for a specific network.
PROCESS:
1. Check auth: if Authorization header present, verify Firebase ID token and derive uid. If x-internal-secret header present, validate against INTERNAL_SYNC_SECRET env var. Return 401 if neither is valid.
2. Validate networkId: must be one of ['exoclick','rollerads','zeydoo','propush'] — return 400 if not.
3. Validate scope: must be 'outbound', 'user-quota', or 'both' — return 400 if not.
4. Check rate limit: max 10 resets/hour per user (user-authenticated calls only) — return 429 if exceeded. Internal-secret calls are not rate limited.
5. If scope includes 'outbound': call limiter.updateSettings({ reservoir: limiterConfig[networkId].reservoir }) on the Bottleneck instance in lib/rate-limit/limiterRegistry.ts to refill the token bucket for the given userId+networkId key.
6. If scope includes 'user-quota': clear the sliding window counter for the uid in lib/rate-limit/userLimiter.ts for all endpoints associated with this networkId.
7. Write audit log entry to auditLogs Firestore collection via Admin SDK: action='network_config_updated', resourceType='rate_limit', resourceId=networkId, metadata={ scope, resetBy: uid||'system', previousReservoir: number }, status='success'. Only write audit log for user-authenticated calls — skip for internal-secret calls to avoid flooding.
8. Return confirmation.
SUCCESS (200): { networkId: string, scope: string, resetAt: string, message: string }
ERRORS:
- 400: invalid networkId or scope
- 401: neither valid Firebase ID token nor valid internal secret
- 429: rate limit exceeded (10/hour for user calls)
- 500: limiter registry update failure
SECURITY: Dual-auth pattern. User calls audit-logged. Internal calls not rate limited. Never resets circuit breaker state — that is handled by POST /api/errors/circuit-breaker/reset.
