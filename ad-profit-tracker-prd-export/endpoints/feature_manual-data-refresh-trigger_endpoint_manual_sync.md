# Feature Manual Data Refresh Trigger Endpoint Manual Sync

POST /api/sync/manual

## Description

REQUEST: POST { networkId?: string (one of 'exoclick'|'rollerads'|'zeydoo'|'propush', optional — omit to trigger all active networks), dateFrom?: string (YYYY-MM-DD, defaults to yesterday), dateTo?: string (YYYY-MM-DD, defaults to yesterday) }
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 immediately if invalid. uid from verified token only.
2. Validate networkId if provided: must be one of the 4 supported networks — return 400 if not.
3. Validate dateFrom/dateTo if provided: valid YYYY-MM-DD, dateFrom <= dateTo, max 90-day range — return 400 if invalid.
4. Check rate limit: if networkId provided, max 10 manual syncs/hour per user (same as per-network sync endpoints). If networkId omitted (sync-all), max 3/hour per user (same as POST /api/networks/sync-all). Return 429 with Retry-After header if exceeded.
5. Determine target networks: if networkId provided, use [networkId]; otherwise query networkConfigs Firestore collection for all documents where userId==uid AND isActive==true.
6. For each target network, verify an API key exists in apiKeys collection where userId==uid AND networkId==network — skip networks without keys and add to skipped array.
7. For each network with a valid key, call the shared server-side helper lib/sync/syncNetworkForUser(uid, networkId, dateFrom, dateTo) which: decrypts the API key using the server-side cryptr/AES helper; calls the appropriate network API (ExoClick Global Statistics with groupByCountry for exoclick; GET /sites/stat with group_by=country for rollerads via axios; Zeydoo CPA statistics endpoint with country grouping via axios; Propush /v5/pub-statistics with geo grouping via axios); applies exponential backoff retry (up to retryAttempts from networkConfigs, default 3, with 1s/2s/4s waits); parses response rows and batch-upserts into adStats Firestore collection in chunks of 500 using Admin SDK batch writes; updates networkConfigs document with lastSyncedAt=serverTimestamp(), lastSyncStatus='success'|'failed', lastSyncError=null|errorMessage; writes audit log entry directly to auditLogs Firestore collection via Admin SDK with action='manual_sync_triggered', resourceType='sync', resourceId=networkId, metadata={ scheduledSync: false, latencyMs, rowsFetched, dateFrom, dateTo, triggeredBy: 'user' }, status='success'|'failure'.
8. Use Promise.allSettled() to fan out across all target networks in parallel — do not let one failure block others.
9. Collect results and return summary.
SUCCESS (200): { triggered: string[], skipped: string[], failed: string[], results: [{ networkId: string, status: 'success'|'failed'|'skipped', rowsFetched: number|null, latencyMs: number|null, errorMessage: string|null }], triggeredAt: string }
ERRORS:
- 400: invalid networkId, invalid date range, range exceeds 90 days
- 401: invalid/expired Firebase ID token
- 429: rate limit exceeded — 3/hour for sync-all, 10/hour for per-network
- 500: Firestore read/write failure or unhandled exception
SECURITY: API keys decrypted server-side only via shared helper, never returned or logged. userId always from verified token — never from request body. All Firestore writes scoped to authenticated uid. Metadata sanitized before audit log write (strip any key/token/secret fields). Rate limited to prevent runaway ad network API calls and Firestore write costs.
