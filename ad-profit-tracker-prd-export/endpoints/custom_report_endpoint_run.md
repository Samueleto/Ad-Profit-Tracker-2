# Custom Report Endpoint Run

POST /api/reports/run

## Description

REQUEST: POST { reportId?: string (optional — if provided, load config from users.savedReports; if omitted, use inline config), metrics: string[] (required if no reportId, subset of 8 allowed), networks: string[] (required if no reportId, subset of 4 networks), countries: string[] (optional, max 50 ISO codes), groupBy: string (required if no reportId, 'daily'|'country'|'network'), dataQuality: string (required if no reportId, 'all'|'anomalies'|'clean'), dateFrom: string (YYYY-MM-DD, required), dateTo: string (YYYY-MM-DD, required), limit?: number (1-100, default 50), cursor?: string (Firestore document ID for pagination) }. Bearer token in Authorization header.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. If reportId provided: read users Firestore document for uid, find report with matching id in savedReports array — return 404 if not found. Merge saved config with any inline overrides (dateFrom/dateTo always required from request body).
3. Validate all config fields: metrics non-empty subset of 8 allowed, networks non-empty subset of 4, groupBy one of 3 values, dataQuality one of 3 values, dateFrom/dateTo valid YYYY-MM-DD with dateFrom <= dateTo and max 90-day range — return 400 if invalid.
4. Check rate limit: max 30 report runs/hour per user — return 429 if exceeded.
5. Check node-cache for key '{uid}_report_run_{hash_of_all_params}'. If cache hit, return with X-Cache: HIT header.
6. Query adStats Firestore collection: filter userId==uid AND date>=dateFrom AND date<=dateTo. If networks is not all 4, filter networkId IN networks (Firestore 'in' operator). If countries provided and <= 10, filter country IN countries. Use getDocs.
7. Apply in-memory filters: (a) dataQuality=='anomalies': keep only validationStatus IN ['anomaly','stale']; (b) dataQuality=='clean': keep only validationStatus=='valid' or null; (c) countries > 10: apply country filter in-memory.
8. Group and aggregate by groupBy dimension. For each group, compute only the requested metrics: revenue (rollerads+zeydoo+propush rows), cost (exoclick rows), netProfit=revenue-cost, roi=(netProfit/cost*100 if cost>0 else null), impressions, clicks, averageCtr, averageCpm — only include fields present in the metrics array.
9. Sort by netProfit descending. Apply cursor-based pagination. Compute summary totals across all results.
10. Store in node-cache with 5-minute TTL.
11. Write audit log entry to auditLogs: action='custom_report_generated', resourceType='report', resourceId=reportId||'adhoc', metadata={ metrics, networks, groupBy, dataQuality, dateFrom, dateTo, rowCount: total }, status='success'.
12. Return paginated results with only the requested metric columns.
SUCCESS (200): { dateFrom, dateTo, config: { metrics, networks, countries, groupBy, dataQuality }, rows: [{ key: string, label: string, [metricKey]: number|null }], summary: { [metricKey]: number|null, totalRows: number }, hasMore: boolean, nextCursor: string|null, cachedAt: string|null }
ERRORS:
- 400: invalid config fields, date range exceeds 90 days
- 401: unauthenticated
- 404: reportId not found in savedReports, or no adStats data for range
- 429: rate limit exceeded (30/hour)
- 500: Firestore read failure
SECURITY: userId from verified token. All Firestore queries scoped to authenticated user's adStats only. Cache keys namespaced by uid. Audit logged. Shares the same 10 exports/hour rate limit budget for export actions — run itself is separate at 30/hour.
