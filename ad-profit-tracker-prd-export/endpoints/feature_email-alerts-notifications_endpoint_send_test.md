# Feature Email Alerts Notifications Endpoint Send Test

POST /api/email/send-test

## Description

REQUEST: POST — no body. Bearer token in Authorization header. Sends a test email to the authenticated user's configured delivery address to verify email delivery is working.
PROCESS:
1. Verify Firebase ID token via admin.auth().verifyIdToken(token) — return 401 if invalid. uid from verified token only.
2. Check rate limit: max 3 test emails/hour per user — return 429 if exceeded. Key: '{uid}_email_test'.
3. Read users Firestore document for uid. Extract users.preferences.alertDeliveryEmail. If not set, fall back to Firebase Auth user record email. If no email found, return 400 with 'No delivery email configured — add one in Settings'.
4. Use transactional email provider SDK (e.g., Nodemailer/Resend/SendGrid/Postmark) configured via EMAIL_PROVIDER_API_KEY and EMAIL_FROM_ADDRESS env vars. Send a test email with subject 'Ad Profit Tracker — Email Delivery Test', body 'This is a test email confirming your alert delivery is configured correctly. You will receive alerts at this address.', and a footer with 'Manage email preferences' link to /settings.
5. Write audit log to auditLogs: action='email_sent', resourceType='notification', resourceId='test', metadata={ recipientEmail, triggeredBy: 'user_test' }, status='success'|'failure'.
6. Update users.preferences.lastTestEmailSentAt = serverTimestamp() via Admin SDK.
7. Return delivery confirmation.
SUCCESS (200): { sent: true, recipientEmail: string, sentAt: timestamp }
ERRORS:
- 400: no delivery email configured
- 401: unauthenticated
- 429: rate limit exceeded (3/hour)
- 502: email provider delivery failure
- 500: Firestore read/write failure
SECURITY: Firebase ID token required. deliveryEmail sourced from Firestore/Firebase Auth only. Rate limited to 3/hour to prevent abuse.
