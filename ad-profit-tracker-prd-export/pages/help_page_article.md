## Description

FUNCTIONALITY: A full article reader page. The article title, category badge, author, last-updated date, and estimated read time display at the top. The article body renders markdown content (using react-markdown with remark-gfm for tables and code blocks) with syntax highlighting for code snippets. A sticky table-of-contents sidebar on desktop auto-generates from H2/H3 headings and highlights the active section as the user scrolls. A 'Was this helpful?' thumbs-up/thumbs-down widget at the bottom calls POST /api/help/feedback with the articleId and rating. A 'Related Articles' section below the feedback widget shows up to 4 articles from the same category, fetched from GET /api/help/articles?category=[category]&limit=4. A breadcrumb trail at the top (Help > [Category] > [Article Title]) links back to /help and the filtered category view. A 'Copy Link' button copies the article URL to clipboard. Video tutorial articles embed a responsive iframe or HTML5 video player. On mount, the page calls PATCH /api/help/articles/[articleId]/view to increment the view count (fire-and-forget, no UI dependency). A 'Back to Help Center' link at the top-left navigates to /help.
DESIGN APPROACH: Single-column content area (max-w-3xl centered) with sticky TOC sidebar on desktop (hidden on mobile). Markdown rendered with consistent typography — headings, code blocks, blockquotes styled with Tailwind prose classes. Progressive disclosure — TOC collapsed on mobile into a 'Jump to section' dropdown. Feedback widget is compact and non-intrusive at the bottom.
STATES: loading — article header shows shimmer title block and meta row, body shows 6 shimmer paragraph blocks, TOC shows 4 shimmer rows; empty — not applicable (article either exists or 404); error_401 — attempt Firebase ID token refresh via getIdToken(true), if refresh fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied — this article requires elevated permissions' with link to /help; error_404 — show 'Article not found' with link back to /help and a 'Browse all articles' CTA; error_500 — inline error banner with Retry button re-calling GET /api/help/articles/[articleId]; success — full article rendered with TOC, feedback widget, and related articles.
SECURITY: Protected route — middleware.ts enforces Firebase Auth. Bearer token on GET /api/help/articles/[articleId]. Articles with requiredPermission field are gated server-side — endpoint returns 403 if user lacks the permission.
