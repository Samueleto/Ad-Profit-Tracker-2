## Description

FUNCTIONALITY: A modal dialog triggered by a clock/calendar icon button on each saved report row in the CustomReportBuilderPage's saved reports sidebar panel. Users configure automated delivery for a specific saved report: (1) Frequency selector — Daily (sent at 8am user timezone), Weekly (day-of-week picker, Mon–Sun), or Monthly (day-of-month picker, 1–28) rendered as a segmented control; (2) Delivery email field — pre-filled with the user's Firebase Auth email, editable to any valid email address; (3) Report date range — a dropdown specifying what date window the scheduled report covers: 'Yesterday', 'Last 7 days', 'Last 30 days', 'Last 90 days', 'This month'; (4) Format toggle — Excel (.xlsx) or PDF; (5) Active toggle — enable/disable the schedule without deleting it. The report name is shown at the top as read-only context. A 'Next delivery' preview label shows the computed next run timestamp based on frequency and timezone. A 'Save Schedule' button calls POST /api/schedules/create. Existing schedules load via GET /api/schedules and pre-populate the form for editing via PATCH /api/schedules/:id. A 'Delete Schedule' button (destructive, requires confirm) calls DELETE /api/schedules/:id. A 'Send Test Now' button triggers POST /api/schedules/:id/send-now for immediate delivery preview.
DESIGN APPROACH: Centered modal (max-w-md) with backdrop overlay using existing ThemeProvider tokens. Compact vertical form layout — frequency at top, email below, date range and format in a two-column row, active toggle at bottom. 'Next delivery' preview in muted text below frequency selector. Progressive disclosure — advanced options (custom timezone override) collapsed under chevron. Destructive delete button at bottom-left, Save at bottom-right.
STATES: loading — 5 shimmer form rows while GET /api/schedules resolves; saving — Save button shows spinner, all inputs disabled; empty (no existing schedule) — form shows defaults (Daily, user email, Last 7 days, Excel, active=true); error_401 — close modal, call getIdToken(true), if refresh fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied' inside modal with close button; error_404 — show 'Report not found' inside modal with close button; error_500 — inline error banner inside modal with Retry; success — 'Schedule saved' toast, modal closes.
SECURITY: Protected — inherits CustomReportBuilderPage Firebase Auth middleware. Bearer token on all API calls. userId always from verified Firebase ID token server-side. Delivery email validated server-side.
