## Description

FUNCTIONALITY: A dedicated report builder page where users construct custom reports by selecting: (1) Metrics — a multi-select checkbox panel with 8 options (Revenue, Cost, Net Profit, ROI%, Impressions, Clicks, CTR%, CPM) with 'Select All' toggle; (2) Date Range — a date range picker (Last 7/14/30/90 days, This Month, Custom with calendar inputs, max 90-day span) independent from the dashboard; (3) Networks — multi-select checkboxes for ExoClick, RollerAds, Zeydoo, Propush with 'All Networks' toggle; (4) Grouping — a single-select segmented control (Daily / Country / Network) determining how rows are aggregated; (5) Data Quality filter — All / Clean Only / Anomalies Only toggle. A live 'Report Preview' panel on the right (desktop) or below (mobile) renders a paginated table of up to 20 rows using GET /api/filters/stats with the current configuration, updating on 300ms debounce after any config change. A row count badge per section shows estimated export counts from GET /api/export/preview. A 'Save Report' button opens a name-input inline field and calls POST /api/reports/save to persist the configuration. A 'Saved Reports' sidebar panel lists all saved reports from GET /api/reports with load, rename (PATCH /api/reports/:id), and delete (DELETE /api/reports/:id) actions. An 'Export' button opens the existing ExportModal pre-configured with the current date range and selected sections. A 'Run Report' button triggers GET /api/filters/stats with full params and renders the complete result in the preview table with pagination.
DESIGN APPROACH: Two-column layout on desktop — left column (380px) for configuration controls, right column for live preview table. Configuration panel uses card sections with clear headings per control group. Progressive disclosure — advanced options (data quality, custom filename) collapsed under chevron. Saved reports panel is a collapsible left drawer on desktop, bottom sheet on mobile. Preview table reuses MobileDataTableWrapper for responsive rendering. Consistent Tailwind semantic color classes throughout.
STATES: loading — left config panel renders immediately with static controls; preview table shows 5 shimmer rows while GET /api/filters/stats resolves on mount; saved reports list shows 3 shimmer rows. empty — preview shows 'No data matches your current configuration. Adjust filters or sync your networks.' with Sync Now link. error_401 — attempt getIdToken(true) Firebase ID token refresh; if refresh fails redirect to / with 'Session expired' toast. error_403 — show 'Access Denied' card with link to /dashboard. error_404 — preview shows 'No data found for selected range and filters' with date range reset button. error_500 — inline error banner in preview panel with Retry button that re-calls GET /api/filters/stats. success — preview table populated with color-coded rows, row count badges updated, Export and Save Report buttons enabled.
SECURITY: Protected route in (app) route group — middleware.ts enforces Firebase Auth before render. Bearer token attached to all API calls. userId always from verified Firebase ID token server-side. Date range capped at 90 days client-side before any API call. Saved report configs stored server-side scoped to authenticated uid only.
