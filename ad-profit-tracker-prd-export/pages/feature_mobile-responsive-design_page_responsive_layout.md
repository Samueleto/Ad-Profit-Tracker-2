## Description

FUNCTIONALITY: A root layout wrapper component (layout.tsx in the (app) route group) that orchestrates the responsive navigation experience across all protected pages. On desktop (md+): renders the existing desktop sidebar on the left (240px fixed width) with main content area filling the remaining width. On mobile (below md): hides the desktop sidebar, renders the MobileNavigationDrawer hamburger trigger in a top header bar (56px height), and renders the MobileBottomNavBar fixed at the bottom. The main content area on mobile has padding-bottom: 56px + safe-area-inset to prevent content from being obscured by the bottom nav. The layout wraps all (app) route group pages — FinancialMetricsDashboardPage, SettingsPage, OnboardingWizardPage, and all dashboard sections — so every page gets responsive nav automatically without per-page changes. The existing ThemeProvider wraps this layout so dark mode applies to all nav chrome. A useMediaQuery hook (extending the existing system preference detection from Dark Mode & UI Theming) provides the md breakpoint boolean to conditionally render nav variants. The layout also adds a mobile-optimized page title in the header bar showing the current route name.
DESIGN APPROACH: Single layout file handles all breakpoints — no duplicate page logic. CSS Grid for the desktop two-column layout (sidebar + content). Flexbox column for mobile (header + content + bottom nav). Tailwind responsive prefixes (md:) throughout — no JavaScript breakpoint switching for layout structure. The useMediaQuery hook is used only for conditional rendering of interactive components (drawer open state), not for layout itself.
STATES: loading: layout shell renders immediately (Server Component shell); inner page content uses Suspense boundaries with loading.tsx per route. error_401: middleware.ts (already created by auth feature) redirects unauthenticated requests to / before layout renders. error_403: not applicable at layout level. error_404: Next.js not-found.tsx handles per route. error_500: Next.js error.tsx handles per route. success: correct nav variant renders per viewport, content area fills remaining space.
SECURITY: Protected route group — middleware.ts enforces Firebase Auth before any (app) layout renders. Layout itself makes no API calls.
