## Description

FUNCTIONALITY: A slide-over panel (triggered by a bell icon in the top navigation bar showing an unread count badge) that displays the user's notification feed. Notifications are grouped into two tabs: 'All' and 'Unread'. Each notification row shows: a semantic icon (sync=arrows, error=red X, export=download, schedule=clock), a title (e.g., 'Sync Complete — ExoClick'), a body message (e.g., '342 rows fetched in 1.2s'), a relative timestamp (e.g., '3 minutes ago'), and a colored left border (green=success, red=error, amber=warning, blue=info). A 'Mark All as Read' button at the top clears the unread badge. Individual notifications have an X dismiss button that calls PATCH /api/notifications/dismiss/:id. A 'Clear All' button (destructive, requires single confirm click) calls DELETE /api/notifications/clear. Clicking a sync-failure notification navigates to the ManualRefreshPanel section on /dashboard. Clicking an export notification does nothing (already downloaded). Clicking a schedule-failure notification navigates to /reports. The panel loads the first 20 notifications on open and supports 'Load more' pagination via GET /api/notifications. The bell icon badge count is driven by users.notifications array filtered to unread items, read client-side from the Zustand dashboard store which is hydrated on app mount.
DESIGN APPROACH: Right-side slide-over panel (360px wide on desktop, full-screen on mobile) with backdrop overlay. Notification rows are compact (56px height) with icon, text block, timestamp, and dismiss button. Grouped tabs at top. Semantic left border color per notification type. Progressive disclosure — body text truncated to one line, expandable on click. Empty state is friendly, not alarming.
STATES: loading — 5 shimmer rows matching notification row structure while GET /api/notifications resolves; empty — bell icon shows no badge, panel shows 'You're all caught up' with a checkmark illustration; error_401 — close panel, call getIdToken(true), if refresh fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied' inside panel with close button; error_404 — show 'No notifications found' with close button; error_500 — inline error banner inside panel with Retry button that re-calls GET /api/notifications; success — notification rows rendered with unread count badge on bell icon, 'Mark All as Read' button active if any unread exist.
SECURITY: Protected — inherits FinancialMetricsDashboardPage Firebase Auth middleware. Bearer token on all API calls. userId always from verified Firebase ID token server-side. Notifications scoped to authenticated user only.
