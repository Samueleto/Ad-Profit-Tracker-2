## Description

FUNCTIONALITY: A full-width card section integrated into the existing FinancialMetricsDashboardPage, positioned below the DailyProfitTrendSection card. Displays a ranked table of countries/GEOs with columns: country flag emoji + name, revenue, cost, net profit, ROI%, impressions, clicks, and a metric share bar (visual percentage of total). A metric toggle in the card header lets users switch the sort dimension between Revenue, Cost, and Profit. A 'Top N' selector (10/20/50) controls how many countries are shown. A free-text country search input filters the visible rows client-side. Each table row is color-coded using the project-wide semantic classes (green-50/text-green-600 for positive ROI, red-50/text-red-600 for negative, amber-50/text-amber-600 for zero/null) derived from per-country colorCode returned by GET /api/roi/breakdown with dimension='country'. A collapsible 'GEO Insights' strip below the table shows: top-performing country, worst-performing country, number of countries with positive ROI, and total countries tracked — collapsed by default with a chevron toggle. Clicking any country row opens a drill-down modal showing that country's daily trend for the selected date range, fetched via GET /api/stats/snapshot with country filter. A 'Network Breakdown' toggle on each row expands an inline sub-row showing per-network contribution for that country. Null revenue/cost cells render as '—' not '$0'. The date range syncs with the parent dashboard shared state — no independent selector. Data loads from GET /api/stats/geo-breakdown for the table and GET /api/roi/breakdown with dimension='country' for colorCode enrichment.
DESIGN APPROACH: Full-width card with sticky card header containing title, metric toggle (Revenue/Cost/Profit segmented control), Top N selector, and country search input all on one row. Dense scannable table with compact row height. Metric share rendered as a thin inline progress bar in the share column. Progressive disclosure — GEO Insights strip collapsed by default, per-network sub-rows expand inline. Null data shown as '—' dash, never coerced to zero. Consistent with project-wide Tailwind semantic color classes.
STATES: loading — table shows 10 shimmer rows matching column structure, card header controls are skeleton placeholders; empty — 'No geographic data for this date range. Sync your networks to see country breakdowns.' with a Sync Now CTA linking to ManualRefreshPanel; error_401 — attempt Firebase ID token refresh via getIdToken(true), if refresh fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied' inline within the card with link to /dashboard; error_404 — show 'No geographic data found for selected range' with a date range reset button; error_500 — show inline error banner inside the card with Retry button that re-calls both GET /api/stats/geo-breakdown and GET /api/roi/breakdown; success — ranked country table rendered with color-coded rows, metric share bars, and GEO Insights strip available to expand.
SECURITY: Protected route — inherits FinancialMetricsDashboardPage Firebase Auth middleware. Bearer token attached to all API calls via Authorization header. Date range capped at 90 days client-side before API call. userId always from verified Firebase ID token server-side. No new auth surface introduced.
