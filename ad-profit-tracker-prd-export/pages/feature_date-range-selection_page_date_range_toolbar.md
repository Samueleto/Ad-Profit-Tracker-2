## Description

FUNCTIONALITY: A persistent date range control bar integrated into the top of FinancialMetricsDashboardPage, above the KPI cards. Users can select from six preset options — Last 7 Days, Last 14 Days, Last 30 Days, Last 90 Days, This Month, and Custom — rendered as a segmented pill control. Selecting any preset immediately updates the shared dashboard date range state (Zustand store or lifted useState at FinancialMetricsDashboardPage level), triggering re-fetches across all active dashboard tabs and sections. Selecting 'Custom' reveals an inline date range picker (two calendar inputs: From and To) using a lightweight date picker library (e.g., react-day-picker or react-datepicker). The custom picker enforces a 90-day maximum span — if the user selects a range exceeding 90 days, an inline validation message appears: 'Maximum range is 90 days' and the Apply button is disabled. Dates are serialized to YYYY-MM-DD format before writing to shared state. A 'data availability' indicator dot appears next to the date range label, sourced from GET /api/stats/dates, showing whether the selected range has complete, partial, or no data. An 'Apply' button confirms custom selections; preset selections apply immediately without confirmation. A small 'Last synced' timestamp badge shows the most recent sync time across all networks from the shared dashboard state.
DESIGN APPROACH: Compact single-row toolbar that does not scroll away — sticky below the sidebar nav on desktop. Preset options as a horizontal segmented control (pill buttons), with the active preset highlighted. Custom date inputs appear inline to the right of the presets when selected, not in a modal. Validation errors appear as a small red text below the date inputs. Consistent with project-wide Tailwind semantic classes. On mobile, presets collapse into a dropdown select and the custom picker opens as a bottom sheet.
STATES: loading — preset buttons render immediately (no loading state needed for the control itself); data availability dot shows a gray spinner while GET /api/stats/dates resolves; empty — dot shows gray 'No data' indicator if GET /api/stats/dates returns empty for the range; error_401 — attempt Firebase ID token refresh via getIdToken(true), if refresh fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied' inline within the toolbar; error_404 — data availability dot shows gray 'No data for this range'; error_500 — data availability dot shows amber warning icon, toolbar remains functional; success — active preset highlighted, data availability dot shows green (complete), amber (partial), or gray (no data) based on GET /api/stats/dates response.
SECURITY: Protected — inherits FinancialMetricsDashboardPage Firebase Auth middleware. Bearer token attached to GET /api/stats/dates. Date range capped at 90 days client-side before any API call. userId always from verified Firebase ID token server-side.
