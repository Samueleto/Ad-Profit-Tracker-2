## Description

FUNCTIONALITY: A dedicated ROI indicator section integrated into the existing FinancialMetricsDashboardPage. Displays a prominent ROI % KPI card with color-coded semantic indicator (green for positive ROI, red for negative ROI, amber for zero or null ROI). Shows the computed ROI value with two decimal places, a directional arrow (↑/↓/→) indicating trend vs prior period, and a percentage change label (e.g., '+12.4% vs prior period'). When ROI is null (totalCost === 0 or no cost data), displays 'N/A' with an amber neutral badge and tooltip explaining 'No cost data available for this period'. A breakeven threshold line on the daily profit trend chart marks the zero-profit axis. Per-network ROI contribution rows show each revenue network's effective yield relative to ExoClick cost. A collapsible 'ROI Breakdown' panel expands to show the formula: (Revenue - Cost) / Cost × 100, with live substituted values. The section reads from GET /api/dashboard/metrics kpis.roi and kpis.roiChange fields — no additional API calls needed.
DESIGN APPROACH: Integrated as a highlighted section within the existing 4-column KPI card grid on /dashboard. ROI card uses a larger font weight than other KPI cards to signal its importance as the primary profitability signal. Color coding uses Tailwind semantic classes: text-green-600/bg-green-50 for positive, text-red-600/bg-red-50 for negative, text-amber-600/bg-amber-50 for zero/null. Breakeven line on chart is a dashed gray horizontal rule at y=0. Progressive disclosure — formula breakdown is collapsed by default.
STATES: loading — ROI card shows shimmer placeholder matching card dimensions, arrow indicator hidden; empty — 'No data for selected range' with amber N/A badge; error_401 — attempt Firebase ID token refresh via getIdToken(true), if fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied' inline within the card with link to /dashboard; error_404 — display 'N/A' with tooltip 'No data found for this date range'; error_500 — show error banner within the ROI card with retry button that re-calls GET /api/dashboard/metrics; success — ROI value rendered with color-coded badge, directional arrow, and prior-period change percentage.
SECURITY: Protected route — inherits FinancialMetricsDashboardPage auth. Bearer token attached to GET /api/dashboard/metrics. userId always from verified Firebase ID token server-side. No additional auth surface introduced.
