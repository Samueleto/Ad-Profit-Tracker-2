## Description

FUNCTIONALITY: A full in-app knowledge base where users can browse and search help content. A prominent search bar at the top accepts free-text queries and calls GET /api/help/search on 300ms debounce, rendering matching articles inline. Below the search bar, a category grid displays all help categories (FAQs, API Guides, Troubleshooting, Video Tutorials, Network Setup, Billing & Account) as clickable cards with icon, title, and article count badge. Clicking a category filters the article list below. The article list renders as scannable rows with title, category badge, estimated read time, and a 'New' badge for articles published in the last 7 days. Clicking any article row navigates to /help/[articleId]. A 'Role-Specific Guides' section surfaces articles tagged with the user's current workspaceRole (fetched from GET /api/rbac/my-permissions on mount) — e.g., Owners see 'Managing Team Members', Members see 'Viewer Guide'. A 'Popular Articles' sidebar panel shows the top 5 most-viewed articles. A 'Contact Support' CTA at the bottom links to an email mailto. All article content is fetched from GET /api/help/articles with optional category and role filters.
DESIGN APPROACH: Two-column layout on desktop — main article list (left, 2/3 width) and sidebar with Popular Articles and role-specific guides (right, 1/3 width). Single-column on mobile. Search bar is prominent and sticky below the page header. Category cards use icon + label in a 3-column grid. Progressive disclosure — category filter collapses on mobile into a horizontal scroll pill row. Consistent ThemeProvider tokens for dark mode.
STATES: loading — category grid shows 6 shimmer cards, article list shows 8 shimmer rows matching row structure; empty — 'No articles found. Try a different search or browse categories.' with a 'Clear Search' button; error_401 — attempt Firebase ID token refresh via getIdToken(true), if refresh fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied' card with link to /dashboard; error_404 — show 'No articles found for this category' with a 'Browse All' button; error_500 — inline error banner with Retry button re-calling GET /api/help/articles; success — category grid and article list populated, role-specific section visible.
SECURITY: Protected route in (app) route group — middleware.ts enforces Firebase Auth. Bearer token on all API calls. Role-specific article filtering uses GET /api/rbac/my-permissions — uid always from verified Firebase ID token server-side.
