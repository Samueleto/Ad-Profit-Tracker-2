## Description

FUNCTIONALITY: Dedicated team management page within the settings area. Displays the current workspace name (editable inline) and a member roster table with columns: avatar, display name, email, role badge (Owner/Admin/Member), joined date, and action menu (Change Role, Remove Member). An 'Invite Member' button opens the InviteMemberModal. The workspace owner sees all controls; Admins can invite and remove Members but cannot change other Admins or the Owner; Members see the roster read-only. A 'Pending Invitations' section below the roster lists outstanding invites with columns: email, invited by, sent date, expiry, and a 'Revoke' button (Admin/Owner only). A 'Leave Workspace' button (non-owners) triggers a confirmation dialog before calling DELETE /api/team/members/:memberId. Workspace owners see a 'Transfer Ownership' option in the action menu for other admins. Role changes call PATCH /api/team/members/:memberId/role. Member removal calls DELETE /api/team/members/:memberId. Pending invite revocation calls DELETE /api/team/invitations/:invitationId.
DESIGN APPROACH: Integrated as a new 'Team' tab on the existing SettingsPage at /settings, keeping all settings consolidated. Dense scannable table with compact row height. Role badges use semantic colors (indigo=Owner, blue=Admin, gray=Member). Action menus use a three-dot dropdown per row. Pending invitations section uses a lighter background to visually separate from active members. Progressive disclosure — advanced workspace settings (rename, transfer ownership) collapsed under a 'Workspace Settings' chevron at the top.
STATES: loading — member table shows 4 shimmer rows matching column structure, pending invitations shows 2 shimmer rows; empty (no other members) — 'You are the only member. Invite your team to collaborate.' with prominent Invite Member CTA; error_401 — attempt Firebase ID token refresh via getIdToken(true), if refresh fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied — you do not have permission to manage this workspace' card with link to /dashboard; error_404 — show 'Workspace not found' with link to /dashboard; error_500 — inline error banner with Retry button that re-calls GET /api/team/members and GET /api/team/invitations; success — member roster table populated with role badges, pending invitations section shown if any exist.
SECURITY: Protected route in (app) route group — middleware.ts enforces Firebase Auth. Role-based UI rendering: Owner sees all controls, Admin sees invite/remove for Members only, Member sees read-only roster. All mutations verified server-side against users.workspaceRole — client role state is display-only.
