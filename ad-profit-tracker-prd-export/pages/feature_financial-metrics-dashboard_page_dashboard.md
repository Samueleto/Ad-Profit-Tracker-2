## Description

FUNCTIONALITY: The primary dashboard page displaying aggregated financial KPIs for the authenticated user's ad network portfolio. Four prominent KPI cards at the top show Total Revenue (sum of RollerAds + Zeydoo + Propush), Total Cost (ExoClick only), Net Profit (Revenue - Cost), and ROI % ((NetProfit/Cost)*100). Each KPI card shows the current period value, a percentage change vs. the prior period, and a directional arrow indicator. A date range selector (Last 7 / 14 / 30 days, This Month, Custom) controls all data on the page. A 'Refresh Data' button calls POST /api/cache/invalidate then re-fetches all KPI data. A data freshness badge ('Live' vs 'Cached · X min ago') is derived from GET /api/cache/status. Below the KPI cards, a daily profit trend line chart (recharts) renders the series from GET /api/stats/trend with metric='profit'. A GEO breakdown table shows top 10 countries by net profit from GET /api/stats/geo-breakdown. Per-network summary rows show each network's primary metric (cost or revenue), last sync time, and sync status badge. The ManualRefreshPanel (already built) is embedded as a collapsible section at the bottom. An 'Export' button opens a modal for Excel/PDF export (links to reporting feature). All data fetching is coordinated through GET /api/stats/summary for KPIs and GET /api/stats/trend for the chart.
DESIGN APPROACH: Dashboard-first dense layout — KPI cards in a 4-column grid (2-column on tablet, 1-column on mobile), chart below spanning full width, GEO table and per-network rows in a 2-column split. Sticky date range selector at top. Progressive disclosure — per-network detail rows collapse by default. Semantic color coding: green for profit/positive ROI, red for loss/negative ROI, amber for neutral/zero.
STATES: loading — four KPI card skeletons with shimmer, chart area shows a gray placeholder rectangle, GEO table shows 5 shimmer rows; empty — 'No data for this date range. Try syncing your networks first.' with a 'Sync Now' CTA button linking to the ManualRefreshPanel; error_401 — attempt Firebase ID token refresh via getIdToken(true), if refresh fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied' card with link to /dashboard; error_404 — show 'No data found for selected range' inline message with date range reset button; error_500 — show error banner at top of page with 'Retry' button that re-fetches all endpoints; success — KPI cards populated with values, trend chart rendered, GEO table populated with country rows.
SECURITY: Protected route — middleware enforces Firebase Auth. Bearer token attached to all API calls. userId always from verified token server-side. Date range capped at 90 days to prevent expensive Firestore scans.
