## Description

FUNCTIONALITY: A persistent control panel section integrated into the existing dashboard that lets users trigger immediate data pulls from all active ad networks or individual networks. The panel displays a 'Refresh All Data Now' primary button that calls POST /api/sync/manual with no networkId (triggers all active networks). Below it, four per-network rows (ExoClick, RollerAds, Zeydoo, Propush) each showing: network name, last synced timestamp (relative, e.g. '2 hours ago'), last sync status badge (success=green, failed=red, never=gray, in-progress=amber spinner), and an individual 'Refresh' icon button. Clicking a per-network refresh calls POST /api/sync/manual with that networkId. After any refresh is triggered, the panel polls GET /api/scheduled/sync-status every 3 seconds (up to 60s or until all triggered networks show a terminal status) to reflect live progress. A 'View Sync History' link opens GET /api/scheduled/sync-history results in a slide-over drawer showing the last 10 sync events per network with rowsFetched, latencyMs, and status. Rate limit feedback: if 429 is returned, the button disables with a countdown timer showing when the next refresh is allowed.
DESIGN APPROACH: Compact card embedded in the dashboard sidebar or top action bar — not a full page. Per-network rows use a dense list layout with status badges inline. Polling is bounded and stops automatically. Destructive actions (refresh all) use a confirmation tooltip on first use.
STATES: loading: skeleton rows for each network with shimmer status badges while initial sync-status loads on mount. empty: 'No networks configured — go to Settings to add API keys' with link to /settings. error_401: attempt Firebase ID token refresh via getIdToken(true) → if fails, redirect to / with 'Session expired' toast. error_403: show 'Access Denied' inline banner with link to /dashboard. error_404: not applicable (sync-status returns defaults for unconfigured networks). error_500: inline error banner with 'Retry' button that re-fetches sync-status. success: all four network rows render with live status badges and last-synced timestamps. in_progress: triggered networks show amber spinner badge; Refresh button for that network is disabled; 'Refresh All' button shows 'Syncing...' with spinner and is disabled. rate_limited: triggered button shows disabled state with countdown 'Available in Xm Ys'.
SECURITY: Protected route — requires authenticated Firebase user. Bearer token attached to all API calls via getIdToken(). Rate limit enforced server-side (3 manual syncs/hour per user for sync-all; 10/hour for per-network). UI reflects rate limit state from 429 responses.
