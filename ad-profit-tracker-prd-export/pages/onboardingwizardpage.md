## Description

FUNCTIONALITY: Multi-step guided wizard that new users see exactly once after their first login. Step 1 — Welcome: displays the user's name (from Firebase Auth displayName), a brief value proposition for Ad Profit Tracker, and a 'Get Started' CTA. Step 2 — Profile Setup: pre-filled displayName field (editable), timezone selector (dropdown of IANA timezone strings), and defaultDateRange selector (Last 7 days / Last 30 days / This month). Submits via PATCH /api/auth/update-profile to persist preferences into users.preferences. Step 3 — API Keys Teaser: explains that API keys for ExoClick, RollerAds, Zeydoo, and Propush are needed to pull data, with a 'Go to Settings' button that routes to /settings and a 'Skip for now' link. Step 4 — Done: animated checkmark, 'Your dashboard is ready' message, and a 'Go to Dashboard' button that routes to /dashboard. Progress bar at top shows current step (1 of 4). Back/Next navigation between steps. On completion, writes onboardingCompletedAt timestamp to users document via PATCH /api/auth/update-profile. Middleware checks users.onboardingCompletedAt — if set, redirect away from /onboarding to /dashboard.
DESIGN APPROACH: Centered card (max-w-lg) with step progress bar at top. Each step occupies the same card — content swaps with a subtle slide transition. Minimal distractions — no sidebar, no nav frame during onboarding. Step indicator shows numbered dots. Back button is ghost-styled; Next/primary action is filled. Skip links are small and de-emphasized.
STATES: Loading (initial): skeleton shimmer in the card body while user profile loads from GET /api/auth/get-user. Submitting (step 2): spinner on Next button, fields disabled. error_401: attempt Firebase ID token refresh → if fails, redirect to / with 'Session expired' toast. error_403: show 'Access Denied' card with link to /dashboard. error_404: user document not found — call POST /api/auth/sync-user to recreate, then retry. error_500: inline error banner with 'Retry' button inside the card. Success: step advances; on final step completion, redirect to /dashboard.
SECURITY: Protected route — requires authenticated Firebase user. Middleware enforces auth. If onboardingCompletedAt already set on users document, redirect to /dashboard immediately. Users can only update their own document via PATCH /api/auth/update-profile.
