## Description

FUNCTIONALITY: A responsive wrapper component applied to all data-heavy tables across the dashboard — GeoBreakdownSection, PerNetworkAnalyticsTabsSection, ComparativeNetworkAnalysisTab, PerformanceBenchmarkingTab, and the SettingsPage activity log. On desktop (md+): renders the standard HTML table with all columns visible. On mobile (below md): switches to a card-list layout where each table row becomes a compact card showing the two most important columns (e.g., country name + net profit for geo table, network name + primary metric for network tables) with a 'Show more' chevron that expands inline to reveal remaining columns. Alternatively, for tables where column scanning is essential (e.g., the network comparison table), wraps the table in a horizontally scrollable container with a subtle left/right scroll shadow indicator so users know more columns exist. The component accepts a columns config prop defining which columns are 'primary' (always visible on mobile) vs 'secondary' (hidden until expanded). Metric share bars (MetricShareBar component) reflow to full-width on mobile. Color-coded row backgrounds (green/red/amber semantic classes) are preserved in card layout. Null cells still render as '—'. The wrapper is a pure Client Component ('use client') since it needs to track expanded card state with useState.
DESIGN APPROACH: Mobile-first progressive enhancement — card layout is the default, table layout is the md: variant. Horizontal scroll tables use overflow-x-auto with -webkit-overflow-scrolling: touch for momentum scrolling on iOS. Scroll shadow indicators use CSS mask-image gradient. Expanded card state is local useState per row — no global state needed. Card layout uses the same ThemeProvider tokens as table rows for consistent dark mode support.
STATES: loading: card layout shows shimmer rectangles matching card dimensions (same count as table skeleton rows defined per parent section). empty: parent section handles empty state — wrapper renders nothing. error_401: parent section handles — wrapper is presentational. error_403: not applicable. error_404: parent section handles. error_500: parent section handles. success: card list or scrollable table rendered per viewport, expanded cards show all columns.
SECURITY: Purely presentational — no API calls, no auth surface. Inherits parent section's auth context.
