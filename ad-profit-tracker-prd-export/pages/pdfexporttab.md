## Description

FUNCTIONALITY: The PDF tab inside the existing ExportModal (already built by Excel Export Functionality). Users configure and initiate PDF report generation. Section checkboxes let users choose what to include: Cover Page (app logo, date range, generated timestamp), Executive Summary (KPI cards: Total Revenue, Total Cost, Net Profit, ROI%), Daily Profit Trend Chart (recharts chart captured as image via html2canvas and embedded), Geographic Breakdown Table (top countries by net profit), Per-Network Stats Tables (one section per network), and Activity Log (audit entries). An orientation selector (Portrait / Landscape) and paper size selector (A4 / Letter) appear below the checkboxes. A row count badge per section (reused from GET /api/export/preview already loaded by the Excel tab) shows estimated rows. An 'Export Now' button calls POST /api/export/pdf, which streams back a .pdf file download. A progress indicator (indeterminate spinner) shows while the server generates the file. On success, the browser triggers automatic download and a 'PDF exported' toast appears. A 'Cancel' button closes the modal. If no data exists for the selected range, Export Now is disabled with tooltip 'No data available for this range'.
DESIGN APPROACH: Reuses the existing ExportModal two-tab layout — this is the second tab ('PDF'). Same compact checkbox rows with row count badges as the Excel tab. Orientation and paper size selectors appear as a two-column inline row below the checkboxes under an 'Options' label. Progressive disclosure — advanced options (custom filename) collapsed under a chevron. Consistent ThemeProvider tokens (dark:bg-gray-900/bg-white) and accent color CSS custom property for the Export Now button.
STATES: loading — 6 shimmer checkbox rows while preview row counts load from GET /api/export/preview (shared with Excel tab, no duplicate fetch); exporting — Export Now button shows spinner and is disabled, all checkboxes and selectors disabled, progress message 'Generating your PDF report...'; empty — Export Now disabled with tooltip 'No data for selected range'; error_401 — close modal, call getIdToken(true) to refresh Firebase ID token, if refresh fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied' inline inside the PDF tab with close button; error_404 — disable Export Now with inline message 'No data found for this date range'; error_500 — show inline error banner inside the PDF tab with 'Retry' button that re-calls POST /api/export/pdf; success — file download triggers automatically, toast 'PDF export complete — file downloaded', modal closes after 1.5s.
SECURITY: Protected — inherits FinancialMetricsDashboardPage Firebase Auth middleware. Bearer token sent with POST /api/export/pdf. userId always from verified Firebase ID token server-side. Export scoped to authenticated user's data only.
