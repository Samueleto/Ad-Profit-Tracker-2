## Description

FUNCTIONALITY: A tabbed section integrated into the existing FinancialMetricsDashboardPage, positioned below the GeoBreakdownSection. Four tabs — ExoClick, RollerAds, Zeydoo, Propush — each rendering a dedicated analytics panel for that network. Each tab displays: (1) a KPI summary row with the network's primary metric (Cost for ExoClick, Revenue for others), impressions, clicks, CTR%, and CPM; (2) a daily trend AreaChart (recharts) showing the primary metric over the selected date range, with color-coded fill (green for revenue networks, amber for cost network), breakeven line at y=0, and interactive tooltips; (3) a GEO breakdown table showing top countries for that network with columns: flag+name, primary metric, impressions, clicks, CTR%, metric share bar, and colorCode row coloring (green/red/amber from GET /api/roi/breakdown?dimension='country' filtered to this network); (4) a 'Network Insights' collapsible strip showing best country, worst country, total countries tracked, and last sync timestamp. A 'Sync This Network' button per tab triggers POST /api/sync/manual with the specific networkId. Clicking any country row opens the existing GeoCountryDrilldownModal. Null cells render as '—'. The MetricToggle, MetricShareBar, TopNSelector, and collapsible insights strip components are reused from GeoBreakdownSection. Date range syncs with parent dashboard shared state — no independent selector. Data loads from GET /api/networks/stats per network tab on activation (lazy load — only fetch when tab is first activated).
DESIGN APPROACH: Tab bar with network name and a status dot (green=synced, red=failed, gray=never) per tab. Each tab panel is a full-width card with KPI row at top, chart below (280px height), then GEO table. Dense but scannable — compact row height in GEO table. Progressive disclosure: Network Insights strip collapsed by default with chevron. Lazy tab loading to avoid 4 simultaneous API calls on mount. Reuse all shared components (MetricToggle, MetricShareBar, TopNSelector, DrawerModal) from GeoBreakdownSection.
STATES: loading — KPI row shows 4 shimmer stat cards, chart area shows 280px shimmer rectangle, GEO table shows 8 shimmer rows; empty — 'No data for [NetworkName] in this date range. Sync this network to see analytics.' with a Sync button; error_401 — attempt Firebase ID token refresh via getIdToken(true), if refresh fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied' inline within the tab panel with link to /dashboard; error_404 — show 'No data found for [NetworkName] in selected range' with date range reset button; error_500 — show inline error banner inside the tab panel with Retry button that re-calls GET /api/networks/stats for this network; success — KPI row populated, recharts AreaChart rendered, GEO table populated with color-coded rows.
SECURITY: Protected route — inherits FinancialMetricsDashboardPage Firebase Auth middleware. Bearer token attached to all API calls. Date range capped at 90 days client-side. userId always from verified Firebase ID token server-side.
