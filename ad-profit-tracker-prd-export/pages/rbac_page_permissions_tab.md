## Description

FUNCTIONALITY: A 'Permissions' tab integrated into the existing TeamManagementPage at /settings/team. Displays a role matrix table showing all four roles (Owner, Admin, Member, Viewer) as columns and all permission categories as rows: Dashboard View, Export Data, Manage API Keys, Sync Networks, View Reports, Create Reports, Manage Team, Manage Billing, View Audit Logs, Manage Benchmarks. Each cell shows a green checkmark or red X indicating whether that role has that permission. A 'Role Descriptions' accordion below the matrix expands to show a plain-language description of each role's intended use case. An 'Effective Permissions' section shows the current authenticated user's own resolved permission set as a list of granted/denied capabilities — useful for members to understand what they can and cannot do. Admins and Owners see a 'Customize Role Permissions' button that opens the RolePermissionsEditModal. A 'Copy Role' dropdown lets Owners duplicate an existing role's permission set as a starting point. Role badge legend at the top maps color codes (indigo=Owner, blue=Admin, gray=Member, teal=Viewer) to role names.
DESIGN APPROACH: Matrix table with sticky first column (permission name) and color-coded column headers per role. Dense but scannable — compact row height. Checkmarks use semantic green/red Tailwind classes. Progressive disclosure — role descriptions collapsed by default. Effective permissions section uses a two-column card layout.
STATES: loading — matrix table shows shimmer rows matching structure, 4 column headers render immediately; empty — not applicable (matrix always shows all permissions); error_401 — attempt Firebase ID token refresh via getIdToken(true), if fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied — insufficient permissions to view role matrix' inline; error_404 — show 'Permissions configuration not found' with retry; error_500 — inline error banner with Retry button re-calling GET /api/rbac/permissions; success — full matrix rendered with checkmarks, effective permissions card populated.
SECURITY: Protected — inherits TeamManagementPage Firebase Auth middleware. Read access for all authenticated workspace members. Edit access restricted to Owner and Admin roles enforced server-side.
