## Description

FUNCTIONALITY: A 'Benchmarks' tab integrated into the FinancialMetricsDashboardPage tab bar, positioned after the Compare tab. Displays the user's actual metrics side-by-side with two reference points: (1) their own historical averages computed from a 90-day rolling window of adStats, and (2) industry benchmark targets stored in the benchmarks Firestore collection. A MetricToggle (reused from ComparativeNetworkAnalysisTab) lets users switch the active metric: ROI%, CTR%, CPM, Revenue, Cost, Impressions, Clicks. Four KPI comparison rows — one per network — each show: actual value (from GET /api/networks/comparison), historical average (from GET /api/benchmarks/performance with type='historical'), industry benchmark (from GET /api/benchmarks/performance with type='industry'), and a MetricShareBar (reused) showing where the user's actual sits relative to the benchmark range. A 'Performance Score' summary card at the top aggregates all metrics into a single 0–100 score with a color-coded gauge (green ≥ 80, amber 50–79, red < 50). A trend comparison strip below the KPI rows shows a sparkline for the user's selected metric over the last 30 days vs the historical average line, rendered with recharts. A 'Benchmark Settings' button opens a modal (BenchmarkSettingsModal) to configure custom industry targets per metric. A data freshness caveat badge appears on any network row where networkStatus.lastSyncStatus is 'failed' or circuitBreakerOpen is true. Null actual values render as '—'. Date range syncs with parent dashboard shared state — no independent selector. Data loads lazily on first tab activation via Promise.all firing GET /api/networks/comparison and GET /api/benchmarks/performance in parallel.
DESIGN APPROACH: Tab panel with a prominent 'Performance Score' gauge card at top-right, metric comparison table as the main content, and sparkline trend strip below. Dense but scannable — compact rows with inline MetricShareBar. Progressive disclosure: historical vs industry toggle on the reference column header. Consistent Tailwind semantic color classes: green-500 for above benchmark, red-500 for below, amber-500 for within 10% of benchmark. Stale-data caveat badges are amber with a warning icon, non-blocking.
