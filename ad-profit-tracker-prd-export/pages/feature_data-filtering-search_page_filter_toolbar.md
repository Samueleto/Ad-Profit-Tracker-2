## Description

FUNCTIONALITY: A persistent secondary filter bar integrated into FinancialMetricsDashboardPage, positioned directly below the existing DateRangeToolbar. Users can filter all dashboard tables and charts by: (1) Network multi-select — checkboxes for ExoClick, RollerAds, Zeydoo, Propush with 'All Networks' toggle; (2) Country/GEO search — a type-ahead input that queries GET /api/filters/options?type=country to populate a dropdown of countries present in the user's adStats data for the active date range; (3) Metric focus toggle — single-select pill group (Revenue / Cost / Profit / ROI) that sets the primary sort metric across all tables; (4) Data quality filter — a toggle to show 'All', 'Anomalies Only', or 'Clean Only' rows, reading from adStats.validationStatus; (5) Free-text search — a debounced input (300ms) that filters visible table rows client-side by country name or network name. An 'Apply Filters' button confirms multi-select changes; single-select pills apply immediately. A 'Clear All' button resets all filters to defaults and writes cleared state to the shared Zustand dashboard store. An active filter count badge appears on a 'Filters' label when any non-default filter is active. Filter state is written to the shared Zustand store (extending the existing dateFrom/dateTo slice with selectedNetworks, selectedCountries, selectedMetric, dataQuality, searchQuery fields) so all dashboard sections re-fetch with the new params.
DESIGN APPROACH: Compact single-row sticky bar matching the DateRangeToolbar height and style. Network checkboxes in a compact inline group. Country search as a combobox with type-ahead. Metric pills as a segmented control. Data quality as a three-state toggle. Free-text search at the far right. On mobile, all controls collapse into a 'Filters' button that opens a bottom sheet with the full filter panel. Active filter badge uses amber dot on the Filters label.
STATES: loading — filter options (country list) show a spinner in the dropdown while GET /api/filters/options resolves; country dropdown shows 'Loading countries...' placeholder; all other controls render immediately from static config. empty — country dropdown shows 'No countries found for this date range' when GET /api/filters/options returns empty. error_401 — attempt Firebase ID token refresh via getIdToken(true); if refresh fails, redirect to / with 'Session expired' toast. error_403 — show 'Access Denied' inline within the toolbar row. error_404 — country dropdown shows 'No data available' with a muted message. error_500 — country dropdown shows amber warning icon with 'Could not load countries' and a retry link; other controls remain functional. success — country combobox populated with countries from user's data, all controls interactive, active filter count badge updates as filters change.
SECURITY: Protected — inherits FinancialMetricsDashboardPage Firebase Auth middleware. Bearer token attached to GET /api/filters/options. Filter state is client-side only — server enforces userId scoping on all downstream API calls. No filter values are trusted server-side without re-validation against the authenticated user's data.
