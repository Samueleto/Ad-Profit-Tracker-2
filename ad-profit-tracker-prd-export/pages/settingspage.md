## Description

FUNCTIONALITY: A dedicated page for managing API keys for ExoClick, RollerAds, Zeydoo, and Propush. The page displays a separate form/card for each ad network. Each form contains a password-style input for the API key with a show/hide toggle, a 'Save' button, and a 'Disconnect' button. The page fetches the connection status (not the keys themselves) on load to show whether each network is 'Connected' or 'Not Connected'. Saving a key triggers a server-side encryption and storage process. Disconnecting removes the key. 
DESIGN APPROACH: The page uses a clean, sectioned layout with a card for each ad network integration. This modular approach makes it easy to add more networks later. The state of each card (e.g., showing a 'Connect' button vs. a 'Connected' status with an 'Update' button) is driven by data from the `/api/keys/status` endpoint. Forms are handled as Client Components within a Server Component page shell.
STATES: Loading: skeleton cards for each network. Empty: forms are visible with placeholder text and 'Save' buttons enabled. Submitting: spinner on the 'Save' button, input field disabled. Success: toast notification 'API key saved successfully' and the card's status updates to 'Connected'. Error: inline error message below the input field and a toast for server errors.
SECURITY: Protected route, requires authenticated user. Redirects to login if unauthenticated. API keys are never sent to or stored on the client.
