## Description

FUNCTIONALITY: Modal overlay triggered on the root route and any protected route redirect. Presents two auth methods: Google OAuth (single button, triggers signInWithPopup with GoogleAuthProvider) and email/password (two fields with show/hide toggle on password). Tab or toggle switches between 'Sign In' and 'Sign Up' modes within the same modal. Sign-in calls signInWithEmailAndPassword; sign-up calls createUserWithEmailAndPassword then writes the user doc to Firestore users collection. On success, onAuthStateChanged fires and the modal closes, redirecting to /dashboard. 'Forgot password?' link triggers sendPasswordResetEmail inline — no separate page. Error messages appear inline below the relevant field.
DESIGN APPROACH: Centered modal with backdrop blur. Minimal two-tab layout (Sign In / Sign Up) to avoid separate routes. Google button visually primary for OAuth, email form secondary. Progressive disclosure — confirm password field only appears in Sign Up tab. Modal is dismissible only after successful auth.
STATES: Loading: spinner on active button, all inputs disabled. Google OAuth loading: button shows spinner. Email error: inline message below field (e.g., 'Wrong password' or 'Email already in use'). Network error: toast with retry. Success: modal closes, router.push('/dashboard').
SECURITY: Public page. Firebase Auth handles credential validation. Rate limiting enforced by Firebase Auth natively. No raw passwords ever touch the Next.js server.
