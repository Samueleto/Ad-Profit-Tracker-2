## Description

FUNCTIONALITY: A fixed bottom navigation bar visible only on mobile viewports (below md breakpoint) providing thumb-friendly access to the four primary sections: Dashboard (grid icon → /dashboard), Networks (activity icon → /dashboard#networks), Settings (gear icon → /settings), and Reports (file icon → /reports). Each tab is a large touch target (full quarter-width, 56px height) with icon centered above a small label. The active tab is highlighted using the project accent color. Tapping a tab navigates using Next.js router.push(). The bar sits above the iOS safe area (padding-bottom: env(safe-area-inset-bottom)) to avoid home indicator overlap on notched devices. On md+ viewports, the bar is hidden (hidden md:hidden). The bar does not duplicate the MobileNavigationDrawer — the drawer handles secondary links while the bottom bar handles primary destinations. A subtle top border separates it from page content. Badge counts (e.g., sync errors) can be overlaid on the Networks icon by reading from the shared Zustand dashboard store's health state.
DESIGN APPROACH: Fixed positioning at bottom:0, full viewport width. Uses existing ThemeProvider tokens for background and border colors. Icon size 24px, label 10px. Active state uses accent color CSS custom property. Safe area inset padding for notched devices. No animations — instant tab switching. Badge is a small red dot (not a number) to keep it simple.
STATES: loading: bar renders immediately with static icons — no loading state. empty: not applicable. error_401: Firebase Auth null → redirect to /. error_403: not applicable. error_404: not applicable. error_500: not applicable. success: active tab highlighted, badge dot visible on Networks if Zustand store health is 'degraded' or 'critical'.
SECURITY: Protected — only renders for authenticated users. Reads Zustand store client-side — no API calls.
