## Description

FUNCTIONALITY: A 'Compare' tab integrated into the FinancialMetricsDashboardPage tab bar, positioned before the four per-network tabs. Displays a side-by-side comparison of all four ad networks (ExoClick, RollerAds, Zeydoo, Propush) across key performance metrics. At the top, a MetricToggle (reused from PerNetworkAnalyticsTabsSection) lets users switch the primary comparison axis: Revenue, Cost, ROI%, CTR%, CPM, Impressions, Clicks. A 4-column comparison card grid shows each network's: primary metric (cost for ExoClick, revenue for others), impressions, clicks, CTR%, CPM, and a MetricShareBar (reused) showing proportional share of total. Below the cards, a grouped bar chart (recharts BarChart) renders all four networks side-by-side for the selected metric, with color-coded bars (amber for ExoClick/cost, green for revenue networks). An efficiency table below the chart shows computed ratios: ROI% (cross-network: (totalRevenue - exoclickCost) / exoclickCost * 100), revenue-per-impression, cost-per-click, and network health status badges (synced/failed/never). A 'Network Ranking' strip ranks networks by the selected metric with ordinal badges (1st/2nd/3rd/4th). Clicking any network card navigates to that network's dedicated tab. A 'Sync All' button triggers POST /api/sync/manual with no networkId. All four networks' data loads in parallel via Promise.all on first tab activation (lazy load). Date range syncs with parent dashboard shared state — no independent selector. Null cells render as '—'.
DESIGN APPROACH: 4-column card grid on desktop (2-column tablet, 1-column mobile). Dense but scannable — compact stat rows within each card. Grouped bar chart at 280px height. Efficiency table uses alternating row shading. Network ranking strip uses medal-style ordinal badges. Progressive disclosure — efficiency ratios collapsed by default with chevron. Consistent Tailwind semantic color classes: green-500 for revenue/positive, amber-500 for cost/ExoClick, red-500 for negative ROI.
STATES: loading — 4 shimmer stat cards matching card dimensions, 280px shimmer rectangle for chart, 4 shimmer rows in efficiency table; empty — 'No data for any network in this date range. Sync your networks to see comparisons.' with Sync All CTA; error_401 — attempt Firebase ID token refresh via getIdToken(true), if refresh fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied' inline within the tab panel with link to /dashboard; error_404 — show 'No data found for selected range' with date range reset button; error_500 — show inline error banner inside the tab panel with Retry button that re-calls all four GET /api/networks/stats calls in parallel; success — 4 network cards populated, grouped bar chart rendered, efficiency table populated, network ranking strip shown.
SECURITY: Protected route — inherits FinancialMetricsDashboardPage Firebase Auth middleware. Bearer token attached to all API calls via Authorization header. Date range capped at 90 days client-side. userId always from verified Firebase ID token server-side.
