## Description

FUNCTIONALITY: A public landing page reached via the invitation link sent by email (URL format: /invite/accept?token=[invitationToken]). On mount, the page extracts the token from the URL query param and calls GET /api/team/invitations/validate to check token validity and retrieve invitation details (workspace name, inviting user's name, role being offered). If valid, displays a card showing: workspace name, inviting user's name, the role being offered, and two CTAs — 'Accept Invitation' (calls POST /api/team/invitations/accept) and 'Decline' (calls POST /api/team/invitations/decline). If the user is not authenticated, the 'Accept' button redirects to the login modal at / with a returnUrl param pointing back to this page so they can authenticate first. If the user is already a member of this workspace, shows 'You are already a member of [Workspace]' with a link to /dashboard. If the token is expired or invalid, shows a clear error with a message to contact the workspace admin. On successful acceptance, the user's Firestore document is updated with workspaceId and workspaceRole, and they are redirected to /dashboard with a welcome toast.
DESIGN APPROACH: Centered card (max-w-sm) on a clean background — minimal chrome, no sidebar, no nav frame. This is a focused single-action page. Workspace name displayed prominently. Role badge shown with description. Accept button is primary/filled with accent color; Decline is ghost-styled. Unauthenticated state shows a 'Sign in to accept' button instead of the two CTAs.
STATES: loading — card body shows shimmer placeholder while GET /api/team/invitations/validate resolves; invalid/expired token — 'This invitation link is invalid or has expired. Please ask your workspace admin to send a new invitation.' with no CTA; already member — 'You are already a member of [Workspace]' with Go to Dashboard link; unauthenticated — card shows invitation details with 'Sign in to accept this invitation' button that redirects to / with returnUrl; error_401 — redirect to / with returnUrl preserved; error_403 — show 'This invitation was sent to a different email address' with close; error_404 — show 'Invitation not found' with contact admin message; error_500 — show error banner with Retry; success — redirect to /dashboard with 'Welcome to [Workspace]!' toast.
SECURITY: Public page (no auth required to view invitation details). Accept action requires Firebase Auth — unauthenticated users are redirected to login with returnUrl. Token is single-use and expires after 7 days. Server verifies token validity and that the accepting user's email matches the invitation email.
