## Description

FUNCTIONALITY: A modal dialog triggered by the 'Benchmark Settings' button in PerformanceBenchmarkingTab. Allows users to configure custom industry benchmark targets per metric per network. Seven metric sections (ROI%, CTR%, CPM, Revenue, Cost, Impressions, Clicks) each display a number input for the user's custom target value and a toggle to switch between 'Use Industry Default' and 'Custom Target'. A 'Reset to Defaults' button per metric row restores the system default. A 'Save All' button submits all changed fields via PATCH /api/benchmarks/settings. A 'Cancel' button closes the modal without saving. Current saved values are pre-populated from GET /api/benchmarks/settings on modal open. Inline validation: numeric fields must be non-negative; ROI% and CTR% capped at 10000; CPM capped at 1000. A 'Last updated' timestamp shows when settings were last saved. Changes are not auto-saved — user must click 'Save All'.
DESIGN APPROACH: Centered modal (max-w-lg) with backdrop overlay. Compact metric rows with label, current default shown in muted text, and input field. 'Use Default' toggle de-emphasizes the input when active. Destructive 'Reset All' button at bottom-left, 'Cancel' and 'Save All' at bottom-right. Saving state disables all inputs and shows spinner on 'Save All' button.
STATES: loading — modal body shows 7 shimmer rows matching metric row structure; empty — not applicable (always shows all 7 metrics); error_401 — close modal and redirect to / with 'Session expired' toast; error_403 — show 'Access Denied' inside modal; error_404 — show 'Settings not found — defaults will be used' with a 'Create Settings' CTA that calls PATCH /api/benchmarks/settings with defaults; error_500 — show error banner inside modal with Retry; success — form pre-populated with saved values, 'Settings saved' toast on successful PATCH.
SECURITY: Protected — inherits parent page Firebase Auth middleware. Bearer token on all API calls. userId always from verified Firebase ID token server-side.
