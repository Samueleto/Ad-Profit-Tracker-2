## Description

FUNCTIONALITY: An 'API Explorer' tab integrated into the FinancialMetricsDashboardPage. It provides a read-only reference panel for developers and power users to inspect raw API data. It features four collapsible accordion sections, one for each ad network (ExoClick, RollerAds, Zeydoo, Propush). Each section displays the API endpoint URL, HTTP method, and required parameters. A syntax-highlighted JSON viewer shows the most recent raw API response, fetched from a dedicated endpoint. A 'Fetch Fresh Sample' button triggers a live test call to the ad network, updating the viewer. Users can copy the JSON response to the clipboard. A field mapping table clearly shows how raw API fields are translated into the app's internal `adStats` schema. A 'Last fetched' timestamp indicates data freshness. If an API key isn't configured, a message with a direct link to the settings page is shown.
DESIGN APPROACH: The UI uses an accordion layout to manage the four network sections, keeping them collapsed by default to maintain a clean interface. A lightweight JSON viewer with syntax highlighting and line numbers (e.g., react-json-view) ensures readability of the raw data. Field mappings are presented in a simple two-column table for clarity. The overall design is informational and tool-like, prioritizing data clarity over decoration.
STATES: The states are managed per-accordion. Loading: The JSON viewer shows a shimmer placeholder while fetching the last response. Empty: If no API key is configured, the accordion body shows 'API key not configured' with a link to /settings. No cached response: 'No response cached yet. Click Fetch Fresh Sample to retrieve one.' is displayed. Fetching fresh sample: The 'Fetch Fresh Sample' button shows a spinner and is disabled. Error: Network or server errors are displayed inline within the accordion body with a retry option.
SECURITY: This is a protected feature, inheriting the authentication from its parent dashboard page. All API calls for raw responses are made server-side, ensuring that no sensitive API keys are ever exposed to the client. The feature is read-only, except for the 'test connection' call which is also a secure, server-mediated action.
