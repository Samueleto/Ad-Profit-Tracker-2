## Description

FUNCTIONALITY: An interactive area/line chart section integrated into the existing FinancialMetricsDashboardPage showing daily net profit trends over a selected date range. Users can toggle between metric views (profit, revenue, cost, ROI) via a segmented control above the chart. A date range selector (Last 7 / 14 / 30 days, This Month, Custom — max 90 days) syncs with the parent dashboard date range. The chart renders a time-series area chart using recharts with: a filled area under the profit line color-coded by overall period profitability (green fill for positive, red fill for negative), individual data point markers colored by per-day colorCode from the API response ('green'|'red'|'amber'), a dashed gray horizontal breakeven line at y=0, a 7-day moving average overlay line (dotted, gray), and interactive tooltips showing date, netProfit, revenue, cost, and ROI% on hover. A 'Pattern Insights' strip below the chart highlights: best day, worst day, longest profitable streak, and period-over-period percent change. Users can click any data point to drill into that day's snapshot via GET /api/stats/snapshot. An 'Export Chart' button triggers PNG download via recharts' built-in export. The section reads from GET /api/roi/compute with groupBy='daily' for the series data and GET /api/roi/breakdown with dimension='daily' for per-point colorCode and roiIndicator enrichment. No new API endpoints are needed — all data comes from existing ROI endpoints.
DESIGN APPROACH: Full-width card below the KPI row on /dashboard. Chart height 320px on desktop, 220px on mobile. Segmented metric toggle (Profit / Revenue / Cost / ROI) sits in the card header alongside the date range selector. Dense but readable — axis labels minimal, grid lines subtle gray. Progressive disclosure: 'Pattern Insights' strip is collapsed by default, expandable with a chevron. Breakeven line labeled 'Break Even' at the right axis edge. Color coding follows project-wide Tailwind semantic classes: green-500 for positive, red-500 for negative, amber-500 for neutral/null. Null data points render as gaps in the line (connectNulls=false) rather than zero.
STATES: loading — chart area shows a gray shimmer rectangle (320px height), metric toggle and date selector are skeleton placeholders, pattern insights strip hidden; empty — 'No profit data for this range. Sync your networks to see trends.' with a 'Sync Now' CTA linking to the ManualRefreshPanel; error_401 — attempt Firebase ID token refresh via getIdToken(true), if refresh fails redirect to / with 'Session expired' toast; error_403 — show 'Access Denied' inline within the chart card with link to /dashboard; error_404 — show 'No data found for selected range' with a date range reset button; error_500 — show error banner inside the chart card with 'Retry' button that re-calls both GET /api/roi/compute and GET /api/roi/breakdown; success — recharts AreaChart rendered with color-coded area fill, data point markers, breakeven line, moving average overlay, and pattern insights strip populated.
SECURITY: Protected route — inherits FinancialMetricsDashboardPage Firebase Auth middleware. Bearer token attached to all API calls via Authorization header. Date range capped at 90 days client-side before API call. userId always from verified Firebase ID token server-side. No new auth surface introduced.
