## Description

FUNCTIONALITY: A slide-in drawer navigation component that replaces the desktop sidebar on mobile viewports (below md breakpoint). Renders a hamburger menu button (☰) in the top-left of the mobile header bar. Tapping it opens a full-height drawer from the left edge with a semi-transparent backdrop overlay. Inside the drawer: the app logo and name at the top, the authenticated user's avatar and display name below it, then navigation links — Dashboard (/dashboard), Settings (/settings), Reports (/reports) — each as a large touch-friendly row (min 48px height) with icon + label. The active route is highlighted using the project accent color from the existing ThemeProvider. A theme toggle (reusing the existing dark/light toggle component from Dark Mode & UI Theming) appears near the bottom of the drawer. A sign-out button at the very bottom calls Firebase Auth signOut(). Tapping any nav link or the backdrop closes the drawer. The drawer is dismissible via swipe-left gesture. On md+ viewports, the drawer is hidden and the desktop sidebar renders instead — no duplicate nav logic.
DESIGN APPROACH: Touch-first — all tap targets minimum 48×48px per WCAG. Drawer width 280px, slides in with 200ms ease-out CSS transition. Backdrop is black at 40% opacity. Uses existing ThemeProvider tokens (dark:bg-gray-900 / bg-white pattern) so dark mode works automatically. Accent color from existing CSS custom properties for active state highlight. Swipe gesture handled via touch event listeners (touchstart/touchend delta check).
STATES: Closed (default): hamburger button visible in header, drawer off-screen. Opening: 200ms slide-in transition with backdrop fade. Open: full drawer visible, backdrop active, body scroll locked. Closing: 200ms slide-out. error_401: Firebase Auth onAuthStateChanged fires null — redirect to / with 'Session expired' toast. error_403: not applicable (nav links are always accessible to authenticated users). error_404: not applicable. error_500: not applicable (static nav, no API calls). success: drawer renders with user avatar and name from Firebase Auth currentUser.
SECURITY: Protected — only renders for authenticated Firebase users. Reads Firebase Auth currentUser client-side — no API call needed. Sign-out calls Firebase Auth signOut() directly.
