We need TypeScript interfaces to represent the data flowing through this feature. These types bridge the API responses from GET /api/roi/compute and GET /api/roi/breakdown into the shape recharts needs to render the chart.

Create the following types in the feature's types folder:

**DailyProfitDataPoint** — Represents a single day on the chart. Fields: `date` (string, ISO format), `netProfit` (number or null), `revenue` (number or null), `cost` (number or null), `roi` (number or null — this is the ROI percentage), `colorCode` ('green' | 'red' | 'amber' | null — comes from the GET /api/roi/breakdown response), `roiIndicator` (string or null — also from breakdown). Null values are intentional: null means no data for that day and should render as a gap in the chart (connectNulls=false).

**ChartMetric** — A union type: `'profit' | 'revenue' | 'cost' | 'roi'`. This is what the segmented metric toggle switches between.

**DateRangeOption** — A union type for the preset options: `'7d' | '14d' | '30d' | 'thisMonth' | 'custom'`.

**DateRange** — An object with `from` (Date) and `to` (Date). The custom range option uses this. Add a validation helper that checks the range doesn't exceed 90 days — if it does, clamp `to` to `from + 90 days`.

**PatternInsights** — Represents the computed insights strip below the chart. Fields: `bestDay` (DailyProfitDataPoint or null), `worstDay` (DailyProfitDataPoint or null), `longestProfitableStreak` (number — count of consecutive days with netProfit > 0), `periodOverPeriodChange` (number or null — percent change vs. the previous equivalent period).

**ComputedChartData** — The final shape passed into the recharts AreaChart. Fields: `dataPoints` (DailyProfitDataPoint[]), `movingAverage` (array of objects with `date` string and `value` number or null — the 7-day moving average), `insights` (PatternInsights), `overallProfitable` (boolean — true if the sum of netProfit across the period is positive, drives the green vs. red area fill color).
