We need to build the `GET /api/networks/stats` route handler. This is the main data source for the PerNetworkAnalyticsTabsSection — every network tab calls this endpoint when it first activates, passing its specific `networkId` along with the shared date range.

Create the route handler at `src/app/api/networks/stats/route.ts`. Here's what it needs to do, in order:

**Authentication first.** Pull the Authorization header, strip 'Bearer ', and call `admin.auth().verifyIdToken(token)`. If this fails for any reason, return 401 immediately. Get `uid` from the verified token — never from query params or the request body.

**Validate inputs.** Read `networkId`, `dateFrom`, `dateTo`, and `groupBy` from the query string. Check that `networkId` is one of the four valid values (`exoclick`, `rollerads`, `zeydoo`, `propush`) — return 400 if not. Validate that `dateFrom` and `dateTo` are valid `YYYY-MM-DD` strings, that `dateFrom <= dateTo`, and that the range is 90 days or fewer — return 400 with a descriptive message for each case. Default `groupBy` to `'total'` if not provided.

**Check the cache.** Build a cache key like `{uid}_network_stats_{networkId}_{dateFrom}_{dateTo}_{groupBy}` and check the NodeCache singleton. If there's a hit, return the cached response immediately with an `X-Cache: HIT` header.

**Query Firestore.** Query the `adStats` collection filtering by `userId == uid`, `networkId == networkId`, `date >= dateFrom`, `date <= dateTo`. Use `getDocs` for a one-time read. If no documents come back, return 404.

**Determine dataRole.** ExoClick is `'cost'`, the other three are `'revenue'`. Use this to decide which field to treat as the primary metric in all aggregations.

**Aggregate based on groupBy:**
- `'total'`: sum the primary metric, impressions, clicks across all rows. Compute weighted average CTR (total clicks / total impressions * 100) and weighted average CPM (primary metric / impressions * 1000). Count unique countries and unique dates.
- `'daily'`: group rows by date, sum per day, compute per-day CTR and CPM. Sort by date ascending.
- `'country'`: group rows by country code, sum per country, compute per-country CTR and CPM. Sort by primary metric descending. Look up the human-readable country name from an ISO 3166-1 alpha-2 lookup (a simple hardcoded map is fine). Compute `metricShare` as (country primary metric / total primary metric * 100).

**Fetch network status.** Read the `networkConfigs` document for this `userId` + `networkId` to get `lastSyncedAt`, `lastSyncStatus`, and `circuitBreakerOpen`. If the document doesn't exist, return nulls for these fields rather than erroring.

**Cache and return.** Store the result in NodeCache with a 5-minute TTL. Return the full `NetworkStatsResponse` payload with a 200 status and `X-Cache: MISS` header.

For error handling: wrap the Firestore reads in try/catch and return 500 with a generic message if they fail — don't leak internal error details.
