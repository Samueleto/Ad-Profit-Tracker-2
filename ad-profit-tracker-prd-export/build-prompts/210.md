We need six reusable components that CustomReportBuilderPage will compose together. Build these in `src/features/reports/components/` before building the page itself.

**MetricSelector** — A card section with a 'Metrics' heading. Shows 8 checkboxes: Revenue, Cost, Net Profit, ROI%, Impressions, Clicks, CTR%, CPM. At the top, a 'Select All' toggle that checks/unchecks all 8 at once. If all are checked and one is unchecked, 'Select All' shows an indeterminate state. Props: `selected: string[]`, `onChange: (metrics: string[]) => void`. The internal metric keys map to the `ALLOWED_METRICS` constants from the types file.

**DateRangePicker** — A card section with a 'Date Range' heading. Shows preset buttons for Last 7 Days, Last 14 Days, Last 30 Days, Last 90 Days, This Month — tapping one sets the range immediately. A 'Custom' option expands a calendar using `react-day-picker`'s range selection mode, showing two months side by side on desktop. Enforce the 90-day max span client-side: if the user picks a range longer than 90 days, show an inline error and don't update the parent state. Props: `preset: string`, `dateFrom: string | null`, `dateTo: string | null`, `onChange: (preset: string, dateFrom: string | null, dateTo: string | null) => void`.

**NetworkSelector** — A card section with a 'Networks' heading. Four checkboxes: ExoClick, RollerAds, Zeydoo, Propush. An 'All Networks' toggle at the top. Same Select All / indeterminate behavior as MetricSelector. Props: `selected: string[]`, `onChange: (networks: string[]) => void`.

**GroupingControl** — A card section with a 'Grouping' heading. A segmented control (three buttons styled as a pill group) for Daily / Country / Network — only one can be active at a time. Props: `value: string`, `onChange: (groupBy: string) => void`.

**DataQualityToggle** — Sits inside a collapsible 'Advanced Options' section (collapsed by default, with a chevron to expand — this is the progressive disclosure pattern). Shows three toggle options: All / Clean Only / Anomalies Only. Props: `value: string`, `onChange: (quality: string) => void`.

**SavedReportsPanel** — A collapsible panel (left drawer on desktop, bottom sheet on mobile) that lists saved reports from `GET /api/reports`. Each row shows the report name, a 'Load' button that fires an `onLoad` callback with the report's config, an inline rename flow (clicking the name makes it editable, pressing Enter calls PATCH /api/reports/:id), and a delete button that calls DELETE /api/reports/:id with a confirmation step. Shows 3 shimmer rows while loading. Shows an empty state message if no saved reports exist. Props: `isOpen: boolean`, `onClose: () => void`, `onLoad: (config: ReportConfig) => void`.

**ReportPreviewTable** — Renders the paginated results from `POST /api/reports/run`. Reuses the existing `MobileDataTableWrapper` component for responsive rendering. Shows only the columns corresponding to the currently selected metrics. Color-codes rows (e.g., red for negative net profit, green for positive). Shows 5 shimmer rows while loading. Shows an empty state ('No data matches your current configuration. Adjust filters or sync your networks.' with a Sync Now link) when the response has zero rows. Shows an inline error banner with a Retry button on 500 errors. Shows a row count badge in the header. Supports pagination — a 'Load More' button appends the next page when `hasMore` is true. Props: `rows: ReportRunRow[]`, `summary: object`, `metrics: string[]`, `isLoading: boolean`, `error: string | null`, `hasMore: boolean`, `onLoadMore: () => void`, `onRetry: () => void`.
