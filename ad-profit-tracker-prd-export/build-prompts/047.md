We need three packages wired up before we can build any of the ExoClick endpoints:

**1. Firebase Admin SDK** — Install `firebase-admin` if it isn't already in the project. Create a shared admin SDK initializer at `src/lib/firestore/admin.ts`. This should initialize the Firebase Admin app once (guard against re-initialization with a check like `getApps().length === 0`) using a service account — either from a JSON file path in the environment or by passing `FIREBASE_SERVICE_ACCOUNT_JSON` as a parsed environment variable. Export the `adminDb` (Firestore instance) and `adminAuth` from this file. Every API route will import from here.

**2. Cryptr (AES encryption/decryption)** — Install `cryptr`. Create a helper at `src/lib/exoclick/crypto.ts` that exports two functions: `encryptApiKey(plaintext: string): string` and `decryptApiKey(ciphertext: string): string`. Both should use a `Cryptr` instance initialized with the `EXOCLICK_ENCRYPTION_SECRET` environment variable. This secret must be set in `.env.local` and in your deployment environment — if it's missing, throw a clear startup error rather than silently failing. Never log the plaintext key anywhere in this file.

**3. ExoClick API client wrapper** — Install `exoclick-api`. Create a wrapper at `src/lib/exoclick/client.ts` that exports a `createExoClickClient(apiKey: string)` factory function. This function should initialize the ExoClick client with the provided API key and return a configured instance ready to call `getGlobalStatistics()`. Keep this thin — just initialization and any default config. The actual API calls will happen in the sync logic, not here.

Also create a small utility at `src/lib/exoclick/sanitize.ts` that exports a `sanitizeRawResponse(row: object): object` function. This strips any fields whose keys contain the words 'key', 'token', 'secret', or 'password' (case-insensitive) from a response object before it gets stored in Firestore. We'll use this in both sync endpoints.
