# Step 375 of 437: Handle all export error states gracefully

We need to make sure every failure mode in the export flow shows the user something helpful rather than a broken UI or a silent failure.

Here's how each error state should be handled in the ExportModal:

**401 (Unauthenticated):** Close the modal immediately. Attempt a Firebase ID token refresh by calling firebaseAuth.currentUser?.getIdToken(true) (the true flag forces a refresh). If the refresh succeeds, the user can try again — show a toast 'Session refreshed, please try again'. If the refresh fails (the user's session is truly expired), redirect to / and show a 'Session expired' toast.

**403 (Forbidden):** Don't close the modal. Show an 'Access Denied' message inside the modal body, replacing the sheet selection UI. Include a close button. This shouldn't happen in normal usage but handle it gracefully.

**404 (No data found):** Disable the Export Now button and show the tooltip 'No data found for this date range'. This can also come from the preview endpoint — if hasData is false in the preview response, apply the same disabled state.

**429 (Rate limited):** Show an inline error banner inside the modal: 'Export limit reached. You can export up to 10 files per hour. Please try again later.' Include a close button on the banner. The Export Now button should remain disabled until the modal is closed and reopened (don't auto-retry).

**500 (Server error):** Show an inline error banner inside the modal with the message 'Something went wrong generating your file.' and a 'Retry' button. The Retry button re-triggers the POST /api/export/excel call with the same parameters. Reset exportStatus to 'idle' so the checkboxes become interactive again.

**Network failure (fetch throws):** Treat this the same as a 500 — show the inline error banner with a Retry button.

**Preview endpoint failure (GET /api/export/preview):** Don't block the user from exporting. Show a subtle inline message 'Row counts unavailable' in place of the badges. The Export Now button should still be enabled (unless hasData is definitively false). The user can still proceed with the export.

**All checkboxes unchecked:** Disable the Export Now button with a tooltip 'Select at least one sheet to export'. This is a client-side validation — don't even make the API call.

**Empty filename in Advanced options:** If the user clears the filename field, fall back to the default filename on the server side — don't show a validation error for an empty filename since it's optional.

Make sure all error banners and toasts use the project's existing notification/toast system for visual consistency.
