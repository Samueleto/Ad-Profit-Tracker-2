We're building the ROI Calculation Engine. Before writing any endpoint or component logic, let's define the TypeScript types that everything will share — this way the API responses, the frontend components, and the shared utility functions all agree on the shape of the data.

Create a types file at `src/features/roi/types/roi.ts`. Define the following interfaces and types:

**ROI compute response** — matches the `GET /api/roi/compute` success payload: fields for `dateFrom`, `dateTo`, `roi` (number or null), `roiIndicator` (union of `'positive' | 'negative' | 'breakeven' | 'no_cost_data'`), `colorCode` (union of `'green' | 'red' | 'amber'`), `totalRevenue`, `totalCost`, `netProfit` (all numbers), `roiChange` (number or null), `roiChangeDirection` (union of `'up' | 'down' | 'flat' | null`), `groupBy` (string), an optional `series` array of per-day/per-country items, and `cachedAt` (string or null).

**ROI breakdown response** — matches `GET /api/roi/breakdown`: `dateFrom`, `dateTo`, `dimension` (union of `'network' | 'country' | 'daily'`), a `breakdown` array where each item has `key`, `label`, `revenue`, `cost`, `netProfit`, `roi` (number or null), `roiIndicator`, `colorCode`, and `contributionPercent`. Plus a `summary` object with `totalRevenue`, `totalCost`, `netProfit`, and `overallRoi` (number or null).

**ROI thresholds** — matches both threshold endpoints: `positiveThreshold`, `warningThreshold`, `criticalThreshold` (all numbers), `targetRoi` (number or null), `alertOnNegative` and `alertOnTargetMiss` (booleans). The GET response also includes `usingDefaults` (string array). The PATCH response includes `updatedAt`.

Also create a shared utility at `src/lib/roi/formula.ts` that exports a `computeRoi(totalRevenue: number, totalCost: number)` function implementing the null-safe formula: `(totalCost > 0) ? ((totalRevenue - totalCost) / totalCost * 100) : null`. Export a `getColorCode(roi: number | null)` function and a `getRoiIndicator(roi: number | null)` function — these will be used by all three API route handlers so the logic stays consistent.

Finally, define the default threshold constants in `src/lib/roi/defaults.ts`: `positiveThreshold: 0`, `warningThreshold: -10`, `criticalThreshold: -10`, `targetRoi: null`, `alertOnNegative: false`, `alertOnTargetMiss: false`.
