We need three API route handlers for managing API keys. All three live under `/api/keys/` and all three must verify the user's Firebase ID token before doing anything. Here's what each one does:

**POST /api/keys/save** — This is the most important one. It receives a `networkId` and a raw `key` from the request body. First, verify the Firebase ID token from the `Authorization: Bearer <token>` header using Firebase Admin — if it's missing or invalid, return 401. Then validate that `networkId` is one of the four supported networks using `isValidNetworkId` — if not, return 400. Validate that `key` is not empty — if it is, return 400. Now encrypt the key using the `encryptKey` utility from `src/lib/encryption.ts`. Never log the raw key anywhere. Finally, do an upsert on the Firestore document at `users/{uid}/apiKeys/{networkId}`: if it exists, update `encryptedKey` and `updatedAt` (using a server timestamp); if it doesn't, create it with all fields including `userId`, `networkId`, `createdAt`, and `updatedAt`. Return `{ networkId, status: 'connected' }` on success.

**GET /api/keys/status** — Verify the token, get the UID. Query the entire `users/{uid}/apiKeys` subcollection from Firestore. Then, for each of the four networks in `SUPPORTED_NETWORKS`, check whether a document exists for that network. Build a response array of `NetworkStatus` objects — if a document exists, status is `'connected'` and include the `updatedAt` timestamp as an ISO string; if not, status is `'not_connected'` and `updatedAt` is null. Return the array. Critically: never include `encryptedKey` in the response. Only confirm existence.

**DELETE /api/keys/delete** — Verify the token, get the UID. Read `networkId` from the request body. Validate it with `isValidNetworkId` — return 400 if invalid. Check that the document at `users/{uid}/apiKeys/{networkId}` actually exists — return 404 if not. Delete the document. Return `{ networkId, status: 'disconnected' }`.

All three endpoints should use a consistent error response shape: `{ error: string, code?: string }`. Add basic rate limiting — you can use a simple in-memory approach or a library like `upstash/ratelimit` if Redis is available. If not, at minimum add a comment noting that rate limiting should be added before production.
