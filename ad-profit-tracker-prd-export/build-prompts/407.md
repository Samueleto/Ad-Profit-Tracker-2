We need tests covering the LoginModal, AuthContext, error handling, and the four API endpoints to make sure nothing breaks as the app grows.

**Unit tests — Auth error mapper (`lib/authErrors.ts`):**
- Each Firebase error code maps to the correct user-friendly string
- Unknown error codes return the generic fallback message
- `auth/popup-closed-by-user` returns null

**Unit tests — AuthContext:**
- `onAuthStateChanged` firing with a user triggers GET `/api/auth/get-user`
- GET `/api/auth/get-user` returning 404 triggers POST `/api/auth/sync-user` as fallback
- `signOut()` clears the user state
- `updateUser()` calls PATCH `/api/auth/update-profile` and updates local state on success
- `updateUser()` reverts local state if PATCH `/api/auth/update-profile` fails
- `deleteAccount()` calls DELETE `/api/auth/delete-account`
- `loading` starts as true and resolves to false after auth state is determined

**Component tests — LoginModal:**
- Renders with Sign In tab active by default
- Switching to Sign Up tab shows the Confirm Password field
- Switching back to Sign In hides the Confirm Password field
- Submitting empty fields shows validation errors without calling Firebase
- Mismatched passwords in Sign Up show an error without calling Firebase
- Shows a spinner on the active button while loading
- Disables all inputs while loading
- Displays inline field errors from the error mapper
- Calls `signInWithEmailAndPassword` on Sign In form submit
- Calls `createUserWithEmailAndPassword` on Sign Up form submit
- Calls `signInWithPopup` when the Google button is clicked

**API endpoint tests:**
- POST `/api/auth/sync-user`: returns 401 with no token; creates a new `users` document if one doesn't exist; updates only `lastLoginAt` if the document already exists; returns the user document on success
- GET `/api/auth/get-user`: returns 401 with no token; returns 404 if the `users` document doesn't exist; returns the full user profile on success
- PATCH `/api/auth/update-profile`: returns 401 with no token; returns 400 if `role` or `uid` is in the request body; updates `displayName` and `preferences` successfully
- DELETE `/api/auth/delete-account`: returns 401 with no token; deletes both the Firestore document and the Firebase Auth user; returns `{ deleted: true }` on success

**Integration test — full sign-up flow:**
Open the app unauthenticated → see LoginModal → switch to Sign Up → enter valid email and matching passwords → submit → Firestore `users` document is created with correct fields → POST `/api/auth/sync-user` is called → modal closes → land on `/dashboard` → refresh the page → still signed in → user profile loads from GET `/api/auth/get-user`.
