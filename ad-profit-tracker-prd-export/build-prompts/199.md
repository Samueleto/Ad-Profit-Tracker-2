We're building the ManualRefreshPanel — a compact card section that integrates into the existing dashboard (not a standalone page). It has several distinct pieces worth building separately so they're testable and reusable.

**SyncStatusBadge component** — Takes a `status: SyncStatus` prop and renders the right visual: green badge for `'success'`, red for `'failed'`, gray for `'never'`, and an amber spinner for `'in_progress'`. Keep it small and inline — it sits next to the network name in a dense list row. This component is purely presentational.

**NetworkSyncRow component** — Renders one row for a single ad network. Props: `networkId: NetworkId`, `lastSyncedAt: Date | null`, `lastSyncStatus: SyncStatus`, `isRefreshing: boolean`, `onRefresh: () => void`. Shows the network name (formatted nicely — 'ExoClick', 'RollerAds', 'Zeydoo', 'Propush'), the `SyncStatusBadge`, the last synced time as a relative string (e.g., '2 hours ago' — use a date utility or `Intl.RelativeTimeFormat`), and a small icon button to trigger a per-network refresh. The icon button should be disabled when `isRefreshing` is true or when the network is currently in-progress. When disabled due to rate limiting, show a countdown label instead of the icon button (e.g., 'Available in 4m 32s').

**SyncHistoryDrawer component** — A slide-over panel (right side) that shows the last 10 sync events per network. It's opened by the 'View Sync History' link and fetches from `GET /api/scheduled/sync-history` when opened. Each row shows: network name, status badge, `rowsFetched`, `latencyMs` (formatted as 'Xs' or 'Xms'), and the timestamp. The drawer should have a close button and an overlay backdrop. It handles its own loading and error states internally.

**ManualRefreshPanel component** — The top-level panel that composes everything. It should:
- Show a 'Refresh All Data Now' primary button at the top. On click, call `POST /api/sync/manual` with no `networkId`. While syncing, change the button to show 'Syncing...' with a spinner and disable it. If rate-limited, disable with a countdown.
- Below the button, render four `NetworkSyncRow` components — one for each of ExoClick, RollerAds, Zeydoo, and Propush — in a dense list.
- On mount, fetch `GET /api/scheduled/sync-status` to populate initial state for all four rows.
- After any refresh is triggered, start polling `GET /api/scheduled/sync-status` every 3 seconds. Stop polling when all triggered networks show a terminal status (`'success'` or `'failed'`), or after 60 seconds, whichever comes first.
- Show a 'View Sync History' link that opens the `SyncHistoryDrawer`.
- Handle the empty state: if no networks are configured, show 'No networks configured — go to Settings to add API keys' with a link to `/settings`.
- Handle the loading state on mount: show shimmer/skeleton rows for each network while the initial sync-status loads.
- Attach the Firebase ID token to every API call via `getIdToken()`. On a 401 response, try `getIdToken(true)` to force-refresh the token and retry once. If that also fails, redirect to `/` with a 'Session expired' toast.
- On 429, parse the `Retry-After` header and start a countdown timer on the affected button.

All API calls should include `Authorization: Bearer <token>` headers. Keep the panel compact — it's embedded in the dashboard sidebar or top action bar, not a full-width page section.
