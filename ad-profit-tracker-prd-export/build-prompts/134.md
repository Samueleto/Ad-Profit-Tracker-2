We need TypeScript types that describe the shape of data flowing through the dashboard — from Firestore documents to the API response to what the frontend components consume. Create these in src/features/dashboard/types/.

Start with the Firestore document shapes. Create an AdStatDocument type that represents a single document from the adStats collection — it should have fields like userId, date (string, YYYY-MM-DD), network (string identifying which network: 'rollerads', 'zeydoo', 'propush', 'exoclick'), revenue (number), cost (number), country (string, ISO code), and any other fields the collection actually has. Create a NetworkConfigDocument type for the networkConfigs collection — userId, networkId, lastSyncedAt (Firestore Timestamp or ISO string), lastSyncStatus (something like 'success' | 'error' | 'pending'), and dataRole (something like 'revenue' | 'cost').

Then define the API response shape for GET /api/dashboard/metrics. This is the DashboardMetricsResponse type. It should match the SUCCESS (200) shape from the endpoint spec exactly: dateFrom, dateTo, a kpis object (totalRevenue, totalCost, netProfit, roi, revenueChange, costChange, profitChange, roiChange — all numbers or null), dailySeries (array of objects with date, revenue, cost, netProfit, roi), topCountries (array with country, countryName, revenue, cost, netProfit, metricShare), perNetwork (array with networkId, dataRole, primaryMetric, lastSyncedAt, lastSyncStatus), and cachedAt (string or null).

Also create a DateRange type for the date range selector — something like { from: string, to: string, label: string } — and a KpiCard type that the KPI card component will use: { label, value, change, changeDirection: 'up' | 'down' | 'neutral', format: 'currency' | 'percent' }.

Keep all these types in one or two files under the types folder. Export everything so components and API routes can import what they need.
