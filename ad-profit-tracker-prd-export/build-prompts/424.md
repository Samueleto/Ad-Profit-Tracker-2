Unit tests for POST /api/networks/rollerads/sync:
- Valid request with a date range syncs correctly and returns rowsFetched, latencyMs, syncedAt
- Invalid date range (dateFrom > dateTo) returns 400
- Date range over 90 days returns 400
- Missing API key returns 404
- Disabled network returns 400 with the right message
- Rate limit exceeded returns 429
- RollerAds API returning a 500 causes the endpoint to return 502 and update lastSyncStatus to 'failed'
- Successful sync updates networkConfigs lastSyncedAt and lastSyncStatus to 'success'
- Successful sync writes an audit log entry with the correct fields
- rawResponse stored in adStats does not contain any fields named key, token, secret, or authorization

Unit tests for GET /api/networks/rollerads/stats:
- Returns grouped-by-day data correctly for a valid date range
- Returns aggregated totals when groupBy='total'
- Returns 400 for missing dateFrom or dateTo
- Returns 400 for a range exceeding 90 days

Unit tests for GET /api/networks/rollerads/stats/by-country:
- Returns countries sorted by revenue descending
- revenueShare percentages sum to approximately 100%
- Respects the limit param
- Returns full countryName alongside country code

Unit tests for GET /api/networks/rollerads/stats/latest:
- Returns the most recent date's aggregated stats
- Returns 404 when no adStats documents exist
- Includes lastSyncedAt and lastSyncStatus from networkConfigs

Unit tests for GET /api/networks/rollerads/raw-response:
- Returns records array and fieldSchema array for a valid date
- Returns 404 when no data exists for the requested date
- fieldSchema contains entries for all expected RollerAds fields (site_id, date, country, impressions, clicks, ctr, revenue, cpm, ecpm)

Unit tests for POST /api/networks/rollerads/scheduled-sync:
- Returns 401 with missing or wrong x-internal-secret
- Returns 401 even with a valid Firebase ID token
- Processes multiple users and returns correct processed/succeeded/failed/skipped counts
- Skips users without a saved API key and adds them to the skipped count
- Continues processing remaining users when one user's sync fails

Auth tests (apply to all five user-facing endpoints):
- No Authorization header → 401
- Malformed JWT → 401
- Expired JWT → 401
- Valid JWT → proceeds normally

For mocking: mock admin.auth().verifyIdToken() to return a fake uid, mock Firestore reads/writes using a Firestore emulator or a mock library, and mock axios to simulate RollerAds API responses (success, 401, 500, network timeout).
