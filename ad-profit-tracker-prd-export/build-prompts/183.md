We need to build all the API route handlers for the team feature. There are quite a few, so let's go through each one carefully. All of them use the verifyBearerToken() helper from lib/team/auth.ts and the Firebase Admin SDK.

GET /api/team/members — Verify the token, get the uid, read that user's Firestore document to find their workspaceId. If workspaceId is missing, return 404. Query the users collection where workspaceId matches, sort by workspaceJoinedAt ascending. Return only safe fields: uid, displayName, email, workspaceRole, workspaceJoinedAt, photoURL — never return API keys or sensitive fields. Response shape: { members: WorkspaceMember[], total: number, workspaceId: string, workspaceName: string }.

GET /api/team/workspace — Verify token, read the user's document, get workspaceId and workspaceRole. Read the workspace owner's document (workspaceId is the owner's uid) to get workspaceName and workspaceCreatedAt. Count members by querying users where workspaceId matches. Return WorkspaceMetadata.

PATCH /api/team/workspace — Verify token, check the user's workspaceRole is 'owner' (403 otherwise). Validate workspaceName is 1-100 chars after trimming (400 if not). Rate limit: max 10 updates/hour per user. Use a Firestore batch write to update workspaceName on ALL member documents in the workspace (it's denormalized). Write an audit log entry to the auditLogs collection with action='workspace_updated', the old and new name, and a server timestamp.

PATCH /api/team/members/:memberId/role — Verify token. Validate the role body is 'admin' or 'member' (never 'owner' — return 400). Read the actor's document, check they're 'owner' or 'admin'. Read the target member's document, verify they're in the same workspace (403 if not). Block changing the owner's role (403). Block admins from changing other admins (403). Rate limit: 20 role changes/hour. Update the target's workspaceRole and updatedAt. Write an audit log with action='role_changed', actorUid, targetUid, oldRole, newRole.

DELETE /api/team/members/:memberId — Verify token. Read actor's document. If actor is removing themselves (memberId === uid), allow if they're not the owner (owners can't leave — they must transfer first). If removing someone else, actor must be 'owner' or 'admin', and admins can only remove members (not other admins or the owner). Verify same workspace. Clear the target's workspaceId and workspaceRole fields (set to null). Write audit log with action='member_removed'. Return 204.

GET /api/team/invitations — Verify token. Get workspaceId. Query workspaceInvitations where workspaceId matches and status is 'pending'. Return the list as WorkspaceInvitationSafe (no token/tokenHash).

POST /api/team/invitations — Verify token. Check actor is 'owner' or 'admin'. Validate email (RFC 5322 format) and role ('admin' or 'member'). Check for duplicate: query workspaceInvitations where workspaceId matches, invitedEmail matches, and status is 'pending' — return 409 if found. Also check users collection for existing member with that email. Rate limit: 10 invitations/hour per user. Generate a UUID v4 token, hash it with hashToken(). Create the workspaceInvitations document with all required fields, expiresAt set to 7 days from now, status='pending'. Send the invitation email (use whatever email service is already in the project, or log the invite URL for now). Return 201 with the created invitation (safe fields only).

DELETE /api/team/invitations/:invitationId — Verify token. Check actor is 'owner' or 'admin'. Read the invitation document, verify it belongs to the same workspace. Update status to 'revoked' and updatedAt. Return 204.

GET /api/team/invitations/validate — Public endpoint (no auth required). Read the token query param. Hash it with hashToken(). Query workspaceInvitations where tokenHash matches. If not found or status isn't 'pending', return 404. Check expiresAt — if expired, update status to 'expired' and return 410. Return invitation details: workspaceName, invitedByName, role, invitedEmail (needed for the accept page to show context).

POST /api/team/invitations/accept — Verify token (auth required). Read the invitationToken from the request body. Hash it, look up the invitation. Verify status is 'pending' and not expired. Verify the authenticated user's email matches invitedEmail (403 if not). Use a Firestore transaction: update the invitation status to 'accepted', set acceptedAt and acceptedByUid; update the user's document with workspaceId and workspaceRole from the invitation. Return 200 with a redirect hint to /dashboard.

POST /api/team/invitations/decline — Verify token. Same lookup as accept. Update status to 'declined'. Return 200.
