We need two packages installed before we can build the report builder UI and backend:

**react-day-picker** — This powers the custom date range calendar inputs on CustomReportBuilderPage. The date range picker needs to support preset options (Last 7/14/30/90 days, This Month) as well as a custom mode where users pick start and end dates from a calendar. Install it with npm and make sure it's imported correctly — react-day-picker v8+ requires its CSS to be imported separately, so add `import 'react-day-picker/dist/style.css'` in the appropriate global styles file or in the component itself.

**node-cache** — This is used server-side in the `POST /api/reports/run` endpoint to cache report results for 5 minutes so repeated runs with the same parameters don't hammer Firestore. Install it and create a shared cache instance at `src/lib/reportCache.ts` that exports a single `nodeCache` instance with a default TTL of 300 seconds (5 minutes). Export it as a singleton so all API route invocations share the same cache instance within the same server process.

Also confirm that `firebase-admin` is already installed and initialized in the project (it should be, since this is an existing Firebase app). If there's already a shared `src/lib/firebaseAdmin.ts` (or similar) that exports the initialized admin app, note where it is — all five report API endpoints will import from it. If it doesn't exist yet, create it: initialize the Firebase Admin SDK using `applicationDefault()` credentials (or the service account already configured in the project's environment), and export the `adminDb` Firestore instance and `adminAuth` for token verification.
