Let's add tests to make sure the network configuration feature works correctly and doesn't break when things change.

**Unit tests for useNetworkConfigs hook:**
- On mount, it calls GET /api/networks/config/list and populates the networks array sorted by displayOrder.
- updateNetworkConfig optimistically updates local state, fires PATCH /api/networks/config/update, and reverts on failure.
- reorderNetworks optimistically reorders and fires PATCH /api/networks/config/reorder, reverts on failure.
- testConnection transitions per-network status through idle → testing → success/error.
- syncAll exposes loading state and result after POST /api/networks/sync-all.
- A 401 response triggers token refresh and retry; sets authExpired if retry also 401.
- resetNetworkConfig calls DELETE /api/networks/config/reset and updates local state.

**Component tests for NetworkCard:**
- Renders network name, status badge, data role label, sync schedule dropdown, and Active toggle.
- Expanding 'Show advanced' reveals the endpoint override field, timeout slider, retry selector, and Test Connection button.
- Toggling the Active switch calls onUpdate with the correct payload.
- Changing the sync schedule dropdown calls onUpdate.
- Clicking 'Save Network Settings' calls onUpdate with all advanced field values.
- Clicking 'Test Connection' shows a spinner, then renders the success result (latency + record count) or error message.
- An inline error banner renders when an error prop is passed, and is dismissible.

**Component tests for NetworkConfigTab:**
- Shows shimmer skeleton cards while loading is true.
- Renders all 4 network cards when data is loaded.
- 'Sync All Now' button calls syncAll and is disabled while syncAllLoading is true.
- A 403 error shows the 'Access Denied' banner.
- A 500 error shows the full-tab error banner with a Retry button.

**Integration tests (end-to-end user flows):**
- Open /settings → click Networks tab → all 4 cards load with correct data from GET /api/networks/config/list.
- Toggle the Active switch on ExoClick → PATCH /api/networks/config/update fires with { networkId: 'exoclick', isActive: false } → card reflects the new state.

- Click Test Connection on Zeydoo → spinner shows → success result renders inline.
- Drag Propush to position 1 → PATCH /api/networks/config/reorder fires with all 4 networks and updated displayOrder values → order persists after page refresh.
- Click Sync All Now → toast shows triggered/skipped/failed summary.
- Simulate a PATCH failure → card reverts to previous value → inline error banner appears → click Retry → PATCH fires again.
