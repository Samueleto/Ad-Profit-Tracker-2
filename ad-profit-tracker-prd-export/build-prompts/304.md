We're building an ad network analytics dashboard. For this step, I need you to connect all the ExoClick components into the app's existing navigation and dashboard structure.

The ExoClick feature should live under a network detail area — something like `/dashboard/networks/exoclick` or as a tab within an existing networks page. Here's how to wire everything together:

**Navigation entry** — Add ExoClick to whatever network list or sidebar the app already has. If there's a 'Networks' section in the nav, add an 'ExoClick' entry that leads to the ExoClick detail area. If the app uses a tab-based layout for network details, create tabs for: Overview, Stats, Countries, Sync, and API Explorer.

**Tab/section layout** — Create a parent layout component for the ExoClick area that renders the five views as tabs or sections:
- 'Overview' tab → `ExoClickSummaryCard`
- 'Stats' tab → `ExoClickStatsView`
- 'Countries' tab → `ExoClickCountryBreakdown`
- 'Sync' tab → `ExoClickSyncPanel`
- 'API Explorer' tab → `ExoClickApiExplorer`

**Shared date range state** — The Stats and Countries tabs should share the same date range. Lift the `dateFrom`/`dateTo` state to the parent layout so switching between those two tabs doesn't reset the selected range. Pass the shared date range down as props.

**Post-sync refresh wiring** — When `useExoClickSync` reports a successful sync, the Overview tab's summary card should reflect the new data. Since `useExoClickLatest` already has a `refetch()` function, make sure the sync panel calls it after success. If the user is currently on the Stats or Countries tab when the sync completes, show a subtle 'Data updated — refresh to see latest' banner or automatically refetch those views too.

**Auth guard** — The entire ExoClick area should be behind the app's existing auth guard. If the user isn't logged in, redirect to the login page. Don't add a new auth guard — reuse whatever the app already has.

**Empty state wiring** — If `GET /api/networks/exoclick/stats/latest` returns 404 (no data synced yet), the Overview tab should show a call-to-action that deep-links directly to the Sync tab so the user knows what to do first.

**Scheduled sync visibility** — The Sync tab should mention that daily automatic syncs are also running via `POST /api/networks/exoclick/scheduled-sync` (handled server-side by a Cloud Scheduler job). Add a small informational note so users understand they don't need to manually sync every day.
