We're building state management for the ComparativeNetworkAnalysisTab. This tab shares date range state with the parent FinancialMetricsDashboardPage and has its own local state for the metric selection, loading/error status, and the API response data.

Here's what needs to be managed:

**Shared state (from parent dashboard)** тАФ The tab should read `dateFrom` and `dateTo` from the parent FinancialMetricsDashboardPage's shared state. It should NOT own these values. When the parent date range changes, the Compare tab should automatically re-fetch data from GET /api/networks/comparison with the new dates. Wire up a useEffect that watches for date range changes and triggers a fresh fetch when they change.

**Local state for this tab:**
- `selectedMetric` тАФ which metric the MetricToggle is set to (default: 'roi'). When this changes, re-call GET /api/networks/comparison with the new metric param. This is a lightweight re-fetch since the endpoint accepts metric as a query param.
- `comparisonData` тАФ the full response payload from GET /api/networks/comparison, which includes the `networks` array, `crossNetwork` derived metrics, and `rankings` array. Initialize as null.
- `loadStatus` тАФ one of: 'idle' | 'loading' | 'success' | 'error' | 'empty'. Drive all the conditional rendering in the tab from this single status field rather than multiple boolean flags.
- `errorCode` тАФ store the HTTP status code of any failed request (401, 403, 404, 500) so the UI can render the right error state.
- `isSyncing` тАФ boolean for the 'Sync All' button spinner state.
- `hasLoaded` тАФ boolean that starts false and flips to true after the first successful load. Use this to implement lazy loading: only fire the initial fetch when the Compare tab is first activated, not on dashboard mount.

**Fetching logic** тАФ Create a `fetchComparisonData` function that:
1. Sets loadStatus to 'loading'
2. Gets the current Firebase ID token via `getIdToken()` from firebase/auth
3. Calls GET /api/networks/comparison with dateFrom, dateTo, and metric as query params, with the token in the Authorization header
4. On 200: sets comparisonData, sets loadStatus to 'success', flips hasLoaded to true
5. On 404: sets loadStatus to 'empty', clears comparisonData
6. On 401: attempts token refresh via `getIdToken(true)`, retries once тАФ if the retry also fails, sets errorCode to 401
7. On 403/500: sets loadStatus to 'error', sets errorCode to the status code

**Sync All logic** тАФ Create a `syncAllNetworks` function that sets isSyncing to true, calls POST /api/sync/manual with no networkId, then on completion (success or failure) sets isSyncing to false and re-calls fetchComparisonData to refresh the data.

Keep all of this in a custom hook (e.g., `useComparativeAnalysis`) so the ComparativeNetworkAnalysisTab component stays clean and just renders based on the state it gets back from the hook.
