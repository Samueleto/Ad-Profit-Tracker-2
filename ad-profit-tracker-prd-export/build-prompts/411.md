Let's add tests to make sure the ComparativeNetworkAnalysisTab and its supporting hook work correctly and don't break under edge cases.

**Unit tests for useComparativeAnalysis hook:**
- Initial state: loadStatus is 'idle', comparisonData is null, hasLoaded is false
- Successful fetch: loadStatus transitions to 'loading' then 'success', comparisonData is populated, hasLoaded flips to true

- 401 response: hook calls getIdToken(true) and retries; if retry succeeds, loadStatus becomes 'success'; if retry fails, redirect to / is triggered
- Metric change: changing selectedMetric triggers a new fetch with the updated metric param
- Date range change: changing dateFrom or dateTo triggers a new fetch
- Sync All success: isSyncing goes true then false, fetchComparisonData is called after
- Sync All failure: isSyncing resets to false, existing comparisonData is preserved
- 90-day cap: passing a date range over 90 days does not fire the API call
