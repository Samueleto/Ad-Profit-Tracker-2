We're building user authentication for a Next.js + Firebase app. For this step, I need you to set up the folder structure for the auth feature.

Create an auth feature area with the following organization:
- A `features/auth/` directory for all auth-related code, with subfolders for `components/` (we'll build the LoginModal here), `hooks/` (for auth state and user data hooks), `context/` (for the AuthContext provider), and `types/` (for TypeScript interfaces).
- A `lib/firebase/` directory for Firebase client-side initialization files — we'll keep the app config, auth instance, and Firestore instance here as separate files so they can be imported cleanly.
- A `lib/firebase-admin/` directory for the Firebase Admin SDK setup used by API routes.
- An `app/api/auth/` directory under the Next.js app router for our four API route handlers: `sync-user/`, `get-user/`, `update-profile/`, and `delete-account/`.

Also create a `.env.local` file (if it doesn't exist) with placeholder keys for the Firebase client config and Admin SDK credentials — we'll fill these in during sdk_setup.

This structure keeps client-side Firebase code completely separate from server-side Admin SDK code, which matters a lot for security — the Admin SDK and its service account credentials should never be bundled into the client.
