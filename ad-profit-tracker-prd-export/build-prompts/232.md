We're building a Zeydoo ad network integration dashboard. There are no dedicated pages for this feature — instead, we need to wire up the UI components that call the six Zeydoo API endpoints and display the results. Here's what to build:

**Zeydoo Stats Overview panel** — This is the main stats display. It should show the user their Zeydoo revenue data for a selected date range. Include a date range picker (default to the last 7 days, max 90 days). When the date range changes, call GET /api/networks/zeydoo/stats with the dateFrom, dateTo, and groupBy params. Show a toggle between 'By Day' and 'Total' views — this maps to the groupBy parameter ('day' or 'total'). Display revenue and other metrics in a clean table or chart depending on the groupBy selection.

**Latest Stats summary card** — A compact card that calls GET /api/networks/zeydoo/stats/latest on load. Show the most recent day's total revenue, impressions, and sync status pulled from the networkConfigs document. If no data exists yet (404 response), show a friendly 'No data yet — try syncing' message with a button to trigger a manual sync.

**Top Countries breakdown** — Below the main stats, show a table of top countries by revenue. Call GET /api/networks/zeydoo/stats/by-country with the same date range as the overview panel, plus a configurable limit (default 20, max 100). Show each country's revenue and its percentage share of total revenue. Let the user adjust the limit with a dropdown.

**Manual Sync control** — A 'Sync Now' button that calls POST /api/networks/zeydoo/sync. Let the user optionally pick a custom date range for the sync (default to yesterday). Show a loading spinner while the sync is running. On success, refresh the stats panels. On failure, show the error clearly.

**API Explorer / Raw Response viewer** — A collapsible section for power users. Include a date picker and a 'Fetch Raw' button that calls GET /api/networks/zeydoo/raw-response?date=YYYY-MM-DD. Display the returned array of raw response objects in a formatted JSON viewer. Also render the schema definition returned by the endpoint so users understand the data structure.

All panels should pass the Firebase ID token in the Authorization header as a Bearer token for every request. Use the Firebase Auth SDK on the client to get the current user's token via getIdToken().
