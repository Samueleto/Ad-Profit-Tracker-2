We're building the state layer for the ThemeSettingsSection. The goal is to keep the section's logic clean and separate from the UI rendering.

Create a custom hook — something like `useThemePreferences` — that handles all the data and interaction logic for the Appearance section. Here's what it needs to manage:

**Fetched state** — On mount, call GET /api/settings/preferences (with the Firebase ID token in the Authorization header) and store the returned `themePreference` and `accentColor`. Track a loading boolean and an error state separately.

**Local working state** — Keep a separate copy of the theme and accent color that the user is currently editing. This is what the segmented control, swatches, and live preview read from. It starts as a copy of the fetched values, but updates immediately as the user interacts — this is the optimistic update layer.

**next-themes integration** — When the local working theme changes, call next-themes' `setTheme()` so the actual page theme switches right away. The hook should also read the current theme from next-themes so the segmented control always reflects reality.

**Accent color application** — When the local accent color changes, apply it as a CSS custom property on the document root (e.g., `document.documentElement.style.setProperty('--accent-color', value)`). Validate hex values before applying — ignore malformed input.

**Save action** — Expose a `savePreferences()` function that sends PATCH /api/settings/preferences with `{ themePreference, accentColor }` from the current working state. Track a saving boolean and expose a success/error result so the UI can show the 'Saved' toast or an error banner.

**Reset action** — Expose a `resetToDefaults()` function that sets the working state back to `{ themePreference: 'light', accentColor: '#6366f1' }` (indigo), applies them via next-themes and the CSS property, and then calls savePreferences() automatically.

**Custom hex input** — Track whether the 'Custom' swatch is selected as a boolean. Expose a setter for the raw hex input string and a debounced version that validates and applies the color after the user stops typing for 400ms.

The ThemeSettingsSection component should import this hook and only deal with rendering — no fetch calls or theme logic in the component itself.
