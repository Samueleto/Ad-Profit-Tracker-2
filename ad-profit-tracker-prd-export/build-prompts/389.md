We're building a Zeydoo ad network integration. For this step, I need you to make sure errors from all six endpoints are handled gracefully on both the server and client sides.

**Server-side error handling:**

For POST /api/networks/zeydoo/sync — if the Zeydoo CPA statistics API call fails (network error, bad credentials, rate limited by Zeydoo, etc.), update the networkConfigs document with lastSyncStatus='failed' and return 502 to the client. Include a human-readable reason in the response body. Don't let a partial batch write leave data in an inconsistent state — if the Firestore batch write fails mid-way, log the error to auditLogs and surface it clearly. If the user's networkConfig is inactive or missing, return 400 with a clear message.

For POST /api/networks/zeydoo/scheduled-sync — this runs for multiple users, so one user's failure must not stop the others. Wrap each user's sync in a try/catch, write a failed audit log entry for that user, and continue to the next. Return a summary of how many succeeded and failed.

For GET /api/networks/zeydoo/stats/latest — when no data exists, return 404 with a body like { code: 'NO_DATA', message: 'No stats available yet' } so the client can distinguish this from a real error.

For GET /api/networks/zeydoo/raw-response — if the rawResponse field is missing from some adStats documents, return whatever is available rather than failing the whole request.

**Client-side error handling:**

In ZeydooSyncControl — distinguish between a 400 (inactive network config, show 'Please activate your Zeydoo integration first'), a 429 (rate limit, show 'Sync limit reached — try again in X minutes' using the reset time from the response), and a 502 (Zeydoo API failure, show 'Zeydoo API is unavailable — your data will sync automatically later').

In ZeydooStatsOverview and ZeydooTopCountries — if the stats fetch fails, show an inline error with a 'Retry' button rather than a full-page error. Keep any previously loaded data visible while showing the error.

In ZeydooLatestCard — 404 is not an error, it's an empty state. Any other error (401, 500) should show a small error indicator.

In ZeydooRawExplorer — if the fetch returns an empty array (no data for that date), show 'No raw data available for this date' rather than an empty JSON array.
