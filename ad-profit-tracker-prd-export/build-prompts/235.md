We're building a financial analytics dashboard for an ad network portfolio. For this step, I need you to build FinancialMetricsDashboardPage at the /dashboard route.

The page has a dense, data-rich layout. Here's how to structure it top to bottom:

**Date range selector (sticky at top)** — Show preset buttons for Last 7, 14, 30 days, This Month, and a Custom option that opens a date picker. This controls all data on the page. Whenever the selection changes, re-fetch from GET /api/dashboard/metrics with the new dateFrom and dateTo query params (formatted as YYYY-MM-DD using date-fns). Cap the custom range at 90 days — show a validation message if the user tries to exceed it. Next to the selector, show a data freshness badge that says either 'Live' or 'Cached · X min ago' — derive this from the cachedAt field in the API response. Add a 'Refresh Data' button that calls POST /api/cache/invalidate then immediately re-fetches GET /api/dashboard/metrics.

**Four KPI cards (grid: 4 columns desktop, 2 tablet, 1 mobile)** — Pull these from the kpis object in the GET /api/dashboard/metrics response:
- Total Revenue (sum of RollerAds + Zeydoo + Propush) — use kpis.totalRevenue and kpis.revenueChange
- Total Cost (ExoClick only) — use kpis.totalCost and kpis.costChange
- Net Profit (Revenue minus Cost) — use kpis.netProfit and kpis.profitChange
- ROI % — use kpis.roi and kpis.roiChange

Each card shows the current value, a percentage change vs. the prior period, and a directional arrow (up/down). Color code semantically: green for positive profit/ROI, red for negative, amber for zero or null.

**Daily profit trend chart (full width)** — Use recharts to render a line chart from the dailySeries array in the response. Each entry has date, revenue, cost, netProfit, and roi. Show at minimum the netProfit line. Add a tooltip on hover showing all four values for that day. Use date-fns to format the x-axis date labels.

**Two-column split below the chart:**
- Left: GEO breakdown table showing the top 10 countries by net profit from the topCountries array. Columns: country name, revenue, cost, net profit, and metric share (as a percentage bar or text). Sort by net profit descending.
- Right: Per-network summary rows from the perNetwork array. Each row shows the network name, its primary metric value (cost for ExoClick, revenue for others), last sync time (formatted with date-fns), and a sync status badge (color-coded: green for success, red for error, amber for pending).

**ManualRefreshPanel** — Embed the already-built ManualRefreshPanel as a collapsible section at the bottom of the page. Default it to collapsed.

**Export button** — Show an 'Export' button in the top bar that opens a modal with options for Excel and PDF export. For now, the modal can be a placeholder that links to the reporting feature.

**Page states to handle:**
- Loading: show four shimmer skeleton cards, a gray placeholder rectangle where the chart goes, and 5 shimmer rows in the GEO table
- Empty (API returns no data): show 'No data for this date range. Try syncing your networks first.' with a 'Sync Now' CTA that scrolls to the ManualRefreshPanel
- Error states: handled separately in error_handling step
- Success: all sections populated as described above

Use firebase/auth to get the current user's ID token (via getIdToken()) and attach it as a Bearer token in the Authorization header on every call to GET /api/dashboard/metrics and POST /api/cache/invalidate.
