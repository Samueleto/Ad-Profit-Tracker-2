We're building the three core UI components that make the app feel native on mobile. All three are Client Components ('use client') since they manage local state and interact with the browser.

**MobileNavigationDrawer**

This replaces the desktop sidebar on mobile. Render a hamburger button (☰) in the top-left of the mobile header bar inside ResponsiveAppLayout. Tapping it opens a 280px-wide drawer that slides in from the left with a 200ms ease-out CSS transition. Behind it, show a semi-transparent black backdrop at 40% opacity that also fades in. Lock body scroll while the drawer is open.

Inside the drawer (top to bottom): the app logo and name, the authenticated user's avatar and display name pulled from Firebase Auth's currentUser (firebase/auth), then navigation links — Dashboard (/dashboard), Settings (/settings), Reports (/reports) — each as a touch-friendly row with a minimum 48×48px tap target, an icon, and a label. Highlight the active route using the project's accent color CSS custom property from the existing ThemeProvider. Near the bottom, include the existing dark/light theme toggle component from the Dark Mode & UI Theming feature. At the very bottom, a sign-out button that calls Firebase Auth's signOut().

Close the drawer when: any nav link is tapped, the backdrop is tapped, or the user swipes left (handle via touchstart/touchend delta — if horizontal delta exceeds 50px leftward, close). On md+ viewports, hide this entire component (hidden md:hidden on the trigger button).

**MobileBottomNavBar**

A fixed bar at the bottom of the screen, visible only below the md breakpoint. Four tabs at equal width, 56px tall: Dashboard (grid icon → /dashboard), Networks (activity icon → /dashboard#networks), Settings (gear icon → /settings), Reports (file icon → /reports). Each tab has a 24px icon centered above a 10px label. Navigate using Next.js router.push() on tap. Highlight the active tab with the accent color.

Add padding-bottom: env(safe-area-inset-bottom) so it clears the iOS home indicator on notched devices. Add a subtle top border using ThemeProvider tokens. For the Networks tab, read from the shared Zustand dashboard store's health state — if health is 'degraded' or 'critical', show a small red dot badge on the Networks icon (just a dot, no number).

**MobileDataTableWrapper**

A wrapper component used by GeoBreakdownSection, PerNetworkAnalyticsTabsSection, ComparativeNetworkAnalysisTab, PerformanceBenchmarkingTab, and the SettingsPage activity log. It accepts a columns config prop where each column is marked as 'primary' (always visible on mobile) or 'secondary' (hidden until expanded), plus the table's row data and a render function.

On desktop (md+): render the standard table as-is. On mobile: switch to a card-list layout where each row becomes a compact card showing only the primary columns, with a chevron button that expands inline to reveal secondary columns. Track expanded state per row with local useState — no global state needed. For tables where horizontal scanning is essential (like the network comparison table), wrap in an overflow-x-auto container with -webkit-overflow-scrolling: touch and CSS mask-image gradient scroll shadow indicators on the left and right edges. Preserve color-coded row backgrounds (green/red/amber semantic classes) in card layout. Null cells render as '—'. MetricShareBar components reflow to full-width on mobile.
