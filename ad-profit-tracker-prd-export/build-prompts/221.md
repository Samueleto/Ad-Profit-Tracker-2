We're building a geographic performance breakdown feature for a marketing analytics dashboard. For this step, I need you to build two UI pieces that integrate into the existing FinancialMetricsDashboardPage: a GeoBreakdownSection card and a GeoCountryDrilldownModal drawer.

**GeoBreakdownSection** — This is NOT a new page. Add it as a full-width card section inside the existing FinancialMetricsDashboardPage, positioned directly below the DailyProfitTrendSection card. It does not have its own date range selector — it reads the date range from the parent dashboard's shared state.

The card header should be sticky and contain four controls on one row: the card title, a segmented metric toggle (Revenue / Cost / Profit) that controls the sort dimension, a Top N selector (10 / 20 / 50 rows), and a free-text country search input that filters rows client-side without re-fetching.

The main content is a dense table with these columns: country (flag emoji + name), revenue, cost, net profit, ROI%, impressions, clicks, and a metric share column that renders as a thin inline progress bar showing that country's percentage of the total for the selected metric. Null revenue or cost cells must render as '—' — never as '$0'.

Each row should be color-coded using the project's existing Tailwind semantic classes: green-50/text-green-600 for positive ROI, red-50/text-red-600 for negative ROI, and amber-50/text-amber-600 for zero or null ROI. The colorCode for each country comes from GET /api/roi/breakdown called with dimension='country'. The table data itself comes from GET /api/stats/geo-breakdown. Both calls use the shared date range and attach a Firebase Bearer token in the Authorization header.

Below the table, add a collapsible 'GEO Insights' strip — collapsed by default with a chevron toggle. When expanded, it shows four stats: top-performing country, worst-performing country, number of countries with positive ROI, and total countries tracked.

Each table row should also support a 'Network Breakdown' inline expand — clicking a toggle on the row expands a sub-row showing per-network contribution for that country (network name, data role, primary metric value, percentage of total).

Clicking any country row opens the GeoCountryDrilldownModal, passing the country code and name.

For loading state: show 10 shimmer rows matching the column structure, with skeleton placeholders in the card header controls. For empty state: show 'No geographic data for this date range. Sync your networks to see country breakdowns.' with a 'Sync Now' CTA linking to the ManualRefreshPanel.

**GeoCountryDrilldownModal** — This is a slide-over drawer that opens from the right side (480px wide on desktop, full-screen on mobile) when a country row is clicked in GeoBreakdownSection. It should have a smooth slide-in animation and a backdrop overlay. On mobile, it should behave as a full-screen sheet with drag-to-dismiss.

At the top, show the selected country's flag emoji and name as the modal title, plus a close (X) button in the top-right corner. Show a label for the active date range.

Below the title, render a recharts LineChart (200px height) showing net profit per day for the selected country over the active date range. This data comes from GET /api/stats/snapshot called with a country filter — you'll need to aggregate the daily data points across the date range.

Below the chart, show a per-network breakdown table with columns: network name, data role (Cost Only / Revenue Only), primary metric value, and percentage of total.

Add a 'View Full Trend' link that navigates to the DailyProfitTrendSection with the country pre-filtered.

For loading state: show a 200px shimmer rectangle where the chart goes and 4 shimmer rows for the network table. For empty state: show 'No data available for [Country] in this period' with a close button.
