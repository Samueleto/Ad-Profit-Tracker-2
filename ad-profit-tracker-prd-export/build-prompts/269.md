# Step 269 of 437: Set up reconciliation state and data hooks

We need clean state management for the reconciliation feature so the three views don't each manage their own fetch logic in isolation. Build a set of custom React hooks that encapsulate all the API calls and local state.

Create a **useReconciliationStatus** hook that calls GET /api/reconciliation/status. It should accept an optional networkId parameter. It should expose: the status data, a loading boolean, an error value, and a refetch function. This hook is used by ReconciliationOverviewPanel and NetworkStatusCard.

Create a **useRunReconciliation** hook that wraps POST /api/reconciliation/run. It should track: whether a run is in progress (isRunning), the last run's result summary, and any error. It should accept networkId (optional), dateFrom, and dateTo as parameters. After a successful run, it should automatically trigger a refetch of the status data so the network cards update without a page reload.

Create a **useAnomalies** hook that calls GET /api/reconciliation/anomalies. It should manage: the current list of anomalies, pagination state (current cursor, hasMore), filter state (networkId, dateFrom, dateTo, severity, anomalyType), loading state for initial load vs. loading more, and an error value. It should expose a loadMore() function that appends the next page to the existing list (don't replace — append). When filters change, reset the list and cursor and reload from scratch.

Create a **useResolveAnomalies** hook that wraps PATCH /api/reconciliation/resolve. It should track: isResolving, resolved count from the last operation, and errors. After a successful resolve, it should remove the resolved document IDs from the anomaly list in local state (optimistic removal) and trigger a status refetch to update anomaly counts on the network cards.

Create a **useValidationRules** hook that calls GET /api/reconciliation/rules and PATCH /api/reconciliation/rules. It should track: the current rules per network, which fields are using defaults (from the usingDefaults array in the response), a saving boolean, and errors. The update function should accept a networkId and a partial rules object, call the PATCH endpoint, and merge the response back into local state.

All hooks should attach the Firebase ID token to every request as a Bearer token in the Authorization header. Pull the token from the Firebase Auth current user using getIdToken(). Handle token expiry by catching 401 responses and triggering a re-authentication flow rather than silently failing.
