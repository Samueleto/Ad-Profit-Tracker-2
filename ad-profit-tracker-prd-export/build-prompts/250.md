# Step 250 of 437: Build AuthContext to manage global auth state

================================================================================

We need a global auth context so every part of the app knows who's signed in, can access the user's profile, and can trigger sign-out without prop drilling.

Create an `AuthContext` and `AuthProvider` that wraps the entire app. Here's what it needs to manage:

**State to track:**
- `user` — the full user profile object from the `users` Firestore collection (not just the Firebase Auth user object — we want the role, preferences, displayName, etc.)
- `firebaseUser` — the raw Firebase Auth user, needed for getting ID tokens
- `loading` — true while we're figuring out the initial auth state on app load
- `error` — any auth-related error

**On app load:** Firebase's `onAuthStateChanged` listener should fire immediately. If there's a signed-in user, call GET `/api/auth/get-user` with their Bearer token to hydrate the full user profile from Firestore. If that returns a 404 (user doc doesn't exist yet), fall back to calling POST `/api/auth/sync-user` to create it, then fetch again. Set `loading` to false once this resolves either way. If there's no signed-in user, set `user` to null and `loading` to false.

**Methods to expose:**
- `signOut()` — calls Firebase's `signOut()`, clears the user state, and redirects to `/`
- `updateUser(data)` — calls PATCH `/api/auth/update-profile` with the provided fields, then updates the local `user` state with the response so the UI reflects changes immediately without a full reload
- `deleteAccount()` — calls DELETE `/api/auth/delete-account`, then clears state and redirects to `/`
- `getIdToken()` — helper that calls `firebaseUser.getIdToken()` and returns the token string, used internally by the API call helpers

**Token handling:** Don't store the ID token in state — Firebase Auth manages token refresh automatically. Always call `getIdToken()` fresh before each API request so you're never sending an expired token.

Wrap the Next.js app with `AuthProvider` in the root layout so every page and component can access auth state via a `useAuth()` hook.
