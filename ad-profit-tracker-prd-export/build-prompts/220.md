We're building a real-time sync monitoring feature for an ad network dashboard. For this step, add a SyncStatusPanel section to the existing dashboard at /dashboard, integrated into the ManualRefreshPanel area — this is not a new route, it's a new section within the existing dashboard page.

Here's what the panel needs to show and do:

**Health Banner** — Stick a banner at the top of the section that shows overall system health: 'All Systems Healthy', 'Degraded', or 'Critical'. Drive this from the `overallHealth` field returned by GET /api/sync/live-state. Color it green, amber, or red respectively. This banner should always be visible even when the cards below are loading.

**Network Status Grid** — Below the banner, show a 2-column grid on desktop and single column on mobile with four cards, one per ad network: ExoClick, RollerAds, Zeydoo, and Propush. Each card shows:
- Network name and icon
- Current sync phase label ('Idle', 'Fetching', 'Writing', 'Complete', 'Failed') — use the `syncPhase` field
- An animated pulsing indicator when the phase is 'Fetching' or 'Writing' (amber color with CSS pulse animation)
- Last synced timestamp displayed as relative time (e.g. '3 minutes ago') from `lastSyncedAt`
- Latest data date from `latestDataDate`
- Row count from `lastRowsFetched`
- Latency in ms from `lastLatencyMs`
- A circuit breaker badge: 'Open' in red or 'Closed' in green based on `circuitBreakerOpen`

Color coding: amber + pulse = in-progress phases, green = 'complete'/'success', red = 'failed', gray = 'idle' or never synced.

Clicking a card should expand inline details showing `circuitBreakerOpenedAt`, `retryCount`, `totalFailureCount`, and `lastErrorCode`. Use a simple toggle — no navigation.

**Recent Activity Feed** — Below the grid, add a collapsible 'Recent Activity' section. Load the last 5 entries from GET /api/sync/activity-feed (pass `limit=5`). Each entry shows: network name, a status badge (success/failure), event label (e.g. 'Scheduled Sync', 'Manual Sync'), rows fetched, and a relative timestamp. The section should be collapsed by default and expandable with a toggle.

**Critical Anomalies Strip** — If GET /api/reconciliation/anomalies returns any results with `severity='critical'`, show an inline alert strip above the grid (below the health banner) with the count of critical anomalies and a 'View Details' link pointing to the reconciliation section of the dashboard.

**Polling Logic** — On mount, call GET /api/sync/live-state immediately. Then poll every 5 seconds if any network has `syncPhase` of 'fetching' or 'writing' (i.e., a sync is actively in progress). Fall back to a 30-second interval when all networks are idle or complete. On unmount, clear all intervals to prevent memory leaks. Also call GET /api/reconciliation/status to get the `overallHealth` field (or use the one from /api/sync/live-state if it includes it — check the response shape and avoid duplicate calls).

**Refresh Button** — Add a small 'Refresh Status' icon button (a circular arrow icon) in the panel header. Clicking it immediately re-fetches GET /api/sync/live-state without triggering a sync. Reset the polling interval timer after a manual refresh.

**Loading State** — While the initial fetch is in progress, show shimmer skeleton placeholders for all four network cards and gray placeholder badges. The health banner can show a neutral 'Checking...' state.

**Empty State** — If all four networks have `isActive=false`, show a message: 'No networks configured — add API keys in Settings' with a link to /settings.

Attach a Firebase ID token (via `getIdToken()` from firebase/auth) as a Bearer token in the Authorization header on every API call.
