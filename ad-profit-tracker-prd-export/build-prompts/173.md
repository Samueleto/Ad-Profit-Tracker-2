We need one API endpoint that powers the entire ComparativeNetworkAnalysisTab. This is GET /api/networks/comparison and it returns a consolidated comparison payload for all four ad networks (ExoClick, RollerAds, Zeydoo, Propush) in a single response.

Here's what the endpoint needs to do, in order:

First, authenticate. Pull the Authorization header, strip 'Bearer ', and verify the token using firebase-admin's admin.auth().verifyIdToken(). If the token is missing or invalid, return 401 immediately. The uid comes exclusively from the verified token — never from query params.

Second, validate the query params. dateFrom and dateTo are required, must be valid YYYY-MM-DD strings, dateFrom must be <= dateTo, and the range can't exceed 90 days. The metric param is optional and defaults to 'roi' — if provided, it must be one of: 'revenue', 'cost', 'roi', 'impressions', 'clicks', 'ctr', 'cpm'. Return 400 with a clear message for any validation failure.

Third, check the node-cache using the key '{uid}_network_comparison_{dateFrom}_{dateTo}_{metric}'. If there's a cache hit, return the cached result immediately with an X-Cache: HIT header.

Fourth, on a cache miss, run a single Firestore getDocs query on the adStats collection filtering by userId == uid AND date >= dateFrom AND date <= dateTo. This one query covers all four networks — don't run separate per-network queries. If no documents come back, return 404.

Fifth, process the results in memory. Group documents by networkId. For each network, compute: primaryMetric (sum of cost for exoclick, sum of revenue for the other three), impressions sum, clicks sum, weighted average CTR, weighted average CPM, and count of unique dates. Then compute cross-network derived metrics: totalRevenue (rollerads + zeydoo + propush revenue), totalCost (exoclick cost), netProfit, overallRoi (null if totalCost is 0), revenuePerImpression (null if no impressions), costPerClick (null if no exoclick clicks). Compute each network's metricShare as a percentage of the total across all four. Rank networks by the requested metric descending, handling ties by assigning the same rank.

Sixth, run a second getDocs on networkConfigs filtered by userId == uid to get lastSyncedAt, lastSyncStatus, and circuitBreakerOpen for each network. Merge this into the per-network results.

Seventh, store the full result in node-cache with a 5-minute TTL, then return it as the 200 response matching the ComparisonResponse shape. Include cachedAt as the current timestamp.

Return 500 with a generic message if any Firestore read throws — don't leak internal error details.
