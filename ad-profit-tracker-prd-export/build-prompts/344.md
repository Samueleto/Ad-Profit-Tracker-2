The API key management feature handles sensitive credentials, so let's make sure every layer is properly secured.

On the three API endpoints — POST /api/keys/save, GET /api/keys/status, and DELETE /api/keys/delete — every request must start by verifying the Firebase ID token from the Authorization header using firebase-admin's `auth().verifyIdToken()`. If the token is missing, malformed, or expired, return 401 immediately. Never proceed without a verified UID.

For POST /api/keys/save, validate that `networkId` is strictly one of the four allowed values: 'exoclick', 'rollerads', 'zeydoo', 'propush'. Reject anything else with a 400. Validate that `key` is a non-empty string. The encryption using `cryptr` must happen server-side only, using a master secret stored in environment variables (or Cloud Secret Manager). The raw key must never appear in server logs — make sure no logging middleware accidentally captures the request body. Store the encrypted key in the apiKeys collection at `users/{uid}/apiKeys/{networkId}`.

For DELETE /api/keys/delete, the UID used to construct the Firestore path must always come from the verified token — never from the request body or query params. This ensures a user can only delete their own keys.

For GET /api/keys/status, confirm the response only contains `networkId`, `status`, and `updatedAt` — never the `encryptedKey` field. Even though it's encrypted, there's no reason to send it to the client.

On the Firestore security rules side, the apiKeys subcollection at `users/{uid}/apiKeys` should only allow reads and writes where `request.auth.uid == userId`. The `createdAt` and `userId` fields should be immutable after creation. Deletions should be allowed (this is a hard delete, not soft delete, based on the endpoint spec).

Add rate limiting to POST /api/keys/save — no more than a handful of saves per minute per user to prevent abuse. Return 429 if exceeded.

Make sure the `ENCRYPTION_SECRET` environment variable is set in your deployment environment and is never committed to source control.
