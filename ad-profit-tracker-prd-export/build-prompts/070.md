We need three packages wired up before we can build the export endpoints:

**1. ExcelJS** — This is what generates the `.xlsx` files. Install `exceljs` at version `^4.3.0`. We'll use it server-side only (inside the API routes), so no client bundle concerns. No special initialization needed — we'll import and instantiate `Workbook` directly in the workbook builder utility.

**2. firebase-admin** — The export endpoints need to verify Firebase ID tokens and query Firestore. If firebase-admin is already initialized elsewhere in the project (common in Next.js apps), just import the existing instance. If not, create a shared initialization file at something like `src/lib/firebase-admin.ts` that calls `admin.initializeApp()` using a service account. Store the service account credentials in environment variables (`FIREBASE_PROJECT_ID`, `FIREBASE_CLIENT_EMAIL`, `FIREBASE_PRIVATE_KEY`) — never hardcode them. Export the `admin.firestore()` and `admin.auth()` instances so the route handlers can import them cleanly. Make sure the initialization is guarded so it only runs once even in Next.js's hot-reload environment (check `if (!admin.apps.length)` before initializing).

**3. node-cache** — The preview endpoint caches row counts for 3 minutes to avoid hammering Firestore on every modal open. Install `node-cache`. Create a shared cache instance in `src/lib/cache.ts` — instantiate `new NodeCache({ stdTTL: 180 })` and export it as a singleton. The preview endpoint will use this to store and retrieve results keyed by `{uid}_export_preview_{dateFrom}_{dateTo}`.

Make sure all three packages are listed in `package.json` and that the project builds cleanly after installation.
