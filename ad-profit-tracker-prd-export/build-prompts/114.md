We need TypeScript interfaces that describe the data shapes flowing through GeoBreakdownSection and GeoCountryDrilldownModal. These types sit between the API responses and the UI components, so getting them right now will make the rest of the build much smoother.

Create the following types in the geo-breakdown types folder:

**GeoCountryRow** — represents one row in the GeoBreakdownSection table. Fields: countryCode (string), countryName (string), flagEmoji (string), revenue (number | null), cost (number | null), netProfit (number | null), roi (number | null), impressions (number | null), clicks (number | null), metricShare (number — 0 to 100, the percentage of total for the currently selected metric), colorCode (string — the semantic color class like 'positive', 'negative', or 'neutral' returned by GET /api/roi/breakdown). Null fields should render as '—' in the UI, never coerced to zero.

**GeoNetworkContribution** — represents one row in the per-network sub-row that expands inline on a country row. Fields: networkName (string), dataRole ('Cost Only' | 'Revenue Only' | 'Both'), primaryMetricValue (number | null), percentageOfTotal (number).

**GeoBreakdownResponse** — the shape of the response from GET /api/stats/geo-breakdown. Fields: countries (GeoCountryRow[]), totalRevenue (number), totalCost (number), totalProfit (number).

**GeoRoiEnrichment** — the shape of the relevant part of GET /api/roi/breakdown response when called with dimension='country'. Fields: countryCode (string), colorCode (string).

**GeoSnapshotDayPoint** — one data point for the recharts LineChart in GeoCountryDrilldownModal. Fields: date (string), netProfit (number | null).

**MetricToggle** — a union type: 'Revenue' | 'Cost' | 'Profit'. Used by the segmented control in the GeoBreakdownSection card header.

**TopNOption** — a union type: 10 | 20 | 50. Used by the Top N selector.

These types are derived from the adStats schema that backs the API endpoints. Make sure null is used (not undefined) for optional numeric fields, since the UI explicitly renders '—' for null values.
