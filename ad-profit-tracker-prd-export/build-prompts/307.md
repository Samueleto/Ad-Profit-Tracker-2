We're building a Propush ad network integration. For this step, I need you to wire all six Propush API endpoints into the frontend so the app can actually talk to them.

Create a Propush API client module (something like `lib/api/propush.ts`) that wraps all six endpoints with typed fetch/axios calls. Each function should automatically attach the Firebase ID token from the current session as a Bearer token in the Authorization header. The six functions to build are:

1. `syncPropushStats(dateFrom?: string, dateTo?: string)` — calls POST /api/networks/propush/sync with the optional date range body.
2. `getPropushStats(dateFrom: string, dateTo: string, groupBy?: 'day' | 'total')` — calls GET /api/networks/propush/stats with the required date params.
3. `getPropushStatsByCountry(dateFrom: string, dateTo: string, limit?: number)` — calls GET /api/networks/propush/stats/by-country.
4. `getLatestPropushStats()` — calls GET /api/networks/propush/stats/latest.
5. `getPropushRawResponse(date: string)` — calls GET /api/networks/propush/raw-response with the required date param.
6. The scheduled sync endpoint (POST /api/networks/propush/scheduled-sync) is internal-only and should NOT be called from the frontend — make sure there's no client-side wrapper for it.

For the token attachment, use Firebase Auth's `getIdToken()` on the current user before each request. If there's no current user, throw an error immediately rather than making an unauthenticated request.

Also wire up the scheduled sync on the backend side: set up a Cloud Scheduler job (or equivalent cron trigger) that calls POST /api/networks/propush/scheduled-sync daily, passing the INTERNAL_SYNC_SECRET in the `x-internal-secret` header. Document where this secret needs to be set in the environment config.

Make sure the API client functions return typed responses — define TypeScript interfaces for the response shapes from each endpoint (stats rows, country breakdown, latest summary, raw response with schema definition).
