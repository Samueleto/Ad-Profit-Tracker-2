We're building the ExportModal — a dialog that lets users configure and trigger an Excel export from the FinancialMetricsDashboardPage. This is a `renderAs: modal` component that gets triggered by the existing 'Export' button on the dashboard. No new route needed.

The modal should be centered on screen with a backdrop overlay, max width `max-w-lg`, using the project's existing ThemeProvider tokens for light/dark mode (`dark:bg-gray-900` / `bg-white`).

**Tabs** — At the top of the modal, show two tabs: 'Excel (.xlsx)' and 'PDF'. Excel should be the default active tab. The PDF tab can be a placeholder for now (show 'Coming soon' or similar). All the logic below applies to the Excel tab.

**Sheet selection** — Show a list of 8 checkbox rows, one per sheet: Summary, Daily Trend, Geographic Breakdown, ExoClick, RollerAds, Zeydoo, ProPush, and Activity Log. Each row has a checkbox, a label, and a row count badge on the right. The row counts come from GET /api/export/preview — call this endpoint when the modal opens (using the current date range from the Zustand store) and populate the badges. While the preview data is loading, show 5 shimmer placeholder rows (the `loading` state). If a sheet has 0 rows, show the badge in a muted style and keep the checkbox enabled (the user can still select it, but the Export Now button will be disabled if ALL selected sheets have 0 rows).

**Date range display** — Below the tabs, show a read-only display of the currently active date range. Read this from the Zustand store — don't add an independent date picker here.

**Advanced options** — Add a collapsible 'Advanced' section with a chevron toggle. Inside: a text input for a custom filename (max 100 chars, alphanumeric/dash/underscore only, show a character count), and a toggle for 'Include column headers' (default on). Keep this collapsed by default.

**Footer buttons** — 'Cancel' closes the modal without doing anything. 'Export Now' calls POST /api/export/excel with the selected sheets, date range, and any advanced options. While the export is in progress (the `exporting` state), show a spinner on the Export Now button, disable it and all checkboxes, and show the message 'Generating your Excel file...' below the sheet list. Disable Export Now with a tooltip 'No data available for this range' if no data exists for the selected range (`hasData: false` from the preview response).

**Success handling** — When the POST response comes back, trigger the file download by creating a blob URL from the binary response and programmatically clicking a hidden anchor tag. Show a 'Export complete — file downloaded' toast. Close the modal after 1.5 seconds.

**Error states** — Handle each error code: 401 closes the modal and attempts `getIdToken(true)` to refresh the Firebase token, redirecting to `/` with a 'Session expired' toast if that fails; 403 shows 'Access Denied' inside the modal with a close button; 404 disables Export Now with 'No data found for this date range'; 500 shows an inline error banner inside the modal with a 'Retry' button.

Use the project's existing Tailwind semantic classes and the accent color CSS custom property for the Export Now button.
