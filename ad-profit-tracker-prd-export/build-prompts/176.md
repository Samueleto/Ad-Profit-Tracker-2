# Step 176 of 437: Build the five network API route handlers

We need five API route handlers. All of them run server-side, which means API keys from the `apiKeys` Firestore collection are read using the Firebase Admin SDK and never exposed to the client.

Before building each route, create a shared server-side utility (something like `src/lib/network-fetcher.ts`) that handles the common logic: reading an API key from Firestore by network ID, making the outbound HTTP call to the ad network, and writing the result to the `cachedResponses` collection with a `fetchedAt` timestamp.

**GET /api/networks/exoclick/raw-response** — Checks Firestore for a cached response for `exoclick`. If one exists, returns it immediately with the `fetchedAt` timestamp. If not, returns a 200 with `{ cached: false, rawResponse: null }` so the UI can show the 'no cached response yet' state. Does NOT automatically hit the ExoClick API — that's what the fresh sample button is for.

**GET /api/networks/rollerads/raw-response** — Same pattern as above but for `rollerads`.

**GET /api/networks/zeydoo/raw-response** — Same pattern for `zeydoo`.

**GET /api/networks/propush/raw-response** — Same pattern for `propush`.

All four raw-response endpoints should also return the `NetworkConfig` for that network (endpoint URL, HTTP method, required params) so the UI has everything it needs in one call. If the `apiKeys` document for that network doesn't exist in Firestore, return a 200 with `{ configured: false }` — the UI will show the 'API key not configured' state with a link to settings.

**POST /api/networks/config/test-connection** — This is the 'Fetch Fresh Sample' action. Accepts a `networkId` in the request body. Reads the API key from Firestore, makes a live call to that network's API endpoint (using the config from `networkConfigs`), saves the response to `cachedResponses`, and returns the full raw response plus latency in milliseconds. If the network call fails, return a structured error response (don't throw a 500) so the UI can display it inline.

All five routes must verify the user is authenticated (check the Firebase Auth session cookie or Authorization header) before doing anything. Return 401 if not authenticated.
