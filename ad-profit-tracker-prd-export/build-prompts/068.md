We need two packages installed before building the API Explorer UI.

First, install `react-json-view`. This gives us a syntax-highlighted, collapsible JSON viewer with line numbers that we'll use to display raw API responses from ExoClick, RollerAds, Zeydoo, and Propush. It's a client-side component, so make sure it's only rendered on the client (use dynamic import with `ssr: false` in Next.js to avoid hydration issues — react-json-view doesn't play well with server-side rendering).

Second, install `zustand`. We'll use this for managing the per-accordion state across the four network sections — things like which accordion is open, the fetched JSON payload for each network, loading states, and error states. Zustand is lightweight and works well for this kind of localized UI state without needing a full context setup.

After installing both, create a quick smoke-test: a minimal zustand store with a single counter value, and a throwaway page that renders a `ReactJson` component with a hardcoded object. Confirm both render without errors, then delete the throwaway page. This catches any SSR or import issues early before we build on top of them.

Also confirm that the Firebase Admin SDK is already initialized somewhere in the project (we'll need it server-side in the API routes to read `apiKeys` and `networkConfigs` from Firestore). If it's not set up yet, initialize it in a shared `src/lib/firebase-admin.ts` file using environment variables for the service account credentials.
