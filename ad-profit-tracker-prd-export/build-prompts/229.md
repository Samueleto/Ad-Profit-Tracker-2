We're building a network configuration management feature for an ad analytics platform. For this step, add a new 'Networks' tab called NetworkConfigTab to the existing SettingsPage at /settings — don't create a new route, just wire in a new tab.

The tab renders a card grid showing all four supported ad networks: ExoClick, RollerAds, Zeydoo, and Propush. Each card is always visible (there's no empty state — all four always appear). On load, call GET /api/networks/config/list to fetch all four network configs. While loading, show skeleton shimmer cards in the same 4-card grid layout, with shimmer on the status badge and schedule dropdown.

Each network card should show: the network name and logo/icon, a color-coded connection status badge (green chip for 'connected', gray for 'not_connected', red for 'error'), a data role label ('Cost Only' for ExoClick, 'Revenue Only' for the other three), a sync schedule dropdown (hourly/daily/weekly) that calls PATCH /api/networks/config/update on change, the last synced timestamp formatted as a relative time (e.g. '2 hours ago'), and an 'Active' toggle that also calls PATCH /api/networks/config/update immediately when flipped.

Each card has a 'Show advanced' chevron that expands to reveal: a custom API endpoint override text field (validated as a URL), a request timeout slider from 5 to 60 seconds, a retry attempts selector from 1 to 5, and a 'Test Connection' button. When 'Test Connection' is clicked, show a spinner on the button and call POST /api/networks/config/test-connection with the networkId. On success, show a green checkmark with the latency in ms and sample record count inline on the card. On failure, show a red X with the error message. Each card also has a 'Save Network Settings' button that sends the current advanced settings via PATCH /api/networks/config/update. Include a 'Reset to Defaults' option per card that calls DELETE /api/networks/config/reset and reloads that card's values.

At the top of the tab, add a 'Sync All Now' button that calls POST /api/networks/sync-all. After it responds, show a summary toast listing which networks were triggered, skipped, and failed.

The four network cards should be draggable and reorderable using dnd-kit. When the user drops a card in a new position, immediately call PATCH /api/networks/config/reorder with the full updated order array (all 4 networks with their new displayOrder values). Optimistically update the UI order before the response comes back, and revert if the call fails.

For the error states: if the list call returns 401, attempt a Firebase ID token refresh and retry once — if it still fails, redirect to / with a 'Session expired' toast. If it returns 403, show an 'Access Denied' inline banner inside the tab. If it returns 404, silently initialize defaults and render the cards with default settings. If any individual card's save or test call returns 500, show an inline error banner on that specific card with a 'Retry' button rather than failing the whole tab.
