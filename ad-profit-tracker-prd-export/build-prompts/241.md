We're building a developer-focused API Response Explorer. This is NOT a new page — it's a new tab called 'API Explorer' added to the existing FinancialMetricsDashboardPage at /dashboard.

The tab contains four collapsible accordion sections, one for each ad network: ExoClick, RollerAds, Zeydoo, and Propush. All four should be collapsed by default so the interface stays clean on first load.

Each accordion section should show:
- The API endpoint URL and HTTP method for that network (e.g., GET /api/networks/exoclick/raw-response)
- The required parameters for that endpoint
- A JSON viewer using react-json-view that displays the most recent raw API response. This data comes from the network's dedicated raw-response endpoint. Use syntax highlighting and line numbers so the JSON is easy to read.
- A 'Fetch Fresh Sample' button that, when clicked, calls /api/networks/config/test-connection for that specific network, triggering a live test call and updating the JSON viewer with the fresh response
- A 'Copy JSON' button that copies the currently displayed response to the clipboard
- A 'Last fetched' timestamp showing when the displayed data was retrieved
- A field mapping table with two columns — 'Raw API Field' and 'Internal adStats Field' — showing exactly how the network's raw response fields map to the app's internal adStats schema fields

Handle these states per accordion section:
- **Loading**: While fetching the last cached response, show a shimmer placeholder where the JSON viewer would be
- **No API key configured**: Instead of the JSON viewer and controls, show a message like 'API key not configured' with a direct link to /settings. Check this against the apiKeys and networkConfigs schemas.
- **No cached response yet**: Show the message 'No response cached yet. Click Fetch Fresh Sample to retrieve one.' in place of the JSON viewer
- **Fetching fresh sample**: Disable the 'Fetch Fresh Sample' button and show a spinner on it while the live call is in progress
- **Error**: Show the error message inline within the accordion body with a 'Retry' button

The overall feel should be tool-like and informational — think developer docs meets live debugger. Keep the design clean and functional, not decorative.
