Show a compact list of log entries: emailType, status (success/failure badge), sentAt timestamp, and triggeredBy label. Add a 'Load more' button if hasMore is true, which fetches the next page using the nextCursor. This gives users visibility into whether their alerts are actually being delivered.

**Summary email frequency** — If the SettingsPage has a general preferences section where users can set notification frequency, make sure any emailSummaryFrequency preference (daily/weekly/monthly) is also surfaced here or nearby, since it controls which summary emails the user receives from POST /api/email/schedule-summary.
