We need a set of reusable components and data-fetching hooks that the ExoClick stats pages will use. Build these before the pages so the pages can import them directly.

**Data-fetching hooks** — Create these in `src/features/exoclick/hooks/`:

- `useExoClickStats(dateFrom: string, dateTo: string, groupBy: 'day' | 'total')` — Calls `GET /api/networks/exoclick/stats` with the given params. Returns `{ data, isLoading, error, refetch }`. Attach the user's Firebase ID token to the `Authorization` header automatically (use the Firebase client SDK's `getIdToken()`).

- `useExoClickStatsByCountry(dateFrom: string, dateTo: string, limit?: number)` — Calls `GET /api/networks/exoclick/stats/by-country`. Same return shape.

- `useExoClickLatest()` — Calls `GET /api/networks/exoclick/stats/latest`. Returns the latest day's summary stats plus `lastSyncedAt` and `lastSyncStatus`.

- `useExoClickRawResponse(date: string)` — Calls `GET /api/networks/exoclick/raw-response` for a specific date. Returns the records array and fieldSchema.

**UI Components** — Create these in `src/features/exoclick/components/`:

- `ExoClickSummaryCard` — Displays a single metric (cost, impressions, clicks, CTR, or CPM) as a stat card with a label, formatted value, and optional trend indicator. Used in both the latest stats summary and the date-range totals view. Accepts `label`, `value`, `format` ('currency' | 'number' | 'percent'), and optional `trend` props.

- `ExoClickStatsTable` — A table showing day-by-day stats. Columns: Date, Cost, Impressions, Clicks, CTR, CPM. Accepts an array of daily stat rows from the `GET /api/networks/exoclick/stats` response. Handles empty state with a message prompting the user to run a sync.

- `ExoClickCountryTable` — A table showing per-country breakdown. Columns: Country (flag + name), Cost, Cost Share (as a progress bar or percentage), Impressions, Clicks, CTR. Accepts the countries array from `GET /api/networks/exoclick/stats/by-country`.

- `ExoClickSyncButton` — A button that triggers `POST /api/networks/exoclick/sync`. Accepts `dateFrom` and `dateTo` props. Shows a loading spinner while the sync is in progress. On success, shows a brief success message with `rowsFetched`. On error, shows the error message. Calls an `onSyncComplete` callback so the parent page can refetch stats.

- `ExoClickApiExplorer` — Shows the raw API response for a selected date. Uses `useExoClickRawResponse`. Renders the `fieldSchema` as a reference table and the raw records as a formatted JSON viewer (use a `<pre>` block or a lightweight JSON viewer component). Includes a date picker to select which day's raw response to view.

All components should handle loading and error states gracefully — show skeleton loaders while data is fetching, and a clear error message with a retry button if the request fails.
