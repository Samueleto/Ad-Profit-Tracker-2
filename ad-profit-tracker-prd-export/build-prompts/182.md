We're building five new endpoints and extending two existing ones. All of them import from the shared types, email client, and rate limiter we set up earlier.

**POST /api/email/send-alert** — This is the core alert delivery endpoint. It supports two authentication paths: a Firebase ID token (for user-initiated calls) or an `x-internal-secret` header (for Cloud Function triggers). Check whichever is present — if neither is valid, return 401. Validate the request body: `type` must be one of the 9 `NotificationType` values, `severity` one of 4 values, `title` max 100 chars, `body` max 500 chars. Read the user's Firestore document from the `users` collection to check `notificationPreferences[type].emailEnabled` — if false, return 200 with `{ sent: false, reason: 'disabled_by_user' }` without sending anything. Get the delivery email from `users.preferences.alertDeliveryEmail`, falling back to the Firebase Auth user record email via `admin.auth().getUser(uid)`. Check the rate limit using key `{uid}_email_alert_{type}` (max 10/hour). Render the alert HTML template with a severity-colored badge (red=error, amber=warning, green=success, blue=info), the title, body text, optional `networkId` context, optional metadata key-value pairs, and a footer linking to `/settings`. Send via the Nodemailer client. Write an audit log to `auditLogs`. Return `{ sent: true, recipientEmail, type, severity }`.

**POST /api/email/send-summary** — Internal-only. Validate `x-internal-secret` first — no Firebase token path. Validate `userId`, `dateFrom`/`dateTo` as `YYYY-MM-DD`, and `summaryType`. Read the user's `users` document, check `notificationPreferences.schedule_delivered.emailEnabled` and `preferences.emailSummaryFrequency`. Query the `adStats` collection for the date range and compute KPIs in memory: `totalRevenue`, `totalCost`, `netProfit`, `roi`, `topCountry`, `bestNetwork`, `daysWithData`. Render the summary HTML template with a KPI table and a 'View Full Dashboard' CTA linking to `/dashboard`. Rate limit: 1/day per summaryType. Write audit log with KPIs in metadata.

**POST /api/email/send-test** — Firebase ID token required. Rate limit: 3/hour per user (key: `{uid}_email_test`). Get delivery email from `users.preferences.alertDeliveryEmail` or Firebase Auth fallback. Send the test email template. Update `users.preferences.lastTestEmailSentAt` to server timestamp. Write audit log with `triggeredBy: 'user_test'`. Return `{ sent: true, recipientEmail, sentAt }`.

**POST /api/email/schedule-summary** — Internal-only, designed for Cloud Scheduler. Validate `x-internal-secret`. Compute yesterday's date range. Query `users` collection for users with `notificationPreferences.schedule_delivered.emailEnabled == true` (or fetch all and filter in memory if the index isn't available — add a comment explaining the tradeoff). Filter to users with `emailSummaryFrequency == 'daily'` or unset. For each qualifying user, call `send-summary` logic directly (not via HTTP). Write a single batch audit log entry. Return `{ processed, sent, skipped, failed, date }`.

**GET /api/email/log** — Firebase ID token required. Validate `limit` (1–50, default 20), optional `type` filter. Query `auditLogs` where `userId == uid AND action == 'email_sent'`, optionally filtering `resourceId == type`. Order by `createdAt DESC`. Use cursor-based pagination with `startAfter`. Map documents to the `EmailAuditLog` shape. Return `{ logs, hasMore, nextCursor }`.

**Extend GET /api/notifications/preferences** — In the existing merge-with-defaults logic, add `emailEnabled` to each type's default object: `true` for the 3 critical types, `false` for the 6 informational ones. Also include `alertDeliveryEmail` and `lastTestEmailSentAt` from `users.preferences` in the response.

**Extend PATCH /api/notifications/preferences** — Accept `emailEnabled` (boolean) per type alongside the existing `enabled` field. Accept `alertDeliveryEmail` (valid email, max 254 chars) as a top-level field. Use Firestore dot-notation partial updates for `notificationPreferences.[type].emailEnabled` and `preferences.alertDeliveryEmail`. Extend the audit log metadata to include `changedEmailTypes` and `alertDeliveryEmailChanged`.
