We're building an automated daily data sync system. For this step, I need you to create the client-side state management that powers the sync monitoring UI — tracking sync status across all four networks (exoclick, rollerads, zeydoo, propush), sync history, and retry operations.

Create a sync status hook (something like `useSyncStatus`) that calls GET /api/scheduled/sync-status on mount and returns the `networks` array and `lastMasterSyncAt` timestamp. Each network entry has `networkId`, `isActive`, `syncSchedule`, `lastSyncedAt`, `lastSyncStatus` (one of 'success', 'failed', 'partial', 'never'), `lastSyncError`, `latestDataDate`, and `nextScheduledSync`. This hook should also expose a `refresh()` function so the UI can poll or manually re-fetch. Poll every 30 seconds while the page is mounted so users see live updates without refreshing.

Create a sync history hook (something like `useSyncHistory`) that calls GET /api/scheduled/sync-history. It should accept an optional `networkId` filter and a `limit` (default 10, max 50). Support cursor-based pagination — expose a `loadMore()` function that appends the next page of results to the existing list using the `nextCursor` from the previous response. The hook should track `history`, `hasMore`, `isLoading`, and `error` states.

Create a retry hook (something like `useRetrySync`) that calls POST /api/scheduled/retry-failed with a `networkId` in the body. It should track `isRetrying` (per network, not globally — so two networks can't be retried simultaneously but independently), `retryError`, and `retryResult`. On success, automatically trigger a refresh of the sync status so the UI reflects the updated `lastSyncStatus`. On 429 (rate limit), surface a specific message like 'You've retried this network too many times this hour — try again later.' On 502, surface the network API error clearly.

All three hooks should attach the Firebase ID token from the currently signed-in user to the Authorization header as a Bearer token. If the token fetch fails or returns 401, redirect to the login page.

Keep these hooks independent so they can be composed into whatever page layout is needed. Don't couple them to any specific component.
