We're building a financial analytics dashboard. For this step, I need you to add the ROIIndicatorSection into the existing FinancialMetricsDashboardPage at /dashboard — this is NOT a new page or route, it's a new highlighted section that slots into the existing 4-column KPI card grid.

The ROI card should visually stand out from the other KPI cards — use a heavier font weight on the ROI percentage value to signal it's the primary profitability signal. The value comes from the `kpis.roi` and `kpis.roiChange` fields already returned by GET /api/dashboard/metrics, so no additional API calls are needed for the main card.

Here's what the ROI card needs to show:

**The main KPI card** — Display the ROI percentage to two decimal places. Apply color coding using Tailwind semantic classes: `text-green-600 bg-green-50` for positive ROI, `text-red-600 bg-red-50` for negative ROI, and `text-amber-600 bg-amber-50` for zero or null ROI. Next to the value, show a directional arrow (↑ for up, ↓ for down, → for flat/null) and a label like '+12.4% vs prior period'. When ROI is null (meaning totalCost === 0 or no cost data exists), show 'N/A' with an amber neutral badge and a tooltip that says 'No cost data available for this period'.

**Breakeven line on the daily profit trend chart** — On whatever recharts chart already shows daily profit trends, add a dashed gray horizontal reference line at y=0. This marks the breakeven threshold visually.

**Per-network ROI contribution rows** — Below the main card, show a small table or list of rows, one per revenue network (rollerads, zeydoo, propush), showing each network's effective yield relative to ExoClick cost. These values come from GET /api/roi/breakdown called with dimension='network'. Call this endpoint when the section mounts, passing the currently selected date range from the dashboard's existing date filter state.

**Collapsible ROI Breakdown panel** — Add a 'ROI Breakdown' toggle below the KPI card. It should be collapsed by default (progressive disclosure). When expanded, show the formula: (Revenue - Cost) / Cost × 100, with the actual live values substituted in — e.g., '($4,200 - $3,100) / $3,100 × 100 = 35.48%'. Pull these numbers from the same `kpis` data already in scope.

**Loading state** — While the dashboard metrics are loading, show a shimmer placeholder that matches the ROI card's dimensions. Hide the directional arrow during loading.

**States to handle:**
- Loading: shimmer card, no arrow
- Empty (no data for range): amber N/A badge, 'No data for selected range'
- error_404: show 'N/A' with tooltip 'No data found for this date range'
- error_500: show an error banner inside the ROI card with a retry button that re-calls GET /api/dashboard/metrics
- error_401: attempt Firebase ID token refresh via `getIdToken(true)`, and if that fails, redirect to / with a 'Session expired' toast notification
- error_403: show 'Access Denied' inline within the card with a link to /dashboard
- Success: full color-coded ROI card with arrow, change label, network rows, and collapsible breakdown

The per-network breakdown rows and the collapsible panel should each have their own loading/error states independent of the main card, since they call GET /api/roi/breakdown separately.
