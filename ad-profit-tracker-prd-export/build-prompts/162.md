We're building five API endpoints for the sync system. Here's what each one does and how to build it:

**GET /api/scheduled/health** — Build this first since it's the simplest and validates the infrastructure. Check the x-internal-secret header against INTERNAL_SYNC_SECRET — return 401 immediately if it doesn't match. Then do a lightweight Firestore read (fetch one document from networkConfigs) to verify database connectivity. Check that INTERNAL_SYNC_SECRET and ENCRYPTION_SECRET environment variables are set. Return a health object with component-level checks: { status: 'healthy' | 'unhealthy', checks: { firestore: 'ok' | 'error', encryptionSecret: 'set' | 'missing', internalSecret: 'set' | 'missing' }, timestamp: string }. Return 503 if any check fails.

**GET /api/scheduled/sync-status** — Verify the Firebase ID token from the Authorization Bearer header using admin.auth().verifyIdToken(). Query the networkConfigs collection for all documents where userId matches the verified uid. For each of the four networks in SUPPORTED_NETWORKS, extract lastSyncedAt, lastSyncStatus, lastSyncError, isActive, and syncSchedule — use safe defaults if a document doesn't exist for that network. Also query adStats for each network (userId==uid AND networkId==network, order by date DESC, limit 1) to get the latestDataDate. Compute nextScheduledSync based on syncSchedule and lastSyncedAt. Return the consolidated status object.

**GET /api/scheduled/sync-history** — Verify the Firebase ID token. Validate query params: networkId must be in SUPPORTED_NETWORKS if provided, limit must be an integer between 1 and 50 (default 10). Query auditLogs where userId==uid AND action=='manual_sync_triggered' AND resourceType=='sync', optionally filtering by resourceId==networkId. Order by createdAt DESC. Use cursor-based pagination with startAfter if a cursor document ID is provided. Fetch limit+1 results to determine hasMore. Map to SyncHistoryEntry shape and return.

**POST /api/scheduled/retry-failed** — Verify the Firebase ID token. Validate that networkId in the request body is in SUPPORTED_NETWORKS. Check rate limiting: max 5 retries per hour per user per network (you can track this in memory for now or use a Firestore counter). Fetch the networkConfigs document for this userId+networkId — if lastSyncStatus isn't 'failed', return 400. Fetch the encrypted API key from apiKeys — return 404 if missing. Determine the retry date from the most recent failed auditLogs entry for this user+network, or default to yesterday. Decrypt the key using the cryptoHelper, call the appropriate network client (same logic as the master sync), batch-upsert results into adStats, update networkConfigs, and write an audit log entry. Return 502 on network API failure.

**POST /api/scheduled/daily-sync** — This is the most complex endpoint. Validate x-internal-secret header — return 401 if wrong. Determine yesterday's date in UTC as YYYY-MM-DD. Query the networkConfigs collection group for all documents where isActive==true AND syncSchedule=='daily'. Group results by userId. For each userId, check which of the four networks have both an active networkConfig AND an API key in the apiKeys collection — skip combinations missing a key. Use Promise.all() to fan out all user+network sync tasks in parallel. For each task: decrypt the API key, call the appropriate network client (ExoClick Global Statistics with groupByCountry; RollerAds GET /sites/stat with group_by=country; Zeydoo CPA statistics with country grouping; Propush /v5/pub-statistics with geo grouping), parse the response rows into AdStatRow objects, batch-upsert into adStats in chunks of 500 using Firestore Admin SDK batch writes. Wrap each network API call in the retryWithBackoff helper (up to retryAttempts from networkConfigs, default 3, with 1s/2s/4s delays). After each attempt, update the networkConfigs document with lastSyncedAt, lastSyncStatus, and lastSyncError. Write one audit log entry per user+network via the auditLogger helper. Collect all results into a DailySyncResponse and return it.

For the four network client wrappers (src/lib/sync/networkClients/), each should accept a decrypted API key and a date string, call the network's API, and return a normalized array of AdStatRow objects. Keep the network-specific parsing logic isolated in these files so the main endpoint stays clean.

The firestoreBatch helper should accept an array of AdStatRow objects and a Firestore db instance, split into chunks of 500, and run batch.set() with merge:true on each document (use a deterministic document ID like userId_networkId_date_country so re-runs are idempotent).

The retryWithBackoff helper should accept an async function and a max retry count, catch errors, wait 1s/2s/4s between attempts, and rethrow after exhausting retries.
