We're building a mobile-responsive navigation and layout system for a Next.js + Firebase app. For this step, I need you to build four components that work together to give every protected page a great mobile experience.

**MobileNavigationDrawer** — This is a slide-in drawer that replaces the desktop sidebar on screens below the `md` breakpoint. Add a hamburger button (☰) to the top-left of a mobile header bar. Tapping it opens a 280px-wide drawer from the left edge with a semi-transparent black backdrop (40% opacity). Inside the drawer, show the app logo and name at the top, then the authenticated user's avatar and display name pulled from Firebase Auth's `currentUser`. Below that, render navigation links for Dashboard (`/dashboard`), Settings (`/settings`), and Reports (`/reports`) — each row must be at least 48px tall with an icon and label side by side. Highlight the active route using the project's accent color CSS custom property from the existing ThemeProvider. Near the bottom, include the existing dark/light theme toggle component. At the very bottom, a sign-out button that calls Firebase Auth's `signOut()`. Tapping any nav link or the backdrop closes the drawer. Also support swipe-left to dismiss using `touchstart`/`touchend` listeners — if the horizontal delta is large enough, close the drawer. The drawer slides in with a 200ms ease-out CSS transition. On `md+` screens, hide this entire component — the desktop sidebar takes over.

**MobileBottomNavBar** — A fixed bar pinned to the bottom of the screen, only visible below `md`. It has four tabs: Dashboard (grid icon → `/dashboard`), Networks (activity icon → `/dashboard#networks`), Settings (gear icon → `/settings`), and Reports (file icon → `/reports`). Each tab takes up a quarter of the full viewport width and is 56px tall with a centered icon (24px) and a 10px label below it. Use Next.js `router.push()` for navigation. Highlight the active tab with the accent color. Add `padding-bottom: env(safe-area-inset-bottom)` so the bar clears the home indicator on notched iOS devices. Read from the shared Zustand dashboard store's health state — if health is `'degraded'` or `'critical'`, show a small red dot badge on the Networks icon. No animations on tab switch. On `md+`, hide this bar entirely.

**ResponsiveAppLayout** — This is the root layout wrapper for the `(app)` route group (`layout.tsx`). It should orchestrate everything: on desktop (`md+`), render the existing desktop sidebar (240px fixed width) on the left with the main content filling the rest, using CSS Grid. On mobile, hide the desktop sidebar, render the MobileNavigationDrawer hamburger trigger in a 56px top header bar, and render MobileBottomNavBar fixed at the bottom. Add `padding-bottom: calc(56px + env(safe-area-inset-bottom))` to the mobile content area so nothing gets hidden behind the bottom nav. This layout wraps all protected pages — FinancialMetricsDashboardPage, SettingsPage, OnboardingWizardPage, and all dashboard sections — so they all get responsive nav automatically. Use Tailwind's `md:` responsive prefixes for layout structure (CSS Grid/Flexbox), not JavaScript. Add a `useMediaQuery` hook (extending the existing system preference detection from the Dark Mode & UI Theming feature) that returns a boolean for the `md` breakpoint — use this only for conditionally rendering interactive components like the drawer's open state, not for layout itself. Show the current route name as a mobile page title in the header bar.

**MobileDataTableWrapper** — A `'use client'` wrapper component that makes data-heavy tables work on mobile. It accepts a `columns` config prop where each column is marked as `'primary'` (always visible on mobile) or `'secondary'` (hidden until expanded). On `md+`, render the standard HTML table with all columns. On mobile, switch to a card-list layout where each table row becomes a compact card showing only the primary columns, with a chevron that expands inline to reveal secondary columns — track expanded state per row with `useState`. For tables where column scanning is essential (like the network comparison table), wrap in a horizontally scrollable container (`overflow-x-auto`, `-webkit-overflow-scrolling: touch`) with CSS `mask-image` gradient shadows on the left and right edges to hint that more columns exist. Apply this wrapper to the tables in GeoBreakdownSection, PerNetworkAnalyticsTabsSection, ComparativeNetworkAnalysisTab, PerformanceBenchmarkingTab, and the SettingsPage activity log. MetricShareBar components inside these tables should reflow to full-width on mobile. Preserve color-coded row backgrounds and null cell rendering (`—`) in the card layout. Use the same ThemeProvider tokens as the table rows so dark mode works consistently.
