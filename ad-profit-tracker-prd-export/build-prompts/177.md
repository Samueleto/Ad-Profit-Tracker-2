We need two API route handlers for the export feature. Here's what each one does:

**GET /api/export/preview** — This is called when the ExportModal opens to populate the row count badges on each sheet checkbox. It's a read-only endpoint that counts rows without generating any file.

Start by verifying the Firebase ID token from the `Authorization: Bearer <token>` header using `admin.auth().verifyIdToken()` — return 401 if missing or invalid. Validate the `dateFrom` and `dateTo` query params using the Zod schema — return 400 if invalid.

Before hitting Firestore, check the node-cache singleton for the key `{uid}_export_preview_{dateFrom}_{dateTo}`. If there's a cache hit, return the cached result immediately with an `X-Cache: HIT` response header.

On a cache miss, query the `adStats` Firestore collection filtering by `userId == uid` AND `date >= dateFrom` AND `date <= dateTo` using `getDocs`. Also query `auditLogs` filtering by `userId == uid` AND `createdAt >= dateFrom` AND `createdAt <= dateTo`. Compute row counts in memory: `summary` is always 1, `daily_trend` is the count of unique dates, `geo_breakdown` is the count of unique country codes, each network sheet (`exoclick`, `rollerads`, `zeydoo`, `propush`) is the count of docs where `networkId` matches, and `activity_log` is the count of auditLogs docs. Sum everything for `totalRows`. Set `hasData` to true if `totalRows > 0`. Store the result in node-cache with a 3-minute TTL, then return the `ExportPreviewResponse` shape.

**POST /api/export/excel** — This is the main export endpoint. It generates and streams back a `.xlsx` file.

Verify the Firebase ID token the same way. Validate the request body with the Zod schema — return 400 with validation details if invalid. Check a rate limit: max 10 exports per hour per user. A simple approach is to use node-cache to store a counter keyed by `{uid}_rate_limit_{currentHour}` — increment on each request and return 429 if it exceeds 10.

Query `adStats` and (if `activity_log` is in the requested sheets) `auditLogs` from Firestore, both scoped to the authenticated user and the requested date range.

If no data is found, return 404.

Build the workbook using ExcelJS: create a new `Workbook` instance, then for each sheet key in the `sheets` array, call `workbook.addWorksheet()` with the right name and populate it with the right columns and rows (see the endpoint spec for the exact column definitions per sheet). Apply header row styling — bold font, `#4F46E5` background fill, white text. Freeze the top row on each sheet using `worksheet.views = [{ state: 'frozen', ySplit: 1 }]`. Set column widths.

Generate the filename: use the provided `filename` or default to `ad-profit-tracker-export-{dateFrom}-{dateTo}`. Write the workbook to a buffer with `workbook.xlsx.writeBuffer()`. Set the response headers (`Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`, `Content-Disposition: attachment; filename={filename}.xlsx`) and send the buffer.

Finally, write an audit log entry to the `auditLogs` Firestore collection recording the export action, which sheets were included, the date range, total row count, and filename.
