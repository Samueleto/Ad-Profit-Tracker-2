We need several reusable pieces that the `EmailAlertPreferencesSection` will be built from. Building these first keeps the section component clean and makes each piece independently testable.

**SeverityBadge** — A small pill/chip component that takes a `severity: Severity` prop and renders with the appropriate color: red background for `error`, amber for `warning`, green for `success`, blue for `info`. Shows the severity label in capitalized text. Used in each toggle row to communicate how important that alert type is at a glance.

**NotificationToggleRow** — A single row in the preferences list. Takes a `NotificationTypeConfig` (label, description, severity, isCritical) plus `emailEnabled: boolean` and an `onChange: (emailEnabled: boolean) => void` callback. Renders the notification type label in bold, the description in muted text below it, the `SeverityBadge` on the right, and an email toggle switch. When the toggle changes, call `onChange` immediately (auto-save behavior is handled by the parent). Show a brief animated checkmark flash after a successful save — accept an optional `saveState: 'idle' | 'saving' | 'saved' | 'error'` prop to drive this.

**MasterEmailToggle** — A prominent toggle row at the top of the section. Takes `allEnabled: boolean` (true if all 9 types have `emailEnabled: true`), `someEnabled: boolean` (true if at least one is enabled — for indeterminate state), and an `onChange: (enabled: boolean) => void` callback. When turned on, it enables all 9 types; when turned off, it disables all 9. Show an indeterminate state visually if some but not all are enabled.

**DeliveryEmailField** — A form field for the alert delivery email address. Takes `value: string`, `onChange`, and `onSave` callbacks, plus a `saveState` prop. Pre-filled with the user's email. Shows a 'Save Email Address' button that calls `onSave` — the button is disabled if the value hasn't changed or is invalid. Validate email format inline (show a red helper text if the format is wrong). Show a loading spinner on the button while saving.

**SendTestButton** — A button that triggers `POST /api/email/send-test`. Takes `onClick`, `isLoading`, and `lastTestSentAt: string | null` props. Shows 'Send Test Alert' as the label, a spinner while loading, and below the button shows 'Last test sent: [relative time]' if `lastTestSentAt` is set (e.g., '2 hours ago'). Disabled while loading.

**EmailLogTable** — A compact table showing the user's email send history from `GET /api/email/log`. Columns: Email Type, Recipient, Status (success/failure badge), Sent At (relative time), Triggered By. Supports pagination via a 'Load more' button using the `nextCursor` from the API. Shows a loading skeleton while fetching. This component is used within `EmailAlertPreferencesSection` under the advanced/collapsed section.
