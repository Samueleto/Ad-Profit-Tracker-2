We're building an API rate limiting and throttling system on top of Firebase. For this step, I need you to create the client-side data fetching and state management layer that the rest of the app will use to interact with the four rate limit endpoints.

Create a custom hook called `useRateLimitStatus` that calls `GET /api/rate-limits/status`. It should accept an optional `networkId` parameter (one of `'exoclick'`, `'rollerads'`, `'zeydoo'`, `'propush'`) — if omitted, it fetches status for all four networks. The hook should expose the `networks` array, the `userQuotas` array, a `polledAt` timestamp, a `loading` boolean, and an `error` state. This hook should support polling — accept a `pollIntervalMs` option so callers can keep the status fresh automatically (a 30-second default is reasonable). Make sure it cancels the interval on unmount.

Create a `useRateLimitConfig` hook that calls `GET /api/rate-limits/config`. This is static configuration data, so it only needs to fetch once. Expose `networkLimiters`, `userQuotaConfig`, `loading`, and `error`.

Create a `useRateLimitViolations` hook that calls `GET /api/rate-limits/violations`. It should accept optional `networkId` and `endpoint` filter params, plus a `limit` (default 20). Implement cursor-based pagination — expose a `loadMore` function that appends the next page to the existing list using the `nextCursor` from the previous response. Expose `violations`, `hasMore`, `loading`, `error`, and `loadMore`.

Create a `useRateLimitReset` hook that wraps `POST /api/rate-limits/reset`. It should accept `networkId` and `scope` (`'outbound'`, `'user-quota'`, or `'both'`), and expose a `reset` function, a `loading` boolean, and `error`. After a successful reset, it should trigger a refresh of the `useRateLimitStatus` hook — the cleanest way to do this is to expose an `onSuccess` callback option or use a shared invalidation mechanism (like a simple counter in context that the status hook watches).

All four hooks must attach the Firebase ID token to every request as a `Bearer` token in the `Authorization` header. Use `firebase/auth`'s `getIdToken()` on the current user to get a fresh token before each request — don't cache the token yourself, Firebase handles refresh automatically.

Put all four hooks in a `features/rate-limits/hooks/` folder and export them from an index file.
