We need to build all seven API endpoints for the error handling system. Each one lives in `src/app/api/errors/` and uses the shared utilities we built in the previous steps. Every single endpoint must call `verifyToken()` first and return 401 immediately if it fails — no exceptions.

**GET /api/errors/retry-state** — Accepts an optional `networkId` query param. If provided, validate it against the four supported networks (return 400 if invalid). Query the `networkConfigs` Firestore collection filtered to the authenticated user's `uid`. For each document, extract the retry state fields and compute three derived values: `isRetryable` (use the `isRetryable()` utility, also check `circuitBreakerOpen==false`), `timeUntilNextRetrySeconds` (`nextRetryAt - now` in seconds, null if no pending retry), and `circuitBreakerAutoResetAt` (`circuitBreakerOpenedAt + 3600s` if the circuit is open). Return the array as `{ networks: [...] }`.

**GET /api/errors/circuit-breaker/status** — No query params. Query all four `networkConfigs` documents for the user. For each, compute `autoResetAt` and flag `isAutoResetPending` if the current time is past `autoResetAt` but the circuit is still marked open. Return a `summary` with counts of healthy vs degraded networks, plus the per-network `circuits` array.

**POST /api/errors/circuit-breaker/reset** — Takes `{ networkId }` in the request body. Validate the network ID (400 if invalid). Check the rate limit (10 resets/hour per user — return 429 if exceeded). Fetch the `networkConfigs` document; return 404 if not found, 400 if `circuitBreakerOpen` is already false. Update the document: set `circuitBreakerOpen=false`, `circuitBreakerOpenedAt=null`, `retryCount=0`, `nextRetryAt=null`, `lastSyncStatus='pending'`, `lastSyncError=null`, `lastErrorCode=null`, `updatedAt=serverTimestamp()`. Write an audit log entry to the `auditLogs` collection with `action='network_config_updated'`, `resourceType='network_config'`, `resourceId=networkId`, and metadata capturing the previous `retryCount` and `errorCode` plus `resetBy: 'user'`.

**GET /api/errors/log** — Accepts optional query params: `networkId`, `errorCode`, `startDate`, `endDate`, `limit` (1–50, default 20), `cursor`. Validate all params (400 on bad values). Query the `auditLogs` collection filtered to `userId==uid`, `status=='failure'`, `resourceType=='sync'`. Apply optional filters for network and date range. Order by `createdAt DESC`. Use `startAfter` cursor pagination — fetch `limit + 1` docs to determine `hasMore`. Enrich each record with `errorCategory` using `getErrorCategory()`. Return the paginated response.

**GET /api/errors/summary** — Accepts optional `dateFrom` and `dateTo` query params (default to last 7 days). Query `auditLogs` for the user within the date range. Separate successes and failures, group by `resourceId` (networkId). For each network compute: `totalAttempts`, `successCount`, `failureCount`, `successRate`, `mostRecentError`, `mostCommonErrorCode`, `averageLatencyMs`. Also query `networkConfigs` to find which networks have `circuitBreakerOpen==true`. Identify the most problematic network by highest `failureCount`. Return the full summary.

**GET /api/errors/retry-config** — No params. Query `networkConfigs` for the user. For each network, extract `retryAttempts`, `timeoutSeconds`, `retryCount`, `nextRetryAt`, `circuitBreakerOpen`. Call `computeBackoffSchedule(retryAttempts)` to generate the full backoff schedule. Return the array.

**PATCH /api/errors/retry-config** — Takes `{ networkId, retryAttempts?, timeoutSeconds? }`. Validate: network ID must be valid, `retryAttempts` must be 1–5 if provided, `timeoutSeconds` must be 5–60 if provided, at least one field must be present. Check rate limit (20 updates/hour). Fetch the document (404 if missing). Build the update payload with only the provided fields plus `updatedAt=serverTimestamp()`. Write an audit log capturing the before/after values of changed fields.
