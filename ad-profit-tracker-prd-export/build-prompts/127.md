We need TypeScript types to keep the sync feature well-typed end-to-end. Create these in a shared types file (e.g., `lib/types/sync.ts` or `src/features/manual-refresh/types.ts`) so both the API route and the frontend components can import from the same place.

**NetworkId type** — A union type: `'exoclick' | 'rollerads' | 'zeydoo' | 'propush'`. Export this and use it everywhere a network identifier is expected — don't use raw strings.

**SyncStatus type** — A union: `'success' | 'failed' | 'never' | 'in_progress'`. This maps to the badge states shown in the ManualRefreshPanel.

**NetworkSyncState interface** — Represents one row in the ManualRefreshPanel. Fields: `networkId: NetworkId`, `lastSyncedAt: Date | null`, `lastSyncStatus: SyncStatus`, `lastSyncError: string | null`.

**ManualSyncRequest interface** — The request body shape for `POST /api/sync/manual`. Fields: `networkId?: NetworkId`, `dateFrom?: string`, `dateTo?: string`.

**ManualSyncResult interface** — One entry in the results array from the endpoint response. Fields: `networkId: NetworkId`, `status: 'success' | 'failed' | 'skipped'`, `rowsFetched: number | null`, `latencyMs: number | null`, `errorMessage: string | null`.

**ManualSyncResponse interface** — The full success response from `POST /api/sync/manual`. Fields: `triggered: string[]`, `skipped: string[]`, `failed: string[]`, `results: ManualSyncResult[]`, `triggeredAt: string`.

**SyncHistoryEntry interface** — Represents one row in the sync history drawer (populated from `GET /api/scheduled/sync-history`). Fields: `networkId: NetworkId`, `status: SyncStatus`, `rowsFetched: number | null`, `latencyMs: number | null`, `triggeredAt: string`, `triggeredBy: 'user' | 'scheduler'`.

**NetworkConfig interface** — Mirrors the `networkConfigs` Firestore document shape. Fields: `networkId: NetworkId`, `isActive: boolean`, `lastSyncedAt: Date | null`, `lastSyncStatus: SyncStatus`, `lastSyncError: string | null`, `retryAttempts: number`.

Make sure all these types are exported and used consistently — the API route handler, the `syncNetworkForUser` helper, and the ManualRefreshPanel components should all import from this shared file rather than defining their own inline types.
