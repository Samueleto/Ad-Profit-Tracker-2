We're building an ad network analytics dashboard. For this step, I need you to set up the client-side state management that powers the ExoClick views — keeping data fetching, loading states, and error handling out of the UI components themselves.

Create a set of custom React hooks (or a context/store if the app already has a pattern for that) to manage the following:

**useExoClickLatest** — Fetches from `GET /api/networks/exoclick/stats/latest`. Exposes: `data` (the latest stats object), `isLoading`, `error`, and a `refetch()` function. The `ExoClickSummaryCard` component uses this. After a successful manual sync, the sync panel should call `refetch()` so the summary card updates automatically.

**useExoClickStats(dateFrom, dateTo, groupBy)** — Fetches from `GET /api/networks/exoclick/stats` whenever the date range or groupBy changes. Debounce the fetch by ~300ms so rapid date picker changes don't fire multiple requests. Exposes: `data` (the stats array and totals), `isLoading`, `error`. Used by `ExoClickStatsView`.

**useExoClickCountries(dateFrom, dateTo, limit)** — Fetches from `GET /api/networks/exoclick/stats/by-country`. Same debounce pattern. Exposes: `countries`, `totalCost`, `isLoading`, `error`. Used by `ExoClickCountryBreakdown`.

**useExoClickSync** — Manages the manual sync flow. Exposes: `triggerSync(dateFrom, dateTo)` which POSTs to `POST /api/networks/exoclick/sync`, `isSyncing` (boolean), `lastResult` (the success response or null), `error`. When `triggerSync` completes successfully, it should also invalidate/refetch the latest stats so the summary card reflects the new data. Used by `ExoClickSyncPanel`.

**useExoClickRawResponse(date)** — Fetches from `GET /api/networks/exoclick/raw-response` for a given date. Fetches when the date changes. Exposes: `records`, `fieldSchema`, `isLoading`, `error`. Used by `ExoClickApiExplorer`.

All hooks should attach the Firebase ID token to every request as a Bearer token in the Authorization header. Use the Firebase Auth `currentUser.getIdToken()` method to get a fresh token before each request — don't cache the token in state since it can expire.

For date range state (shared between the stats view and country breakdown), consider lifting it to a shared parent component or a small context so both views stay in sync when the user changes the date range.
