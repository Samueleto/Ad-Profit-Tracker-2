We need three API route handlers. Here's what each one does:

**GET /api/health** ‚Äî This is a public liveness probe, no auth needed. Cloud Run calls this to decide if an instance is alive. Do a lightweight Firestore connectivity check by calling `db.collection('users').limit(1).get()` wrapped in the `withTimeout()` utility with a 2-second limit. Record how long it takes (`firestoreLatencyMs`). Check memory usage via `process.memoryUsage()` and convert `heapUsed` to MB. Determine the overall status: return `'healthy'` if Firestore responds under 1 second and memory is under 400MB; return `'degraded'` if Firestore responds but took over 1 second; return `'unhealthy'` (with HTTP 503) if Firestore times out or throws. Also include `uptime` from `process.uptime()` and `version` from `process.env.npm_package_version` or a fallback. Use the `HealthResponse` type for the response shape.

**GET /api/ready** ‚Äî Also public, no auth. Cloud Run calls this during startup to decide when to send traffic. Check that all four required env vars are set (`FIREBASE_PROJECT_ID`, `INTERNAL_SYNC_SECRET`, `EMAIL_PROVIDER_API_KEY`, `EMAIL_FROM_ADDRESS`) ‚Äî if any are missing, return HTTP 503 with `{ ready: false, failedCheck: 'envVars' }`. Then check that Firebase Admin is initialized using `isFirebaseAdminInitialized()` ‚Äî if not, return HTTP 503 with `{ ready: false, failedCheck: 'firebaseAdmin' }`. If both pass, return HTTP 200 with `{ ready: true, checks: { envVars: 'ok', firebaseAdmin: 'ok' } }`. Never log or return the actual values of env vars ‚Äî only their presence. Use the `ReadinessResponse` type.

**GET /api/deployment/status** ‚Äî This one requires auth and is for admin tooling only. Extract the Bearer token from the `Authorization` header and verify it using `admin.auth().verifyIdToken(token)` ‚Äî return 401 if missing or invalid. Take the `uid` from the verified token (never from the request body or query params). Look up the user's Firestore document at `users/{uid}` and check that `workspaceRole` is `'owner'` or `'admin'` ‚Äî return 403 if not. Then read the deployment metadata from environment variables: `CLOUD_RUN_SERVICE_NAME`, `CLOUD_RUN_REGION`, `K_REVISION`, `K_SERVICE`. Compute `uptimeSeconds` from `process.uptime()`. Determine the `environment` label from `DEPLOYMENT_ENV` or fall back to `NODE_ENV`. Return the `DeploymentStatusResponse` shape. If any env var read throws unexpectedly, return 500.
