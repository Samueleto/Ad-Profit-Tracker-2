We're building a financial metrics dashboard. For this step, I need you to add a DateRangeToolbar as a persistent, sticky control bar at the top of the existing FinancialMetricsDashboardPage, sitting above the KPI cards.

This toolbar is not a separate page — it integrates directly into FinancialMetricsDashboardPage as a sticky section just below the sidebar nav on desktop. It should never scroll away.

Here's what the toolbar needs to do:

**Preset controls** — Render six preset options as a horizontal segmented pill control: Last 7 Days, Last 14 Days, Last 30 Days, Last 90 Days, This Month, and Custom. The active preset should be visually highlighted. Clicking any preset (except Custom) immediately applies the date range — no confirmation needed. Use date-fns to calculate the actual start/end dates for each preset (e.g., `subDays`, `startOfMonth`) and serialize them to YYYY-MM-DD format before writing to shared state.

**Custom date picker** — When the user selects 'Custom', reveal an inline date range picker to the right of the pill buttons (not in a modal). Use react-day-picker to render two calendar inputs: a 'From' date and a 'To' date. Enforce a 90-day maximum span client-side — if the selected range exceeds 90 days, show a small red validation message below the inputs reading 'Maximum range is 90 days' and disable the 'Apply' button. When the range is valid, the 'Apply' button confirms the selection and writes to shared state.

**Data availability indicator** — Next to the date range label, show a small dot that reflects whether the selected range has data. This dot calls GET /api/stats/dates (with the current date range as query params and a Firebase Bearer token in the Authorization header). While the request is in flight, show a gray spinner. Based on the response, show green for complete data, amber for partial, or gray for no data.

**Last synced badge** — Show a small 'Last synced' timestamp badge in the toolbar, reading from the shared dashboard state (not from a separate API call).

**Mobile behavior** — On mobile, collapse the six preset pills into a single dropdown select element. When 'Custom' is selected on mobile, open the date picker as a bottom sheet instead of inline.

Style everything with Tailwind semantic classes, consistent with the rest of the project. The toolbar should be a single compact row on desktop.
