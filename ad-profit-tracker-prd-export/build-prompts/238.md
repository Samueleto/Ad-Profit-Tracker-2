We're building a financial analytics dashboard. For this step, I need you to add two new integrated sections to the existing FinancialMetricsDashboardPage — both render as tabs or sections within that page, not as standalone routes.

**PerNetworkAnalyticsTabsSection** — This goes below the existing GeoBreakdownSection on the dashboard. Build a tab bar with four tabs: ExoClick, RollerAds, Zeydoo, and Propush. Each tab label should include a small status dot — green if the network's last sync succeeded, red if it failed, and gray if it's never been synced. Pull this status from the `networkStatus.lastSyncStatus` field returned by `GET /api/networks/stats`.

Each tab panel is a full-width card with three sections stacked vertically:

1. **KPI summary row** — Four stat cards showing: primary metric (Cost for ExoClick, Revenue for the other three), Impressions, Clicks, CTR%, and CPM. While loading, show shimmer placeholder cards for all four.

2. **Daily trend AreaChart** — Use recharts to render a 280px tall area chart showing the primary metric over the selected date range. Load this data by calling `GET /api/networks/stats` with `groupBy=daily`. Color the fill green for revenue networks (RollerAds, Zeydoo, Propush) and amber for ExoClick (the cost network). Add a breakeven line at y=0 and interactive tooltips showing the exact value on hover. While loading, show a 280px shimmer rectangle.

3. **GEO breakdown table** — Load country-level data from `GET /api/networks/stats` with `groupBy=country` (not from `/api/roi/breakdown` directly — use the network-scoped endpoint). Columns: flag + country name, primary metric, impressions, clicks, CTR%, and a MetricShareBar. Color-code rows using the colorCode logic from GeoBreakdownSection (green/red/amber). Reuse the MetricShareBar, MetricToggle, and TopNSelector components from GeoBreakdownSection. Clicking any country row opens the existing GeoCountryDrilldownModal. Null cells render as '—'. While loading, show 8 shimmer rows.

4. **Network Insights strip** — A collapsible section (collapsed by default, chevron to expand) showing: best country, worst country, total countries tracked, and last sync timestamp. Reuse the collapsible insights strip pattern from GeoBreakdownSection.

Each tab also has a 'Sync This Network' button that calls `POST /api/sync/manual` with the specific `networkId`.

**Lazy loading is critical** — only call `GET /api/networks/stats` when a tab is first activated. Don't fire all four requests on mount. Once a tab's data is loaded, cache it in component state so switching back doesn't re-fetch.

The date range comes from the parent dashboard's shared state — there is no independent date selector on this section. When the parent date range changes, invalidate all tab data and re-fetch the active tab.

**ApiExplorerTab** — Add a fifth tab to the same tab bar labeled 'API Explorer'. This is a read-only reference panel. Build four accordion sections, one per network (ExoClick, RollerAds, Zeydoo, Propush), all collapsed by default.

Each accordion section shows:
1. The API endpoint URL and HTTP method the app uses for that network
2. Required parameters (date range, groupBy, access token field name)
3. A syntax-highlighted JSON code block showing the most recent raw API response, fetched from the appropriate per-network endpoint: `GET /api/networks/exoclick/raw-response`, `GET /api/networks/rollerads/raw-response`, `GET /api/networks/zeydoo/raw-response`, or `GET /api/networks/propush/raw-response`. Show a 'Loading last response...' shimmer while fetching. Include a 'Copy Response' button that copies the JSON to clipboard and a 'Last fetched' timestamp.
4. A field mapping table (two columns: API field → App field) showing how raw API fields map to the internal `adStats` schema fields. Include type annotations.
5. A 'Fetch Fresh Sample' button that calls `POST /api/networks/config/test-connection` for that network and displays the response inline.

If no API key is configured for a network (detectable from the response), show 'API key not configured — go to Settings' with a link to /settings instead of the JSON viewer.

Use a lightweight JSON formatter for syntax highlighting — no heavy dependencies needed. Monospace font for JSON blocks, sans-serif for the mapping table.
