We need the Firebase Admin SDK available to all seven error handling API routes. All of them use `firebase-admin` to verify ID tokens and read/write Firestore.

Check if the project already has a Firebase Admin initialization file (something like `src/lib/firebase/admin.ts` or `src/lib/firebaseAdmin.ts`). If it does, we'll just import from there — don't create a second initialization. If it doesn't exist yet, create one now.

The initialization file should:
- Import `firebase-admin` and initialize the app using `initializeApp()` with `applicationDefault()` credentials (or using a service account JSON if the project already has one configured)
- Export a `db` instance from `getFirestore()` for Firestore access
- Export an `auth` instance from `getAuth()` for token verification
- Guard against double-initialization with a check like `getApps().length === 0` before calling `initializeApp()`

Make sure the following environment variables are documented (add them to `.env.example` if the project has one):
- `GOOGLE_APPLICATION_CREDENTIALS` (path to service account JSON, for local dev)
- Or `FIREBASE_PROJECT_ID`, `FIREBASE_CLIENT_EMAIL`, `FIREBASE_PRIVATE_KEY` if using explicit credential config

Also create a shared token verification helper at `src/lib/errors/auth.ts` that exports a `verifyToken(request: Request)` function. It should pull the `Authorization` header, strip the `Bearer ` prefix, call `admin.auth().verifyIdToken(token)`, and return the decoded token (or throw if invalid). All seven error API routes will call this helper — we don't want to repeat the same auth logic in every file.

Finally, make sure `firebase-admin` is listed in `package.json` dependencies and installed.
