We need to make sure the DailyProfitTrendSection fails gracefully in every scenario without breaking the rest of the FinancialMetricsDashboardPage.

**Loading state:** While either GET /api/roi/compute or GET /api/roi/breakdown is in flight, show a gray shimmer rectangle at 320px height (220px on mobile) where the chart would be. Render skeleton placeholders for the metric toggle and date range selector in the card header. Hide the Pattern Insights strip entirely during loading — don't show a skeleton for it.

**Empty state:** If both APIs return successfully but the merged series data is an empty array (no data points for the selected range), show: 'No profit data for this range. Sync your networks to see trends.' with a 'Sync Now' CTA button. This button should open or link to the ManualRefreshPanel. Don't show the chart or Pattern Insights strip in this state.

**error_404:** If either API returns 404, show 'No data found for selected range' inside the chart card area. Include a 'Reset Date Range' button that resets the selector back to the default (Last 7 days) and re-fetches.

**error_500:** If either API returns 500, show an error banner inside the chart card (not a full-page error — the rest of the dashboard should still be usable). The banner should say something like 'Failed to load profit data' and include a 'Retry' button. The Retry button re-calls both GET /api/roi/compute and GET /api/roi/breakdown simultaneously.

**error_401:** As described in the security step — attempt getIdToken(true) force-refresh, retry once. If the retry fails, redirect to / with a 'Session expired' toast. This should be handled at the API call layer so it applies to all three endpoints consistently.

**error_403:** Show 'Access Denied' inline within the chart card with a link to /dashboard. Since the user is already on /dashboard, this link should scroll to the top of the page or just be a visual anchor.

**Partial failure (one API succeeds, one fails):** If GET /api/roi/compute succeeds but GET /api/roi/breakdown fails (or vice versa), treat it as an error_500 and show the retry banner. Don't try to render a partially-enriched chart — the colorCode data from /api/roi/breakdown is required for correct rendering.

**Snapshot drill-down errors:** If GET /api/stats/snapshot returns an error when the user clicks a data point, show an error message inside the snapshot panel itself (not a toast, not a redirect). Include a 'Close' button. Don't close the panel automatically on error.

**Section isolation:** All errors in DailyProfitTrendSection must be contained within the card. They should never cause the FinancialMetricsDashboardPage to crash or show a full-page error. Wrap the section in an error boundary as a safety net.
