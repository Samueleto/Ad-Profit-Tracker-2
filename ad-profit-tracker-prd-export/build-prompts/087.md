We're using Firestore as the database. For this step, set up the collection structure, composite indexes, and security rules that the RollerAds endpoints depend on.

The RollerAds integration reads from and writes to four Firestore collections. These may already exist in the project for other network integrations — if so, just add the RollerAds-specific indexes without touching existing data or rules.

**adStats** — This is the main data collection. Each document stores one day + country combination for one user. The document ID should be a deterministic composite key like `{userId}_{networkId}_{date}_{country}` so upserts are idempotent (writing the same day twice doesn't create duplicates). Fields: `userId`, `networkId` (always `'rollerads'`), `date` (YYYY-MM-DD string), `country` (ISO alpha-2), `revenue`, `impressions`, `clicks`, `ctr`, `cpm`, `cost` (always null for RollerAds), `rawResponse` (map), `syncedAt` (timestamp).

**apiKeys** — Stores encrypted API keys per user per network. Fields: `userId`, `networkId`, `encryptedKey`, `createdAt`, `updatedAt`. This likely already exists.

**networkConfigs** — Stores per-user network configuration. Fields: `userId`, `networkId`, `isActive`, `syncSchedule`, `endpointOverride`, `timeoutSeconds`, `lastSyncedAt`, `lastSyncStatus`, `lastSyncError`. This likely already exists.

**auditLogs** — Append-only log of sync actions. Fields: `userId`, `action`, `resourceType`, `resourceId`, `metadata` (map), `status`, `errorMessage`, `createdAt`.

Create the following composite indexes in your `firestore.indexes.json`:
- `adStats`: `userId ASC` + `networkId ASC` + `date ASC` (for stats queries with date range)
- `adStats`: `userId ASC` + `networkId ASC` + `date DESC` (for latest stats query)
- `adStats`: `userId ASC` + `networkId ASC` + `date ASC` + `country ASC` (for by-country queries)
- `networkConfigs`: `networkId ASC` + `isActive ASC` + `syncSchedule ASC` (for the scheduled-sync cross-user query)

In `firestore.rules`, make sure the `adStats` and `auditLogs` collections are only readable and writable by the Admin SDK (server-side only) — no direct client access. The `networkConfigs` and `apiKeys` collections should only be accessible by the owning user (`request.auth.uid == resource.data.userId`).

Deploy the indexes with `firebase deploy --only firestore:indexes` and the rules with `firebase deploy --only firestore:rules`.
