We're building a financial analytics dashboard. For this step, I need you to add a DailyProfitTrendSection to the existing FinancialMetricsDashboardPage at /dashboard. This is NOT a new route — it's a new full-width card section that sits directly below the KPI row on that existing page.

The section is built around a recharts AreaChart. Here's what it needs to do and look like:

**Card layout:** Full-width card. The card header has two things side by side: a segmented metric toggle (four options — Profit, Revenue, Cost, ROI) on the left, and a date range selector on the right (Last 7 / Last 14 / Last 30 days, This Month, and Custom). The Custom option should open a date picker capped at 90 days max — enforce this client-side before making any API call. The chart area itself is 320px tall on desktop and 220px on mobile.

**Data sources:** Call GET /api/roi/compute with groupBy='daily' to get the time-series data for the chart. Also call GET /api/roi/breakdown with dimension='daily' to get per-point enrichment data — specifically the colorCode ('green' | 'red' | 'amber') and roiIndicator for each day. Merge these two responses by date before rendering. When the user clicks a data point, call GET /api/stats/snapshot for that specific day to show a drill-down view.

**Chart features to implement:**
- Area fill under the profit line, color-coded by overall period profitability: green-500 fill if the period is net positive, red-500 fill if net negative
- Individual data point dot markers colored per-day using the colorCode from the /api/roi/breakdown response (green-500, red-500, or amber-500)
- A dashed gray horizontal reference line at y=0, labeled 'Break Even' at the right axis edge
- A 7-day moving average overlay rendered as a dotted gray line — compute this client-side from the series data
- Interactive tooltips on hover showing: date, netProfit, revenue, cost, and ROI%
- Null data points should render as gaps (set connectNulls=false), not as zeros
- Axis labels should be minimal; grid lines should be subtle gray
- An 'Export Chart' button in the card header that triggers PNG download using recharts' built-in export capability

**Pattern Insights strip:** Below the chart, add a collapsible strip (collapsed by default, toggled with a chevron). When expanded, it shows four highlights computed from the loaded data: best day (highest netProfit), worst day (lowest netProfit), longest profitable streak (consecutive days with positive netProfit), and period-over-period percent change compared to the equivalent prior period.

**States to handle:**
- Loading: show a gray shimmer rectangle at 320px height where the chart goes; render skeleton placeholders for the metric toggle and date selector; hide the Pattern Insights strip entirely
- Empty: show 'No profit data for this range. Sync your networks to see trends.' with a 'Sync Now' CTA that links to the ManualRefreshPanel
- error_404: show 'No data found for selected range' with a button to reset the date range back to the default
- error_500: show an error banner inside the chart card with a 'Retry' button that re-calls both GET /api/roi/compute and GET /api/roi/breakdown
- error_401: attempt a Firebase ID token force-refresh via getIdToken(true); if that fails, redirect to / with a 'Session expired' toast
- error_403: show an 'Access Denied' message inline within the chart card with a link to /dashboard
- Success: fully rendered recharts AreaChart with all overlays, markers, and the Pattern Insights strip available to expand

The metric toggle controls which field drives the chart's Y-axis and area fill — switching between Profit (netProfit), Revenue, Cost, and ROI% should re-render the chart with the appropriate data field without re-fetching if the data is already loaded.
