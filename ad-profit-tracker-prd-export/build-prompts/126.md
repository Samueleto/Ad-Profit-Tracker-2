We need clear TypeScript interfaces for all the data shapes this feature works with. Define these in src/lib/sync/types.ts so every endpoint and utility can import from one place.

NetworkConfig — represents a document in the networkConfigs Firestore collection. Fields: userId (string), networkId (one of 'exoclick' | 'rollerads' | 'zeydoo' | 'propush'), isActive (boolean), syncSchedule (string, e.g. 'daily'), lastSyncedAt (Firestore Timestamp or null), lastSyncStatus ('success' | 'failed' | 'partial' | 'never'), lastSyncError (string or null), retryAttempts (number, default 3).

AdStatRow — represents a document in the adStats collection. Fields: userId (string), networkId (string), date (string in YYYY-MM-DD format), country (string), impressions (number), clicks (number), revenue (number), and any other network-specific metrics. Also include createdAt and updatedAt timestamps.

ApiKey — represents a document in the apiKeys collection. Fields: userId (string), networkId (string), encryptedKey (string — the AES-encrypted API key), createdAt (Firestore Timestamp).

AuditLog — represents a document in the auditLogs collection. Fields: userId (string), action (string, e.g. 'manual_sync_triggered'), resourceType (string, e.g. 'sync'), resourceId (string — the networkId), status ('success' | 'failure'), metadata (object containing scheduledSync boolean, latencyMs number, rowsFetched number, date string), createdAt (Firestore Timestamp).

SyncSummaryItem — the per-network result object used in the daily-sync response. Fields: userId, networkId, status ('success' | 'failed' | 'skipped'), rowsFetched (number), errorMessage (string or null).

DailySyncResponse — the full response shape for POST /api/scheduled/daily-sync. Fields: date, usersProcessed, networksTriggered, networksFailed, networksSkipped, totalRowsFetched, durationMs, summary (array of SyncSummaryItem).

SyncStatusNetwork — one entry in the GET /api/scheduled/sync-status response. Fields: networkId, isActive, syncSchedule, lastSyncedAt, lastSyncStatus, lastSyncError, latestDataDate, nextScheduledSync.

SyncHistoryEntry — one entry in the GET /api/scheduled/sync-history response. Fields: id, networkId, status, rowsFetched, latencyMs, scheduledSync, date, createdAt.

Also export a SUPPORTED_NETWORKS constant — a readonly array of the four network IDs — so validation logic across all endpoints can import it from one place rather than duplicating the list.
