We're building an ad network integration manager. For this step, I need you to build the SettingsPage at /settings — a page where users can connect, update, and disconnect their API keys for four ad networks: ExoClick, RollerAds, Zeydoo, and Propush.

The page shell should be a Server Component that checks authentication on load and redirects to login if the user isn't signed in. Inside it, render a Client Component section that handles all the interactive form logic.

Layout-wise, show one card per ad network, stacked vertically in a clean sectioned layout. Each card should display the network name and its current connection status. When the page first loads, fetch from GET /api/keys/status to determine which networks are already connected. While that fetch is in flight, show skeleton placeholder cards for all four networks.

Each card has two states:
- **Not connected**: Show a password-style input field for the API key (with a show/hide toggle button), a 'Save' button, and a placeholder hint like 'Enter your API key'. The input should never pre-fill with any key value — keys are never sent to the client.
- **Connected**: Show a 'Connected' status badge with the last updated timestamp, an 'Update Key' button that reveals the input form again, and a 'Disconnect' button.

Use react-hook-form to manage each network's form independently. Each form should validate that the key field isn't empty before submitting. When the user clicks 'Save', call POST /api/keys/save with the networkId and the entered key. While the request is in flight, show a spinner on the Save button and disable the input field. On success, show a toast notification saying 'API key saved successfully' and update that card's status to 'Connected' without a full page reload. On error, show an inline error message below the input and a toast for server-level errors.

When the user clicks 'Disconnect', call DELETE /api/keys/delete with the networkId. On success, update that card back to the 'Not connected' state and show a toast confirming the disconnection.

Important: the raw API key value must never be stored in component state beyond what's needed to submit the form. Clear the input field immediately after a successful save. The page should feel snappy — status updates should reflect in the UI right away without waiting for a re-fetch of /api/keys/status.
