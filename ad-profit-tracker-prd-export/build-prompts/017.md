We're building an API rate limiting and throttling system for four ad networks (ExoClick, RollerAds, Zeydoo, ProPush). For this step, I need you to set up the folder structure that everything else will live in.

Create a `lib/rate-limit/` directory — this is the core of the feature. Inside it, we'll need files for: `limiterRegistry.ts` (the in-memory Bottleneck limiter instances, keyed by userId+networkId), `limiterConfig.ts` (the static Bottleneck configuration per network — things like maxConcurrent, minTime, reservoir size), `userLimiter.ts` (the in-memory sliding window counters for user-facing endpoint quotas), and `userLimiterConfig.ts` (the static config for per-endpoint user quotas like 10/hour for manual sync).

On the API side, create a `src/app/api/rate-limits/` directory with four route files: `status/route.ts`, `reset/route.ts`, `config/route.ts`, and `violations/route.ts`. These will map to our four endpoints.

Also create a `lib/auth/` directory if it doesn't already exist — we'll put the Firebase token verification helper there.

Keep the `lib/rate-limit/` files completely separate from the API routes. The route handlers will import from `lib/rate-limit/` but the library files should have no knowledge of HTTP — they're just logic and state.
