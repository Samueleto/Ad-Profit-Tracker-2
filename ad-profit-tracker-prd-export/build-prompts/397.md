We need the FilterToolbar and its connected endpoints to fail gracefully. Here's how each error state should be handled:

**Country combobox errors**:
- While GET /api/filters/options is loading: show a spinner inside the dropdown and 'Loading countries...' as placeholder text. All other toolbar controls should render immediately from static config — don't block the whole toolbar on this one async call.
- If the response returns an empty array: show 'No countries found for this date range' as a non-interactive message in the dropdown.
- If the endpoint returns 404: show 'No data available' with a muted style in the dropdown.
- If the endpoint returns 500: show an amber warning icon with 'Could not load countries' and a clickable 'Retry' link that re-fires the request. The rest of the toolbar (networks, metric pills, data quality, search) must remain fully functional.
- If the endpoint returns 401: trigger the token refresh flow (see security section). Don't show a raw error in the dropdown.

**GET /api/filters/stats errors** (when dashboard sections re-fetch after filter changes):
- 400 (bad params): this shouldn't happen if the client validates before sending, but if it does, show a small inline error near the affected dashboard section with 'Invalid filter combination' and a 'Reset filters' link.
- 401: trigger token refresh flow.
- 404 (no data matching filters): show an empty state in the affected dashboard section — something like 'No data matches your current filters' with a 'Clear filters' button that calls clearAllFilters() in the store.
- 500: show an error state in the affected section with a retry button. Don't clear the filter state — the user's selections should persist so they can retry.

**Preset errors**:
- GET /api/filters/presets failing: show a subtle 'Could not load saved filters' message near the presets dropdown. Don't block the rest of the toolbar.
- POST /api/filters/presets 400 (max presets reached): show an inline message 'You've reached the maximum of 20 saved filters. Delete one to save a new one.'
- POST /api/filters/presets 429 (rate limited): show 'Too many saves — please wait a moment before saving again.'
- DELETE /api/filters/presets/:presetId 404: show 'This preset no longer exists' and remove it from the local list.
- Any preset write failure (500): show a toast notification 'Could not save filter preset — please try again.'

**General toolbar errors**:
- 403 on any endpoint: show 'Access Denied' as an inline message within the toolbar row. Don't redirect.
- Network offline: show a subtle 'You appear to be offline' banner. Disable the Apply Filters button and country combobox. Static controls (metric pills, data quality toggle, free-text search) can still work client-side.
- All error messages should be user-friendly — no raw error codes, stack traces, or technical jargon visible to the user.
