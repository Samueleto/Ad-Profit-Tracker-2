We need two things initialized before any of the Propush endpoints can work:

**1. Firebase Admin SDK** — Install `firebase-admin` if it isn't already in the project. Create a singleton initializer at `src/lib/firebase/admin.ts`. It should check if an Admin app is already initialized (to avoid re-initializing on hot reloads in development) and then call `initializeApp()` using a service account. Store the service account credentials in environment variables — never hardcode them. Export the `auth`, `firestore`, and any other Admin services you'll need from this file so every API route imports from one place.

**2. Axios and the Propush API client** — Install `axios`. Create a small Propush API client at `src/lib/propush/api-client.ts`. This should export a function (something like `fetchPropushStats`) that accepts a decrypted API key, a date range, and calls the Propush API v5 at `/v5/pub-statistics` with `groupBy=geo`. Handle the HTTP call here so the endpoint handlers stay clean. Set a reasonable timeout (e.g., 30 seconds) since external API calls can be slow.

**3. Crypto helper** — Create a `src/lib/propush/crypto.ts` file that exports a `decryptApiKey(encryptedKey: string): string` function. This should use AES decryption (via the `cryptr` package or Node's built-in `crypto` module) with a secret stored in an environment variable (e.g., `ENCRYPTION_SECRET`). This helper will be used by both the manual sync and the scheduled sync endpoints to safely retrieve stored API keys.

Make sure all secrets (`FIREBASE_SERVICE_ACCOUNT`, `ENCRYPTION_SECRET`, `INTERNAL_SYNC_SECRET`) are documented in a `.env.example` file so it's clear what needs to be set up in each environment.
