We need several reusable components that are shared across TeamManagementPage and InviteMemberModal. Build these before the pages so the pages can import them directly.

RoleBadge — A small inline badge component that takes a WorkspaceRole prop and renders it with the right color: indigo for 'owner', blue for 'admin', gray for 'member'. Capitalize the label. Used in the member roster table and on the AcceptInvitationPage invitation card.

MemberTableRow — A single row in the member roster table. Takes a WorkspaceMember and the current user's role as props. Shows: avatar (photoURL with a fallback initial if null), display name, email, RoleBadge, formatted join date, and a three-dot action menu. The action menu should show 'Change Role' and 'Remove Member' options — but only render these if the current user has permission (owner sees both for all non-owners; admin sees both for members only; member sees nothing). Also show 'Transfer Ownership' in the action menu for the owner when viewing an admin row. Fires callbacks: onChangeRole(member), onRemoveMember(member), onTransferOwnership(member). For the current user's own row, show 'Leave Workspace' instead of remove (and hide it for the owner).

PendingInvitationRow — A row for the pending invitations table. Takes a WorkspaceInvitationSafe and the current user's role. Shows: invitedEmail, invitedByName, formatted sent date, formatted expiry date, and a 'Revoke' button (only visible to Admin/Owner). Fires an onRevoke(invitation) callback.

MemberTableSkeleton — A loading placeholder that renders 4 shimmer rows matching the member table column structure (avatar circle, two text lines, badge shape, date, action menu dot). Used while GET /api/team/members is loading.

InvitationTableSkeleton — Similar shimmer placeholder with 2 rows matching the pending invitations table structure.

RoleSelector — A segmented control component used inside InviteMemberModal. Takes the selected role ('admin' | 'member') and an onChange callback. Shows two options side by side. Below each option, render a short muted description: Admin = 'Can invite members, manage roles, and access all workspace features'; Member = 'Can access workspace features but cannot manage team settings'. Highlights the selected option with the accent color.

ConfirmDialog — A reusable modal dialog for destructive confirmations. Takes: title, message, confirmLabel, confirmVariant ('danger' | 'primary'), and onConfirm/onCancel callbacks. Used for 'Remove Member', 'Leave Workspace', 'Revoke Invitation', and 'Transfer Ownership' confirmations. Shows a spinner on the confirm button while the action is in progress.

All components should use the existing ThemeProvider tokens and CSS custom properties for colors — don't hardcode hex values.
