We're using Firebase on both the client and server sides of this feature. Before building anything, let's make sure both SDKs are properly configured.

On the server side, we need Firebase Admin SDK (`firebase-admin`) initialized and ready to use in our API routes. Check if there's already a shared Firebase Admin instance in the project (something like `src/lib/firebaseAdmin.ts`). If there is, great — we'll import from there. If not, create one now. It should initialize the Admin SDK once using a service account (via `FIREBASE_SERVICE_ACCOUNT_KEY` or individual `FIREBASE_PROJECT_ID`, `FIREBASE_CLIENT_EMAIL`, `FIREBASE_PRIVATE_KEY` environment variables) and export a singleton `adminDb` (Firestore instance) and `adminAuth` (Auth instance). Make sure initialization is guarded so it doesn't run multiple times in Next.js's hot-reload environment.

On the client side, we need `firebase/auth` available so the PreferencesCard can get the current user's ID token to attach to API requests. Again, check if there's already a shared Firebase client config in the project. If not, create one at `src/lib/firebaseClient.ts` that initializes the Firebase app and exports `getAuth`. Store the Firebase config values (`NEXT_PUBLIC_FIREBASE_API_KEY`, etc.) in `.env.local`.

Make sure all environment variables are documented in a `.env.example` file so other developers know what's needed.
