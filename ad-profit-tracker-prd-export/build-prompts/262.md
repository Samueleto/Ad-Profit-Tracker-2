# Step 262 of 437: Set up state management for network configs

We need a state layer that keeps the NetworkConfigTab's data and interactions clean — the UI components shouldn't be doing their own fetching or managing stale data directly.

Create a custom hook called useNetworkConfigs that handles everything the NetworkConfigTab needs. It should:

**Loading and fetching:** On mount, call GET /api/networks/config/list and store the returned array of 4 network config objects. Expose a loading boolean (true while the initial fetch is in flight) and an error state. The hook should attach the Firebase ID token as a Bearer token on every API call — fetch it fresh from the Firebase auth SDK each time so it's never stale.

**Per-network updates:** Expose an updateNetworkConfig function that takes a networkId and a partial config object (matching the PATCH /api/networks/config/update request shape). This should optimistically update the local state immediately, then fire the PATCH call. If the call fails, revert the local state to what it was before and surface the error so the card can show its inline error banner.

**Reordering:** Expose a reorderNetworks function that takes the new ordered array (after a drag-and-drop). Optimistically reorder the local networks array by displayOrder, then call PATCH /api/networks/config/reorder. Revert on failure.

**Test connection:** Expose a testConnection function that takes a networkId. Track per-network testing state (a map of networkId → { status: 'idle'|'testing'|'success'|'error', latencyMs, sampleRecordCount, errorMessage }). Call POST /api/networks/config/test-connection and update the per-network result when it resolves.

**Sync all:** Expose a syncAll function that calls POST /api/networks/sync-all. Track a syncAllLoading boolean and a syncAllResult object ({ triggered, skipped, failed }) for the toast.

**Reset:** Expose a resetNetworkConfig function that takes a networkId, calls DELETE /api/networks/config/reset, and on success replaces that network's entry in local state with the returned default config.

**Token refresh on 401:** If any API call returns 401, the hook should attempt to refresh the Firebase ID token once (using the auth SDK's token refresh) and retry the original call. If the retry also returns 401, set an authExpired flag that the page can use to redirect.

Keep the networks array sorted by displayOrder at all times so the UI always renders in the right order without extra sorting logic in the components.
