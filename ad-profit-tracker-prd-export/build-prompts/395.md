We need to make sure every error state in PerNetworkAnalyticsTabsSection and ApiExplorerTab fails gracefully and gives the user a clear path forward.

**PerNetworkAnalyticsTabsSection error states:**

- **Loading** — When a tab is first activated and data is being fetched, show: 4 shimmer stat cards in the KPI row, a 280px shimmer rectangle where the chart will be, and 8 shimmer rows in the GEO table. The tab label should show a subtle loading indicator (a small spinner or pulsing dot) so the user knows something is happening.

- **Empty (404)** — When `GET /api/networks/stats` returns 404 (no data for this network in the selected date range), show a centered message inside the tab panel: 'No data for [NetworkName] in this date range. Sync this network to see analytics.' Include a 'Sync Now' button that calls `POST /api/sync/manual` for that network. Don't show the KPI row, chart, or GEO table.

- **Error 401** — Attempt token refresh via `getIdToken(true)`. If the refresh succeeds, automatically retry the failed request. If the refresh fails, redirect to / with a 'Session expired' toast. Don't show an error state inside the tab — handle it at the navigation level.

- **Error 403** — Show an 'Access Denied' message inline within the tab panel. Include a link to /dashboard (in case they navigated here via a deep link somehow). Don't show the KPI row, chart, or GEO table.

- **Error 500** — Show an inline error banner inside the tab panel: 'Something went wrong loading [NetworkName] data.' with a 'Retry' button that re-calls `GET /api/networks/stats` for this specific network (all three groupBy calls). The rest of the dashboard should remain functional — only this tab panel shows the error.

- **Sync failure** — If `POST /api/sync/manual` fails, show an error toast: 'Sync failed for [NetworkName]. Please try again.' Re-enable the Sync button. Don't change the tab's data state.

- **Partial data** — If the three parallel `GET /api/networks/stats` calls (total, daily, country) partially succeed (e.g., total and daily succeed but country returns 500), show the KPI row and chart with the data that loaded, and show an inline error message in the GEO table section only: 'Country breakdown unavailable. Retry.' with a retry button for just that call.

**ApiExplorerTab error states:**

- **Loading** — Accordion headers render immediately. Inside an expanded accordion, show 'Loading last response...' as a shimmer block where the JSON viewer will be.

- **No API key configured** — If the raw-response endpoint indicates no API key is set up, show inside the accordion: 'API key not configured — go to Settings' with a Next.js Link to /settings. Don't show the JSON viewer or field mapping table.

- **Error 404 (no cached response)** — Show: 'No response cached yet. Click Fetch Fresh Sample to retrieve one.' The 'Fetch Fresh Sample' button should be prominently visible in this state.

- **Error 500 on raw-response fetch** — Show an error message inside the accordion with a 'Retry' button that re-fetches from the appropriate raw-response endpoint.

- **Test connection failure** — If `POST /api/networks/config/test-connection` fails, show the error response inline below the button (don't use a toast — keep it in context). Re-enable the 'Fetch Fresh Sample' button.

**General principles** — Errors in one tab or accordion should never affect other tabs or accordions. Always show the most actionable next step (Retry, Sync, go to Settings). Error messages should use the network's display name (e.g., 'ExoClick', not 'exoclick').
