We need TypeScript interfaces for everything the benchmarking feature touches — the Firestore `benchmarks` document, the API request/response shapes, and the metric types. This gives us type safety across the API routes and UI components.

Create `types/benchmarking/index.ts` (or split into multiple files if you prefer) with the following:

**Firestore Document Types:**
- `MetricKey` — a union type: `'roi' | 'ctr' | 'cpm' | 'revenue' | 'cost' | 'impressions' | 'clicks'`
- `MetricTargetEntry` — `{ customTarget: number | null, useDefault: boolean, updatedAt: Timestamp | null }`
- `BenchmarkDocument` — the full Firestore `benchmarks` document shape matching all fields: `id`, `userId`, `metricTargets` (a `Record<MetricKey, MetricTargetEntry>`), `systemDefaults` (a `Record<MetricKey, number>`), `historicalWindowDays`, `performanceScore`, `performanceScoreComputedAt`, `createdAt`, `updatedAt`

**API Response Types:**
- `BenchmarkPerformanceResponse` — the full response shape from `GET /api/benchmarks/performance`: includes `dateFrom`, `dateTo`, `metric`, `networkId`, `actual` (`{ value: number | null, unit: string }`), `historicalAverage` (`{ value: number | null, daysUsed: number, windowDays: number }`), `industryBenchmark` (`{ value: number, isCustom: boolean, source: 'custom' | 'system_default' }`), `gaps` (`{ vsHistorical: number | null, vsIndustry: number | null }`), `performanceRatio`, `trend` (`'above' | 'below' | 'at' | 'no_data'`), `cachedAt`
- `BenchmarkSettingsMetric` — one entry in the settings response: `{ metric: MetricKey, customTarget: number | null, useDefault: boolean, effectiveTarget: number, systemDefault: number, unit: string, isCustom: boolean, updatedAt: Timestamp | null }`
- `BenchmarkSettingsResponse` — the full `GET /api/benchmarks/settings` response: `{ historicalWindowDays: number, metricTargets: BenchmarkSettingsMetric[], lastUpdatedAt: Timestamp | null }`
- `BenchmarkSettingsPatchBody` — the request body for `PATCH /api/benchmarks/settings`: `{ metricTargets?: Partial<Record<MetricKey, { customTarget?: number | null, useDefault?: boolean }>>, historicalWindowDays?: number }`

**Validation Helpers** — In `lib/benchmarks/validation.ts`, create a `validateMetricTarget(metric: MetricKey, value: number): string | null` function that returns an error message string if the value is out of range, or null if valid. Apply the caps: ROI% and CTR% max 10000, CPM max 1000, Revenue and Cost max 10000000, all must be non-negative. Also export `validateHistoricalWindowDays(days: number): string | null` that checks the 7–365 range.
