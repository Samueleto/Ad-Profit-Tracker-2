We're building the Help & Documentation Center on Next.js + Firebase. For this step, connect the help pages to the rest of the app so users can actually reach them and navigate naturally between them.

**Navigation entry points**

Add a 'Help' link to the app's main navigation (sidebar or top nav, wherever the app's other nav links live). This should link to /help. If the app has a bottom navigation bar on mobile, add a Help icon there too. Also add a help icon or '?' button in the app header that links to /help — this is a common pattern users expect.

**Breadcrumb and internal navigation**

On HelpArticlePage, the breadcrumb trail (Help > [Category] > [Article Title]) needs to work correctly:
- 'Help' links to /help
- The category segment links to /help with that category pre-selected. Since HelpCenterPage manages category selection in state, pass the category as a query param (e.g., /help?category=faq) and have HelpCenterPage read that param on mount to pre-select the right category filter

The 'Back to Help Center' link at the top of HelpArticlePage should also go to /help (not browser back — that could take users somewhere unexpected).

Related article links at the bottom of HelpArticlePage navigate to /help/[articleId] for the respective article.

**Route protection**

Both /help and /help/[articleId] live inside the (app) route group, so middleware.ts already enforces Firebase Auth. Double-check that both routes are covered by the middleware matcher — they should be, but verify the matcher pattern includes /help and /help/* paths.

**Deep linking**

Article URLs (/help/[articleId]) should be shareable. If a user pastes a link and isn't logged in, middleware should redirect them to the login page and then back to the article after sign-in. Make sure the redirect-after-login flow preserves the /help/[articleId] destination.

**Toast notifications**

Wire up the following toast notifications using whatever toast/notification system the app already uses:
- After a successful 'Copy Link' action on HelpArticlePage: 'Link copied to clipboard'
- After successful feedback submission: 'Thanks for your feedback!'
- On session expiry (401 after token refresh fails): 'Session expired' toast before redirecting to /

**ThemeProvider**

Both pages should respect the app's existing ThemeProvider tokens for dark mode. Make sure the shimmer loading states, category cards, article rows, and markdown prose styles all use theme-aware colors rather than hardcoded light-mode values.
