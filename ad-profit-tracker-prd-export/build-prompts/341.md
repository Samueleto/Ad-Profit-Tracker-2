The DailyProfitTrendSection inherits the FinancialMetricsDashboardPage's Firebase Auth middleware, but we need to make sure every API call this section makes is properly secured.

**Bearer token on every request:** All three API calls — GET /api/roi/compute, GET /api/roi/breakdown, and GET /api/stats/snapshot — must include the Firebase ID token as a Bearer token in the Authorization header. Get the token via getIdToken() from firebase/auth. Don't cache the token across requests — call getIdToken() fresh each time (Firebase caches it internally and handles refresh automatically).

**Token refresh on 401:** If any of the three API calls returns a 401, attempt a force-refresh of the Firebase ID token using getIdToken(true). Then retry the failed request once with the new token. If the retry also returns 401, treat it as a session expiry: redirect the user to / and show a 'Session expired' toast. Don't retry more than once — avoid infinite refresh loops.

**Client-side date range cap:** Before making any API call, validate that the selected date range is 90 days or fewer. If the user somehow submits a range longer than 90 days (e.g., via the Custom picker), clamp it to 90 days and show a brief notice. This is a client-side guard — the server should also enforce this, but we're not adding new endpoints here.

**No sensitive data in URLs:** The date range params go in query strings, which is fine. Make sure the Firebase ID token is only ever sent in the Authorization header — never in a query param or the URL.

**userId from token only:** The section never sends a userId in the request body or query params. The server-side handlers for /api/roi/compute, /api/roi/breakdown, and /api/stats/snapshot should extract userId from the verified Firebase ID token. The client just sends the token and trusts the server to scope the data correctly.

**403 handling:** If any call returns 403, show an 'Access Denied' message inline within the chart card with a link to /dashboard. Don't redirect — the user is already on /dashboard, so just show the inline error state.
