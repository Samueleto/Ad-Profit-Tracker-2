The ManualRefreshPanel needs to handle a range of error conditions gracefully. Here's how each one should behave:

**Initial load failure (500 on GET /api/scheduled/sync-status)** — If the first sync-status fetch fails, show an inline error banner inside the panel (not a full-page error) with the message 'Unable to load sync status' and a 'Retry' button that re-fetches sync-status. Don't show skeleton rows indefinitely — resolve to this error state after the fetch fails.

**Polling failure** — If a polling call to GET /api/scheduled/sync-status fails, don't crash or show a banner. Just silently retry on the next 3-second interval. If 3 consecutive polls fail, stop polling and show a small warning indicator near the status badges: 'Status may be outdated — click to refresh' that manually re-triggers a single fetch.

**Trigger failure (500 on POST /api/sync/manual)** — If the POST call itself fails with a 500, show an inline error notification (toast or banner) saying 'Sync failed to start — please try again'. Remove the affected networks from the triggered set immediately so their buttons re-enable. Don't leave the UI stuck in a 'Syncing...' state.

**Per-network sync failure** — The POST /api/sync/manual response includes a `failed` array and per-network `status` fields. If a network comes back as 'failed' in the results, update its status badge to red and show the errorMessage (sanitized and truncated) as a tooltip on hover over the badge. Don't show a global error — surface it inline on the specific row.

**Rate limit (429)** — Read the Retry-After header value (in seconds). Disable the affected button (either 'Refresh All' or the specific network's Refresh button) and show a countdown: 'Available in Xm Ys'. Tick this down every second. When it reaches zero, re-enable the button. If Retry-After is missing from the response, default to a 60-second countdown.

**401 — token expired** — As described in security: try getIdToken(true) first. If that fails, redirect to / with a 'Session expired' toast.

**403 — access denied** — Show an 'Access Denied' inline banner inside the ManualRefreshPanel with a link to /dashboard. Don't redirect.

**Network offline** — If a fetch fails due to a network error (not an HTTP error code), show 'No internet connection — retrying...' and retry with exponential backoff (2s, 4s, 8s) up to 3 times before showing a permanent error banner with a manual Retry button.

**Sync history load failure** — If GET /api/scheduled/sync-history fails, show an error state inside the SyncHistoryDrawer ('Unable to load history — try again') with a Retry button that re-fetches. Don't close the drawer automatically.
