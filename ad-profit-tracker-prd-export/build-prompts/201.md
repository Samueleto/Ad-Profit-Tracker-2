We need several reusable components that the FinancialMetricsDashboardPage will assemble together. Build these in src/features/dashboard/components/.

KpiCard — This is used four times across the top of the dashboard (Total Revenue, Total Cost, Net Profit, ROI %). It takes a label, a formatted value, a percentChange number (or null), and a changeDirection ('up' | 'down' | 'neutral'). Show the label at the top, the value prominently in the middle, and a directional arrow with the percentage change at the bottom. Use semantic color coding: green for positive profit/ROI, red for negative, amber for zero or null. When in loading state (pass an isLoading prop), show a shimmer skeleton instead of the real content. The card should look good in a 4-column grid on desktop, 2-column on tablet, and 1-column on mobile.

DateRangeSelector — A sticky control that lets users pick Last 7 days, Last 14 days, Last 30 days, This Month, or Custom. For the Custom option, show two date inputs (from/to). It should call an onChange callback with a DateRange object whenever the selection changes. Cap the custom range at 90 days and show a validation message if the user tries to exceed it. This component controls all data on the FinancialMetricsDashboardPage.

ProfitTrendChart — A line chart built with recharts showing daily profit over time. It takes the dailySeries array from the DashboardMetricsResponse (each item has date, revenue, cost, netProfit). Render a LineChart with a line for netProfit, an XAxis showing dates formatted with date-fns, a YAxis with currency formatting, a Tooltip showing all three metrics on hover, and a ResponsiveContainer so it fills its parent width. Add 'use client' at the top since recharts uses browser APIs. Show a gray placeholder rectangle when isLoading is true.

GeoBreakdownTable — Shows the topCountries array from the API response. Columns: Country (flag emoji + name), Revenue, Cost, Net Profit, Share %. Color the Net Profit cell green or red based on value. Show 5 shimmer rows when loading. Show an empty state message if the array is empty.

NetworkSummaryRow — Shows one network's data: network name/logo, its primary metric value (revenue or cost depending on dataRole), last sync time formatted as a relative time ('2 hours ago' using date-fns), and a status badge (green dot for 'success', red for 'error', amber for 'pending'). This component is used once per network in a collapsible section, but since it has meaningful local state logic (status badge color, relative time formatting) it's worth extracting.

CacheStatusBadge — A small badge that shows either 'Live' (green) or 'Cached · X min ago' (gray). It takes the cachedAt timestamp from the API response and computes how many minutes ago it was using date-fns. Used in the dashboard header next to the Refresh Data button.
