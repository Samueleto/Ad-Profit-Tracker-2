# Step 374 of 437: Handle auth expiry and nav failure states gracefully

The mobile nav components are mostly static, so there aren't many error paths — but the ones that exist need to be handled cleanly.

**Session expiry in MobileNavigationDrawer** — As covered in security, when `onAuthStateChanged` fires null, redirect to `/` and show a 'Session expired' toast. Make sure the toast is visible even if the drawer is currently open (the drawer should close first, then the toast appears, then the redirect happens — all within the same tick so it feels instant). Use whatever toast/notification system is already in the project.

**Sign-out failure** — `Firebase Auth signOut()` can fail in rare cases (network issues). Wrap it in a try/catch. If it fails, show a brief error notification ('Sign out failed — try again') and re-enable the sign-out button. Don't leave the user stuck with a disabled button.

**Missing user data in drawer** — If `currentUser` exists but `displayName` or `photoURL` is null (e.g., the user signed up with email/password and never set a profile), show a fallback: use the first letter of their email as an avatar initial with a colored background, and show their email address instead of a display name. Don't show a broken image or an empty name slot.

**MobileDataTableWrapper edge cases** — If a table has zero rows, the wrapper should render nothing (the parent section handles the empty state). If a table row is expanded and the data updates (e.g., a background refresh), keep the expanded state as-is — don't collapse it. If the `columns` config prop is missing or malformed, fall back to rendering all columns as primary so no data is hidden.

**Hydration mismatch prevention** — Because `useMediaQuery` returns `false` on the server, there's a risk of a flash where the mobile layout briefly renders on desktop before hydration. Suppress this by wrapping the conditional nav rendering in a `useEffect`-gated `isMounted` boolean — only render the viewport-specific nav variants after the component has mounted on the client. Show the desktop layout as the default server-rendered shell since most traffic is desktop.
