HOW TO USE THESE PROMPTS
------------------------

Each build prompt file is SELF-CONTAINED with all entity context embedded.

Files are numbered for guaranteed alphabetical ordering on all platforms:
  - 000.md (this README file - start here!)
  - 001.md (first step)
  - 002.md (second step)
  - ... and so on

Simple numerical naming ensures perfect ordering on Windows, macOS, and Linux.
Check the file header inside each file to see what it contains.

1. Open each step file in sequential order (001.md, 002.md, 003.md...)
2. Read the task description at the top
3. Read any "discovery files" listed (prevents naming conflicts)
4. Copy the entire PROMPT section
5. Paste it into your AI coding assistant (Claude, GPT, Cursor, etc.)
6. After implementation, copy the VERIFICATION PROMPT
7. Paste the verification prompt to test your work
8. Re-paste the verification prompt multiple times to iteratively improve
9. Check troubleshooting section if you encounter issues
10. Complete each step before moving to the next

Tips:
  - All referenced entities (pages, schemas, endpoints) are embedded inline
  - Individual entity files in pages/, schemas/, endpoints/ provide full details
  - design-system.md at root has complete design specifications
