We're building the PDF export tab for an ad analytics dashboard. This is the second tab inside the existing ExportModal component (the first tab is already built for Excel export). Do NOT create a new route or a new modal — add a 'PDF' tab to the existing two-tab ExportModal layout.

The PdfExportTab lives at /dashboard inside the ExportModal. Here's what it needs to show and do:

At the top, render a list of section checkboxes — one for each of these six options: Cover Page, Executive Summary, Daily Profit Trend Chart, Geographic Breakdown Table, Per-Network Stats Tables (one checkbox that includes all four networks), and Activity Log. Each checkbox row should show a row count badge next to the label. These row counts come from GET /api/export/preview, which is already fetched by the Excel tab — do not make a duplicate fetch. Reuse whatever shared state or context the Excel tab is already using for that preview data. While the preview data is loading, show 6 shimmer placeholder rows instead of the checkboxes.

Below the checkboxes, add an 'Options' label and a two-column inline row with two selectors: an orientation selector (Portrait / Landscape) and a paper size selector (A4 / Letter). These should feel compact and inline, not stacked.

Below that, add a collapsible 'Advanced' section using a chevron toggle. Inside it, show a custom filename input field. This is progressive disclosure — collapsed by default.

At the bottom, show two buttons: 'Cancel' (closes the modal) and 'Export Now'. The Export Now button should call POST /api/export/pdf with the selected sections, orientation, paper size, date range (from wherever the modal gets its date range), and optional filename. While the export is running, show an indeterminate spinner on the Export Now button, disable it, disable all checkboxes and selectors, and show a progress message 'Generating your PDF report...' below the button.

On success, the browser should automatically trigger a file download (handle the binary response from the endpoint and use a Blob + anchor click pattern to trigger the download). Show a toast notification 'PDF export complete — file downloaded' and close the modal after 1.5 seconds.

If no data exists for the selected range (the preview shows zero rows and the endpoint would return 404), disable the Export Now button and show a tooltip on hover that says 'No data available for this range'.

Style everything to match the existing ExportModal: use the same ThemeProvider tokens (dark:bg-gray-900 / bg-white), the same compact checkbox row style as the Excel tab, and the same accent color CSS custom property for the Export Now button. The two tabs should feel visually identical in structure.
