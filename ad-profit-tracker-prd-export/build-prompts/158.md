Now we build the six API route handlers. These are the core of the ExoClick integration. Here's what each one does:

**POST /api/networks/exoclick/sync** — The manual sync trigger. Verify the Firebase ID token from the `Authorization: Bearer <token>` header using `adminAuth.verifyIdToken()` — return 401 immediately if invalid. Validate `dateFrom` and `dateTo` from the request body (both optional, default to yesterday; must be valid YYYY-MM-DD; dateFrom <= dateTo; max 90-day range). Check a rate limit of 10 manual syncs per hour per user — store counts in Firestore or in-memory with a TTL (Firestore is safer for serverless). Fetch the user's `networkConfigs` document for `networkId='exoclick'` — return 400 if `isActive=false`. Fetch the encrypted API key from the `apiKeys` collection — return 404 if not found. Decrypt it using `decryptApiKey` from your crypto helper. Call `createExoClickClient(decryptedKey)` and invoke `getGlobalStatistics()` with `groupByCountry()`, `setTimezone()` (from user preferences or UTC), `applyFilter({ date: { from, to } })`, and `enableDetailedResults()`. Record latency. Parse the response rows and call `upsertAdStatBatch` with the parsed records — sanitize each row's raw response using `sanitizeRawResponse` before storing. Update `networkConfigs` with `lastSyncedAt`, `lastSyncStatus='success'`. Write an audit log entry directly via Admin SDK. Return `{ networkId, rowsFetched, dateFrom, dateTo, latencyMs, syncedAt }`.

**GET /api/networks/exoclick/stats** — Verify token. Validate `dateFrom`, `dateTo`, and optional `groupBy` ('day' or 'total', default 'day'). Query `adStats` where `userId==uid AND networkId=='exoclick' AND date>=dateFrom AND date<=dateTo`, ordered by date ASC. If `groupBy='total'`, sum all rows into a single totals object. If `groupBy='day'`, group by date summing across countries. Return the structured response.

**GET /api/networks/exoclick/stats/by-country** — Verify token. Validate `dateFrom`, `dateTo`, and optional `limit` (1–100, default 20). Query `adStats` filtering on the same fields plus `country != null`. Group by country in memory, sum cost/impressions/clicks. Sort by total cost descending, slice to limit. Compute each country's `costShare` as a percentage of total cost. Map country codes to full names (use a small lookup object or a package). Return the structured response.

**GET /api/networks/exoclick/raw-response** — Verify token. Validate `date` param (required, valid YYYY-MM-DD). Query `adStats` where `userId==uid AND networkId=='exoclick' AND date==date`. Map each document to `{ country, rawResponse }`. If no documents found, return 404. Also return a `fieldSchema` array describing the ExoClick API fields (hardcode this as a constant — it describes what the raw response contains). Return the full response.

**POST /api/networks/exoclick/scheduled-sync** — This is an internal endpoint called by a cron job, NOT by users. Validate the `x-internal-secret` header against `process.env.INTERNAL_SYNC_SECRET` — return 401 immediately if missing or wrong. Query `networkConfigs` for all documents where `networkId=='exoclick' AND isActive==true AND syncSchedule=='daily'` using the Admin SDK. For each matching user, fetch and decrypt their API key, call the ExoClick API for yesterday's date, upsert results via `upsertAdStatBatch`, update `networkConfigs`, and write an audit log. Track processed/succeeded/failed/skipped counts. Return the summary.

**GET /api/networks/exoclick/stats/latest** — Verify token. Query `adStats` where `userId==uid AND networkId=='exoclick'`, order by `date DESC`, limit 1 to find the latest date. Return 404 if no documents. Re-query for all documents on that date, aggregate totals. Fetch `networkConfigs` for `lastSyncedAt` and `lastSyncStatus`. Return the full summary response.

For all user-facing endpoints, always derive `userId` from the verified token — never trust a userId from the request body or query params.
