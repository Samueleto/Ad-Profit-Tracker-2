We need a `PreferencesCard` Client Component that will be embedded in the existing SettingsPage at `/settings`. This is the main UI piece for this feature — it renders as a card below the existing API key cards, consistent with the page's existing card-per-section layout.

The component accepts the initial preferences as props (passed down from the SettingsPage Server Component, which fetches them via GET /api/settings/preferences on page load). It should be a `'use client'` component since it manages local state and handles user interactions.

**Layout and controls inside the card:**
- A 'General Preferences' heading with a divider below it
- **Timezone** — a searchable combobox (not a plain `<select>`) that lets users type to filter the list of IANA timezone strings. The full list is long (~600 entries), so filtering as you type is essential for usability. Show the current value pre-selected.
- **Display Currency** — a standard dropdown with the supported ISO 4217 currency codes (USD, EUR, GBP, JPY, CAD, AUD, CHF, CNY, INR, MXN, BRL, SGD, HKD, NOK, SEK, DKK, NZD, ZAR, KRW, AED)
- **Default Date Range** — a dropdown with the four options: 'Last 7 days', 'Last 14 days', 'Last 30 days', 'This month' (mapping to the enum values `last_7_days`, `last_14_days`, `last_30_days`, `this_month`)
- **Notification Preferences** — two toggle switches: 'Daily summary email' and 'Weekly report email'. Use a switch/toggle component, not checkboxes.
- A 'Save Preferences' button at the bottom of the card

**Behavior:**
- Each control change should call PATCH /api/settings/preferences immediately on blur or change (auto-save per field). To make the API call, get the current user's ID token via `getAuth().currentUser?.getIdToken()` from `firebase/auth` and attach it as a Bearer token in the Authorization header.
- The 'Save Preferences' button at the bottom does a final PATCH with all current values — useful as a confirmation action.
- While the save is in flight, show a spinner on the 'Save Preferences' button and disable all controls.
- On success, show a toast notification: 'Preferences saved'.
- If the user has no saved preferences yet (the props come in with defaults), show a subtle 'Defaults applied' label near the heading.

**Loading state:** When the parent Server Component is fetching, show a skeleton card with 4 rows of shimmer placeholders matching the card's structure.

**Error states:** If a save fails with 401, attempt to refresh the Firebase ID token and retry once. If it still fails, redirect to `/` with a 'Session expired' toast. If it fails with 403, show an 'Access Denied' banner inside the card with a link to `/dashboard`. If it fails with 500, show an inline error banner with a 'Retry' button.
