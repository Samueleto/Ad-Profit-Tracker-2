Now we need to connect the auth system to the app's navigation so unauthenticated users can't access protected routes, and the LoginModal appears at the right times.

**Route protection:** Create a `withAuth` higher-order component or a `ProtectedRoute` wrapper that checks the `useAuth()` hook. If `loading` is true, show a full-page loading spinner (don't flash the login modal or the protected content). If `user` is null after loading completes, show the LoginModal overlay on top of the current route — don't redirect away, just block access with the modal. If `user` is present, render the children normally.

**Root route (/):** This is where the LoginModal lives by default. If a user is already signed in and visits `/`, redirect them straight to `/dashboard` — they shouldn't see the login modal if they're already authenticated.

**After successful auth:** Once the LoginModal's auth flow completes and `onAuthStateChanged` fires, the AuthContext updates `user`, and the modal should close automatically. The router should then push to `/dashboard`. Make sure this push happens after the AuthContext has finished hydrating the full user profile from GET `/api/auth/get-user` — not just after Firebase Auth resolves — so `/dashboard` has the full user object (including `role` and `preferences`) available immediately.

**Sign out:** The `signOut()` method in AuthContext handles clearing state and redirecting to `/`. Make sure any in-flight API requests are cancelled or ignored after sign-out to avoid state updates on unmounted components.

**Delete account:** After `deleteAccount()` completes, the user should land on `/` with the LoginModal visible (since they're now unauthenticated). Make sure the Firestore `users` document deletion and Firebase Auth account deletion both happen via DELETE `/api/auth/delete-account` — don't do either client-side.

**Deep linking:** If an unauthenticated user visits a protected route directly (e.g., `/dashboard`), show the LoginModal overlay on that route. After they sign in, they should land on `/dashboard` (where they were trying to go), not be redirected somewhere unexpected.
