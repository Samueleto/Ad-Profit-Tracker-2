We're building eight route handlers under src/app/api/stats/. Before writing the handlers themselves, create a set of shared helpers in src/lib/stats/ that all of them will use — this avoids duplicating the same logic eight times.

Shared helpers to build first:
- A date validation helper that checks YYYY-MM-DD format, validates dateFrom <= dateTo, and enforces the 90-day max range
- A networkId validation helper that checks against the NETWORK_IDS constant
- An aggregation helper that takes an array of AdStatDocument rows and computes totals: summing cost from exoclick rows, summing revenue from revenue-network rows, computing netProfit and roi
- A rate limiter helper using Firestore to track request counts per user per hour (used by backfill and delete endpoints)
- The ISO 3166-1 alpha-2 country name lookup map in src/lib/geo/countryNames.ts

Now build each endpoint:

**GET /api/stats/snapshot** — Verify token, validate date and optional networkId, query adStats where userId==uid AND date==date (plus networkId filter if provided). If no docs, return 404. If groupBy='country', group by networkId+country. If groupBy='total' (default), aggregate into per-network totals. Compute cross-network summary (totalRevenue, totalCost, netProfit, roi). Return the SnapshotResponse shape.

**DELETE /api/stats/snapshot** — Verify token, validate date and optional networkId, check rate limit (5/hour). Query matching adStats docs. If none, return 404. Batch delete in chunks of 500 using the Admin SDK. Write an auditLog entry. Return deletedCount.

**GET /api/stats/trend** — Verify token, validate dateFrom, dateTo (max 90 days), optional networkId and metric (default 'profit'). Query adStats for the date range. Group by date in memory, aggregate per day. Build a { date, value } series for the requested metric. Compute metadata: min, max, average, total, percentChange, and 7-day moving average. Return TrendResponse.

**GET /api/stats/summary** — Verify token, validate date range (max 90 days). Query all adStats in range. Aggregate totals across all networks and dates. Compute per-network subtotals. Fetch networkConfigs for this user to get lastSyncedAt and lastSyncStatus per network. Return SummaryResponse.

**GET /api/stats/geo-breakdown** — Verify token, validate params including optional limit (1–100, default 20). Query adStats filtering out null country rows. Group by country, aggregate metrics. Sort by requested metric descending, slice to limit. Compute percentage share. Enrich with country names from the lookup map. Return GeoBreakdownResponse.

**GET /api/stats/coverage** — Verify token, validate date range. Query adStats for the range. Enumerate every calendar day from dateFrom to dateTo and check which networks have data — mark each as 'present', 'missing', or 'partial'. Query auditLogs for sync events in the range. Compute coverage stats and identify gap dates. Return CoverageResponse.

**POST /api/stats/backfill** — Verify token, validate networkId and date range (max 90 days), check rate limit (3/hour). Determine target networks (from request or from active networkConfigs). For each network, verify an apiKey exists. Call syncNetworkForUser(uid, networkId, dateFrom, dateTo) for each valid network using Promise.allSettled(). Update networkConfigs and write auditLog entries. Return BackfillResponse with triggered/skipped/failed arrays.

**GET /api/stats/dates** — Verify token, validate optional params (default dateFrom to 90 days ago, dateTo to today). Query adStats for the range. Extract unique dates, compute which networks have data per date and whether each date is complete (all 4 networks) or partial. Sort descending. Return DatesResponse.

All eight endpoints must call verifyToken at the very top and return 401 immediately if it fails. All Firestore queries must include userId==uid as a mandatory filter.
