We need two things wired up before any of the Zeydoo endpoints can work: the Firebase Admin SDK (for auth verification, Firestore reads/writes, and audit logging) and an Axios instance configured for the Zeydoo CPA API.

For Firebase Admin, create an initialization file in `src/lib/firebase/admin.ts`. Use the standard pattern for Next.js — check if an Admin app is already initialized before calling `initializeApp()` to avoid duplicate initialization errors during hot reloads. Export the `admin` instance, a `db` reference to Firestore, and an `auth` reference. The service account credentials should come from environment variables (either a JSON blob in `FIREBASE_SERVICE_ACCOUNT_JSON` or individual fields like `FIREBASE_PROJECT_ID`, `FIREBASE_CLIENT_EMAIL`, `FIREBASE_PRIVATE_KEY`). Never hardcode credentials.

For the Zeydoo API client, create `src/lib/networks/zeydoo/client.ts`. Set up an Axios instance with the Zeydoo CPA statistics base URL already configured. The API key will be passed in at call time (it's fetched and decrypted per-request from Firestore), so don't bake it into the instance. Add a reasonable timeout (30 seconds) and set the Accept header to `application/json`. Export a typed function like `fetchZeydooStats(apiKey: string, dateFrom: string, dateTo: string)` that calls the correct Zeydoo endpoint with the `groupBy=country` parameter and returns the raw response.

Add all required environment variables to `.env.local` (with placeholder values) and document them in `.env.example` so other developers know what's needed. The required vars are: Firebase credentials, and `INTERNAL_SYNC_SECRET` (used to secure the scheduled-sync endpoint).
