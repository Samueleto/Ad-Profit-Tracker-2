We need TypeScript types and constants that both the frontend components and the API endpoints can share. Create these in `src/features/reports/types/report.ts`.

First, define the constants that act as the source of truth for allowed values — these will be used for validation in the API endpoints and for rendering options in the UI:

```
ALLOWED_METRICS = ['revenue', 'cost', 'netProfit', 'roi', 'impressions', 'clicks', 'ctr', 'cpm']
ALLOWED_NETWORKS = ['exoclick', 'rollerads', 'zeydoo', 'propush']
ALLOWED_GROUPBY = ['daily', 'country', 'network']
ALLOWED_DATA_QUALITY = ['all', 'anomalies', 'clean']
ALLOWED_DATE_PRESETS = ['last_7', 'last_14', 'last_30', 'last_90', 'this_month', 'custom']
```

Then define these TypeScript interfaces:

**SavedReport** — Matches the shape stored in the `users` collection's `savedReports` array: `id` (string), `name` (string), `metrics` (string[]), `networks` (string[]), `countries` (string[]), `groupBy` (string), `dataQuality` (string), `dateRangePreset` (string), `dateFrom` (string | null), `dateTo` (string | null), `createdAt` (Firestore Timestamp or ISO string), `updatedAt` (Firestore Timestamp or ISO string).

**ReportConfig** — The live configuration state used by CustomReportBuilderPage while the user is building a report. Same fields as SavedReport minus `id`, `name`, `createdAt`, `updatedAt`. This is what gets passed to the run and preview API calls.

**ReportRunRow** — A single row returned by `POST /api/reports/run`: `key` (string), `label` (string), and then optional number fields for each metric (`revenue?`, `cost?`, `netProfit?`, `roi?`, `impressions?`, `clicks?`, `ctr?`, `cpm?`).

**ReportRunResponse** — The full response shape from `POST /api/reports/run`: `dateFrom`, `dateTo`, `config` (ReportConfig), `rows` (ReportRunRow[]), `summary` (partial metric map + `totalRows`), `hasMore` (boolean), `nextCursor` (string | null), `cachedAt` (string | null).

**SavedReportsListResponse** — Response from `GET /api/reports`: `reports` (SavedReport[]), `total` (number).

Also create `src/lib/reportValidation.ts` with a `validateReportConfig()` function that takes a partial config object and returns `{ valid: boolean, errors: string[] }`. It should check every field against the constants above — non-empty arrays, valid enum values, date format, 90-day max range. Both `POST /api/reports/save` and `POST /api/reports/run` will import and call this function.
