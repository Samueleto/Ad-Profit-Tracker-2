We're using Firestore as the database. This step sets up the indexes and collection structure needed for the email feature to query efficiently.

First, the `auditLogs` collection. Several endpoints write to this collection (`POST /api/email/send-alert`, `POST /api/email/send-summary`, `POST /api/email/send-test`, `POST /api/email/schedule-summary`) and `GET /api/email/log` reads from it. Each audit log document should have: `userId`, `action` (e.g., `'email_sent'`), `resourceType` (e.g., `'notification'`), `resourceId` (the email type or `'test'` or `'summary_report'`), `metadata` (a map with fields like `emailType`, `recipientEmail`, `subject`, `networkId`, `severity`, `triggeredBy`, `summaryType`, `kpis`), `status` (`'success'` or `'failure'`), and `createdAt` (server timestamp). If this collection already exists in the project, we're just adding email-related entries to it ‚Äî no structural changes needed.

Second, the `users` collection needs two new fields under `preferences`: `alertDeliveryEmail` (string, nullable) and `lastTestEmailSentAt` (timestamp, nullable). And under `notificationPreferences.[type]`, each of the 9 notification types needs an `emailEnabled` boolean field. The 9 types are: `sync_failure`, `sync_success`, `export_complete`, `export_failure`, `schedule_delivered`, `schedule_failed`, `reconciliation_anomaly`, `circuit_breaker_opened`, `rate_limit_exceeded`. These fields don't need a migration ‚Äî they'll be written on first save and read with defaults applied in code.

Third, add Firestore composite indexes in `firestore.indexes.json` for:
- `auditLogs`: composite index on `userId ASC + action ASC + createdAt DESC` (for `GET /api/email/log` queries)
- `auditLogs`: composite index on `userId ASC + action ASC + resourceId ASC + createdAt DESC` (for filtered log queries by type)
- `users`: if you want to query users by `notificationPreferences.schedule_delivered.emailEnabled` in `POST /api/email/schedule-summary`, add a single-field index on that nested field ‚Äî otherwise the schedule-summary endpoint will fetch all users and filter in memory (document this tradeoff in a comment).

Deploy the indexes with `firebase deploy --only firestore:indexes`.
