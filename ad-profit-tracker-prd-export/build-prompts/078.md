We need a few packages installed and configured before building the help center.

On the frontend, install react-markdown and remark-gfm. These are used on HelpArticlePage to render the article body markdown — remark-gfm adds support for tables, strikethrough, and code blocks that plain react-markdown doesn't handle. Make sure these are imported correctly as ESM modules (react-markdown v9+ is ESM-only, so if the project uses CommonJS you may need to handle that with a dynamic import or next.config.js transpilePackages setting).

On the backend (API routes), install node-cache. This is used in GET /api/help/search to cache search results per user for 2 minutes, avoiding repeated full Firestore reads on every keystroke. Create a shared cache instance at src/lib/helpCache.ts that exports a single NodeCache instance with a default TTL of 120 seconds — this way all API routes share the same cache rather than each creating their own.

Firebase Admin SDK (firebase-admin) should already be initialized in the project. If there's a shared admin initialization file (like src/lib/firebaseAdmin.ts), make sure it exports the Firestore admin instance — all five help endpoints will import from there. If it doesn't exist yet, create it: initialize the app with admin.initializeApp() using the service account credentials from environment variables, and export both the admin auth instance and the Firestore db instance.

On the frontend, firebase/auth is used for getting the current user's ID token to attach as a Bearer token on API calls. Make sure the Firebase client SDK is initialized and accessible — the help pages will call getIdToken() on the current user before each API request.
