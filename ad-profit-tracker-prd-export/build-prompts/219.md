We're building a manual data refresh control panel that lives inside the existing dashboard — not as a standalone page, but as a persistent section embedded in the dashboard sidebar or top action bar at /dashboard.

The panel has two main parts:

**Refresh All button** — A prominent 'Refresh All Data Now' primary button at the top. On first click, show a confirmation tooltip (something like 'This will sync all active networks — continue?') before actually firing. Once confirmed, call POST /api/sync/manual with no networkId in the body. While syncing is in progress, replace the button label with 'Syncing...' and show a spinner, and disable it entirely. If the server returns a 429, disable the button and show a countdown timer like 'Available in 4m 32s' that ticks down in real time based on the Retry-After header value.

**Per-network rows** — Below the main button, show four rows: one each for ExoClick, RollerAds, Zeydoo, and Propush. Each row shows the network name, a relative 'last synced' timestamp (e.g. '2 hours ago' — update this live), a status badge (green for success, red for failed, gray for never, amber spinner for in-progress), and a small 'Refresh' icon button on the right. Clicking a per-network Refresh button calls POST /api/sync/manual with that network's networkId ('exoclick', 'rollerads', 'zeydoo', or 'propush'). While that network is syncing, disable its individual Refresh button and show the amber spinner badge. Apply the same 429 countdown logic per-network if that individual button gets rate limited.

**Sync History drawer** — Add a 'View Sync History' link that, when clicked, fetches GET /api/scheduled/sync-history and opens a slide-over drawer on the right side of the screen. The drawer shows the last 10 sync events per network in a compact list, each entry displaying the network name, status badge, rows fetched, latency in ms, and a formatted timestamp. The drawer should be closeable with an X button or by clicking outside.

**Polling** — After any refresh is triggered (either all or per-network), start polling GET /api/scheduled/sync-status every 3 seconds. Use the response to update the status badges and last-synced timestamps in real time. Stop polling when all triggered networks show a terminal status (success or failed), or after 60 seconds — whichever comes first.

**States to handle:**
- Loading (on mount): show shimmer skeleton rows for all four networks while the initial sync-status call resolves
- Empty: if no networks are configured, hide the rows and show 'No networks configured — go to Settings to add API keys' with a link to /settings
- In-progress: triggered networks show amber spinner; their individual Refresh buttons are disabled
- Rate limited: affected button disabled with countdown
- Error states: covered in error_handling step
- Success: all four rows render with live badges and timestamps

Keep the layout compact and dense — this is a sidebar/action-bar component, not a full-width page section. Use a card container with a clear heading like 'Data Sync' or 'Refresh Data'.
