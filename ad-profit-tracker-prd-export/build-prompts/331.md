We're making sure the geographic performance breakdown feature is properly secured. This feature introduces no new auth surface — it inherits the Firebase Auth middleware from FinancialMetricsDashboardPage — but there are several specific security behaviors to verify and enforce.

**Bearer token on every API call** — Every call to GET /api/stats/geo-breakdown, GET /api/roi/breakdown, and GET /api/stats/snapshot must include the Firebase ID token as a Bearer token in the Authorization header. Fetch the token using Firebase Auth's getIdToken() immediately before each request. Do not cache the token in component state between requests — always get a fresh one (Firebase's SDK handles caching and refresh internally, so getIdToken() without force=true is fine for normal calls).

**Date range cap** — Before making any API call, validate that the requested date range is 90 days or fewer. If the parent dashboard somehow passes a range longer than 90 days, truncate it to 90 days from the end date before sending it to the API. This should be enforced in the data-fetching hooks, not just in the UI.

**userId from server-side token only** — The userId used in any API query must come from the verified Firebase ID token on the server side. The client should never send a userId in the request body or query params — the server extracts it from the token. Make sure GeoBreakdownSection and GeoCountryDrilldownModal do not include userId in their API requests.

**401 handling with token refresh** — When any of the three API calls returns a 401, attempt one token refresh using getIdToken(true) (force refresh). If the refresh succeeds, retry the original request exactly once with the new token. If the refresh fails or the retried request also returns 401, redirect to / and show a 'Session expired' toast. Do not retry more than once — avoid infinite loops.

**403 handling** — If any API call returns a 403, show an 'Access Denied' message inline within the relevant component (inside the card for GeoBreakdownSection, inside the modal for GeoCountryDrilldownModal). Include a link to /dashboard. Do not redirect automatically.

**No sensitive data in client state** — Make sure the Firebase ID token is never stored in React state, localStorage, or any client-accessible store. Always fetch it on demand from the Firebase Auth SDK.
