We need TypeScript types and interfaces for everything the filter feature touches — request params, response shapes, Zustand store state, and the filter preset data structure. Create these in `src/features/filters/` so they can be imported by both the API routes and the frontend components.

Create `src/features/filters/types.ts` with the following:

**Filter state types** (used by the Zustand store):
- `FilterState`: the shape of the filter slice — `selectedNetworks: string[]`, `selectedCountries: string[]`, `selectedMetric: MetricKey`, `dataQuality: DataQualityOption`, `searchQuery: string`
- `MetricKey`: union type of `'revenue' | 'cost' | 'profit' | 'roi' | 'impressions' | 'clicks' | 'ctr' | 'cpm'`
- `DataQualityOption`: union type of `'all' | 'anomalies' | 'clean'`

**API response types** (matching the endpoint SUCCESS shapes exactly):
- `CountryOption`: `{ country: string, countryName: string, hasData: boolean }`
- `NetworkOption`: `{ networkId: string, label: string, dataRole: 'cost' | 'revenue', hasData: boolean }`
- `MetricOption`: `{ metric: MetricKey, label: string, unit: string }`
- `FilterOptionsResponse`: the full response from GET /api/filters/options — `{ dateFrom, dateTo, countries?, networks?, metrics?, cachedAt }`
- `FilterStatsRow`: `{ key, label, revenue, cost, netProfit, roi, impressions, clicks, ctr, cpm, rowCount }` — matching the rows array from GET /api/filters/stats
- `FilterStatsResponse`: the full response from GET /api/filters/stats including `rows`, `summary`, `filters`, `hasMore`, `nextCursor`, `cachedAt`
- `SearchResult`: `{ type: 'country' | 'network', key, label, revenue, cost, netProfit, roi, impressions, clicks, daysWithData }` — matching GET /api/filters/search results
- `SearchResponse`: full response from GET /api/filters/search
- `FilterPreset`: `{ id: string, name: string, networks: string[], countries: string[], metric: MetricKey, dataQuality: DataQualityOption, createdAt: any }` — used by GET/POST/DELETE /api/filters/presets
- `PresetsResponse`: `{ presets: FilterPreset[] }`

**Request param types** (for use in API route handlers):
- `FilterOptionsParams`: the validated query params for GET /api/filters/options
- `FilterStatsParams`: the validated query params for GET /api/filters/stats
- `FilterSearchParams`: the validated query params for GET /api/filters/search
- `CreatePresetBody`: the POST body for POST /api/filters/presets

Also create `src/features/filters/constants.ts` that re-exports the default filter state values — `DEFAULT_FILTER_STATE: FilterState` with `selectedNetworks: []` (meaning all), `selectedCountries: []`, `selectedMetric: 'profit'`, `dataQuality: 'all'`, `searchQuery: ''`. This is what the 'Clear All' button resets to.
