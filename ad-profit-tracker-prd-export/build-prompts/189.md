We need several reusable components that make up the DailyProfitTrendSection. Build these before wiring up the full section so each piece can be developed and tested independently.

**MetricToggle** — A segmented control with four options: Profit, Revenue, Cost, ROI. Accepts the current `ChartMetric` value and an `onChange` callback. The active segment should be visually distinct. Use Tailwind semantic classes for styling — no custom colors. This sits in the card header.

**DateRangePicker** — Shows five options: Last 7 Days, Last 14 Days, Last 30 Days, This Month, and Custom. For the Custom option, show a date range input (two date pickers, from/to). Enforce the 90-day maximum using the clamp helper from the types utils — if the user picks a range over 90 days, automatically adjust the end date and show a brief inline note ('Max range is 90 days'). Accepts current `DateRangeOption`, current `DateRange`, and an `onChange` callback. This also sits in the card header, to the right of MetricToggle.

**ProfitAreaChart** — The main recharts chart. Takes `ComputedChartData` and the active `ChartMetric` as props. Renders a recharts AreaChart at 320px height (220px on mobile via responsive container). Key behaviors:
- Area fill color: green-500 (with opacity) when `overallProfitable` is true, red-500 when false
- connectNulls={false} so null data points render as gaps
- A dashed gray horizontal ReferenceLine at y=0 labeled 'Break Even' at the right edge
- A dotted gray Line for the 7-day moving average overlay (from `movingAverage` array)
- Custom dot renderer on the main line: each dot's fill color maps from the point's `colorCode` — green-500 for 'green', red-500 for 'red', amber-500 for 'amber', gray-400 for null
- Custom tooltip showing date, netProfit, revenue, cost, and ROI% for the hovered point
- Minimal axis labels, subtle gray grid lines
- Each data point should be clickable — clicking fires an `onPointClick` callback with the `DailyProfitDataPoint` (the parent will use this to call GET /api/stats/snapshot)
- An 'Export Chart' button in the top-right corner of the chart that triggers PNG download using recharts' built-in `toBlob` or the chart ref's export method

**PatternInsightsStrip** — A collapsible strip below the chart. Collapsed by default, toggled with a chevron icon. When expanded, shows four stat tiles: Best Day (date + netProfit), Worst Day (date + netProfit), Longest Profitable Streak (N days), and Period-over-Period Change (±X%). Each tile uses the appropriate semantic color (green for positive values, red for negative, amber for neutral). Accepts a `PatternInsights` object as props. If `insights` is null or all fields are null, hide the strip entirely.

**ChartLoadingSkeleton** — A gray shimmer rectangle at 320px height (220px on mobile) that matches the chart's dimensions exactly. Used during the loading state. Also include skeleton placeholders for the MetricToggle and DateRangePicker in the card header.

All components should use Tailwind classes only — no inline styles except where recharts requires it for SVG elements.
