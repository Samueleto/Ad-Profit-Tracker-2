We need to configure Firestore so the help center queries run efficiently and the data is properly secured.

First, indexes. The helpArticles collection needs composite indexes to support the queries our endpoints will run:
- isPublished (ascending) + updatedAt (descending) — for the main article list sorted by most recently updated
- isPublished (ascending) + category (ascending) + updatedAt (descending) — for category-filtered article lists
- isPublished (ascending) + viewCount (descending) — for the Popular Articles sidebar (top 5 most-viewed)

Add these to your firestore.indexes.json and deploy them. Without these, Firestore will reject the compound queries with an error and give you a link to create the index — better to set them up now.

Second, security rules. Update your firestore.rules for the helpArticles collection:
- Any authenticated user can read documents (the server-side API handles permission filtering, but Firestore rules need to allow the admin SDK reads — note that Admin SDK bypasses client rules, so these rules apply to direct client reads if any)
- Only authenticated users can create documents
- Update and delete should be restricted — for now, only allow if the request is coming from an admin context. Since all writes go through the Admin SDK in our API routes, you can lock down client-side writes entirely for helpArticles (deny all client writes)
- The createdAt field must be immutable — add a rule that rejects updates where createdAt changes

Also make sure the auditLogs collection (used by POST /api/help/feedback to log feedback submissions) allows authenticated writes via the Admin SDK. If auditLogs rules don't exist yet, add them: deny all client reads and writes — only the Admin SDK touches this collection.

Finally, seed a few sample help articles into Firestore so the pages have something to display during development. Create 6–8 documents in the helpArticles collection covering the categories: faq, api_guide, troubleshooting, video_tutorial, network_setup, and account. Set isPublished: true on all of them.
