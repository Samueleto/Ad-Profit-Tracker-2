We're extending the existing Zustand dashboard store to support the new FilterToolbar. The store already has a dateFrom/dateTo slice — we need to add filter fields alongside those without breaking anything that already reads from the store.

Add these new fields to the existing dashboard Zustand store:

- **selectedNetworks** — an array of network IDs (strings). Default is an empty array, which means 'All Networks' is active. When the array has items, only those networks are active.
- **selectedCountries** — an array of ISO alpha-2 country codes (strings). Default is an empty array, meaning no country filter is applied.
- **selectedMetric** — a single string, one of 'revenue' | 'cost' | 'profit' | 'roi'. Default is 'profit'.
- **dataQuality** — a single string, one of 'all' | 'anomalies' | 'clean'. Default is 'all'.
- **searchQuery** — a string, default empty string ''. This is the debounced free-text search value.

Also add a **stagedNetworks** and **stagedCountries** field to hold the user's in-progress selections before they hit 'Apply Filters'. These are separate from the committed selectedNetworks/selectedCountries so that the dashboard doesn't re-fetch on every checkbox click — only when the user explicitly applies.

Expose these actions from the store:
- `setStagedNetworks(networks: string[])` — updates staged (not committed) network selection
- `setStagedCountries(countries: string[])` — updates staged country selection
- `applyFilters()` — copies staged values into selectedNetworks and selectedCountries
- `setSelectedMetric(metric: string)` — updates metric immediately
- `setDataQuality(quality: string)` — updates data quality immediately
- `setSearchQuery(query: string)` — updates the debounced search query
- `clearAllFilters()` — resets selectedNetworks, selectedCountries, stagedNetworks, stagedCountries, selectedMetric, dataQuality, and searchQuery all back to their defaults

Also add a derived selector `activeFilterCount` that computes how many filter dimensions are currently non-default (so the badge in FilterToolbar can read a single number). Count: selectedNetworks non-empty = +1, selectedCountries non-empty = +1, selectedMetric !== 'profit' = +1, dataQuality !== 'all' = +1, searchQuery non-empty = +1.

Make sure the existing dateFrom/dateTo fields and their actions are untouched. Any dashboard section that already subscribes to the store should continue working exactly as before.
