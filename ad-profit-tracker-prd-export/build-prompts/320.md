Now we need to connect the CustomReportBuilderPage to the rest of the app and make sure all the shared pieces are properly integrated.

**Navigation**

Add a 'Reports' link to the main app navigation (sidebar or top nav, wherever the other main sections live). It should point to /reports. If the app uses an active link indicator, make sure /reports highlights correctly.

**ExportModal integration**

The 'Export' button on CustomReportBuilderPage should open the existing ExportModal component. Pass it the current date range (dateFrom, dateTo derived from the current config) and the selected sections/metrics so the modal opens pre-configured. The modal should call `GET /api/export/excel` or `GET /api/export/pdf` depending on what the user selects — make sure those calls also include the Bearer token from `getAuthHeader()`.

**MobileDataTableWrapper integration**

The preview table uses the existing `MobileDataTableWrapper` component. Wire it up so it receives the rows from `useReportPreview` (or `useRunReport` when in run mode). The columns should be dynamic — only show columns for the metrics the user has selected. For example, if the user only selected Revenue and Clicks, only those two metric columns should appear in the table alongside the grouping key column (date, country, or network name depending on `groupBy`).

**Saved Reports drawer toggle**

Add a 'Saved Reports' button near the top of the page (or in the header area) that toggles the sidebar drawer open/closed on desktop. On mobile, this button should open the bottom sheet. The drawer/sheet should close when the user clicks Load on a report (since the config is now applied and they probably want to see the preview).

**Toast notifications**

Wire up toast notifications for these events using whatever toast library the app already uses:
- Save Report success: 'Report saved successfully'
- Save Report failure: 'Failed to save report. Please try again.'
- Rename success: 'Report renamed'
- Delete success: 'Report deleted'
- Run Report error: 'Failed to run report. Please try again.'
- Session expired (401 after token refresh fails): 'Session expired. Please sign in again.' then redirect to /

**Route protection**

Make sure /reports is inside the `(app)` route group so the existing middleware.ts auth guard covers it. You shouldn't need to add anything new here — just confirm the file is in the right place (`app/(app)/reports/page.tsx`) so it inherits the group's auth protection.

**Deep linking to a saved report**

If the URL includes a `?reportId=<id>` query param when the page loads, `useSavedReports` should automatically call `loadReport` with the matching report after the saved reports list loads. This lets users share or bookmark a specific report configuration. If the reportId doesn't exist in their saved reports, just ignore the param and load with defaults.
