We need next-themes set up so the app can switch between Light, Dark, and System (OS-following) themes without a flash of the wrong theme on page load.

Install the next-themes package. Then wrap the entire app with the ThemeProvider from next-themes. In Next.js, this goes in the root layout file (app/layout.tsx if using the App Router, or pages/_app.tsx if using the Pages Router — use whichever the project already has). Set the ThemeProvider's attribute to 'class' so it adds a class to the HTML element that Tailwind (or whatever CSS approach the project uses) can pick up. Set defaultTheme to 'system' so new users automatically follow their OS preference.

To prevent the flash of unstyled content (FOUC) that happens when the page loads before the theme is applied, make sure the ThemeProvider is configured with enableSystem set to true. The next-themes package handles this automatically when set up correctly, but double-check that the HTML element gets the right class before the page renders.

For the accent color side of theming, we'll handle that with CSS custom properties (CSS variables) rather than next-themes — next-themes handles light/dark switching, and we'll store the accent color separately. Set up a global CSS variable --accent-color in the project's global stylesheet with the default indigo hex value. We'll update this variable dynamically when the user picks a different accent.

After setup, verify that toggling the theme class on the HTML element actually changes the visual appearance of the page — if the project uses Tailwind, make sure darkMode is set to 'class' in tailwind.config.
