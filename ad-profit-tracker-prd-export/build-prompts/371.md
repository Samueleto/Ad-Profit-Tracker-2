We need to make sure the geographic performance breakdown feature fails gracefully across all error scenarios. Both GeoBreakdownSection and GeoCountryDrilldownModal have multiple API calls that can fail independently, and each needs a clear, helpful response.

**GeoBreakdownSection error states:**

- **Loading** — Show 10 shimmer rows that match the column structure (same widths, same number of columns). The card header controls (metric toggle, Top N selector, search input) should also show as skeleton placeholders during the initial load so the layout doesn't jump.

- **Empty (no data)** — When GET /api/stats/geo-breakdown returns an empty array, show: 'No geographic data for this date range. Sync your networks to see country breakdowns.' with a 'Sync Now' CTA that links to or opens the ManualRefreshPanel.

- **error_401** — Attempt Firebase ID token refresh via getIdToken(true). If refresh succeeds, retry the request. If refresh fails, redirect to / with a 'Session expired' toast notification.

- **error_403** — Show 'Access Denied' inline within the card body (not a full-page error). Include a link to /dashboard.

- **error_404** — Show 'No geographic data found for selected range' with a date range reset button that resets the dashboard date range to the default (e.g., last 7 or 30 days).

- **error_500** — Show an inline error banner inside the card (not a full-page error) with a 'Retry' button. The Retry button must re-call both GET /api/stats/geo-breakdown AND GET /api/roi/breakdown together — not just one of them.

- **Partial failure** — If GET /api/stats/geo-breakdown succeeds but GET /api/roi/breakdown fails, render the table without color-coding (treat all rows as neutral/no color) and show a small non-blocking warning: 'ROI color coding unavailable.' If the reverse happens (ROI succeeds, geo fails), show the full error state.

**GeoCountryDrilldownModal error states:**

- **Loading** — Show a 200px shimmer rectangle in the chart area and 4 shimmer rows in the network breakdown table area.

- **Empty** — Show 'No data available for [Country Name] in this period' with a close button. Use the actual country name in the message.

- **error_401** — Close the modal immediately, then redirect to / with a 'Session expired' toast.

- **error_403** — Show 'Access Denied' inside the modal body with a close button.

- **error_404** — Show 'No data found for this country' inside the modal with a close button.

- **error_500** — Show an error banner inside the modal with a 'Retry' button that re-calls GET /api/stats/snapshot for the same country and date range.

Make sure error states don't bleed between components — an error in the modal should not affect the GeoBreakdownSection table, and vice versa.
