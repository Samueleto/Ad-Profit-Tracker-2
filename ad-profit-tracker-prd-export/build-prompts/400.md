# Step 400 of 437: Handle all error states in the report builder

The report builder makes a lot of API calls, and each one can fail in different ways. Let's make sure every failure is handled gracefully so users always know what happened and what to do next.

**Preview table errors (GET /api/filters/stats)**

- **401**: Attempt `getIdToken(true)` token refresh and retry once. If the retry also fails, show a 'Session expired' toast and redirect to /.
- **403**: Replace the preview table with an 'Access Denied' card that includes a link to /dashboard.
- **404**: Show 'No data found for selected range and filters' with a 'Reset Date Range' button that sets the preset back to 'last_30' and triggers a new preview call.
- **500**: Show an inline error banner inside the preview panel (not a full-page error) with a 'Retry' button that re-calls `GET /api/filters/stats` with the same config.
- **Empty result (200 but zero rows)**: Show 'No data matches your current configuration. Adjust filters or sync your networks.' with a 'Sync Now' link.
- **Network failure / timeout**: Show 'Unable to load preview. Check your connection.' with a Retry button.

**Run Report errors (POST /api/reports/run)**

- **400**: Show an inline error near the Run button: 'Invalid configuration. Please check your selected metrics and date range.'
- **401**: Same token refresh + redirect flow as above.
- **404**: Show 'No data found for the selected configuration and date range.' in the preview area.
- **429**: Show 'You've run too many reports this hour. Please wait a few minutes.' — don't redirect, just block the action.
- **500**: Show an error banner in the preview panel with a Retry button.

**Save Report errors (POST /api/reports/save)**

- **400 (max 20 reports)**: Show 'You've reached the maximum of 20 saved reports. Delete one to save a new one.' as an inline message near the Save button.
- **400 (other validation)**: Show 'Invalid report configuration.' inline.
- **401**: Token refresh + redirect.
- **429**: Show 'Too many saves this hour. Please try again later.'
- **500**: Show 'Failed to save report. Please try again.' as a toast.

**Rename errors (PATCH /api/reports/:id)**

Since rename is optimistic, if the PATCH fails: revert the name in the UI immediately, show a toast 'Failed to rename report. Please try again.'

**Delete errors (DELETE /api/reports/:id)**

Since delete is optimistic, if the DELETE fails: restore the report in the list immediately, show a toast 'Failed to delete report. Please try again.'

**Load saved reports errors (GET /api/reports)**

- **401**: Token refresh + redirect.
- **404**: This means the user document doesn't exist yet — treat it the same as an empty list (show 'No saved reports yet').
- **500**: Show 'Failed to load saved reports.' with a Retry button inside the sidebar panel.

**Row count badge errors (GET /api/export/preview)**

If this call fails, just hide the badges silently — they're supplementary info and a failure here shouldn't disrupt the main experience.

**General principles**

Create a shared `handleApiResponse` utility that takes a fetch Response and handles the 401 refresh-and-retry flow in one place. All hooks should use this utility. Never show raw error messages or stack traces to the user — always translate to plain English. Log the raw error to the console for debugging.
