We're building a Next.js app with Firebase authentication. For this step, I need you to build the LoginModal component — a centered modal overlay with a backdrop blur that handles all authentication flows in one place.

The modal lives at the root route (/) and also appears whenever a user tries to access a protected route. It should not be dismissible until the user successfully authenticates — there's no close button or backdrop click to escape.

Inside the modal, show two tabs: 'Sign In' and 'Sign Up'. Both tabs share the same modal shell, so switching between them is instant with no route changes.

**Sign In tab:** Show a Google OAuth button (make this visually primary — it should stand out) and below it an email/password form. The password field needs a show/hide toggle. Include a 'Forgot password?' link that calls Firebase's `sendPasswordResetEmail` inline — no separate page needed, just show a confirmation message in place. When the user submits the email form, call `signInWithEmailAndPassword` from `firebase/auth`.

**Sign Up tab:** Same Google button at the top. Email/password form, but this time also show a 'Confirm Password' field (only visible in this tab — progressive disclosure). When submitted, call `createUserWithEmailAndPassword` from `firebase/auth`. After the Firebase account is created, immediately write the user document to Firestore's `users` collection with all the fields from the users schema: uid, email, displayName, photoURL, authProvider set to 'email', role set to 'user', createdAt as a server timestamp, lastLoginAt as a server timestamp, and an empty preferences map.

**Google OAuth (both tabs):** Call `signInWithPopup` with `GoogleAuthProvider` from `firebase/auth`. After success, the `onAuthStateChanged` listener will handle the rest.

**After any successful auth:** Call POST `/api/auth/sync-user` with the user's Firebase ID token as a Bearer token in the Authorization header. Once that resolves, close the modal and push the user to `/dashboard`.

**Loading states:** While any auth operation is in progress, show a spinner on the active button and disable all inputs. The Google button and email form should have independent loading states — clicking Google shouldn't freeze the email form.

**Error messages:** Show inline, directly below the relevant field. Don't use a generic error banner — put 'Wrong password' under the password field, 'Email already in use' under the email field, etc. For network errors (not field-specific), show a toast notification with a retry option.
