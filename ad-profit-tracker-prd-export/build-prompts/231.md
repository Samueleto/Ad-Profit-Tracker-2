We're building an ad network analytics dashboard. For this step, I need you to build the frontend pages that let users view their ExoClick statistics, trigger manual syncs, and explore raw API responses.

There are no standalone page routes to create here — instead, these views should be integrated as tabs or sections within the existing network/dashboard area of the app. Here's what each view needs to do:

**ExoClick Overview / Summary Card** — This is a compact summary widget that calls `GET /api/networks/exoclick/stats/latest` on load. It should show the most recently synced day's totals: cost, impressions, clicks, CTR, CPM, and how many countries contributed data. Also show the `lastSyncedAt` timestamp and `lastSyncStatus` from the response. If the API returns 404 (no data yet), show a friendly prompt telling the user to run their first sync. If `lastSyncStatus` is `'failed'`, show a warning badge.

**ExoClick Stats View** — A date-range stats panel that calls `GET /api/networks/exoclick/stats` with `dateFrom`, `dateTo`, and `groupBy` query params. Give the user a date range picker (default to the last 7 days) and a toggle between 'By Day' and 'Total' grouping. When `groupBy='day'`, render a line or bar chart showing daily cost, impressions, and clicks over time. When `groupBy='total'`, show a summary table with aggregate totals. Below the chart, always show the `totals` object from the response (total cost, impressions, clicks). Cap the selectable range at 90 days and show an error if the user tries to exceed it.

**ExoClick Country Breakdown View** — Calls `GET /api/networks/exoclick/stats/by-country` with the same date range and a configurable `limit` (default 20, max 100). Show results as a sortable table with columns: country flag + name, cost, impressions, clicks, CTR, and cost share percentage. The `costShare` field from the response should render as a small progress bar or percentage badge. Let the user sort by any column client-side.

**ExoClick Manual Sync Panel** — A simple panel with a date range picker (defaults to yesterday) and a 'Sync Now' button. On click, POST to `POST /api/networks/exoclick/sync` with the selected `dateFrom` and `dateTo`. While the sync is running, show a loading spinner and disable the button. On success, show the response details: `rowsFetched`, `dateFrom`, `dateTo`, and `latencyMs`. On error, show the specific error message (rate limited, network disabled, no API key, etc.). Add a note near the button explaining the 10 syncs/hour limit.

**API Explorer View** — A developer-facing panel that calls `GET /api/networks/exoclick/raw-response` for a selected date. Show a date picker (single date, not a range). On load, display the raw response records in a collapsible JSON viewer — one expandable row per country. Below the raw data, render the `fieldSchema` array from the response as a reference table showing field name, type, and description. If the API returns 404, show 'No data for this date — try syncing first.'
