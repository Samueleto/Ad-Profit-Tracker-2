We need several UI components before building the full PerformanceBenchmarkingTab and BenchmarkSettingsModal pages. These components are either reused across both, or complex enough to warrant building and testing in isolation.

**1. BenchmarkScoreGauge** (`components/benchmarking/BenchmarkScoreGauge.tsx`)

A circular gauge that displays the 0–100 performance score. The gauge arc should be color-coded: green-500 for scores ≥ 80, amber-500 for 50–79, and red-500 for < 50. Show the numeric score prominently in the center. Accept a `score: number | null` prop — when null, show a shimmer placeholder circle. Accept a `computedAt: string | null` prop to show a small 'Last updated' timestamp below the score. This is the prominent card at the top-right of PerformanceBenchmarkingTab.

**2. BenchmarkComparisonRow** (`components/benchmarking/BenchmarkComparisonRow.tsx`)

A single row in the metric comparison table. Shows: network name/icon, actual value (formatted with the right unit — '%' for roi/ctr, '$' for cpm/revenue/cost, raw number for impressions/clicks), historical average value, industry benchmark value, and a MetricShareBar (imported from the existing ComparativeNetworkAnalysisTab components) showing where the actual sits relative to the benchmark range. The actual value cell should be color-coded: green-500 if actual > industry benchmark, red-500 if below, amber-500 if within 10%. Null actual values render as '—'. If the network's `lastSyncStatus` is `'failed'` or `circuitBreakerOpen` is true, show an amber warning badge with a warning icon on the row. Accept a `loading: boolean` prop — when true, render a shimmer placeholder row instead of real data.

**3. BenchmarkSparkline** (`components/benchmarking/BenchmarkSparkline.tsx`)

A recharts-based sparkline showing the user's selected metric over the last 30 days (a `LineChart` with a thin line) plus a dashed reference line for the historical average. Mark this component with `'use client'` since recharts requires it. Accept props: `data: Array<{ date: string, actual: number | null, historicalAverage: number | null }>`, `metric: MetricKey`, `unit: string`. Use a `ResponsiveContainer` so it fills its parent width. Keep it compact — around 120px tall. When `data` is empty or all nulls, show a 120px shimmer rectangle placeholder.

**4. BenchmarkMetricRow** (`components/benchmarking/BenchmarkMetricRow.tsx`)

A single row in the BenchmarkSettingsModal form. Shows the metric label, the system default value in muted text, a number input for the custom target, and a toggle switch for 'Use Industry Default' vs 'Custom Target'. When the toggle is set to 'Use Default', gray out and disable the number input. A 'Reset to Default' button per row restores the input to the system default value and sets the toggle back to 'Use Default'. Accept props: `metric: MetricKey`, `systemDefault: number`, `currentTarget: number | null`, `useDefault: boolean`, `unit: string`, `onChange: (metric, value, useDefault) => void`. When `loading` is true, render a shimmer row.
