We're building a scheduled report delivery feature that lets users configure automated email delivery of their saved reports. For this step, set up the folder structure so everything has a home before we start writing real code.

On the frontend side, create a scheduled-reports feature area. Inside it, you'll want folders for: components (we'll have the ScheduleReportModal and its sub-pieces), hooks (for fetching and mutating schedules), and types (the TypeScript interfaces for schedule objects).

On the API side, create a folder at src/app/api/schedules/ to house all five route handlers we'll need: the create endpoint, the list endpoint, the update endpoint, the delete endpoint, and the send-now endpoint. The routes will be:
- src/app/api/schedules/route.ts (handles GET list and POST create)
- src/app/api/schedules/[id]/route.ts (handles PATCH update and DELETE)
- src/app/api/schedules/[id]/send-now/route.ts (handles POST send-now)

Also create a shared utilities folder at src/lib/schedules/ for things like the nextRunAt computation logic and date range resolution — these will be shared between multiple API routes so they shouldn't live inside any single route file.
