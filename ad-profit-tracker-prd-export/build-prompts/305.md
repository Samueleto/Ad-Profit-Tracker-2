We're building a RollerAds ad network integration. For this step, I need you to connect all six RollerAds API endpoints to the rest of the application so data flows correctly from the backend to wherever it's consumed.

Start with the manual sync trigger. Wherever the app has a 'Sync Now' button or similar control for ad networks, wire it up to call POST /api/networks/rollerads/sync. Pass the user's Firebase ID token in the Authorization header. If the user has selected a custom date range, include dateFrom and dateTo in the request body (YYYY-MM-DD format). On success, the response includes rowsFetched, latencyMs, and syncedAt — use these to show a confirmation message like 'Synced 42 rows for 2024-01-15'.

For the scheduled sync, POST /api/networks/rollerads/scheduled-sync is an internal endpoint only — it should never be called from the frontend. Make sure it's wired up to your Cloud Scheduler (or equivalent cron trigger) to fire daily, passing the INTERNAL_SYNC_SECRET in the x-internal-secret header. Double-check that no client-side code references this endpoint.

For stats display, wire GET /api/networks/rollerads/stats into any date-range stats view. Pass dateFrom, dateTo, and groupBy ('day' or 'total') as query params. The response data array drives time-series charts; the totals object drives summary numbers.

For the country breakdown view, wire GET /api/networks/rollerads/stats/by-country with dateFrom, dateTo, and an optional limit param. The response includes a revenueShare percentage and full countryName for each entry — use both in the table or chart.

For dashboard summary cards, wire GET /api/networks/rollerads/stats/latest — no params needed. This returns the most recent day's aggregated revenue, impressions, clicks, ctr, cpm, and countryCount, plus lastSyncedAt and lastSyncStatus from networkConfigs. Show lastSyncStatus as a badge (success/failed) next to the card.

For the API Explorer tab, wire GET /api/networks/rollerads/raw-response with a date param. The response includes a records array (one entry per country with its rawResponse snapshot) and a fieldSchema array describing each RollerAds field. Render fieldSchema as a reference table and records as the raw data viewer.

Make sure every frontend call to these endpoints attaches the current user's Firebase ID token. Use Firebase Auth's getIdToken() to retrieve it fresh before each request — don't cache tokens long-term since they expire after an hour.
