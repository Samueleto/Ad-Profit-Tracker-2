We need TypeScript interfaces that describe the data flowing through this feature — from the Firestore `adStats` and `networkConfigs` documents, through the `GET /api/networks/stats` endpoint, and into the React components. Let's define these in the `types/` folder we created.

Create a file like `src/features/network-analytics/types/index.ts` with the following:

**NetworkId** — a union type: `'exoclick' | 'rollerads' | 'zeydoo' | 'propush'`. Use this everywhere instead of raw strings.

**DataRole** — `'cost' | 'revenue'`. ExoClick is cost, the other three are revenue.

**NetworkStatsSummary** — the aggregated totals object: `primaryMetric`, `impressions`, `clicks`, `averageCtr`, `averageCpm`, `countryCount`, `daysWithData` — all numbers.

**NetworkStatsDailyPoint** — one entry in the time-series array: `date` (string), `primaryMetric`, `impressions`, `clicks`, `ctr`, `cpm` — all numbers.

**NetworkStatsCountryRow** — one row in the GEO breakdown: `country` (ISO alpha-2 code), `countryName`, `primaryMetric`, `impressions`, `clicks`, `ctr`, `cpm`, `metricShare` — all numbers except the string fields.

**NetworkStatus** — the network config status: `lastSyncedAt` (timestamp or null), `lastSyncStatus` (string), `circuitBreakerOpen` (boolean).

**NetworkStatsResponse** — the full API response shape matching what `GET /api/networks/stats` returns: `networkId`, `dataRole`, `dateFrom`, `dateTo`, `groupBy`, `summary`, optional `series` array, optional `countries` array, `networkStatus`, `cachedAt` (string or null).

**AdStatDocument** — the shape of a raw `adStats` Firestore document: `userId`, `networkId`, `date`, `country`, `impressions`, `clicks`, `cost` (optional), `revenue` (optional), `ctr`, `cpm`. This is used inside the API route when reading from Firestore.

**NetworkConfigDocument** — the shape of a `networkConfigs` Firestore document: `userId`, `networkId`, `lastSyncedAt`, `lastSyncStatus`, `circuitBreakerOpen`.

Keep all these in one file for now — the feature is self-contained enough that splitting into multiple type files would be overkill.
