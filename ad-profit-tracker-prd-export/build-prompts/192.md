We need to build the `LoginModal` component — this is the only UI piece for the auth feature, and it handles both sign-in and sign-up in a single modal overlay.

**Overall structure:**
The modal renders as a centered overlay with a backdrop blur effect. It is not dismissible by clicking the backdrop or pressing Escape — the user must complete authentication to close it. This is intentional: the modal appears on the root route `/` and on any protected route redirect, so there's no unauthenticated state to fall back to.

**Two-tab layout (Sign In / Sign Up):**
Use a tab or toggle at the top of the modal to switch between modes. Both tabs live in the same component — no routing involved. The Sign Up tab adds a confirm password field that isn't shown in Sign In mode (progressive disclosure).

**Google OAuth button:**
This should be visually prominent — treat it as the primary action. On click, call `signInWithPopup` from `firebase/auth` using the `GoogleAuthProvider` instance from `lib/firebase/auth.ts`. While the popup is loading, show a spinner on the button and disable all inputs. On success, call `POST /api/auth/sync-user` with the user's ID token (get it via `user.getIdToken()`), then close the modal and push to `/dashboard`.

**Email/password form:**
Two fields: email and password (with a show/hide toggle on the password field). In Sign Up mode, add a confirm password field. Validate that passwords match before submitting. Sign In calls `signInWithEmailAndPassword`; Sign Up calls `createUserWithEmailAndPassword` then immediately writes the user doc to Firestore's `users` collection (using the client-side Firestore instance from `lib/firebase/firestore.ts`) AND calls `POST /api/auth/sync-user`. Show inline error messages below the relevant field — not as toasts. Map Firebase Auth error codes to friendly messages (e.g., `auth/wrong-password` → 'Wrong password', `auth/email-already-in-use` → 'Email already in use').

**Forgot password:**
A 'Forgot password?' link below the password field (Sign In tab only). Clicking it calls `sendPasswordResetEmail` from `firebase/auth` inline — no separate page or route. Show a success message inline when the email is sent.

**Loading states:**
While any async operation is in progress, show a spinner on the active button and disable all inputs. The Google button and email submit button should have independent loading states so they don't interfere with each other.

**After success:**
On successful auth (either method), `onAuthStateChanged` will fire. The modal should close and the router should push to `/dashboard`.
