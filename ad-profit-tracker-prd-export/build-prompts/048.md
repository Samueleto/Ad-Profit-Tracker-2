We need two packages set up before building the RollerAds endpoints: axios for making HTTP calls to the RollerAds API, and cryptr for encrypting and decrypting API keys server-side. Firebase Admin SDK should already be initialized in the project — if not, set that up too.

First, install the packages. We need `axios` at version `^1.6.0` specifically (the sync endpoints depend on this version), and `cryptr` for AES encryption.

Next, check whether the project already has a Firebase Admin SDK initialization file (something like `src/lib/firebase-admin.ts` or `src/lib/firebaseAdmin.ts`). If it exists, leave it alone. If it doesn't exist, create it — initialize the Admin SDK using `firebase-admin` with the service account credentials from environment variables (`FIREBASE_PROJECT_ID`, `FIREBASE_CLIENT_EMAIL`, `FIREBASE_PRIVATE_KEY`). Export the `admin` instance, `db` (Firestore), and `auth` so all API routes can import them from one place.

Then create the cryptr helper at `src/lib/encryption.ts`. This should export two functions: `encryptApiKey(plaintext: string): string` and `decryptApiKey(ciphertext: string): string`. Both use a `Cryptr` instance initialized with the `ENCRYPTION_SECRET` environment variable. This secret must be the same one used by the existing `/api/keys/save` endpoint — do not create a new secret. Add a guard that throws a clear error if `ENCRYPTION_SECRET` is not set, so misconfiguration fails loudly at startup rather than silently at runtime.

Finally, create the RollerAds axios client factory at `src/lib/rollerads/client.ts`. Export a function `createRollerAdsClient({ baseURL, apiKey, timeoutSeconds }: RollerAdsClientOptions)` that returns an `axios.create()` instance preconfigured with: the provided `baseURL` (defaulting to `https://api.rollerads.com`), an `Authorization: Bearer {apiKey}` header, `Content-Type: application/json`, and a timeout of `timeoutSeconds * 1000` milliseconds. The sync endpoints will call this factory — they never construct axios instances directly.

Add `ENCRYPTION_SECRET` and `INTERNAL_SYNC_SECRET` to your `.env.local` (and document them in `.env.example` if the project has one). `INTERNAL_SYNC_SECRET` is used by the scheduled-sync endpoint to authenticate Cloud Scheduler calls.
