We need TypeScript types and config definitions that all four endpoints will share. This keeps the response shapes consistent and makes the route handlers much cleaner.

Create a `lib/rate-limit/types.ts` file with the following types:

- `NetworkId` — a union type: `'exoclick' | 'rollerads' | 'zeydoo' | 'propush'`. Export a `VALID_NETWORKS` constant array with these four values so we can validate incoming query params against it.

- `NetworkLimiterStatus` — the per-network status shape returned by GET /api/rate-limits/status: `{ networkId: NetworkId, isThrottled: boolean, reservoir: number, reservoirMax: number, nextReservoirRefreshAt: string | null, running: number, queued: number, circuitBreakerOpen: boolean, retryCount: number }`

- `UserQuotaStatus` — the per-endpoint quota shape: `{ endpoint: string, limit: number, windowSeconds: number, remaining: number, resetAt: string }`

- `NetworkLimiterConfig` — the static config shape from `limiterConfig.ts`: `{ networkId: NetworkId, maxConcurrent: number, minTimeMs: number, reservoir: number, reservoirRefreshIntervalMs: number, reservoirRefreshAmount: number, description: string }`

- `UserQuotaConfig` — the static quota config shape: `{ endpoint: string, limit: number, windowSeconds: number, description: string }`

- `ViolationRecord` — the audit log violation shape: `{ id: string, networkId: string | null, endpoint: string, limitType: 'outbound' | 'user-quota', limitValue: number, windowSeconds: number, retryAfterSeconds: number | null, createdAt: FirebaseFirestore.Timestamp }`

- `ResetScope` — a union type: `'outbound' | 'user-quota' | 'both'`. Export a `VALID_SCOPES` constant array.

In `lib/rate-limit/userLimiterConfig.ts`, define the quota config for each rate-limited endpoint: `/api/sync/manual` (10/hour), `/api/stats/backfill` (3/hour), `/api/scheduled/retry-failed` (5/hour per network), `/api/reconciliation/run` (10/hour), `/api/errors/circuit-breaker/reset` (10/hour). Each entry should match the `UserQuotaConfig` type.

In `lib/rate-limit/userLimiter.ts`, set up the in-memory sliding window counter. Use a Map keyed by `uid:endpoint` storing an array of timestamps. Export `getRemainingQuota(uid, endpoint)` and `clearQuota(uid, endpoint)` functions. The sliding window should drop timestamps older than `windowSeconds` before counting.
