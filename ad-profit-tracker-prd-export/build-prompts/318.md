We need to make sure next-themes is properly set up at the app level so theme switching works everywhere — not just inside the ThemeSettingsSection.

In the Next.js app root (either `_app.tsx` or the root layout if using the App Router), wrap the entire app with next-themes' `ThemeProvider`. Configure it with:
- `attribute='class'` so it adds a class to the `<html>` element (this is the standard approach for Tailwind or CSS class-based dark mode)
- `defaultTheme='system'` so new users start with their OS preference
- `enableSystem={true}` to allow the 'System' option to work
- `disableTransitionOnChange={false}` — we want smooth transitions when switching

Make sure your global CSS (or Tailwind config) is set up to respond to the dark mode class. If using Tailwind, confirm `darkMode: 'class'` is in the config. If using plain CSS, make sure there are `.dark` class overrides for background, text, and card colors.

For the accent color, since it's applied as a CSS custom property (`--accent-color`) on the document root, make sure your global stylesheet references this variable in the right places — buttons, links, active states, focus rings. This way, changing the accent color in ThemeSettingsSection cascades everywhere automatically.

Finally, wire the ThemeSettingsSection into the existing SettingsPage. Find where the SettingsPage tab bar is defined and add an 'Appearance' tab that renders ThemeSettingsSection. The tab label should say 'Appearance' and can use a palette or paint-brush icon. Make sure the tab is positioned logically within the existing tab order (typically after account/profile settings).

Do a quick smoke test to confirm: switching to dark mode on the Appearance tab should make the entire app go dark — not just the settings section.
