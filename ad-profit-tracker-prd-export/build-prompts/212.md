We're building the NotificationCenterPanel — a right-side slide-over panel that opens when the user clicks a bell icon in the top navigation bar. This is the only page-level UI for this feature (it renders as a panel on /dashboard), but it has enough internal complexity that we should build its pieces as composable components.

Here's what to build:

**BellIconTrigger** — The bell icon that lives in the top nav bar. Shows an unread count badge when unreadNotificationCount > 0. The badge should cap display at 99+ for large counts. Clicking it opens the NotificationCenterPanel. The unread count comes from the Zustand dashboard store (which is hydrated on app mount from the users document). This component goes into the existing top navigation bar.

**NotificationCenterPanel** — The main slide-over panel. 360px wide on desktop, full-screen on mobile. Opens from the right with a backdrop overlay that closes the panel when clicked. Inside: a header with 'Notifications' title, a 'Mark All as Read' button (only active if unreadCount > 0, calls PATCH /api/notifications/mark-all-read), and a 'Clear All' button (destructive — requires a single confirm click before calling DELETE /api/notifications/clear). Below the header, two tabs: 'All' and 'Unread'. The active tab filters which notifications are shown. The notification list loads the first 20 from GET /api/notifications on panel open. A 'Load more' button at the bottom calls GET /api/notifications with the nextCursor when hasMore is true.

**NotificationRow** — A single notification item, 56px tall. Left side: a colored left border (green for success, red for error, amber for warning, blue for info). Then a semantic icon (arrows icon for sync types, red X for error types, download icon for export types, clock icon for schedule types). Then a text block: title in bold, body text truncated to one line (expandable on click to show full text). Timestamp in the top-right corner as relative time (e.g., '3 minutes ago', '2 hours ago'). A small X dismiss button on the right that calls PATCH /api/notifications/dismiss/:id. Unread notifications should have a subtle background tint. Clicking the row (not the dismiss button) navigates based on type: sync_failure goes to the ManualRefreshPanel section on /dashboard, schedule_failed goes to /reports, all other types do nothing on click.

**NotificationSkeletonRow** — A shimmer placeholder that matches the 56px height and layout of NotificationRow. Show 5 of these while GET /api/notifications is loading.

**NotificationEmptyState** — Shown when there are no notifications (or no unread ones on the Unread tab). Shows a checkmark illustration and 'You're all caught up' message. Friendly, not alarming.

**NotificationErrorBanner** — An inline error banner inside the panel (not a full-screen takeover) shown when GET /api/notifications returns a 500. Has a Retry button that re-calls the endpoint. For 401 errors, close the panel, call getIdToken(true) to force-refresh the token, and if that fails redirect to / with a 'Session expired' toast. For 403, show 'Access Denied' with a close button. For 404, show 'No notifications found' with a close button.

Make sure the relative timestamps update automatically (re-render every minute or use a library like date-fns formatDistanceToNow). The panel should be accessible — trap focus inside when open, support Escape key to close, and have proper ARIA labels on the bell icon badge and panel.
