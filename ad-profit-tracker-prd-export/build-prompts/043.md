We need two server-side utilities set up before we can build the API endpoints: Firebase Admin for talking to Firestore, and `cryptr` for encrypting API keys.

First, install the required packages: `firebase-admin`, `cryptr`, and `react-hook-form` (the last one is for the frontend forms we'll build later).

**Firebase Admin setup** — Create a shared Firebase Admin initialization file at `src/lib/firebase-admin.ts`. This should initialize the Admin SDK using a service account. The service account credentials should come from environment variables (never hardcoded). A common pattern is to store the service account JSON as a base64-encoded environment variable called `FIREBASE_SERVICE_ACCOUNT_BASE64`, decode it at runtime, and pass it to `initializeApp`. Make sure the initialization is guarded so it only runs once (check if an app is already initialized before calling `initializeApp`). Export a `db` constant that gives you a Firestore instance ready to use.

**Encryption utility** — Create `src/lib/encryption.ts`. This file should initialize `cryptr` using a master secret pulled from an environment variable called `ENCRYPTION_SECRET`. Export two functions: `encryptKey(plaintext: string): string` and `decryptKey(ciphertext: string): string`. The raw key should never be logged anywhere in this file. Add a comment warning future developers not to log the plaintext value.

**Environment variables** — Add `FIREBASE_SERVICE_ACCOUNT_BASE64` and `ENCRYPTION_SECRET` to your `.env.local` file (and document them in `.env.example` without real values). The `ENCRYPTION_SECRET` should be a long random string — at least 32 characters. These must never be exposed to the client, so make sure neither variable is prefixed with `NEXT_PUBLIC_`.

Both utilities should only ever be imported in server-side code (API routes, Server Components, or server actions). Add a note at the top of each file making this clear.
