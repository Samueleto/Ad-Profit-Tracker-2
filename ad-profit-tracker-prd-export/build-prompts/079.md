We need the Firebase Admin SDK set up so our API routes can talk to Firestore and verify auth tokens.

Install `firebase-admin` if it isn't already in the project. Then create a singleton initializer at `src/lib/firebase-admin.ts` (or wherever the project keeps server-side Firebase setup). The key thing here is to initialize it only once — Next.js can hot-reload in development and you'll get errors if you try to initialize the app multiple times. Check `admin.apps.length > 0` before calling `initializeApp()`, and if it's already initialized, just export the existing app.

For the initialization config, use the `FIREBASE_PROJECT_ID` environment variable. In production on Cloud Run, the service account credentials are picked up automatically from the environment (Application Default Credentials), so you don't need to bundle a service account JSON file. In local development, you'll need to either set `GOOGLE_APPLICATION_CREDENTIALS` pointing to a downloaded service account key, or use the Firebase emulator.

Export three things from this module:
- The initialized `admin` object
- A `db` reference to `admin.firestore()` — we'll use this in the health check and deployment status endpoints
- A `auth` reference to `admin.auth()` — we'll use this to verify Firebase ID tokens in the deployment status endpoint

Also add a helper function `isFirebaseAdminInitialized()` that returns `admin.apps.length > 0` — the GET /api/ready endpoint needs this to confirm the SDK is ready before the instance accepts traffic.

Make sure `FIREBASE_PROJECT_ID` is documented in `.env.example` and add it to your local `.env.local` for development.
