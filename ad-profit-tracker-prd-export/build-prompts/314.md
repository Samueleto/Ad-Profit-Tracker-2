We need to connect the PerNetworkAnalyticsTabsSection and ApiExplorerTab into the existing FinancialMetricsDashboardPage and make sure everything talks to each other correctly.

**Placement** — Add PerNetworkAnalyticsTabsSection directly below the existing GeoBreakdownSection in FinancialMetricsDashboardPage. Don't create a new route — this is a section within the existing /dashboard page.

**Date range wiring** — The parent FinancialMetricsDashboardPage already has a shared date range state (dateFrom and dateTo). Pass these down as props to PerNetworkAnalyticsTabsSection. The section should not have its own date picker — it reads from the parent only. Make sure the section re-renders and re-fetches when the parent date range changes.

**GeoCountryDrilldownModal wiring** — When a user clicks a country row in any network's GEO table, it should open the existing GeoCountryDrilldownModal. The modal needs the country code and the current date range. Make sure the modal receives the correct country and date range context regardless of which network tab the user is on.

**Sync button wiring** — The 'Sync This Network' button in each tab calls `POST /api/sync/manual`. After a successful sync, the tab's data should refresh automatically (the `syncNetwork()` function from `useNetworkAnalytics` handles this). Show a loading spinner on the button while the sync is in progress and disable it to prevent double-clicks. Show a success toast when sync completes and an error toast if it fails.

**API Explorer tab placement** — The ApiExplorerTab should appear as the fifth tab in the same tab bar as the four network tabs inside PerNetworkAnalyticsTabsSection. It should not be a separate section — it lives inside the same tab bar component.

**Settings link** — The 'API key not configured' message in ApiExplorerTab links to /settings. Make sure this uses the app's existing router (Next.js Link component) so it's a client-side navigation, not a full page reload.

**Shared component reuse** — Double-check that MetricToggle, MetricShareBar, TopNSelector, and the collapsible insights strip are imported from wherever GeoBreakdownSection defines them, not re-implemented. If they're not already exported from GeoBreakdownSection's file, refactor them into a shared components location so both sections can import them.

**Tab persistence** — When the user navigates away from the dashboard and comes back, the active tab should reset to ExoClick (the default). Don't persist tab state to the URL or localStorage — keep it ephemeral.
