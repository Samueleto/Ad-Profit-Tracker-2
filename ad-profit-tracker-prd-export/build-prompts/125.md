We need TypeScript interfaces for the four Firestore collections this feature touches: `adStats`, `apiKeys`, `networkConfigs`, and `auditLogs`. Create these at `src/lib/propush/types.ts` (or a shared `src/types/firestore.ts` if the project already has one).

**`AdStatDocument`** — Represents one row of synced publisher statistics. Fields: `id` (string), `userId` (string), `networkId` (string, will be `'propush'`), `date` (string, YYYY-MM-DD), `geo` (string, country code), `revenue` (number), `impressions` (number, optional), `clicks` (number, optional), `cost` (null — Propush doesn't provide cost data), `rawResponse` (Record<string, unknown> — the sanitized raw API response stored for the API Explorer), `createdAt` (Firestore Timestamp), `updatedAt` (Firestore Timestamp).

**`ApiKeyDocument`** — Stores encrypted credentials. Fields: `id` (string), `userId` (string), `networkId` (string), `encryptedKey` (string), `createdAt` (Firestore Timestamp), `updatedAt` (Firestore Timestamp).

**`NetworkConfigDocument`** — Tracks per-user network configuration and sync state. Fields: `id` (string), `userId` (string), `networkId` (string), `isActive` (boolean), `syncSchedule` (string, e.g., `'daily'`), `lastSyncedAt` (Firestore Timestamp or null), `lastSyncStatus` (`'success' | 'failed' | 'pending'` or null), `createdAt` (Firestore Timestamp), `updatedAt` (Firestore Timestamp).

**`AuditLogDocument`** — Records sync attempts for accountability. Fields: `id` (string), `userId` (string), `networkId` (string), `action` (string, e.g., `'sync'`), `status` (`'success' | 'failed'`), `dateFrom` (string), `dateTo` (string), `recordsUpserted` (number, optional), `errorMessage` (string, optional), `createdAt` (Firestore Timestamp).

Also define the shape of the Propush API v5 response — create a `PropushApiResponse` type and a `PropushStatRow` type that matches what `/v5/pub-statistics` returns (geo, revenue, and any other fields). This will make the API client and sync logic type-safe.

Finally, define the request/response shapes for each of the six endpoints as TypeScript interfaces — this makes the route handlers much easier to write correctly.
