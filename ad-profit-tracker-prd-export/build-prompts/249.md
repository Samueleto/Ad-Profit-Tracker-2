We're building a Help & Documentation Center for a Next.js + Firebase app. For this step, build the two pages that make up the help system: HelpCenterPage at /help and HelpArticlePage at /help/[articleId].

**HelpCenterPage (/help)**

This is the main knowledge base landing page. At the top, render a sticky search bar that fires GET /api/help/search on a 300ms debounce as the user types (minimum 1 character). Search results should render inline below the search bar, replacing the normal article list while a query is active. Each result row should show the title, category badge, read time, and which field matched (title/summary/tags).

Below the search bar, show a category grid with 6 cards in a 3-column layout: FAQs, API Guides, Troubleshooting, Video Tutorials, Network Setup, and Billing & Account. Each card shows an icon, label, and an article count badge pulled from the `categoryGroups` array returned by GET /api/help/articles. Clicking a card filters the article list below it.

The article list renders as scannable rows — each row shows the article title, category badge, estimated read time, and a 'New' badge if the article was published within the last 7 days (compare `updatedAt` to today minus 7 days). Clicking a row navigates to /help/[articleId].

On the right sidebar (desktop only, 1/3 width), show a 'Popular Articles' panel with the top 5 articles sorted by viewCount — you can derive this from the articles returned by GET /api/help/articles. Below that, show a 'Role-Specific Guides' section. On mount, call GET /api/rbac/my-permissions to get the user's workspaceRole, then call GET /api/help/articles with the matching role filter to surface relevant articles (e.g., owners see 'Managing Team Members', members see 'Viewer Guide').

At the bottom of the page, add a 'Contact Support' CTA that opens a mailto link.

On mobile, collapse the category grid into a horizontal scrollable pill row. Use a single-column layout — sidebar content moves below the article list.

Handle these states: loading shows 6 shimmer cards in the category grid and 8 shimmer rows in the article list; empty (no results for a search or category) shows 'No articles found. Try a different search or browse categories.' with a 'Clear Search' button; error_404 for an empty category shows 'No articles found for this category' with a 'Browse All' button; error_500 shows an inline error banner with a Retry button that re-calls GET /api/help/articles.

**HelpArticlePage (/help/[articleId])**

This is the full article reader. On mount, call GET /api/help/articles/:articleId to load the article, and fire PATCH /api/help/articles/:articleId/view as a fire-and-forget call (don't await it or show any UI based on its response).

At the top, render a breadcrumb trail: Help > [Category] > [Article Title], where 'Help' links to /help and the category links to /help with that category pre-selected. Add a 'Back to Help Center' link at the top-left and a 'Copy Link' button that copies the current URL to the clipboard.

Below the breadcrumb, show the article header: title, category badge, author name, last-updated date, and estimated read time.

Render the article body using react-markdown with the remark-gfm plugin (for tables and code blocks). Apply Tailwind's prose classes for consistent typography — headings, code blocks, and blockquotes should all look polished. For articles with a videoUrl, embed a responsive iframe or HTML5 video player.

On desktop, show a sticky table-of-contents sidebar (1/3 width) that auto-generates from H2 and H3 headings in the markdown. Highlight the active section as the user scrolls using an IntersectionObserver. On mobile, collapse the TOC into a 'Jump to section' dropdown.

At the bottom, render a 'Was this helpful?' widget with thumbs-up and thumbs-down buttons. When the user clicks either, call POST /api/help/feedback with the articleId and rating ('helpful' or 'not_helpful'). Show a thank-you message after submission and disable the buttons so they can't submit twice.

Below the feedback widget, show a 'Related Articles' section with up to 4 articles from the same category, fetched from GET /api/help/articles?category=[category]&limit=4.

Handle these states: loading shows shimmer blocks for the title, meta row, body paragraphs, and TOC; error_403 shows 'Access Denied — this article requires elevated permissions' with a link back to /help; error_404 shows 'Article not found' with a link to /help and a 'Browse all articles' CTA; error_500 shows an inline error banner with a Retry button.
