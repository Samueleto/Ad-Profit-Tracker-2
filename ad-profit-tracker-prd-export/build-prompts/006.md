On the backend side, create a folder for the audit API routes. We'll need four route handlers:
- `POST /api/audit/log` — for writing new log entries
- `GET /api/audit/logs` — for fetching paginated, filtered logs
- `GET /api/audit/logs/export` — for CSV export
- `DELETE /api/audit/logs/clear` — for purging logs

Also create a shared utility file at something like `src/lib/auditLogger.ts` — this will be a helper that other endpoints in the app call to write audit log entries without duplicating the fetch logic everywhere.
