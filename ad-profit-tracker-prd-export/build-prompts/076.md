We're using Firebase on both the client (firebase/auth and firebase/firestore) and the server (firebase-admin). The client SDK is likely already set up for the existing app, but we need to make sure the Firebase Admin SDK is properly initialized for the new team API routes — this is what lets our server-side code verify ID tokens and read/write Firestore securely.

First, check if there's already a firebase-admin initialization file in the project (often at lib/firebase-admin.ts or similar). If there is, we can reuse it. If not, create one now. It should initialize the Admin SDK using a service account — store the service account credentials as environment variables (FIREBASE_PROJECT_ID, FIREBASE_CLIENT_EMAIL, FIREBASE_PRIVATE_KEY). Use a singleton pattern so the app doesn't re-initialize on every request in development (Next.js hot reloads can cause this).

Export two things from this file: the initialized admin app, and a Firestore admin instance (admin.firestore()). All our team API routes will import from here.

Also create a shared token verification helper at lib/team/auth.ts. It should export a verifyBearerToken(request) function that: reads the Authorization header, strips the 'Bearer ' prefix, calls admin.auth().verifyIdToken(token), and returns the decoded token (which includes the uid). If the header is missing or the token is invalid/expired, it should throw a typed error that the route handlers can catch and return a 401 response.

Finally, make sure the client-side Firebase packages (firebase/auth and firebase/firestore) are installed — these are used by the frontend pages. If they're already in package.json, no action needed.
