We're building an ad profit tracker app. For this step, I need you to build the OnboardingWizardPage at the route /onboarding — a four-step guided wizard that new users see exactly once after their first login.

The overall layout is a centered card (max-w-lg) with no sidebar or nav frame — just the card on a clean background. At the top of the card, show a progress bar and numbered step dots (e.g. '1 of 4'). Each step swaps in and out with a subtle slide transition. The Back button is ghost-styled, the primary Next/action button is filled. Skip links are small and visually de-emphasized.

**Step 1 — Welcome:** Pull the user's displayName from Firebase Auth and greet them by name. Show a short value proposition for Ad Profit Tracker and a 'Get Started' CTA that advances to Step 2. While the user profile is loading from GET /api/auth/get-user, show a skeleton shimmer inside the card body.

**Step 2 — Profile Setup:** Use react-hook-form to manage a form with three fields: a displayName text input (pre-filled from the user's current displayName, editable), a timezone dropdown populated with IANA timezone strings, and a defaultDateRange selector with options 'Last 7 days' (value: last7days), 'Last 30 days' (last30days), and 'This month' (thisMonth). When the user taps Next, submit the form via PATCH /api/auth/update-profile with the displayName and preferences (timezone and defaultDateRange). While submitting, show a spinner on the Next button and disable all fields.

**Step 3 — API Keys Teaser:** A purely informational step. Explain that API keys for ExoClick, RollerAds, Zeydoo, and Propush are needed to pull data. Show a 'Go to Settings' button that routes to /settings, and a smaller 'Skip for now' link. Both options should advance the wizard — 'Skip for now' should also set onboardingSkipped: true when writing the completion record.

**Step 4 — Done:** Show an animated checkmark (CSS or a lightweight animation), a 'Your dashboard is ready' message, and a 'Go to Dashboard' button that routes to /dashboard. When this step is reached (either via 'Go to Settings' on Step 3 or arriving at Step 4), call PATCH /api/auth/update-profile with onboardingCompletedAt: 'serverTimestamp' and, if the user skipped, onboardingSkipped: true.

Make sure the Back button works between steps (Step 1 has no Back). The progress bar and step dots update as the user moves forward and backward.
