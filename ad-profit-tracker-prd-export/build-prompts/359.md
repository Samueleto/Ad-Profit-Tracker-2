We're making sure the POST /api/export/pdf endpoint is properly secured. Most of this is already baked into the endpoint logic, but let's explicitly verify and tighten each security layer.

Authentication: Every request to POST /api/export/pdf must include a valid Firebase ID token in the Authorization header as a Bearer token. The endpoint must call admin.auth().verifyIdToken(token) and extract the uid from the verified result. If the token is missing, malformed, or expired, return 401 immediately — before touching Firestore or doing any computation. Never trust a uid sent in the request body.

Data scoping: Every Firestore query in this endpoint — both the adStats collection and the auditLogs collection — must filter by userId == uid where uid comes from the verified token. There should be no code path where a user can request data belonging to another user. Double-check that the cache key also includes the uid: '{uid}_export_preview_{dateFrom}_{dateTo}' — a cache hit for one user must never serve data to a different user.

Filename sanitization: If the client provides a filename, validate it server-side against the pattern /^[a-zA-Z0-9_-]{1,100}$/ before using it in the Content-Disposition header. Never interpolate an unsanitized filename into a header — this could allow header injection.

Rate limiting: The 10 exports/hour budget is shared between POST /api/export/pdf and POST /api/export/excel. Make sure the rate limit counter uses a key scoped to the uid (e.g. '{uid}_export_rate_limit') and that both endpoints increment and check the same counter. Return 429 with a clear message if the limit is hit.

No sensitive data in exports: The PDF must never include Firebase API keys, service account credentials, other users' data, or any internal system metadata. The audit log entry written after export should be scoped to the exporting user's auditLogs document.

On the frontend side, the PdfExportTab must call getIdToken(true) to get a fresh token before each export request (the true flag forces a refresh if the token is close to expiring). The token must only be sent in the Authorization header — never in the URL or request body.
