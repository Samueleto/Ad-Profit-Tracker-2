We need TypeScript types and Firestore helper functions for the `adStats` collection so all six endpoints work with consistent, type-safe data.

**Types** — Create `src/lib/firestore/types.ts` (or add to an existing types file). Define an `AdStat` interface with all fields from the adStats schema: `id: string`, `userId: string`, `networkId: string`, `date: string`, `country: string | null`, `cost: number | null`, `revenue: number | null`, `impressions: number | null`, `clicks: number | null`, `ctr: number | null`, `cpm: number | null`, `rawResponse: Record<string, unknown>`, `syncedAt: FirebaseFirestore.Timestamp`, `createdAt: FirebaseFirestore.Timestamp`. Make optional fields explicit with `| null` rather than `?` — this makes it clear these fields exist but may be absent for certain network types.

Also define a `AdStatUpsertPayload` type that omits `id`, `createdAt`, and `userId` (those are set by the server, not the caller) and makes all nullable fields optional. This is what the sync logic will pass when writing records.

**Firestore collection helper** — Create `src/lib/firestore/adStats.ts`. Export:
- `getAdStatsCollection(db: Firestore)` — returns a typed collection reference
- `upsertAdStatBatch(db: Firestore, userId: string, records: AdStatUpsertPayload[])` — takes an array of records and writes them to Firestore in batches of 500 (Firestore's batch write limit). For each record, generate a deterministic document ID from `userId + networkId + date + country` (e.g., using a hash or a concatenated string like `${userId}_exoclick_${date}_${country ?? 'all'}`). This makes upserts idempotent — re-syncing the same date won't create duplicates. Set `syncedAt` to `FieldValue.serverTimestamp()` on every write. Set `createdAt` to `FieldValue.serverTimestamp()` only on create (use `{ merge: true }` with explicit field handling, or check existence first).

The deterministic document ID approach is important — it means `POST /api/networks/exoclick/sync` and `POST /api/networks/exoclick/scheduled-sync` can both call `upsertAdStatBatch` safely without worrying about duplicates.
