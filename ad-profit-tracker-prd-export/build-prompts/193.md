We need several reusable pieces that the OnboardingWizardPage will be built from. Build these components before the page so the page can import them by name.

First, build an OnboardingLayout wrapper component at src/features/onboarding/components/OnboardingLayout.tsx. This renders a centered card (max-w-lg, centered both horizontally and vertically on the page) with a clean white background and subtle shadow. It accepts a children prop and renders them inside the card. No sidebar, no nav — just the card on a neutral background.

Second, build a StepProgressBar component at src/features/onboarding/components/StepProgressBar.tsx. It takes two props: currentStep (1-4) and totalSteps (4). It renders a row of four numbered dots at the top of the card. The current step dot is filled/highlighted; completed steps are filled with a muted color; upcoming steps are outlined. Below the dots, show a text label like 'Step 2 of 4'. Keep it simple and clean.

Third, build four step content components — one per wizard step — each as a self-contained component that receives whatever props it needs:
- WelcomeStep (src/features/onboarding/components/WelcomeStep.tsx): accepts a displayName string prop. Shows a greeting using the name, a short value proposition blurb for Ad Profit Tracker, and a 'Get Started' button that triggers an onNext callback prop.
- ProfileSetupStep (src/features/onboarding/components/ProfileSetupStep.tsx): accepts defaultValues (OnboardingFormValues) and onSubmit callback. Uses react-hook-form internally. Renders a displayName text input (pre-filled, required), a timezone dropdown (a select element populated with a reasonable list of common IANA timezone strings — at minimum cover US, EU, and Asia/Pacific zones), and a defaultDateRange select with options 'Last 7 days', 'Last 30 days', 'This month' mapping to 'last7days', 'last30days', 'thisMonth'. Shows a spinner on the submit button and disables fields when an isSubmitting prop is true.
- ApiKeysTeaserStep (src/features/onboarding/components/ApiKeysTeaserStep.tsx): purely informational. Explains that API keys for ExoClick, RollerAds, Zeydoo, and Propush are needed to pull data. Has a 'Go to Settings' button (onGoToSettings callback) and a small de-emphasized 'Skip for now' link (onSkip callback).
- DoneStep (src/features/onboarding/components/DoneStep.tsx): shows an animated checkmark (CSS animation is fine — no heavy library needed), a 'Your dashboard is ready' heading, and a 'Go to Dashboard' filled button (onGoToDashboard callback).

All step components should use the OnboardingFormValues and UpdateProfilePayload types from the types file we created earlier.
