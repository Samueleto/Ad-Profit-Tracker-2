The ThemeSettingsSection makes two API calls — GET /api/settings/preferences and PATCH /api/settings/preferences — and also applies user-provided values directly to the DOM as CSS. Both of these need to be handled carefully.

**API authentication** — Every call to GET and PATCH /api/settings/preferences must include the Firebase ID token as a Bearer token in the Authorization header. Use `getIdToken()` from the Firebase Auth SDK to get the current token before each request. Don't cache the token in a variable for reuse — always call `getIdToken()` fresh (or use `getIdToken(true)` to force a refresh if needed). The server-side middleware on these endpoints already validates the token and extracts the userId — the client just needs to make sure it always sends it.

**Token expiry handling** — If a 401 comes back from either endpoint, try `getIdToken(true)` to force a token refresh and retry the request once. If it fails again after the refresh, that means the session is truly expired — redirect the user to `/` and show a 'Session expired' toast. Don't retry in a loop.

**CSS injection prevention** — The accent color gets applied directly as a CSS custom property on the document root. This is safe for hex values, but you need to validate the input before applying it. Only apply the value if it matches a strict hex color pattern: `#` followed by exactly 3 or 6 hexadecimal characters. Reject anything else silently (don't apply it, don't crash). The 8 preset swatches are hardcoded so they're always safe — only the custom hex input needs this validation.

**Value sanitization on save** — Before sending the PATCH request, validate that `themePreference` is one of the three allowed values (`'light'`, `'dark'`, `'system'`) and that `accentColor` passes the hex validation. Don't send the request if either value is invalid. This is a defense-in-depth measure — the server should also validate, but the client shouldn't send garbage.

**403 and 404 handling** — If the GET returns 403, show an 'Access Denied' message inline within the ThemeSettingsSection (not a redirect). If it returns 404, show the default controls (Light theme, indigo accent) with a small note saying 'Settings not found — saving will create them.'
