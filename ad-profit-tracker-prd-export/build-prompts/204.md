We need several UI components that the ComparativeNetworkAnalysisTab will assemble together. Build these as standalone, importable pieces before wiring up the full tab.

NetworkComparisonCard — A card that displays one network's stats. Takes a NetworkComparisonItem as a prop. Shows: the network name and a color-coded header (amber for ExoClick, green for the revenue networks), the primaryMetric prominently labeled as either 'Cost' or 'Revenue' based on dataRole, then compact stat rows for impressions, clicks, CTR%, and CPM. Below the stats, include a MetricShareBar — a horizontal bar showing this network's proportional share of the total (the metricShare value as a percentage). At the bottom, show a small status badge: green dot for lastSyncStatus 'synced', red dot for 'failed', gray dot for 'never'. Null/missing values render as '—'. The whole card should be clickable (fires an onNetworkClick callback with the networkId). Use Tailwind classes: amber-500 for ExoClick, green-500 for revenue networks, red-500 for negative values.

ComparisonBarChart — A recharts BarChart that shows all four networks side-by-side for the selected metric. Takes the networks array and the current metric as props. Renders grouped bars, color-coded (amber for ExoClick, green for others). Height is fixed at 280px. Axes should be labeled. Tooltips should show the formatted metric value. Must have 'use client' at the top. Handle the case where all values are zero gracefully.

EfficiencyTable — Shows the cross-network derived metrics from the crossNetwork object: overall ROI%, revenue-per-impression, and cost-per-click. Takes a CrossNetworkMetrics prop. Uses alternating row shading. Null values render as '—'. The whole section is collapsed by default with a chevron toggle (progressive disclosure).

NetworkRankingStrip — A horizontal strip showing all four networks ranked by the selected metric. Takes the rankings array as a prop. Each network gets an ordinal badge: 1st gets a gold medal style, 2nd silver, 3rd bronze, 4th a plain badge. Shows the network name and metric value next to each badge. Tied networks share the same rank badge.

All four components should be typed using the interfaces from the comparison types file.
