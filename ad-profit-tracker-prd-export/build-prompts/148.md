We need TypeScript types for the RBAC data structures and a central place to define the system default permissions. These will be used by all four API endpoints and the frontend components.

In src/features/rbac/types/, create a permissions.ts file with the following types:
- A WorkspaceRole type: union of 'owner' | 'admin' | 'member' | 'viewer' | 'analyst'
- A PermissionKey type: union of all 9 permission keys — 'canViewDashboard' | 'canExportData' | 'canManageApiKeys' | 'canSyncNetworks' | 'canViewReports' | 'canCreateReports' | 'canManageTeam' | 'canViewAuditLogs' | 'canManageBenchmarks'
- A PermissionMap type: Record<PermissionKey, boolean>
- A RolePermissionsMatrix type: Record<WorkspaceRole, PermissionMap>
- A WorkspacePermissionsDoc interface matching the workspacePermissions Firestore schema: id, workspaceId, rolePermissions (partial — only non-owner roles stored), customizedRoles, createdAt, updatedAt, updatedByUid
- A UserRbacFields interface for the RBAC fields on the users collection: workspaceRole, effectivePermissions (Partial<PermissionMap>), permissionsUpdatedAt
- Response types for each API endpoint: PermissionsMatrixResponse (for GET /api/rbac/permissions), MyPermissionsResponse (for GET /api/rbac/my-permissions), PermissionCheckResponse (for POST /api/rbac/check)

In src/lib/rbac/, create a systemDefaults.ts file that exports the SYSTEM_DEFAULTS constant — a RolePermissionsMatrix object with the default permission values for each role. Based on the spec: owner gets all true (computed inline, but include it for completeness); admin gets all 9 permissions true; member gets canViewDashboard, canViewReports, canSyncNetworks true and the rest false; viewer gets only canViewDashboard and canViewReports true; analyst gets canViewDashboard, canViewReports, canCreateReports, canManageBenchmarks true and the rest false.

Also export a PERMISSION_KEYS array (all 9 keys) and an EDITABLE_ROLES array (['admin', 'member', 'viewer', 'analyst']) — these are used for validation in the API endpoints. Export a mergeWithDefaults(rolePermissions: Partial<RolePermissionsMatrix>) helper function that takes a stored rolePermissions map and fills in any missing permission keys with system default values, returning a complete RolePermissionsMatrix.
