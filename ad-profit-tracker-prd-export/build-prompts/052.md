We need a few SDKs in place before building the manual refresh feature. Some of these may already be set up in the project — check first and only add what's missing.

**Firebase Admin SDK** — This is used server-side in the `POST /api/sync/manual` endpoint to verify ID tokens (`admin.auth().verifyIdToken()`), read from Firestore (`networkConfigs`, `apiKeys`, `adStats`, `auditLogs` collections), and write batch updates. If the project already initializes Firebase Admin somewhere (like `lib/firebase-admin.ts`), reuse that singleton — don't initialize it again. If it's missing, set it up with the service account credentials from environment variables (`FIREBASE_PROJECT_ID`, `FIREBASE_CLIENT_EMAIL`, `FIREBASE_PRIVATE_KEY`). Make sure the Admin SDK is only ever imported in server-side files — never in client components.

**Firebase Auth (client-side)** — The `firebase/auth` package is used in the ManualRefreshPanel to call `getIdToken()` (and `getIdToken(true)` for forced refresh on 401). If the project already has a Firebase client config initialized, just confirm `getIdToken` is accessible from the auth instance. If not, set up the Firebase client app with the standard env vars (`NEXT_PUBLIC_FIREBASE_*`).

**axios** — Used inside `syncNetworkForUser` to call the four ad network APIs (ExoClick, RollerAds, Zeydoo, Propush). Install if not already present.

**cryptr** — Used server-side only to decrypt stored API keys before passing them to the network APIs. Install and set up with a `CRYPTR_SECRET_KEY` environment variable. This key must never be exposed to the client.

Double-check that all sensitive env vars (`FIREBASE_PRIVATE_KEY`, `CRYPTR_SECRET_KEY`) are in `.env.local` and listed in `.env.example` with placeholder values so other developers know they're needed.
