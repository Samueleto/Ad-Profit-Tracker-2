We're building the FilterToolbar — a persistent secondary filter bar that sits directly below the existing DateRangeToolbar on the FinancialMetricsDashboardPage. It needs to match the DateRangeToolbar's height and visual style. Build this as a collection of composable sub-components that the FilterToolbar assembles.

**NetworkCheckboxGroup** — A compact inline group of four checkboxes: ExoClick, RollerAds, Zeydoo, Propush. Include an 'All Networks' toggle that selects/deselects all four at once. When all four are checked, the state should be equivalent to 'All Networks'. This is a controlled component — it receives `selectedNetworks: string[]` and an `onChange` callback. No API call needed — the network list comes from the static networks config.

**CountryCombobox** — A type-ahead combobox for selecting countries. It receives a list of `CountryOption` objects (from GET /api/filters/options?type=country), a loading state, an error state, and `selectedCountries: string[]` plus an `onChange` callback. While loading, show 'Loading countries...' as placeholder text with a spinner in the dropdown. If the options list is empty, show 'No countries found for this date range'. On error, show an amber warning icon with 'Could not load countries' and a retry link. Supports multi-select — selected countries appear as removable chips inside the input. Filters the dropdown list as the user types (client-side filtering of the already-loaded options list, not a new API call on each keystroke).

**MetricPillGroup** — A segmented control / pill group for single-selecting the primary metric. Options: Revenue, Cost, Profit, ROI. Receives `selectedMetric: MetricKey` and `onChange`. Applies immediately on click (no 'Apply' button needed). Style as pill-shaped buttons where the active one is visually highlighted.

**DataQualityToggle** — A three-state toggle: All / Anomalies Only / Clean Only. Receives `dataQuality: DataQualityOption` and `onChange`. Applies immediately on click.

**FilterSearchInput** — A free-text search input at the far right of the toolbar. Receives `searchQuery: string` and `onChange`. Debounce the onChange call by 300ms so it doesn't fire on every keystroke. Filters are applied client-side against visible table rows (country name or network name). Show a small search icon inside the input.

**FilterToolbar** — The main component that assembles all the above. It should:
- Render as a sticky single-row bar directly below the DateRangeToolbar
- Show NetworkCheckboxGroup, CountryCombobox, MetricPillGroup, DataQualityToggle, and FilterSearchInput in a horizontal layout
- Include an 'Apply Filters' button that commits multi-select changes (networks and countries) to the Zustand store — single-select controls (metric, dataQuality) and the search input apply immediately
- Include a 'Clear All' button that resets all filter state to DEFAULT_FILTER_STATE and writes that to the Zustand store
- Show an active filter count badge (amber dot) on a 'Filters' label when any filter is non-default — count the number of active non-default filters
- On mobile (below md breakpoint), collapse all controls into a single 'Filters' button that opens a bottom sheet containing the full filter panel with all the same controls in a vertical stacked layout
- Call GET /api/filters/options?type=country (with the current dateFrom/dateTo from the existing Zustand store) when the component mounts, and re-fetch when the date range changes
- Read and write filter state from/to the Zustand dashboard store (the selectedNetworks, selectedCountries, selectedMetric, dataQuality, searchQuery fields)

Make sure the FilterToolbar is exported as the default export so it can be dropped into the FinancialMetricsDashboardPage.
