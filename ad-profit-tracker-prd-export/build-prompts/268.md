We're building an ad network sync dashboard. For this step, I need you to set up the data-fetching and state management layer that the error monitoring panels will use.

Since this is a Next.js app, use React Query (or SWR if that's already in the project) for server state — it handles caching, background refresh, and loading/error states cleanly for these read-heavy panels.

**Circuit Breaker State** — Create a `useCircuitBreakerStatus` hook that fetches from `GET /api/errors/circuit-breaker/status`. It should auto-refresh every 30 seconds (these statuses can change when background syncs run). Expose the `summary`, `circuits` array, loading state, error state, and a `refetch` function. Also create a `useResetCircuitBreaker` mutation hook that POSTs to `POST /api/errors/circuit-breaker/reset` with a `networkId`. On success, it should invalidate the circuit breaker status query so the panel refreshes automatically.

**Retry State** — Create a `useRetryState` hook that fetches from `GET /api/errors/retry-state`. Accept an optional `networkId` parameter — when provided, pass it as a query param. Cache for 15 seconds (retry state changes frequently during active sync cycles). Expose the `networks` array, loading, and error.

**Error Log State** — Create a `useErrorLog` hook that fetches from `GET /api/errors/log`. Accept filter params: `networkId`, `errorCode`, `startDate`, `endDate`, `limit`, and `cursor`. When any filter changes, reset to the first page. Support infinite loading — each 'load more' appends the next page using the `nextCursor` from the previous response. Expose the flat list of all loaded errors, `hasMore`, `loadMore` function, loading states for initial load vs. loading more, and error state.

**Error Summary State** — Create a `useErrorSummary` hook that fetches from `GET /api/errors/summary`. Accept `dateFrom` and `dateTo` as parameters (default to last 7 days). When the date range changes, re-fetch. Cache for 60 seconds — this is an aggregation query and doesn't need to be real-time.

**Retry Config State** — Create a `useRetryConfig` hook that fetches from `GET /api/errors/retry-config`. Also create a `useUpdateRetryConfig` mutation hook that PATCHes `PATCH /api/errors/retry-config`. On success, invalidate the retry config query and also invalidate the retry state query (since `retryAttempts` affects the backoff schedule shown there).

All hooks should read the Firebase ID token from the existing auth context and attach it as a `Bearer` token in the `Authorization` header on every request. If any request returns 401, redirect to the login page. Store filter state for the error log (network, error code, date range) in URL search params so users can share or bookmark filtered views.
