We need clear TypeScript types so the API responses, component props, and hook return values all agree on the shape of the data. Create these in the `features/sync-status/types/` folder.

**NetworkSyncState** — represents one network's live status as returned by `GET /api/sync/live-state`. Fields: `networkId` (one of `'exoclick' | 'rollerads' | 'zeydoo' | 'propush'`), `isActive: boolean`, `syncPhase: 'idle' | 'fetching' | 'writing' | 'complete' | 'failed'`, `lastSyncStatus: string`, `lastSyncedAt: string | null` (ISO timestamp), `lastSyncError: string | null`, `latestSyncedAt: string | null`, `latestDataDate: string | null`, `lastRowsFetched: number | null`, `lastLatencyMs: number | null`, `circuitBreakerOpen: boolean`, `circuitBreakerOpenedAt: string | null`, `lastErrorCode: string | null`, `retryCount: number`, `totalFailureCount: number`, `timeUntilNextScheduledSync: number | null`.

**LiveStateResponse** — the full response shape from `GET /api/sync/live-state`: `overallHealth: 'healthy' | 'degraded' | 'critical'`, `networks: NetworkSyncState[]`, `polledAt: string`.

**ActivityFeedEntry** — one item from `GET /api/sync/activity-feed`: `id: string`, `networkId: string`, `status: 'success' | 'failure'`, `eventLabel: 'Scheduled Sync' | 'Manual Sync' | 'Backfill' | 'Retry'`, `rowsFetched: number | null`, `latencyMs: number | null`, `dateFrom: string | null`, `dateTo: string | null`, `createdAt: string`.

**ActivityFeedResponse** — the paginated wrapper: `feed: ActivityFeedEntry[]`, `hasMore: boolean`, `nextCursor: string | null`.

**SyncProgressUpdate** — the request body for `PATCH /api/sync/progress`: `networkId: string`, `phase: 'fetching' | 'writing' | 'complete' | 'idle'`, `rowsFetched?: number`, `totalRows?: number`, `errorMessage?: string`.

Also add a small utility type `OverallHealth = 'healthy' | 'degraded' | 'critical'` and a `NetworkId = 'exoclick' | 'rollerads' | 'zeydoo' | 'propush'` union type — these get reused across components and API routes.

Finally, add a few pure utility functions in `features/sync-status/utils/`: one that formats a timestamp as a relative string ('3 minutes ago'), one that maps a `syncPhase` value to its display label and semantic color class, and one that maps `overallHealth` to its banner text and color.
