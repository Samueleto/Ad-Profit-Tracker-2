The API Explorer reads from two existing Firestore collections — `apiKeys` and `networkConfigs` — and we'll also need a place to cache the most recent raw API response for each network so the 'Last fetched' timestamp and cached JSON viewer work correctly.

First, confirm the `apiKeys` collection structure. Each document should be keyed by network name (e.g., `exoclick`, `rollerads`, `zeydoo`, `propush`) and contain at minimum an `apiKey` field (and any other auth credentials that network requires). These are read server-side only — never sent to the client.

Second, confirm the `networkConfigs` collection. Each document (again keyed by network name) should store the endpoint URL, HTTP method, and required parameters for that network's API. This is what the API Explorer displays in the reference panel.

Third, add a `cachedResponses` subcollection or top-level collection to store the most recent raw API response per network. Each document should have: `networkId` (string), `rawResponse` (the full JSON payload as a map), `fetchedAt` (timestamp), and `httpStatus` (number). This lets the API Explorer show the last fetched response without hitting the ad network on every page load.

Set up Firestore security rules so that `apiKeys` is only readable server-side (no client reads allowed — admin SDK only). `networkConfigs` and `cachedResponses` should be readable by authenticated users but not writable from the client.
