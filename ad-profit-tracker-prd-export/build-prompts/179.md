We need five API route handlers for the report builder. All of them share the same auth pattern: read the `Authorization: Bearer <token>` header, call `adminAuth.verifyIdToken(token)` from the Firebase Admin SDK, and return 401 immediately if it fails. The `uid` always comes from the verified token — never from the request body. Import `adminDb` and `adminAuth` from the shared Firebase admin file, `nodeCache` from `src/lib/reportCache.ts`, and `validateReportConfig` from `src/lib/reportValidation.ts`.

**POST /api/reports/save** (`src/app/api/reports/save/route.ts`) — After auth, validate the request body using `validateReportConfig()` plus the name field (1-100 chars). Read the user's `users` Firestore document and check that `savedReports.length < 20` — return 400 with 'Maximum 20 saved reports allowed' if at the limit. Generate a UUID v4 for the new report's `id`. Use `adminDb` with `arrayUnion` to append the new report object to `users.savedReports`. Also write an entry to the `auditLogs` Firestore collection with `action: 'preferences_updated'`, `resourceType: 'report'`, and the report metadata. Return 201 with the created report object. Rate limit: 20 saves/hour per user.

**GET /api/reports** (`src/app/api/reports/route.ts`) — After auth, read the user's `users` document with `adminDb`. Extract `savedReports`, sort by `createdAt` descending, and return `{ reports, total }`. If `savedReports` is empty or missing, return `{ reports: [], total: 0 }` — not a 404. Return 404 only if the `users` document itself doesn't exist.

**PATCH /api/reports/[id]** and **DELETE /api/reports/[id]** (both in `src/app/api/reports/[id]/route.ts`) — For PATCH: validate the `:id` param and at least one body field, read the users document, find the matching report by id (404 if not found), merge the changes, then use a Firestore transaction with `arrayRemove` (old object) + `arrayUnion` (updated object) to replace it atomically. Set `updatedAt` to the current timestamp. Write an audit log. Rate limit: 30 updates/hour. For DELETE: find the report by id (404 if not found), use `arrayRemove` to pull it out, write an audit log with `action: 'deleted'`. Rate limit: 20 deletes/hour. Return `{ deleted: true, reportId }` on success.

**POST /api/reports/run** (`src/app/api/reports/run/route.ts`) — This is the most complex one. After auth, if `reportId` is provided, load the saved config from the user's `savedReports` array (404 if not found) and merge with the request's `dateFrom`/`dateTo`. Validate the full config with `validateReportConfig()`. Check `nodeCache` for a key like `{uid}_report_run_{hash}` — if hit, return cached result with `X-Cache: HIT` header. Query the `adStats` Firestore collection filtering by `userId == uid`, `date >= dateFrom`, `date <= dateTo`, and optionally `networkId IN networks` (up to 4 values, safe for Firestore's `in` operator). Apply in-memory filters for `dataQuality` and countries arrays larger than 10. Group and aggregate the results by the `groupBy` dimension, computing only the metrics listed in the config. Sort by `netProfit` descending, apply cursor-based pagination (default limit 50, max 100), and compute summary totals. Cache the result for 5 minutes. Write an audit log with `action: 'custom_report_generated'`. Return the `ReportRunResponse` shape. Rate limit: 30 runs/hour.
