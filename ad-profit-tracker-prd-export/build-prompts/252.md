We need a dedicated state manager for the ManualRefreshPanel so the component itself stays clean and all the sync/polling logic lives in one place.

Create a custom hook (something like `useManualRefresh`) that the ManualRefreshPanel imports. Here's what it needs to manage:

**Sync status state** — Track the current sync status for each of the four networks (ExoClick, RollerAds, Zeydoo, Propush). For each network, store: lastSyncedAt (timestamp or null), lastSyncStatus ('success' | 'failed' | 'never' | 'in_progress'), and lastSyncError (string or null). On mount, fetch GET /api/scheduled/sync-status to populate this initial state. While that first fetch is in progress, expose an `isInitialLoading` flag so the panel can show skeleton rows.

**Triggered networks tracking** — Keep a set of networkIds that are currently in-progress (triggered but not yet terminal). This drives which rows show the amber spinner and which Refresh buttons are disabled. When a POST /api/sync/manual call succeeds, add the triggered networkIds to this set. When polling detects a terminal status for a network, remove it from the set.

**Polling logic** — After any network is added to the triggered set, start a polling interval that calls GET /api/scheduled/sync-status every 3 seconds. On each poll response, update the per-network status state and remove any networks from the triggered set that now show a terminal status. Stop polling when the triggered set is empty OR when 60 seconds have elapsed since polling started — whichever comes first. Make sure to clean up the interval on component unmount so there are no memory leaks.

**Rate limit state** — For each button (the 'Refresh All' button and each per-network Refresh button), track whether it's rate limited and when the rate limit expires. When a POST /api/sync/manual call returns 429, read the Retry-After header (value in seconds), record the expiry time, and expose a per-button countdown value. Run a 1-second interval to tick the countdown down. When the countdown reaches zero, re-enable the button.

**Trigger actions** — Expose two functions: `triggerAll()` (calls POST /api/sync/manual with no networkId) and `triggerNetwork(networkId)` (calls POST /api/sync/manual with that networkId). Both should: attach the Firebase ID token as a Bearer token in the Authorization header (use `getIdToken()` from firebase/auth), handle the response, update triggered state, and kick off polling. Both should be no-ops if the relevant button is already rate limited or in-progress.

**Sync history state** — Track whether the history drawer is open, whether history is loading, and the history data itself (array of sync events). Expose `openHistory()` which fetches GET /api/scheduled/sync-history and sets the drawer open, and `closeHistory()`.

Keep all API calls in this hook — the ManualRefreshPanel component should only call hook functions and read hook state, with no fetch logic of its own.
