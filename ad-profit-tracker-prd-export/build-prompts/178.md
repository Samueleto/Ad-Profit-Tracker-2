We're building the `POST /api/export/pdf` endpoint — this is the heart of the feature. It takes a request with date range, section selections, orientation, paper size, and optional filename, then generates and streams back a PDF file.

Here's the full flow to implement:

**Authentication:** Extract the Bearer token from the Authorization header and verify it with `admin.auth().verifyIdToken(token)`. If it's missing or invalid, return 401 immediately. Always use the `uid` from the verified token — never trust a userId from the request body.

**Validation:** Check that `dateFrom` and `dateTo` are present, valid YYYY-MM-DD strings, that dateFrom is not after dateTo, and that the range is 90 days or fewer (return 400 otherwise). Validate `sections` is a non-empty array where every item is one of the 9 keys in `ALLOWED_SECTIONS`. Validate `orientation` is `'portrait'` or `'landscape'` and `paperSize` is `'a4'` or `'letter'`. If `filename` is provided, check it's alphanumeric with dashes/underscores only and max 100 chars.

**Rate limiting:** Check a shared rate limit counter (using node-cache or the same mechanism as the Excel export) — max 10 exports per hour per user across both PDF and Excel. Return 429 if exceeded.

**Data fetching:** Check node-cache for the key `{uid}_export_preview_{dateFrom}_{dateTo}`. On a cache hit, use the cached data. On a miss, run Firestore queries in parallel with `Promise.all()`: query the `adStats` collection filtering by `userId == uid` AND `date >= dateFrom` AND `date <= dateTo`. If `sections` includes `'activity_log'`, also query `auditLogs` with `userId == uid` AND `createdAt >= dateFrom` AND `createdAt <= dateTo`. If `adStats` is empty and the requested sections aren't limited to just `'cover_page'` or `'activity_log'`, return 404.

**PDF generation:** Create a new jsPDF instance with the requested orientation and paper size. Set document properties (title, author, subject, creator). Then loop through the requested sections in order and build each one:
- `cover_page`: centered app name, subtitle, date range, generated timestamp, horizontal rule, then a new page
- `executive_summary`: compute totalRevenue (rollerads+zeydoo+propush revenue), totalCost (exoclick cost), netProfit, ROI% from adStats in memory. Render with `autoTable()` as a 2-column [Metric, Value] table. Format all currency to 2 decimal places.
- `daily_trend`: group adStats by date, compute per-day revenue/cost/netProfit/ROI. Render as a table sorted by date ASC with columns [Date, Revenue, Cost, Net Profit, ROI%].
- `geo_breakdown`: group by country, enrich country codes with ISO 3166-1 names from the static lookup. Render sorted by netProfit DESC with columns [Country, Revenue, Cost, Net Profit, ROI%, Impressions, Clicks].
- Per-network sections (`exoclick`, `rollerads`, `zeydoo`, `propush`): filter adStats to that networkId, render with columns [Date, Country, Primary Metric, Impressions, Clicks, CTR%, CPM]. Add a header noting whether it's Cost Only or Revenue Only.
- `activity_log`: render auditLogs with columns [Timestamp, Action, Resource, Status, Details].

After all sections, loop through every page and add a centered footer: `Page X of Y`.

**Response:** Generate the filename (use provided or default to `ad-profit-tracker-report-{dateFrom}-{dateTo}`). Call `doc.output('arraybuffer')`, convert to a Node.js Buffer, set `Content-Type: application/pdf` and `Content-Disposition: attachment; filename={filename}.pdf` headers, and send the buffer.

**Audit log:** After a successful response, write a document to the `auditLogs` Firestore collection via the Admin SDK with action `'manual_sync_triggered'`, resourceType `'sync'`, resourceId `'export'`, metadata containing exportType, sectionsIncluded, dateFrom, dateTo, orientation, paperSize, filename, and status `'success'`.
