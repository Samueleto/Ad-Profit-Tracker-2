We're building a financial metrics dashboard. For this step, I need you to add the FilterToolbar as a new section directly below the existing DateRangeToolbar on the FinancialMetricsDashboardPage at /dashboard. This is NOT a new route — it integrates into the existing page as a persistent sticky bar.

The FilterToolbar is a compact single-row bar that matches the height and visual style of the DateRangeToolbar above it. It contains five controls, laid out left to right:

1. **Network multi-select** — Inline checkboxes for ExoClick, RollerAds, Zeydoo, and Propush, plus an 'All Networks' toggle that checks/unchecks all four. Changes are staged until the user clicks 'Apply Filters'.

2. **Country/GEO combobox** — A type-ahead input that calls GET /api/filters/options?type=country with the active dateFrom/dateTo from the Zustand store when the user opens or focuses the dropdown. While the request is in flight, show a spinner and 'Loading countries...' placeholder text. If the response comes back empty, show 'No countries found for this date range'. If it errors, show an amber warning icon with 'Could not load countries' and a retry link. On success, populate a searchable dropdown of country objects (country code + country name). Country selections are also staged until 'Apply Filters'.

3. **Metric focus pills** — A segmented control with four options: Revenue, Cost, Profit, ROI. Single-select. Applies immediately without needing 'Apply Filters' — tapping a pill writes selectedMetric to the Zustand store right away.

4. **Data quality toggle** — A three-state toggle: All / Anomalies Only / Clean Only. Reads from adStats.validationStatus on the backend. Also applies immediately on change.

5. **Free-text search** — A debounced input (300ms delay) at the far right of the bar. Filters visible table rows client-side by country name or network name. Writes searchQuery to the Zustand store after the debounce fires.

At the right end of the bar, include an 'Apply Filters' button that commits the staged network and country selections to the Zustand store. Next to it, a 'Clear All' button that resets every filter field (selectedNetworks, selectedCountries, selectedMetric, dataQuality, searchQuery) back to their defaults in the store.

Add an active filter count badge — a small amber dot or count indicator — on a 'Filters' label that appears whenever any filter is in a non-default state. Count each active filter dimension (e.g., if networks are filtered AND a country is selected, badge shows 2).

On mobile, collapse all five controls into a single 'Filters' button that opens a bottom sheet containing the full filter panel with the same controls stacked vertically. The amber badge still appears on the collapsed button when filters are active.

The toolbar should be sticky so it stays visible as the user scrolls down through the dashboard tables and charts. All filter state lives in the Zustand store — the toolbar reads from and writes to the store, and the downstream dashboard sections re-fetch their data whenever the relevant store fields change.
