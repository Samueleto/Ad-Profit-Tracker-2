We're building a RollerAds ad network integration. For this step, I need you to make sure every endpoint is properly secured — no data leaks, no unauthorized access, and no way for one user to see another's data.

For the five user-facing endpoints (POST /api/networks/rollerads/sync, GET /api/networks/rollerads/stats, GET /api/networks/rollerads/stats/by-country, GET /api/networks/rollerads/stats/latest, GET /api/networks/rollerads/raw-response), every single one must verify the Firebase ID token using admin.auth().verifyIdToken() at the very top of the handler — before touching Firestore or doing any other work. Extract the uid exclusively from the verified token result, never from the request body or query params. Return 401 immediately if the token is missing, malformed, or expired.

For POST /api/networks/rollerads/sync specifically, add a rate limit check: max 10 manual syncs per hour per user. Store the count in a way that resets hourly (a Firestore counter document keyed by uid + hour, or an in-memory store if you have a single server instance). Return 429 with a clear message if exceeded. This prevents a user from hammering the RollerAds API and running up Firestore write costs.

For POST /api/networks/rollerads/scheduled-sync, this endpoint must NEVER accept a Firebase ID token — it's internal only. Validate the x-internal-secret header against the INTERNAL_SYNC_SECRET environment variable at the very top. Return 401 immediately if it's missing or doesn't match. Make sure this endpoint is not listed in any client-side API modules or route configs.

For the decrypted RollerAds API key in POST /api/networks/rollerads/sync and POST /api/networks/rollerads/scheduled-sync: decrypt it server-side using the cryptr/AES helper, use it immediately to make the axios call, and then let it go out of scope. Never log it, never include it in any response, never store it in a variable that gets serialized. The rawResponse stored in adStats must be sanitized — strip any fields whose names contain 'key', 'token', 'secret', or 'authorization' before writing to Firestore.

All Firestore queries across all six endpoints must be scoped to the authenticated user's uid. Never do a collection-group query from a user-facing endpoint — the scheduled-sync uses Admin SDK for cross-user queries, but that's internal only.

Make sure INTERNAL_SYNC_SECRET and the AES/cryptr encryption secret are stored as environment variables and never hardcoded or committed to source control.
