We're building the ROI Calculation Engine. Now let's build the frontend components that make up the ROIIndicatorSection — the highlighted section that integrates into the existing FinancialMetricsDashboardPage at `/dashboard`. This section reads from the `kpis.roi` and `kpis.roiChange` fields already returned by `GET /api/dashboard/metrics`, so no additional API calls are needed from these components.

**ROIKPICard** — The main KPI card that sits within the existing 4-column KPI grid. It should display the ROI percentage to two decimal places with a larger font weight than the other KPI cards to signal its importance. Apply color-coded Tailwind classes based on the value: `text-green-600 bg-green-50` for positive ROI, `text-red-600 bg-red-50` for negative, `text-amber-600 bg-amber-50` for zero or null. Show a directional arrow (↑ for positive, ↓ for negative, → for zero) next to the value. Below the main value, show the prior-period change label (e.g., '+12.4% vs prior period'). When ROI is null (no cost data), display 'N/A' with an amber badge and a tooltip that says 'No cost data available for this period'. During loading, show a shimmer placeholder that matches the card's dimensions — hide the arrow indicator during this state.

**ROIBreakevenLine** — A thin Recharts reference line component to be added to the existing daily profit trend chart. It renders as a dashed gray horizontal rule at `y=0` with a small label 'Breakeven'. This is a lightweight wrapper around Recharts' `ReferenceLine` component so it can be dropped into the existing chart without restructuring it.

**ROINetworkContributionRow** — A single row showing one revenue network's effective yield relative to ExoClick cost. Displays the network name, its revenue contribution, and its per-network ROI with the same color-coding logic as ROIKPICard. These rows are listed under the main KPI card for each of the revenue networks (rollerads, zeydoo, propush).

**ROIBreakdownPanel** — A collapsible panel that sits below the KPI card. Collapsed by default (progressive disclosure). When expanded, shows the ROI formula spelled out: `(Revenue - Cost) / Cost × 100` with the actual live values substituted in, e.g., `($1,250 - $800) / $800 × 100 = 56.25%`. Use a smooth expand/collapse animation. The expand/collapse toggle should be a small chevron button.

**ROIIndicatorSection** — The parent component that composes all of the above. It receives the `kpis.roi`, `kpis.roiChange`, and any per-network data as props from FinancialMetricsDashboardPage. It handles all the display states: loading (shimmer), empty ('No data for selected range' with amber N/A badge), error_404 ('N/A' with tooltip), error_500 (error banner with retry button), and success (full rendered section). This is the component that gets imported and placed into the existing dashboard page.
