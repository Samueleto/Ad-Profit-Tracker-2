We're building a Zeydoo ad network integration. For this step, I need you to set up the state management layer that the Zeydoo UI components will use to fetch data and track sync status.

Create a set of custom React hooks that encapsulate all the data fetching and mutation logic, keeping the components clean.

**useZeydooStats(dateFrom, dateTo, groupBy)** — Fetches from GET /api/networks/zeydoo/stats. Re-fetches automatically when any of the three params change. Returns { data, isLoading, error, refetch }. The data should be shaped for easy rendering — an array of daily rows or a single totals object depending on groupBy.

**useZeydooLatest()** — Fetches from GET /api/networks/zeydoo/stats/latest on mount. Returns { data, isLoading, error, refetch }. Should handle the 404 case gracefully — when the API returns 404, set data to null and isLoading to false without treating it as an error state.

**useZeydooTopCountries(dateFrom, dateTo, limit)** — Fetches from GET /api/networks/zeydoo/stats/by-country. Re-fetches when params change. Returns { countries, totalRevenue, isLoading, error }.

**useZeydooSync()** — Manages the manual sync flow against POST /api/networks/zeydoo/sync. Returns { triggerSync, isSyncing, lastSyncResult, error }. The triggerSync function takes optional dateFrom and dateTo params. After a successful sync, it should signal to the stats hooks that they need to refetch — you can do this by exposing a syncVersion counter that the stats hooks can include as a dependency.

**useZeydooRawResponse(date)** — Fetches from GET /api/networks/zeydoo/raw-response lazily (only when explicitly triggered, not on mount). Returns { fetch, data, schema, isLoading, error }.

All hooks should get the current user's Firebase ID token via getIdToken() before each request and attach it as a Bearer token. Handle token refresh transparently — if getIdToken() returns a fresh token, use it. Don't cache tokens in state.

For error handling, distinguish between network errors, auth errors (401), and API errors (400, 502) so the UI can show appropriate messages for each case.
