The ComparativeNetworkAnalysisTab needs to handle several distinct failure modes gracefully. Each one has a specific UI treatment — let's make sure none of them show a blank screen or a raw error message.

**Loading state** — While the initial fetch is in progress, show: 4 shimmer stat cards that match the dimensions of the real network cards, a 280px-tall shimmer rectangle where the grouped bar chart will appear, and 4 shimmer rows in the efficiency table area. The MetricToggle and tab header should still be visible during loading so the user knows where they are.

**Empty state (404)** — When GET /api/networks/comparison returns 404 (no data for the selected date range), show a centered message: 'No data for any network in this date range. Sync your networks to see comparisons.' Include a 'Sync All' CTA button that triggers POST /api/sync/manual. Do not show the card grid, chart, or efficiency table in this state.

**Error 401 (session expired)** — If the token refresh and retry both fail (see security step), redirect to `/` and show a 'Session expired' toast. This should be a clean redirect — don't leave the user on a broken dashboard page.

**Error 403 (access denied)** — Show 'Access Denied' as an inline message within the tab panel. Include a link to /dashboard. Do not show a full-page error or redirect. The other tabs should still be accessible.

**Error 500 (server error)** — Show an inline error banner inside the tab panel (not a full-page takeover). The banner should say something like 'Failed to load comparison data.' and include a 'Retry' button. The Retry button should re-call GET /api/networks/comparison (and any other parallel calls) with the current date range and metric. The banner should be dismissible.

**Partial data** — If some networks return data but others don't (e.g., one network has no stats for the date range), render the cards that have data and show '—' for all null metric values in the cards that don't. Don't treat partial data as an error — the efficiency table and chart should still render with whatever data is available.

**Sync All failure** — If POST /api/sync/manual fails, show a brief toast notification ('Sync failed. Please try again.') and reset the isSyncing spinner. Don't block the UI or show a full error state — the existing comparison data should remain visible.

**Network card null values** — Any metric value that is null or undefined in the API response should render as '—' in the card. Never show 'NaN', 'Infinity', 'undefined', or empty cells. Apply this consistently across the card grid, efficiency table, and ranking strip.
