We need six API route handlers for the filter system. All of them live under `src/app/api/filters/` and all of them start by verifying the Firebase ID token from the Authorization header using `admin.auth().verifyIdToken(token)` — reject with 401 immediately if the token is missing or invalid. The `uid` always comes from the verified token, never from query params or the request body.

**GET /api/filters/options** (`options/route.ts`) — Returns available filter options for the user's data in a date range. Accept query params: `type` (required, one of 'country'|'network'|'metric'|'all'), `dateFrom`, `dateTo` (both required, YYYY-MM-DD), and optional `networkId`. Validate that dateFrom <= dateTo and the range doesn't exceed 90 days — return 400 otherwise. Check node-cache with key `{uid}_filter_options_{type}_{dateFrom}_{dateTo}_{networkId||all}` and return with `X-Cache: HIT` header on a hit. On miss, query the `adStats` Firestore collection filtering by `userId==uid` and the date range. For `type=country`: extract unique non-null country codes, enrich with the ISO country name map from `src/lib/filters/countries.ts`, sort alphabetically by name. For `type=network`: return all four networks from the networks config, with `hasData: true/false` based on whether any adStats docs exist for that network in the range. For `type=metric`: return the static metrics array from `src/lib/filters/metrics.ts` — no Firestore query needed. For `type=all`: combine all three. Cache for 5 minutes and return.

**GET /api/filters/stats** (`stats/route.ts`) — The main filtered data endpoint. Accept query params: `dateFrom`, `dateTo` (required), `networks` (comma-separated), `countries` (comma-separated), `metric` (default 'profit'), `dataQuality` (default 'all'), `searchQuery` (max 100 chars), `groupBy` ('daily'|'country'|'network', default 'daily'), `limit` (1–100, default 50), `cursor`. Validate all params — return 400 on any invalid value. Cache key is a hash of all params namespaced by uid. Build the Firestore query on `adStats` with userId + date range, then add `networkId IN [...]` if networks provided (Firestore 'in' operator is safe here since we have max 4 networks). If more than 10 countries are requested, fetch without country filter and apply in-memory. After fetching, apply in-memory filters: dataQuality filtering against `validationStatus` field ('anomaly'/'stale' for anomalies, 'valid'/null for clean), and searchQuery case-insensitive match against country or networkId. Group and aggregate by the `groupBy` dimension — compute revenue (sum of rollerads+zeydoo+propush rows), cost (sum of exoclick rows), netProfit, roi (null-safe: null if cost is 0), impressions, clicks, ctr, cpm. Sort by the requested metric descending. Compute summary totals across ALL filtered rows (not just the current page), then apply cursor pagination and slice to limit+1 to detect hasMore. Cache 5 minutes.

**GET /api/filters/search** (`search/route.ts`) — Keyword search across the user's filter dimensions. Accept: `q` (required, 1–100 chars, trim whitespace), `dateFrom`, `dateTo`, `scope` ('country'|'network'|'all', default 'all'), `limit` (1–50, default 20). Fetch the user's adStats for the date range, then apply in-memory search: normalize q to lowercase, match against country ISO code AND enriched country name for country scope, match against networkId for network scope. Group matching docs by their primary key, aggregate metrics per group, sort by netProfit descending, slice to limit. Enrich country results with full names. Cache 3 minutes (shorter TTL since search is more query-specific).

**GET /api/filters/presets** (`presets/route.ts`) — Returns the user's saved filter presets. Read the `users` Firestore document for the authenticated uid. Extract `savedFilterPresets` array — if null/undefined, return `{ presets: [] }` (not 404). Sort by `createdAt` descending. Return the array.

**POST /api/filters/presets** (`presets/route.ts`) — Saves a new filter preset. Parse and validate the request body: `name` (1–50 chars), `networks` (subset of the 4 valid network IDs), `countries` (array of ISO codes, max 50), `metric` (valid MetricKey), `dataQuality` (valid DataQualityOption). Check rate limit (20 saves/hour per uid — use node-cache or a simple in-memory counter). Read the user's `users` document and check that `savedFilterPresets.length < 20` — return 400 with 'Maximum 20 presets allowed' if at the limit. Generate a UUID for the preset id, build the preset object with `createdAt: FieldValue.serverTimestamp()`, and use `arrayUnion` to append it to `users.savedFilterPresets`. Also set `users.updatedAt = FieldValue.serverTimestamp()`. Write an audit log entry to the `auditLogs` collection with `action='preferences_updated'`, `resourceType='preferences'`, `resourceId='filter_preset'`, and metadata including the preset id and name. Return 201 with the created preset.

**DELETE /api/filters/presets/:presetId** (`presets/[presetId]/route.ts`) — Deletes a saved preset. Extract `presetId` from the URL path. Validate it's non-empty. Check rate limit (20 deletes/hour per uid). Read the user's `users` document, find the preset with matching id in `savedFilterPresets` — return 404 if not found. Use `arrayRemove` to remove the preset object, update `users.updatedAt`. Write an audit log entry with `action='preferences_updated'` and `metadata={ presetId, action: 'deleted' }`. Return `{ deleted: true, presetId }`.
