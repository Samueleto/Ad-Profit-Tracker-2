We need a couple of shared pieces set up before building the filter endpoints — a caching layer and some static lookup data that multiple endpoints will share.

First, set up node-cache for the filter API routes. If the project already has a node-cache instance (likely from existing stats endpoints), reuse that singleton — don't create a second one. If it doesn't exist yet, create a shared cache instance at `src/lib/cache.ts` (or wherever the project keeps singletons) with a default TTL of 300 seconds (5 minutes). The filter endpoints will use TTLs of 300s for options and stats, and 180s for search results.

Second, create a static ISO 3166-1 alpha-2 country lookup map at `src/lib/filters/countries.ts`. This should export a plain object mapping two-letter country codes to their full English names (e.g., `{ US: 'United States', DE: 'Germany', FR: 'France', ... }`). This map is used by GET /api/filters/options, GET /api/filters/search, and GET /api/filters/stats to enrich raw country codes from Firestore into human-readable names. Include all commonly used country codes — at minimum the ones likely to appear in ad network data (US, UK, DE, FR, BR, IN, CA, AU, MX, IT, ES, NL, PL, RU, JP, KR, etc.).

Third, create a static metric config at `src/lib/filters/metrics.ts` that exports the full list of supported metrics as an array of `{ metric, label, unit }` objects. The supported metrics are: revenue, cost, profit, roi, impressions, clicks, ctr, cpm. This is the canonical list that GET /api/filters/options returns for `type=metric` and that all filter endpoints use for validation.

Fourth, create a constants file at `src/lib/filters/networks.ts` that exports the four supported networks: exoclick, rollerads, zeydoo, propush — each with their networkId, display label, and dataRole ('cost' for exoclick, 'revenue' for the others). This is the source of truth used across all filter endpoints.

Make sure firebase-admin is already initialized in the project (it should be from existing endpoints). If there's a shared `src/lib/firebase-admin.ts` initializer, we'll import from that in the API routes.
