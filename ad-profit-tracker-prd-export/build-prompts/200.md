We need several reusable components that the SyncStatusPanel will compose together. Build these in `features/sync-status/components/`.

**NetworkStatusCard** — The core card for a single network. Takes a `NetworkSyncState` object as a prop. Shows: the network name and a network icon (you can use simple text logos or small SVG icons for ExoClick, RollerAds, Zeydoo, Propush), a status phase badge (using the `syncPhaseDisplay` utility for color and label — amber with a CSS pulse animation for `fetching`/`writing`, green for `complete`, red for `failed`, gray for `idle`), the last synced timestamp formatted as relative time ('3 minutes ago' using `formatRelativeTime`), the latest data date, row count from the last sync, latency in ms, and a circuit breaker badge ('Open' in red, 'Closed' in green). When the card is clicked, expand an inline section showing `circuitBreakerOpenedAt`, `retryCount`, `totalFailureCount`, and `lastErrorCode`. Use a simple toggle — no modal, just an inline expand/collapse within the card. The card should have a loading skeleton variant (shimmer effect) that renders when a `loading` prop is true.

**HealthBanner** — Takes `overallHealth: OverallHealth` as a prop. Renders a sticky banner at the top of the section using the `healthDisplay` utility. 'All Systems Healthy' in green, 'Degraded' in amber, 'Critical' in red. Should be visually prominent — full-width bar with an icon and text.

**ActivityFeedList** — Takes an array of `ActivityFeedEntry` items. Renders a compact timeline list. Each row shows: a colored status dot (green for success, red for failure), the `eventLabel`, the network name, rows fetched, latency, and the relative timestamp. Keep rows compact — this is a secondary info area. If the array is empty, show 'No recent activity'.

**AnomalyAlertStrip** — Takes a `criticalCount: number` prop and an `onViewDetails` callback. Renders a dismissible inline alert strip in red/amber when `criticalCount > 0`. Shows 'X critical anomalies detected' and a 'View Details' link that calls `onViewDetails`. Hidden entirely when `criticalCount === 0`.

**RefreshButton** — A small icon button (a circular arrow icon) that triggers an immediate re-fetch. Takes an `onRefresh` callback and an `isLoading` boolean — shows a spinning animation while loading. This is distinct from triggering a sync; it just re-fetches the current status.

All components should use the project's existing styling approach (Tailwind, CSS modules, or whatever the project uses — check and match). Make sure the pulse animation for in-progress states is done with CSS (not JavaScript intervals) so it's performant.
