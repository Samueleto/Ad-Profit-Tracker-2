We need the Firebase Admin SDK set up so all eight stat endpoints can authenticate users and read/write Firestore.

Install firebase-admin if it isn't already in the project. Create a shared Firebase Admin initialization module at src/lib/firebase/admin.ts. This module should initialize the Admin SDK exactly once (use a singleton pattern — check if an app is already initialized before calling initializeApp). Pull the service account credentials from environment variables: FIREBASE_PROJECT_ID, FIREBASE_CLIENT_EMAIL, and FIREBASE_PRIVATE_KEY. The private key will have escaped newlines in the env file, so make sure to replace \\n with \n when reading it.

Export two things from this module: the initialized admin app, and a pre-built Firestore instance (admin.firestore()). Every route handler will import the Firestore instance from here rather than initializing its own.

Also create a shared auth verification helper at src/lib/firebase/verifyToken.ts. This function should take a request object, extract the Bearer token from the Authorization header, call admin.auth().verifyIdToken(token), and return the decoded token (which includes the uid). If the header is missing or the token is invalid/expired, it should throw a typed error that the route handlers can catch and turn into a 401 response. This helper will be called at the top of every single one of the eight stat endpoints.

Add all required environment variables to .env.local and document them in .env.example so other developers know what's needed.
