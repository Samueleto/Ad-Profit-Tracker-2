We need both the Firebase client SDK (for the browser) and the Firebase Admin SDK (for API routes) set up before we build anything else.

**Client-side Firebase (`lib/firebase/`):**
Install `firebase`. Create three files:
1. `app.ts` — Initialize the Firebase app using `initializeApp()` with your project config. Use a guard (`getApps().length === 0`) so it only initializes once, which matters in Next.js where modules can be evaluated multiple times.
2. `auth.ts` — Export a `getAuth()` instance from the initialized app. Also export a `GoogleAuthProvider` instance — we'll use this for the Google OAuth button in LoginModal.
3. `firestore.ts` — Export a `getFirestore()` instance. We'll use this to write user documents after sign-up.

All Firebase client config values (API key, project ID, etc.) should come from `NEXT_PUBLIC_` environment variables in `.env.local` so they're safely embedded in the client bundle.

**Server-side Firebase Admin (`lib/firebase-admin/`):**
Install `firebase-admin`. Create an `admin.ts` file that initializes the Admin SDK using `initializeApp()` with a service account. Use the same single-initialization guard. Export `adminAuth` (from `getAuth()`) and `adminDb` (from `getFirestore()`) — these are what our four API routes will import.

The Admin SDK credentials (service account JSON or individual key fields) must only live in non-`NEXT_PUBLIC_` environment variables — they should never be accessible in the browser. Store the service account private key, client email, and project ID as separate env vars, or base64-encode the full JSON and decode it at runtime.

Add all required environment variable keys to `.env.local` with placeholder values, and document what each one is for with a comment.
