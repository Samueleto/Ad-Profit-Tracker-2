We're building a historical data storage and analytics feature. For this step, I need you to create a set of custom React hooks that wrap all eight stats API endpoints and manage their loading, error, and data states for use across the dashboard.

Here's what each hook should do:

**useSnapshot(date, networkId?, groupBy?)** — Calls GET /api/stats/snapshot. Takes a date string (YYYY-MM-DD), an optional networkId, and an optional groupBy ('country' or 'total'). Returns the snapshot data including the networks array and summary object. Should re-fetch automatically when any of the params change. Don't fire the request if date is empty.

**useTrend(dateFrom, dateTo, networkId?, metric?)** — Calls GET /api/stats/trend. Manages the time-series data and metadata (min, max, average, movingAverage7d, percentChange). Only fires when both dateFrom and dateTo are set and the range is valid (dateFrom <= dateTo).

**useSummary(dateFrom, dateTo)** — Calls GET /api/stats/summary. Returns the aggregated totals across all networks plus the perNetwork breakdown. Only fires when both dates are provided.

**useGeoBreakdown(dateFrom, dateTo, networkId?, metric?, limit?)** — Calls GET /api/stats/geo-breakdown. Returns the ranked countries array and totals. Only fires when both dates are set.

**useCoverage(dateFrom, dateTo, networkId?)** — Calls GET /api/stats/coverage. Returns the coverage map, gaps array, and syncHistory. Only fires when both dates are set.

**useDates(networkId?, dateFrom?, dateTo?)** — Calls GET /api/stats/dates. This one is special — it should fire on mount even without explicit date params since the endpoint has sensible defaults (last 90 days). Use this to power date pickers and calendars that need to know which dates have data.

**useBackfill()** — Returns a trigger function that POSTs to POST /api/stats/backfill. Takes networkId (optional), dateFrom, and dateTo. Manages a loading state and the results summary (triggered, skipped, failed arrays). Should expose the last result so the UI can show what happened.

**useDeleteSnapshot()** — Returns a trigger function that calls DELETE /api/stats/snapshot. Takes date and optional networkId. Manages loading state and exposes the deletedCount from the response.

For all hooks: attach the Firebase ID token to every request as a Bearer token in the Authorization header. Fetch the token from the current Firebase Auth user before each request — don't cache it long-term since tokens expire after an hour. Each hook should expose { data, loading, error } at minimum. For the mutation hooks (useBackfill, useDeleteSnapshot), also expose a reset function to clear the result state.

Create a shared utility function (something like getAuthHeaders()) that gets the current user's ID token and returns the Authorization header object — all hooks should use this so token fetching logic isn't duplicated eight times.

Keep all the API call logic inside these hooks. Pages and components should never call fetch() directly — they just call these hooks and react to the state they return.
