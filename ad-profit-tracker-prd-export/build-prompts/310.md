We're building an ad network sync dashboard. For this step, I need you to integrate the five error monitoring panels into the existing dashboard and connect them to each other.

**Dashboard integration** — Add an 'Error Monitoring' section or tab to the existing dashboard. Place the `ErrorSummaryWidget` in the dashboard header area alongside the other health stats — it's compact and should always be visible. The other four panels (`CircuitBreakerStatusPanel`, `ErrorLogPanel`, `RetryStatePanel`, `RetryConfigPanel`) should live in a dedicated 'Error Monitoring' tab or expandable section. Don't bury them — if any circuit is open, surface a warning banner at the top of the dashboard that links directly to the circuit breaker panel.

**Cross-panel connections** — Wire the panels together so they feel like one coherent view:
- When the user clicks a network name in `ErrorSummaryWidget`, it should pre-filter the `ErrorLogPanel` to that network (update the URL param and scroll to the log panel)
- When a circuit breaker reset succeeds in `CircuitBreakerStatusPanel`, also refresh `RetryStatePanel` (since `retryCount` resets to 0)
- When `RetryConfigPanel` saves new settings, refresh `RetryStatePanel` (the backoff schedule changes)
- The `RetryStatePanel` 'next retry' countdown should tick down in real time — update the displayed `timeUntilNextRetrySeconds` every second using a local interval, without re-fetching from the API

**Alert banner** — On any page load, check `GET /api/errors/circuit-breaker/status`. If `summary.networksDegraded > 0`, show a dismissible warning banner at the top of the dashboard: 'X network(s) have open circuit breakers — sync is paused.' Include a 'View details' link that scrolls to or opens the CircuitBreakerStatusPanel. Dismiss state should be stored in session storage (not local storage) so it reappears on the next visit if circuits are still open.

**Empty and onboarding states** — If the user has no `networkConfigs` documents yet (all panels return empty arrays), show a single friendly message: 'No networks configured yet. Add a network to start monitoring sync health.' Link to the network configuration flow.

**Loading coordination** — The `ErrorSummaryWidget` in the header loads first and independently. The tab panels load lazily — only fetch when the user opens the 'Error Monitoring' tab, not on initial dashboard load. This keeps the dashboard fast for users who don't need to check errors on every visit.
