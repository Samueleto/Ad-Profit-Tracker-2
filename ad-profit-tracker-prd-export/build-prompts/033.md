- `src/app/api/reports/route.ts` — GET to list saved reports
- `src/app/api/reports/[id]/route.ts` — PATCH and DELETE for a specific report
- `src/app/api/reports/run/route.ts` — POST to execute a report and get results

Finally, add a shared utility file at `src/lib/reportValidation.ts` — we'll put all the field validation logic here so both the save and run endpoints can share it without duplicating code.
