We're building an audit logging feature for Ad Profit Tracker. For this step, add a new 'Activity Log' tab to the existing SettingsPage at /settings — don't create a new route, just wire a new tab into the existing tab structure.

The ActivityLogTab is a dense, scannable table UI. Here's what to build:

**Filter bar (always visible above the table):**
A sticky filter bar with three controls: an action type multi-select (options: api_key_saved, api_key_deleted, network_config_updated, network_config_reordered, network_config_reset, network_connection_tested, manual_sync_triggered, preferences_updated, profile_updated, account_deleted), a date range picker with presets (last 7 days, 30 days, 90 days, and a custom range), and a free-text search input (searches against the metadata/details field, max 100 characters). Changing any filter should re-fetch from GET /api/audit/logs with the updated query params.

**Table layout (5 columns):**
- Timestamp: show relative time (e.g., '2 hours ago') using date-fns, with the absolute formatted date/time appearing on hover as a tooltip
- Action type: render as a badge with semantic colors — green for success actions, red for delete actions (api_key_deleted, account_deleted), blue for config actions, amber for sync actions
- Resource: display the resourceType and resourceId together (e.g., 'ExoClick API Key', 'RollerAds Config')
- Details: a concise summary derived from the metadata field — for config updates show changed fields like 'syncSchedule: daily → hourly', for sync show triggered/skipped/failed counts
- Status: success/failure badge

Each row should be expandable inline (accordion style, no modals) to reveal the full raw metadata JSON payload, nicely formatted.

Pagination shows 25 rows per page. Display a row count like 'Showing 1–25 of 142 events'. Include a 'Load more' button that appends the next page using the nextCursor from GET /api/audit/logs — don't replace the existing rows, append to them.

**Bottom action bar:**
- Export to CSV button: calls GET /api/audit/logs/export with the current active filters and triggers a file download
- Clear All Logs button: destructive, styled in red. Clicking it opens a confirmation dialog (two-step: first click shows the dialog, second click confirms). On confirmation, call DELETE /api/audit/logs/clear, then refresh the table. Show a success toast after clearing.

**States to handle:**
- Loading: show shimmer placeholder rows matching the 5-column table structure. The filter bar should remain interactive during loading.
- Empty (no logs yet): show an illustration with the message 'No activity recorded yet — actions like saving API keys and updating settings will appear here.'
- Empty (filters returned nothing): show 'No logs found' with a 'Reset filters' CTA that clears all active filters
- Error states: covered in error_handling step

Use firebase/auth to get the current user's ID token via currentUser.getIdToken() and attach it as a Bearer token in the Authorization header for all three API calls (GET /api/audit/logs, GET /api/audit/logs/export, DELETE /api/audit/logs/clear). Use date-fns for all timestamp formatting.
