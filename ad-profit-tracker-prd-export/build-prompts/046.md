We need the Firebase Admin SDK properly initialized for the server-side audit endpoints, and the Firebase Auth client SDK available on the frontend for attaching tokens to requests.

On the backend, the audit endpoints (`POST /api/audit/log`, `GET /api/audit/logs`, `GET /api/audit/logs/export`, `DELETE /api/audit/logs/clear`) all need to verify Firebase ID tokens using `admin.auth().verifyIdToken()` and write/read from Firestore using the Admin SDK. If the project already has a Firebase Admin singleton initialized somewhere (common in Next.js projects to avoid re-initializing on every hot reload), point the audit routes at that. If not, create one now at `src/lib/firebaseAdmin.ts`. It should initialize the app once using `getApps().length === 0` guard, pulling credentials from environment variables (`FIREBASE_PROJECT_ID`, `FIREBASE_CLIENT_EMAIL`, `FIREBASE_PRIVATE_KEY`). Export a `db` (Firestore Admin instance) and `auth` (Auth Admin instance) from this file.

On the frontend, the ActivityLogTab page needs `firebase/auth` to get the current user's ID token and attach it as a Bearer token to API calls. Make sure the Firebase client SDK is already initialized in the project (there's usually a `src/lib/firebase.ts` or similar). If it doesn't exist yet, create it using the project's Firebase config environment variables (`NEXT_PUBLIC_FIREBASE_*`). Export a `getAuthToken()` helper that calls `auth.currentUser?.getIdToken()` — the audit components will use this to authenticate every request.

Also install `date-fns` if it's not already in the project — we'll use it in the ActivityLogTab for formatting timestamps (relative time like '2 hours ago' with the absolute time on hover).

Make sure all the Firebase config values are in `.env.local` and that none of the Admin SDK credentials are prefixed with `NEXT_PUBLIC_` (those would be exposed to the browser).
