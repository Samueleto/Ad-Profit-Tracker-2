We're building seven API route handlers for the notification system. All of them live under /api/notifications and use the Firebase Admin SDK to read and write the users Firestore collection. Here's what each one needs to do:

**GET /api/notifications** — The main feed endpoint. Verify the Firebase ID token from the Authorization header using admin.auth().verifyIdToken(). Read the user's users document and extract the notifications array and unreadNotificationCount. Filter out isDismissed=true items, then optionally filter to isRead=false if filter='unread'. Sort by createdAt descending. Apply cursor-based pagination using the cursor query param (find the notification with that id, slice from the next one). Fetch limit+1 items to determine hasMore. Return { notifications, unreadCount, hasMore, nextCursor }. Default limit is 20, max is 50.

**PATCH /api/notifications/dismiss/:id** — Mark one notification as dismissed. Verify token, validate the :id param. Rate limit to 60 dismissals/hour per user. Find the notification in the array — return 404 if not found or already dismissed. Use a Firestore transaction with arrayRemove (old object) + arrayUnion (updated object with isDismissed=true, isRead=true). If it was previously unread, decrement unreadNotificationCount (floor at 0). Update notificationsUpdatedAt with serverTimestamp().

**PATCH /api/notifications/mark-all-read** — Mark every unread notification as read. Verify token. Rate limit to 30/hour. Read the notifications array, update all isRead=false items to isRead=true. Write the full updated array back. Set unreadNotificationCount=0 and notificationsUpdatedAt=serverTimestamp(). Return { updated: number, unreadCount: 0 }.

**DELETE /api/notifications/clear** — Wipe all notifications. Verify token. Rate limit to 10/hour. Set users.notifications=[], unreadNotificationCount=0, notificationsUpdatedAt=serverTimestamp(). Write an audit log entry to the auditLogs Firestore collection with action='preferences_updated', resourceType='notification', resourceId='all', metadata={ clearedCount, action: 'clear_all' }, status='success'.

**POST /api/notifications/create** — Dual-auth endpoint. If Authorization header is present, verify the Firebase ID token and use that uid. If x-internal-secret header is present, validate it against the INTERNAL_SYNC_SECRET env var and use userId from the request body. Validate all fields — type must be one of the nine NotificationType values, severity one of four, title max 100 chars, body max 300 chars. Check the user's notificationPreferences — if preferences[type].enabled === false, return 200 with { created: false, reason: 'disabled_by_user' }. Generate a UUID v4 for the notification id. If the notifications array is at or near 100 items, prune the oldest dismissed/read ones first. Use arrayUnion to append the new notification and increment unreadNotificationCount by 1.

**GET /api/notifications/preferences** — Read the user's notification preferences. Verify token. Read users.notificationPreferences. Merge with system defaults (all nine types default to enabled: true). Return the full list with an isDefault flag on each.

**PATCH /api/notifications/preferences** — Update preferences. Verify token. Rate limit to 20/hour. Validate that each key in the request body is a valid NotificationType and each value has enabled as a boolean. Use Firestore dot-notation partial updates (e.g., 'notificationPreferences.sync_failure.enabled') so you don't overwrite other preferences. Write an audit log entry to auditLogs. Return the full merged preferences.

For rate limiting, use an in-memory store or a lightweight Redis/Firestore-based counter — whichever fits the project's existing patterns. Make sure every endpoint returns the correct HTTP status codes as specified.
