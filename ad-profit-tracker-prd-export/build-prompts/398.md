# Step 398 of 437: Handle errors gracefully in the API Explorer

================================================================================

The API Explorer makes live network calls to external ad network APIs, which means things will go wrong — networks will be down, API keys will expire, rate limits will be hit. We need to handle all of this gracefully without breaking the whole tab or confusing the user.

Here's how errors should be handled per scenario:

**Loading cached response fails** (loadCachedResponse for any of the four networks): Show an inline error message inside the accordion body — something like 'Failed to load cached response.' with a 'Retry' button. The error should only appear in the affected accordion, not affect the others. The Retry button should call loadCachedResponse again for that network.

**Fetch Fresh Sample fails** (/api/networks/config/test-connection returns an error): Show the error inline in the accordion body. Re-enable the 'Fetch Fresh Sample' button. If the error is a network timeout, say 'Request timed out — the network may be slow.' If it's a 401/403 from the ad network (meaning the API key is invalid or expired), say 'API key may be invalid or expired. Check your settings.' with a link to /settings. For other errors, show the HTTP status and a generic 'Something went wrong' message.

**API key not configured**: This isn't an error state — it's an expected empty state. Show the 'API key not configured' message with the /settings link. Don't show an error banner.

**No cached response yet**: Also not an error — show 'No response cached yet. Click Fetch Fresh Sample to retrieve one.' Don't show an error banner.

**Copy to clipboard fails**: The clipboard API can fail (e.g., in non-HTTPS environments or if the user denies permission). If it fails, show a small tooltip or notification saying 'Copy failed — try selecting the text manually.'

**Malformed JSON in the response**: If a raw-response endpoint returns something that isn't valid JSON, don't crash the react-json-view component. Catch the parse error and show 'Response could not be parsed as JSON. Raw response: [show as plain text]' instead.

All error states should be stored in the Zustand store's error field per network and cleared when a new successful fetch completes or when clearError is called.
