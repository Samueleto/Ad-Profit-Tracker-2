Let's add tests to make sure the notification system works correctly and doesn't break when things change.

Unit tests for the Zustand notifications slice:
- fetchNotifications sets isLoading, then populates notifications and unreadCount on success.
- fetchMoreNotifications appends to the existing array (doesn't replace it) and updates hasMore and nextCursor.
- dismissNotification removes the item optimistically and rolls back on API failure.
- markAllRead sets all isRead to true and unreadCount to 0 optimistically, rolls back on failure.
- clearAll empties the array optimistically, rolls back on failure with the original array.
- setUnreadCount updates only the unreadCount field without touching the notifications array.

Unit tests for error handling:
- A 401 response triggers getIdToken(true) and retries the request.
- A 500 response sets the error state in the store.
- A 429 response on dismiss shows the rate limit toast without rolling back (since the dismiss didn't execute).
- A 404 on dismiss leaves the optimistic removal in place.

Component tests for NotificationCenterPanel:
- Renders 5 shimmer rows while isLoading is true.
- Renders notification rows with correct icon, border color, title, body, and timestamp when notifications are present.
- 'Unread' tab filters to only isRead=false items.
- Clicking the X on a row calls dismissNotification with the correct id.
- 'Mark All as Read' button is disabled when unreadCount is 0.
- 'Clear All' button requires a confirm click before firing clearAll.
- 'Load more' button is hidden when hasMore is false.
- Empty state shows 'You're all caught up' when notifications array is empty.
- Inline error banner with Retry button appears when error state is set.

Component tests for NotificationBellIcon:
- Badge shows the correct unreadCount from the store.
- Badge is hidden when unreadCount is 0.
- Clicking the bell calls openPanel.

Integration tests (full user flows):
- Open panel → notifications load → dismiss one → it disappears → mark all as read → badge clears → clear all → empty state shows.
- Open panel → network error → retry → notifications load.
- Dismiss a notification → API fails → notification reappears → toast shown.
- Click a sync_failure notification → panel closes → page scrolls to ManualRefreshPanel.
- Click a schedule_failed notification → panel closes → /reports loads.

Mock all Firebase auth calls (getIdToken) and all fetch calls to the notification API endpoints in tests. Don't make real network calls.
