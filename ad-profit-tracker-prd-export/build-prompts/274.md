We're building the state layer for the DailyProfitTrendSection. The goal is to keep all data fetching, merging, and derived computations out of the component itself so the chart stays clean and testable.

Create a custom hook — something like useDailyProfitTrend — that the DailyProfitTrendSection component will call. Here's what it needs to manage:

**Inputs it accepts:**
- The selected date range (start date, end date, or a named preset like 'last7', 'last14', 'last30', 'thisMonth')
- The selected metric ('profit' | 'revenue' | 'cost' | 'roi')

**What it fetches:**
When the date range changes, it should fire both GET /api/roi/compute (with groupBy='daily' and the date range params) and GET /api/roi/breakdown (with dimension='daily' and the same date range) in parallel using Promise.all. It should NOT re-fetch when only the metric changes — the metric toggle just controls which field is displayed from already-loaded data.

**Data merging:**
After both calls return, merge the results by date. Each merged data point should have: date, netProfit, revenue, cost, roi, colorCode, and roiIndicator. Store this merged array as the canonical series data.

**Derived values computed from the series:**
- The 7-day moving average array (compute this from netProfit values, handling fewer than 7 points gracefully by using whatever points are available)
- Overall period profitability (sum of netProfit — positive or negative — used to set the area fill color)
- Pattern insights: best day (max netProfit with its date), worst day (min netProfit with its date), longest profitable streak (consecutive days where netProfit > 0), and period-over-period percent change (compare the current period total to the equivalent prior period — you'll need to fetch the prior period data or compute it from available data)

**State it exposes:**
- seriesData: the merged array
- movingAverage: the 7-day MA array
- patternInsights: { bestDay, worstDay, longestStreak, periodOverPeriodChange }
- isOverallPositive: boolean for area fill color
- status: 'idle' | 'loading' | 'success' | 'empty' | 'error'
- errorType: null | 401 | 403 | 404 | 500
- retry: a function that re-fires both API calls

**Snapshot drill-down:**
Create a separate small hook — useDaySnapshot — that accepts a date string and fetches GET /api/stats/snapshot for that day. It should be lazy (only fires when a date is provided) and expose the snapshot data plus its own loading/error state.

**Date range sync:**
The hook should accept the date range from the parent FinancialMetricsDashboardPage so the DailyProfitTrendSection stays in sync with the dashboard's global date range selector. When the parent date range changes, the hook re-fetches automatically.

Attach the Firebase Auth Bearer token to every API call — get the current user's ID token via getIdToken() from firebase/auth and include it as an Authorization header.
