We need several reusable components that both HelpCenterPage and HelpArticlePage will share. Build these before the pages so the pages can import them directly.

**CategoryCard** — Used in HelpCenterPage's category grid. Takes a category name, icon, article count, and whether it's currently selected. Renders as a clickable card with the icon, category label, and a count badge. Selected state should be visually distinct (highlighted border or background). Fires an onSelect callback with the category value. On mobile, this will be used in a horizontal scroll pill row instead of a grid.

**ArticleRow** — Used in HelpCenterPage's article list. Takes a HelpArticleListItem and renders a scannable row with: article title, category badge (color-coded by category), estimated read time, and a 'New' badge if isNew is true. The whole row is clickable and navigates to /help/[articleId]. Keep it compact — this is a list item, not a card.

**PopularArticlesSidebar** — Used in HelpCenterPage's right sidebar. Takes an array of up to 5 HelpArticleListItem objects (the most-viewed articles, sorted by viewCount). Renders them as a numbered list with title and read time. Each item links to /help/[articleId]. Shows a shimmer loading state when data is loading.

**TableOfContents** — Used in HelpArticlePage's sticky sidebar. Takes the article body markdown string and parses out all H2 and H3 headings to build a navigation list. Highlights the currently active heading as the user scrolls (use an IntersectionObserver). Each item is a smooth-scroll anchor link. On mobile, renders as a 'Jump to section' dropdown select instead of a sidebar list. Shows nothing if the article has fewer than 2 headings.

**FeedbackWidget** — Used at the bottom of HelpArticlePage. Shows 'Was this helpful?' with a thumbs-up and thumbs-down button. On click, calls POST /api/help/feedback with the articleId and rating. While submitting, disable both buttons and show a spinner. After submission, replace the buttons with a thank-you message ('Thanks for your feedback!'). If the API returns 429 (already submitted), show 'You've already rated this article' instead. Handles error state with a brief retry option.

**ArticleShimmer** — A loading skeleton used in both pages. Accepts a variant prop: 'list' renders 8 shimmer rows matching ArticleRow's structure, 'article' renders a shimmer title block + meta row + 6 paragraph blocks matching HelpArticlePage's layout, 'category-grid' renders 6 shimmer cards matching CategoryCard's size. Use CSS animation for the shimmer effect, consistent with the app's ThemeProvider tokens for dark mode.

All components should use the project's existing ThemeProvider tokens for colors so dark mode works automatically.
