We're building the PdfExportTab component — the second tab inside the existing ExportModal. This component handles all the UI for configuring and triggering a PDF export. It should feel visually consistent with the Excel tab that's already there.

**Layout:** The tab has three main areas: a list of section checkboxes, an options row, and an action bar at the bottom.

**Section checkboxes:** Show one checkbox row per section in this order: Cover Page, Executive Summary, Daily Profit Trend Chart, Geographic Breakdown Table, Per-Network Stats Tables (ExoClick, RollerAds, Zeydoo, ProPush), and Activity Log. Each row shows the section label and a row count badge on the right — these badge counts come from the `GET /api/export/preview` response that the Excel tab already fetched and passed down as a prop (don't re-fetch it). While the preview data is loading, show 6 shimmer placeholder rows instead of the real checkboxes. All checkboxes should be checked by default.

**Options row:** Below the checkboxes, show an 'Options' label with two inline selectors side by side: an Orientation selector (Portrait / Landscape) and a Paper Size selector (A4 / Letter). These should look like a compact two-column row, not a full-width stacked layout.

**Progressive disclosure:** Add a collapsed 'Advanced' section with a chevron toggle. When expanded, show a 'Custom filename' text input (max 100 chars, alphanumeric/dash/underscore only — validate on change and show an inline error if the format is wrong).

**Action bar:** At the bottom, two buttons side by side: 'Cancel' (closes the modal) and 'Export Now'. The Export Now button should be disabled with a tooltip 'No data available for this range' when the preview shows no data. When the user clicks Export Now, call `POST /api/export/pdf` with the selected sections, orientation, paper size, date range (passed as props from the parent modal), and optional filename. While the request is in flight, show an indeterminate spinner on the button, disable all inputs, and show a progress message 'Generating your PDF report...' below the options.

**Success handling:** When the response comes back, trigger the file download automatically (create a blob URL from the response buffer, click it programmatically, then revoke it). Show a toast 'PDF export complete — file downloaded'. Close the modal after 1.5 seconds.

**Error states:**
- 401: close the modal, call `getIdToken(true)` to force-refresh the Firebase ID token, redirect to `/` with a 'Session expired' toast if refresh fails
- 403: show an inline 'Access Denied' message inside the tab with a close button
- 404: disable Export Now with an inline message 'No data found for this date range'
- 500: show an inline error banner with a 'Retry' button that re-calls the endpoint

Use the project's existing ThemeProvider tokens for colors (dark: bg-gray-900 / light: bg-white) and the accent color CSS custom property for the Export Now button — match the Excel tab's styling exactly.
