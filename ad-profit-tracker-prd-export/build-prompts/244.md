We're building a custom report builder for an ad analytics dashboard. For this step, build the CustomReportBuilderPage at the /reports route. This is a fairly complex page, so let me walk through each piece.

The overall layout is two-column on desktop: a fixed 380px left column for all the configuration controls, and a right column for the live preview table. On mobile, the config stacks above the preview.

**Left column — Configuration panel**

Build five card sections, each with a clear heading:

1. **Metrics** — A multi-select checkbox panel with exactly these 8 options: Revenue, Cost, Net Profit, ROI%, Impressions, Clicks, CTR%, CPM. Include a 'Select All' toggle at the top that checks or unchecks all eight at once.

2. **Date Range** — Use `react-day-picker` to build a date range picker. Offer these preset buttons: Last 7 Days, Last 14 Days, Last 30 Days, Last 90 Days, This Month, and Custom. When the user picks Custom, show two calendar inputs (date from / date to). Enforce a 90-day maximum span client-side — if the user selects a range wider than 90 days, show an inline warning and disable the Run/Preview buttons. This date range is independent from any dashboard-level date filter.

3. **Networks** — Multi-select checkboxes for ExoClick, RollerAds, Zeydoo, and Propush. Include an 'All Networks' toggle.

4. **Grouping** — A single-select segmented control with three options: Daily, Country, Network. Only one can be active at a time.

5. **Data Quality** — A three-way toggle: All / Clean Only / Anomalies Only.

Put the Data Quality section under a collapsible chevron labeled 'Advanced Options' so it's hidden by default (progressive disclosure).

**Right column — Live Preview panel**

This panel shows a paginated table of up to 20 rows. It calls `GET /api/filters/stats` with the current configuration, debounced 300ms after any config change. Use `MobileDataTableWrapper` for responsive rendering — this component already exists in the project, so import and reuse it here.

The preview has these states:
- **Loading**: show 5 shimmer placeholder rows while the API call resolves
- **Empty**: show 'No data matches your current configuration. Adjust filters or sync your networks.' with a 'Sync Now' link
- **Error 500**: show an inline error banner with a Retry button that re-calls `GET /api/filters/stats`
- **Error 404**: show 'No data found for selected range and filters' with a date range reset button
- **Success**: populate the table with color-coded rows

Also show a row count badge on each config section — these come from `GET /api/export/preview` and show estimated export counts. Update these badges whenever the config changes (same 300ms debounce).

**Action buttons**

- **Run Report**: calls `POST /api/reports/run` with the full current config and renders the complete result in the preview table with pagination (up to 100 rows, cursor-based).
- **Save Report**: when clicked, reveals an inline name-input field. On submit, calls `POST /api/reports/save` with the current config. After a successful save, show a success toast and refresh the saved reports list.
- **Export**: opens the existing ExportModal, pre-populated with the current date range and selected sections. This triggers `GET /api/export/excel` or `GET /api/export/pdf` depending on what the user picks in the modal.

Disable the Export and Save Report buttons until the preview has successfully loaded at least once.

**Saved Reports sidebar**

On desktop, this is a collapsible left drawer that slides in from the left edge. On mobile, it's a bottom sheet. It lists all saved reports fetched from `GET /api/reports`. Each report in the list shows its name and has three actions: Load (applies that report's config to the left panel and triggers a preview refresh), Rename (inline edit that calls `PATCH /api/reports/:id`), and Delete (calls `DELETE /api/reports/:id` with a confirmation prompt). While the list is loading, show 3 shimmer rows.

On initial page load, fetch both `GET /api/reports` (for the sidebar) and trigger the first preview call simultaneously.
