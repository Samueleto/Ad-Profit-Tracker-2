We're building a historical ad stats storage and retrieval system. For this step, set up the folder structure that all eight API endpoints will live in.

Create an API route area under src/app/api/stats/ with individual route files for each endpoint we'll build: snapshot (handles both GET and DELETE), trend, summary, geo-breakdown, coverage, backfill, and dates. Each of these will become its own route handler file.

Also create a shared server-side library folder at src/lib/stats/ — this is where we'll put reusable helpers for things like date validation, Firestore query builders, and aggregation logic that multiple endpoints share. Create a src/lib/sync/ folder as well — this is where the shared syncNetworkForUser helper lives (used by both the backfill endpoint and the manual sync feature).

Finally, create a src/lib/geo/ folder for the static ISO 3166-1 alpha-2 country code lookup map that the geo-breakdown endpoint needs.

The project uses Next.js with Firebase as the backend, so all route handlers should follow the Next.js App Router convention (route.ts files with exported GET, POST, DELETE functions).
