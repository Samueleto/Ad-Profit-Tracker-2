We need to connect the ComparativeNetworkAnalysisTab into the existing FinancialMetricsDashboardPage so it behaves like a first-class tab alongside the four per-network tabs.

Here's what needs to be wired up:

**Tab registration** — Add 'Compare' as a tab entry in the FinancialMetricsDashboardPage tab bar. It should appear as the first tab, before the ExoClick, RollerAds, Zeydoo, and Propush tabs. The tab label should be 'Compare' and its panel should render ComparativeNetworkAnalysisTab.

**Shared date range** — The parent FinancialMetricsDashboardPage already manages a shared dateFrom/dateTo state that the per-network tabs use. Pass these values down to ComparativeNetworkAnalysisTab as props (or via context if that's how the other tabs receive them). The Compare tab must not own its own date range — it reads from the parent.

**Lazy activation** — The Compare tab should only trigger its data fetch when the user first clicks on it. Use the tab activation event (or a visibility/active flag) to signal the `useComparativeAnalysis` hook to fire its initial fetch. The other per-network tabs should not be affected by this.

**Network card navigation** — When a user clicks one of the four network cards inside ComparativeNetworkAnalysisTab, it should switch the active tab in FinancialMetricsDashboardPage to that network's dedicated tab. Wire this up so clicking the ExoClick card activates the ExoClick tab, RollerAds card activates the RollerAds tab, and so on. This is a tab switch, not a route change.

**Sync All integration** — The 'Sync All' button inside the tab calls POST /api/sync/manual. If the parent dashboard or other tabs have their own data refresh mechanisms, make sure a successful Sync All also signals them to refresh — you don't want the Compare tab to show fresh data while the per-network tabs still show stale data. If there's a shared refresh trigger in the parent, hook into it.

**URL state (optional but preferred)** — If the parent dashboard already syncs the active tab to a URL query param (e.g., ?tab=compare), make sure 'compare' is a valid value and that deep-linking to ?tab=compare activates the Compare tab correctly. If the dashboard doesn't do URL-synced tabs yet, note this as a gap but don't implement it from scratch.
