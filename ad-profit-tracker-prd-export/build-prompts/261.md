We need a clean way to manage the connection state for all four ad networks without the SettingsPage component becoming a mess of useState calls.

Create a custom hook — something like useApiKeyStatus — that handles fetching and updating the connection state for all four networks. Here's what it should do:

On mount, call GET /api/keys/status and store the results as a map keyed by networkId (e.g., `{ exoclick: { status: 'connected', updatedAt: ... }, rollerads: { status: 'not_connected', updatedAt: null }, ... }`). Expose a loading state so the page can show skeletons while this initial fetch is in progress, and an error state if the fetch fails.

Expose two update functions that the individual network cards can call:
- `markConnected(networkId, updatedAt)` — updates a single network's status to 'connected' locally after a successful save. This should be optimistic: update the UI immediately without re-fetching from the server.
- `markDisconnected(networkId)` — updates a single network's status to 'not_connected' locally after a successful delete.

Also manage per-network submission state separately — each card needs to know if its own save or delete request is in flight. You can do this with a simple set of networkIds that are currently 'submitting'. Expose `setSubmitting(networkId)` and `clearSubmitting(networkId)` helpers.

The hook should also handle the Firebase auth token. Before any API call (save, status, delete), it needs to get the current user's ID token using Firebase Auth's `getIdToken()` and attach it as a Bearer token in the Authorization header. Centralize this token-fetching logic in a shared utility so it's not duplicated across the three API calls.

Keep the raw key value out of this hook entirely — that stays inside react-hook-form within each ApiKeyForm component and is discarded after submission.
