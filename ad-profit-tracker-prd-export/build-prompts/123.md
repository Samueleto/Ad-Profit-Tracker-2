We need TypeScript interfaces and types that all six RollerAds endpoints will share. Create these in `src/lib/rollerads/types.ts`.

Define the following types:

**`AdStatDocument`** — Represents one document in the `adStats` Firestore collection. Fields: `userId: string`, `networkId: string`, `date: string`, `country: string | null`, `revenue: number`, `impressions: number`, `clicks: number`, `ctr: number`, `cpm: number | null`, `cost: null` (always null for RollerAds), `rawResponse: Record<string, unknown>`, `syncedAt: FirebaseFirestore.Timestamp`.

**`NetworkConfigDocument`** — Represents a document in the `networkConfigs` collection. Fields: `userId: string`, `networkId: string`, `isActive: boolean`, `syncSchedule: 'daily' | 'manual' | string`, `endpointOverride: string | null`, `timeoutSeconds: number`, `lastSyncedAt: FirebaseFirestore.Timestamp | null`, `lastSyncStatus: 'success' | 'failed' | null`, `lastSyncError: string | null`.

**`ApiKeyDocument`** — Represents a document in the `apiKeys` collection. Fields: `userId: string`, `networkId: string`, `encryptedKey: string`, `createdAt: FirebaseFirestore.Timestamp`, `updatedAt: FirebaseFirestore.Timestamp`.

**`AuditLogDocument`** — Represents a document in the `auditLogs` collection. Fields: `userId: string | null` (null for scheduled syncs), `action: string`, `resourceType: string`, `resourceId: string`, `metadata: Record<string, unknown>`, `status: 'success' | 'failure'`, `errorMessage: string | null`, `createdAt: FirebaseFirestore.Timestamp`.

**`RollerAdsStatRow`** — Represents one row from the RollerAds `/sites/stat` API response. Fields: `site_id: string`, `date: string`, `country: string`, `impressions: number`, `clicks: number`, `ctr: number`, `revenue: number`, `cpm: number`, `ecpm: number`.

**`RollerAdsFieldSchema`** — Represents one entry in the field schema array returned by the raw-response endpoint. Fields: `field: string`, `type: string`, `description: string`.

Also populate `src/lib/rollerads/fieldSchema.ts` with the static `ROLLERADS_FIELD_SCHEMA` array — an array of `RollerAdsFieldSchema` objects describing all nine fields from the RollerAds `/sites/stat` response: `site_id`, `date`, `country`, `impressions`, `clicks`, `ctr`, `revenue`, `cpm`, `ecpm`. Include the field name, type (`string` or `number`), and a plain-English description for each.

Finally, create a small helper function `sanitizeRawResponse(obj: Record<string, unknown>): Record<string, unknown>` in `src/lib/rollerads/sanitize.ts`. This strips any keys whose names contain `key`, `token`, `secret`, or `authorization` (case-insensitive) from an object before it gets stored as `rawResponse` in Firestore. The sync endpoints call this before writing.
