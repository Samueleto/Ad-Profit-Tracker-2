Now that the hooks exist, we need to connect them to the actual dashboard UI so the data flows through correctly.

Here's what needs to be wired up:

**Date picker / calendar integration** — The useDates hook should power any date picker or calendar component in the dashboard. When the calendar renders, call useDates() to get the list of dates that have data. Use the isComplete and networks fields from each date entry to visually mark dates — for example, fully covered dates could show as solid, partial dates as dotted, and missing dates as greyed out. This gives users a visual sense of data availability before they pick a date.

**Snapshot view** — Whatever component shows a single-day snapshot should call useSnapshot with the currently selected date. When the user changes the date (via the calendar or a date input), the hook re-fetches automatically. Wire the groupBy toggle (if there is one) to pass 'country' or 'total' to the hook. The networks array and summary object from the response should flow directly into the display.

**Trend chart** — The trend chart component should call useTrend with the selected date range, networkId filter, and metric selector. When the user changes the metric dropdown (revenue, cost, profit, roi, impressions, clicks), pass the new metric to the hook — it re-fetches and the chart updates. The movingAverage7d array from the metadata should be available to overlay on the chart if needed.

**Summary panel** — Any summary/overview panel should call useSummary with the current date range. The perNetwork array should feed into a per-network breakdown table or card set. Wire the lastSyncedAt and lastSyncStatus fields from each network entry into status indicators.

**Geo breakdown** — The geo/country breakdown component should call useGeoBreakdown. Wire the metric selector and limit control to the hook params. The metricShare field on each country should drive any percentage bar or pie chart visualization.

**Coverage map** — The coverage/calendar heatmap component should call useCoverage. The gaps array (dates with missingNetworks) should be highlighted visually. The syncHistory array should feed into a sync log or timeline if one exists.

**Backfill trigger** — Any 'Re-sync' or 'Backfill' button should call the trigger function from useBackfill. After it completes, show the results summary (how many networks triggered, skipped, or failed). On success, invalidate or re-fetch the relevant hooks (useCoverage, useDates, useSnapshot for the affected date range) so the UI reflects the newly fetched data.

**Delete snapshot** — Any delete button on a snapshot should call the trigger from useDeleteSnapshot with the date and optional networkId. After deletion, show the deletedCount in a confirmation message and re-fetch useDates and useCoverage so the calendar updates to reflect the removed data.

Make sure that when the backfill or delete operations complete, the stale data in other hooks gets refreshed — either by calling their refetch functions or by resetting their state so they re-run on next render.
