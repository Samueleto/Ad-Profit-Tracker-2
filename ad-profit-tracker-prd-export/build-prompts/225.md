We're building an Excel export feature for a financial metrics dashboard. For this step, I need you to build the ExportModal — a modal dialog that renders inside the existing FinancialMetricsDashboardPage at /dashboard, triggered by the 'Export' button already present on that page.

The modal should be a centered overlay (max-w-lg) using the project's existing ThemeProvider tokens — so dark:bg-gray-900 / bg-white backgrounds, consistent Tailwind semantic classes. It should have a backdrop overlay behind it.

At the top of the modal, show two tabs: 'Excel (.xlsx)' and 'PDF', with Excel selected by default. For now we're only wiring up the Excel tab — the PDF tab can exist but doesn't need to do anything yet.

The main body of the Excel tab has two sections:

1. **Date range display** — Read the currently active date range from the Zustand store (dateFrom and dateTo). Show it as a read-only display like 'Jan 1, 2024 – Jan 31, 2024'. This is not an independent date picker — it just mirrors whatever the dashboard is already filtered to.

2. **Sheet selection** — Show five checkbox rows, one per sheet: Summary (KPI totals), Daily Trend (date-series rows), Geographic Breakdown (country rows), Per-Network Stats (one sheet per network — this counts as one checkbox that includes exoclick, rollerads, zeydoo, and propush), and Activity Log (audit entries). Each checkbox row should show a row count badge next to the label. These counts come from GET /api/export/preview — call this endpoint when the modal opens, passing the current dateFrom/dateTo from the Zustand store. While those counts are loading, show 5 shimmer placeholder rows (the 'loading' state). All checkboxes should be checked by default.

3. **Advanced options** — Collapsed by default under a 'Advanced' chevron toggle. When expanded, show a custom filename text input (optional, max 100 chars) and an 'Include Headers' toggle (on by default).

At the bottom, show two buttons: 'Cancel' (closes modal, no action) and 'Export Now' (accent color using the CSS custom property). The Export Now button calls POST /api/export/excel with the selected sheets, the date range from the Zustand store, and any advanced options the user set. While the export is running (the 'exporting' state), show an indeterminate spinner on the Export Now button, disable it, disable all checkboxes, and show the message 'Generating your Excel file...' below the sheet list.

If the preview endpoint returns hasData: false, disable the Export Now button and show a tooltip on hover: 'No data available for this range'.

On a successful export response, the browser should automatically trigger a file download (the response is a binary .xlsx stream — handle this by creating a blob URL and clicking it programmatically). After the download triggers, show a toast 'Export complete — file downloaded' and close the modal after 1.5 seconds.

The POST /api/export/excel request body should map the sheet checkboxes to the correct sheet keys: 'summary', 'daily_trend', 'geo_breakdown', and the per-network checkbox should expand to ['exoclick', 'rollerads', 'zeydoo', 'propush'], plus 'activity_log'. Include the Bearer token from Firebase Auth in the Authorization header for both API calls.
