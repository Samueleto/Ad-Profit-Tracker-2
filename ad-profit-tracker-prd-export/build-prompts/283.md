We're building the state layer for the in-app notification system. The NotificationCenterPanel and the bell icon badge both need to react to the same shared state, so we'll extend the existing Zustand dashboard store rather than create isolated component state.

Add a notifications slice to the Zustand dashboard store with the following shape: a notifications array (the loaded notification objects from GET /api/notifications), an unreadCount number (the denormalized count from the users document — this is what drives the bell badge), a hasMore boolean, a nextCursor string or null, a isOpen boolean (whether the panel is visible), a isLoading boolean, a isLoadingMore boolean, and an error field.

Expose these actions on the store:
- openPanel / closePanel: toggle isOpen. openPanel should trigger fetchNotifications if the notifications array is empty.
- fetchNotifications: calls GET /api/notifications (limit=20, no cursor), sets isLoading, replaces the notifications array on success, sets unreadCount from the response's unreadCount field, sets hasMore and nextCursor.
- fetchMoreNotifications: calls GET /api/notifications with the stored nextCursor, sets isLoadingMore, appends results to the existing notifications array.
- dismissNotification(id): optimistically removes the notification from the local array and decrements unreadCount, then calls PATCH /api/notifications/dismiss/:id. If the API call fails, re-insert the notification at its original position and restore unreadCount.
- markAllRead: optimistically sets all notifications' isRead to true and sets unreadCount to 0, then calls PATCH /api/notifications/mark-all-read. Rolls back on failure.
- clearAll: optimistically empties the notifications array and sets unreadCount to 0, then calls DELETE /api/notifications/clear. Rolls back on failure.
- setUnreadCount(count): a simple setter so the app can hydrate the badge count on mount without loading the full notifications array.

On app mount (in the root layout or dashboard layout), read the unreadNotificationCount field from the users Firestore document using the Firebase client SDK and call setUnreadCount with it. This means the bell badge is populated immediately on load without waiting for the panel to open. Do NOT set up a real-time Firestore listener — just a one-time read on mount is sufficient. The users document fields relevant here are unreadNotificationCount and notificationsUpdatedAt.

All API calls in the store actions must attach the current Firebase ID token as a Bearer token. Use getIdToken() from firebase/auth on the current user before each call. If getIdToken() fails (user signed out), dispatch a sign-out action and redirect to /.

The NotificationCenterPanel should read all its data from this store and call these actions — it should have no local fetch logic of its own.
