This tab should be positioned before the four per-network tabs and renders as ComparativeNetworkAnalysisTab — it does NOT get its own route, it lives inside the existing dashboard page.

The tab has several distinct sections, so let's walk through each one:

**MetricToggle** — At the top, reuse the existing MetricToggle component to let users switch the primary comparison axis between: Revenue, Cost, ROI%, CTR%, CPM, Impressions, and Clicks. This selection drives everything else on the page.

**4-Column Network Card Grid** — Below the toggle, show a card for each of the four networks: ExoClick, RollerAds, Zeydoo, and Propush. Each card displays: the primary metric (cost for ExoClick since it's the spend network, revenue for the other three), impressions, clicks, CTR%, CPM, and a MetricShareBar (reuse the existing one) showing that network's proportional share of the total. On desktop this is 4 columns, tablet 2 columns, mobile 1 column. Clicking any card should navigate to that network's dedicated tab. Null values should render as '—' — never show undefined or NaN.

**Grouped Bar Chart** — Use recharts BarChart to render all four networks side-by-side for whichever metric is currently selected. Set the chart height to 280px. Color-code bars: amber-500 for ExoClick/cost, green-500 for the revenue networks. This chart should re-render whenever the MetricToggle selection changes.

**Efficiency Table** — Below the chart, show a table with computed cross-network ratios pulled from the `crossNetwork` object in the API response: overall ROI% (formula: (totalRevenue - exoclickCost) / exoclickCost * 100), revenue-per-impression, and cost-per-click. Also show a network health status badge per network (synced/failed/never) sourced from the `networkStatus` field on each network object. Use alternating row shading. This section should be collapsed by default with a chevron to expand it (progressive disclosure).

**Network Ranking Strip** — A horizontal strip that ranks all four networks by the currently selected metric, showing ordinal medal-style badges (1st/2nd/3rd/4th). Pull rank values from the `rankings` array in the API response. Handle ties by showing the same rank for tied networks.

**Sync All Button** — Include a 'Sync All' button that calls POST /api/sync/manual with no networkId parameter. This should show a loading spinner while in progress.

**Data Loading** — When the Compare tab is first activated (lazy load, not on dashboard mount), fire GET /api/networks/comparison in parallel with any other needed calls using Promise.all. Pass the current dateFrom and dateTo from the parent dashboard's shared date range state — this tab has no independent date selector. Also pass the currently selected metric as a query param.

**States to handle:**
- Loading: show 4 shimmer stat cards matching the card dimensions, a 280px shimmer rectangle where the chart goes, and 4 shimmer rows in the efficiency table
- Empty (404 response): show 'No data for any network in this date range. Sync your networks to see comparisons.' with a Sync All CTA button
- Error 500: show an inline error banner inside the tab panel (not a full-page error) with a Retry button that re-fires all four GET /api/networks/stats calls in parallel
- Error 403: show 'Access Denied' inline within the tab panel with a link to /dashboard
- Success: all four cards populated, chart rendered, efficiency table populated, ranking strip shown

Use Tailwind semantic color classes consistently: green-500 for revenue/positive values, amber-500 for cost/ExoClick, red-500 for negative ROI.
