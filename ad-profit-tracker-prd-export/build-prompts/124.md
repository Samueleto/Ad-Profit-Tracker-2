We need TypeScript interfaces for the four Firestore collections this feature touches: `adStats`, `apiKeys`, `networkConfigs`, and `auditLogs`. Having these typed upfront means every endpoint gets autocomplete and type safety when reading from or writing to Firestore.

Create a types file at `src/lib/networks/zeydoo/types.ts` (or a shared `src/types/firestore.ts` if the project already has one).

**adStats** — This is the main data collection. Each document represents one row of stats for a specific user, network, date, and country. Fields: `id` (string), `userId` (string), `networkId` (string, always `'zeydoo'` for this feature), `date` (string, YYYY-MM-DD), `country` (string, ISO code), `revenue` (number), `impressions` (number, optional), `clicks` (number, optional), `conversions` (number, optional), `cost` (null — Zeydoo doesn't provide cost data), `rawResponse` (a map/object of the sanitized original API response row), `createdAt` and `updatedAt` (Firestore Timestamps).

**apiKeys** — Stores encrypted API credentials. Fields: `userId` (string), `networkId` (string), `encryptedKey` (string), `iv` (string, the initialization vector for decryption), `createdAt` and `updatedAt` (Timestamps). Also export a helper type `DecryptedApiKey` that's just `{ key: string }` for what the decrypt function returns.

**networkConfigs** — Per-user network configuration. Fields: `userId` (string), `networkId` (string), `isActive` (boolean), `syncSchedule` (string, e.g. `'daily'`), `lastSyncedAt` (Timestamp or null), `lastSyncStatus` (`'success' | 'failed' | 'pending' | null`), `createdAt` and `updatedAt` (Timestamps).

**auditLogs** — Immutable log entries. Fields: `userId` (string), `action` (string, e.g. `'zeydoo_sync'`), `status` (`'success' | 'failed'`), `metadata` (a flexible object for extra context like date range, row count, error message), `createdAt` (Timestamp).

Also create a Zod validation schema (or plain validation helpers) for the date range parameters that multiple endpoints share — validate that dates are in YYYY-MM-DD format and that the range doesn't exceed 90 days. Export this as `validateDateRange(dateFrom: string, dateTo: string)`.
