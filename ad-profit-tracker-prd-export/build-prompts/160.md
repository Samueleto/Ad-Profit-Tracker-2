**POST /api/networks/zeydoo/sync** — The manual sync trigger. First, verify the Firebase ID token from the `Authorization: Bearer <token>` header using `admin.auth().verifyIdToken()` to get the `uid`. Validate `dateFrom` and `dateTo` from the request body using `validateDateRange` (both default to yesterday if not provided). Check a rate limit of 10 requests per hour per user — store this in Firestore or in-memory with a TTL. Fetch the `apiKeys` document for this user and network, then decrypt the key using a server-side helper (never log or return the plaintext key). Call `fetchZeydooStats()` with the decrypted key and date range. Parse the response — for each row, build an `AdStat` document with `networkId='zeydoo'`, `cost=null`, and the `rawResponse` field set to a sanitized copy of the row (strip any fields that look like credentials). Upsert into `adStats` using Firestore batch writes in chunks of 500. Update `networkConfigs` with `lastSyncedAt=serverTimestamp()` and `lastSyncStatus='success'`. Write an `auditLogs` entry. If the Zeydoo API call fails, update `networkConfigs` with `lastSyncStatus='failed'` and return 502.

**POST /api/networks/zeydoo/scheduled-sync** — The internal cron endpoint. Validate the `x-internal-secret` header against the `INTERNAL_SYNC_SECRET` environment variable — reject with 401 if it doesn't match. Do a collection-group query on `networkConfigs` for all documents where `networkId=='zeydoo'`, `isActive==true`, and `syncSchedule=='daily'`. For each matching config, fetch and decrypt that user's API key, call the Zeydoo API for yesterday's stats, batch-upsert into `adStats`, update `networkConfigs`, and write an `auditLogs` entry. Process users sequentially or in small batches to avoid hammering the Zeydoo API. Return a summary of how many users were synced and how many failed.

**GET /api/networks/zeydoo/stats** — Verify the Firebase ID token. Validate `dateFrom`, `dateTo` (required), and `groupBy` (`'day'` or `'total'`, default `'day'`) from query params. Query `adStats` filtered by `userId`, `networkId=='zeydoo'`, and the date range. If `groupBy='day'`, group results by date and sum revenue and other metrics per day. If `groupBy='total'`, sum everything into a single object. Return the structured result.

**GET /api/networks/zeydoo/stats/by-country** — Verify token. Validate `dateFrom`, `dateTo`, and `limit` (1–100, default 20). Query `adStats` for this user and network within the date range. Group by country in memory, summing revenue and metrics. Sort by total revenue descending, slice to `limit`. Calculate each country's percentage share of total revenue. Return the list.

**GET /api/networks/zeydoo/stats/latest** — Verify token. Query `adStats` for this user where `networkId=='zeydoo'`, ordered by `date` descending, limit 1 — this gives the most recent date that has data. If no documents exist, return 404. Do a second query for all `adStats` documents on that date. Aggregate totals. Also fetch the `networkConfigs` document to include `lastSyncedAt` and `lastSyncStatus` in the response. Return the full summary.

**GET /api/networks/zeydoo/raw-response** — Verify token. Validate the `date` query param (required, YYYY-MM-DD). Query `adStats` for this user, `networkId=='zeydoo'`, and the specified date. Map over results and extract just the `rawResponse` field from each document. Also return a schema definition object that describes the fields in the raw response (for use in an API Explorer UI). Return both the data array and the schema.
