We need TypeScript types that represent the onboarding-related fields on the users collection, so the onboarding wizard and the API endpoint are working with the same shape.

Create a types file at src/features/onboarding/types/onboarding.ts. In it, define the following:

First, a UserPreferences interface with two optional fields: timezone (string — an IANA timezone string) and defaultDateRange (a union type: 'last7days' | 'last30days' | 'thisMonth').

Second, extend or reference the existing User type (wherever it lives in the project) to include the four new onboarding fields: onboardingCompletedAt (a Firestore Timestamp or null), onboardingSkipped (boolean or undefined), and the preferences map using the UserPreferences interface above. If the project already has a User type, add these fields to it rather than creating a duplicate.

Third, define an OnboardingFormValues interface that represents what react-hook-form will manage on Step 2 of the wizard: displayName (string, required), timezone (string, required), and defaultDateRange (the same union type as above, required).

Fourth, define an UpdateProfilePayload interface that matches the request body shape for PATCH /api/auth/update-profile — this should include optional fields for displayName, onboardingCompletedAt (typed as the string literal 'serverTimestamp' since the client sends that string and the server converts it), onboardingSkipped (boolean), and preferences (UserPreferences).

Keep everything exported so the page components, hooks, and API route can all import from this single source of truth.
