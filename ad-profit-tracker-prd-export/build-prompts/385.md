We need to make sure every failure mode in the NetworkConfigTab is handled gracefully and gives users clear, actionable feedback.

**Initial load failures (GET /api/networks/config/list):**
- 401: Attempt a Firebase ID token refresh and retry once. If the retry also returns 401, redirect to '/' using the Next.js router and show a 'Session expired. Please sign in again.' toast.
- 403: Show an 'Access Denied — you don't have permission to view network settings.' inline banner inside the Networks tab. Don't redirect.
- 404: This means the user has no config documents yet. Silently initialize defaults client-side and render all 4 cards with default values. No error shown to the user.
- 500: Show a full-tab error banner with a 'Retry' button that re-calls GET /api/networks/config/list.
- Network offline: Show a 'Unable to load network settings — check your connection.' banner with a Retry button.

**Per-card save failures (PATCH /api/networks/config/update):**
- Revert the optimistic update in local state.
- Show an inline error banner on the specific card that failed (not a global error). The banner should say something like 'Failed to save ExoClick settings.' with a 'Retry' button that re-submits the same change.
- 400 validation errors: Show the specific field error next to the relevant input (e.g. 'Timeout must be between 5 and 60 seconds') rather than a generic message.

**Reorder failures (PATCH /api/networks/config/reorder):**
- Revert the card order to what it was before the drag.
- Show a brief toast: 'Couldn't save new order — please try again.'

**Test Connection failures (POST /api/networks/config/test-connection):**
- 404 (no API key saved): Show inline on the card: 'No API key saved for this network. Add one in the API Keys settings.'
- 502 (network API unreachable): Show inline: 'Connection failed — [error message from response].'
- 500: Show inline: 'Test failed due to a server error. Try again.'
- Timeout: If the test-connection call takes longer than the configured timeoutSeconds, show 'Connection timed out.' inline.

**Sync All failures (POST /api/networks/sync-all):**
- 429 (rate limited): Show a toast: 'Sync limit reached — you can sync up to 3 times per hour.'
- 500: Show a toast: 'Sync failed — please try again in a moment.'
- Partial results (some triggered, some failed): Show the summary toast with the breakdown — don't treat partial success as a full failure.

**Reset failures (DELETE /api/networks/config/reset):**
- 404: Show inline on the card: 'Nothing to reset — this network hasn't been configured yet.'
- 500: Show inline: 'Reset failed. Please try again.'

Make sure all error messages use plain language — no HTTP status codes, no stack traces, no technical jargon visible to the user. Error banners on cards should be dismissible. Toasts should auto-dismiss after 5 seconds.
