We're building the state layer for the mobile responsive navigation. There are two pieces to wire up here.

**Drawer open/close state** — The MobileNavigationDrawer needs to track whether it's open or closed. This state lives in the ResponsiveAppLayout (the parent that renders both the hamburger button and the drawer), so the hamburger button in the header bar can toggle it and the drawer itself can close itself when a link is tapped or the backdrop is clicked. Pass the open boolean and a close callback down as props to MobileNavigationDrawer. Don't put this in global state — it's purely local UI state and `useState` in the layout is the right tool. When the drawer opens, lock body scroll (`document.body.style.overflow = 'hidden'`) so the page behind doesn't scroll. When it closes, restore scroll. Clean this up properly if the component unmounts.

**useMediaQuery hook** — Build a `useMediaQuery(query: string): boolean` hook that takes a CSS media query string (e.g., `'(min-width: 768px)'`) and returns whether it currently matches. Use `window.matchMedia` and listen for changes with `addEventListener('change', ...)`. On the server (SSR), return `false` as the default to avoid hydration mismatches — the correct value will kick in after hydration on the client. Memoize the event listener cleanup in a `useEffect` return. Export a convenience wrapper `useIsMobile()` that calls `useMediaQuery('(max-width: 767px)')` so components don't have to remember the breakpoint value. This hook is used in ResponsiveAppLayout to conditionally render the drawer's interactive open/close behavior, and in MobileBottomNavBar to read the Zustand health state only when on mobile (avoiding unnecessary store subscriptions on desktop).

**Zustand store integration** — MobileBottomNavBar reads the `health` field from the existing shared Zustand dashboard store to decide whether to show the red badge dot on the Networks tab. Don't create a new store — just import and subscribe to the existing store's health state. The badge should reactively update if health changes while the user is on the page (e.g., a background sync completes and health goes from 'degraded' back to 'healthy').
