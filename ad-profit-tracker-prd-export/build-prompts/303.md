A few things to make sure are wired up correctly:

**Tab activation:** The ActivityLogTab should only start fetching data when the tab is actually active — not on SettingsPage mount. **Auth guard:** The SettingsPage should already be a protected route. Confirm that the ActivityLogTab is covered by the same auth guard — it shouldn't be reachable by unauthenticated users. If the SettingsPage doesn't already have route protection, add a check using firebase/auth's onAuthStateChanged and redirect to / if no user is found.
