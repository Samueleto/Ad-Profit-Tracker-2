We need three packages set up before building the DateRangeToolbar:

1. react-day-picker — This is the calendar UI component we'll use for the custom date range picker (the 'From' and 'To' calendar inputs that appear when the user selects 'Custom'). Install it and make sure its CSS is imported globally or in the component — react-day-picker requires its own stylesheet to render correctly. Check the version you're installing (v8+ has a different API than v7) and note which API we'll be using.

2. date-fns — This is our date utility library for formatting, parsing, and doing arithmetic on dates. We'll use it to serialize dates to YYYY-MM-DD format, calculate the difference between two dates (to enforce the 90-day max), and format the 'Last synced' timestamp badge. Install it if it's not already in the project.

3. zustand — This is the state management library for the shared dashboard date range state. The DateRangeToolbar writes to this store, and all other dashboard tabs and sections read from it to know which date range to fetch data for. Install it if it's not already present. We'll set up the actual store shape in the schema_models step.

After installing, do a quick sanity check: import DayPicker from react-day-picker in a throwaway component and make sure it renders without errors. Import format from date-fns and make sure it can format a date. Import create from zustand and make sure it's available.
