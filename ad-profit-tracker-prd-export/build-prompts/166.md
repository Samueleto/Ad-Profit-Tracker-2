We're building six API endpoints that form the complete reconciliation system. Every single one starts by verifying the Firebase ID token from the Authorization header using admin.auth().verifyIdToken() — reject with 401 if it's missing or invalid. The uid comes only from the verified token, never from the request body.

**POST /api/reconciliation/run** — This is the main engine. It accepts an optional networkId (must be one of 'exoclick', 'rollerads', 'zeydoo', 'propush'), and optional dateFrom/dateTo (YYYY-MM-DD, max 90-day range, both default to yesterday). Check the rate limit first (max 10/hour per user). If no networkId is provided, query networkConfigs for the user's active networks. For each target network, load its validationRules from networkConfigs (merge with SYSTEM_DEFAULTS for any null fields). Then fetch all adStats documents for that network and date range. Run six validation checks on each document: revenue/cost range check (critical if >10x threshold, warning otherwise), CTR sanity check, null/missing required fields check, duplicate detection (same userId+networkId+date+country combo), cross-network consistency check (for dates where all 4 networks have data), and stale data check (syncedAt older than 48 hours for daily-schedule networks). Build the anomalyFlags array for each document and write back validationStatus + anomalyFlags + validationCheckedAt using Firestore batch writes in chunks of 500. Update each network's networkConfigs document with lastValidationStatus, lastValidationAt, and anomalyCount. If any anomalies were found, write an audit log entry to auditLogs. Return the full reconciliation summary.

**GET /api/reconciliation/status** — Accepts optional networkId. Queries networkConfigs for the user's networks, then for each network fetches the 5 most recent adStats documents with validationStatus in ['anomaly', 'stale']. Computes an overall health score: 'healthy' (all clean, no open circuits), 'degraded' (some anomalies), 'critical' (any critical anomalies or open circuit breaker). Returns the consolidated status.

**GET /api/reconciliation/anomalies** — Accepts networkId, dateFrom (default 30 days ago), dateTo (default today), severity, anomalyType, limit (1–100, default 20), and cursor for pagination. Queries adStats for documents with validationStatus in ['anomaly', 'stale'] within the date range. Apply severity and anomalyType filters in-memory on the anomalyFlags array. Use cursor-based pagination (fetch limit+1 to detect hasMore). Return the paginated list with a summary of criticalCount, warningCount, and affectedNetworks.

**PATCH /api/reconciliation/resolve** — Accepts documentIds (array, max 50) and resolution ('acknowledged' | 'corrected' | 'false_positive') and optional note (max 500 chars). Check rate limit (max 20/hour). For each document ID, fetch the adStats doc and verify userId == uid — silently skip any that don't belong to the user (don't reveal they exist). For verified documents, set isReconciled=true, reconciledAt=serverTimestamp(), validationStatus='valid', and append a resolution object to each anomaly flag. Decrement anomalyCount on the affected networkConfigs documents. Write an audit log entry. Return resolved count and skipped count.

**GET /api/reconciliation/rules** — Accepts optional networkId. Queries networkConfigs for the user's networks, extracts validationRules for each, merges with SYSTEM_DEFAULTS, and returns the effective rules with a usingDefaults array indicating which fields are falling back to defaults.

**PATCH /api/reconciliation/rules** — Accepts networkId (required) and a partial rules object. Validate each provided field against its allowed range (maxDailyRevenueUSD 0–1000000, maxCtrPercent 0–100, etc.). At least one rule field must be provided. Check rate limit (max 20/hour). Fetch the networkConfigs document — return 404 if not found. Use Firestore dot-notation field paths for partial map updates (e.g., 'validationRules.maxDailyRevenueUSD') so you only overwrite the provided fields. Write an audit log with the before/after values. Return the full merged rules.
