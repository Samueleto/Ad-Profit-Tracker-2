We're building the DateRangeToolbar and its three sub-components. These will be integrated into FinancialMetricsDashboardPage as a sticky section above the KPI cards — not a standalone page or route.

Start with the three sub-components, then assemble them into DateRangeToolbar:

**PresetPills** — Renders the six preset options (Last 7 Days, Last 14 Days, Last 30 Days, Last 90 Days, This Month, Custom) as a horizontal segmented pill control. The active preset should be visually highlighted using Tailwind semantic classes. Tapping any preset calls setPreset() from the Zustand store immediately — no confirmation needed. On mobile (small screens), this collapses into a native dropdown select element instead of the pill row. Accept the current preset as a prop and an onChange callback.

**CustomDatePicker** — Only renders when the active preset is 'custom'. Shows two calendar inputs side by side (From and To) using react-day-picker. As the user picks dates, call setCustomRange() on the store. Show an 'Apply' button that calls applyCustomRange() — this button should be disabled if the pending range exceeds 90 days. If the range exceeds 90 days, show a small red validation message below the inputs: 'Maximum range is 90 days'. Use date-fns to calculate the day difference for validation display. On mobile, this panel should open as a bottom sheet overlay rather than inline.

**DataAvailabilityDot** — A small indicator dot that shows the data coverage status for the selected range. It reads dataAvailability from the Zustand store. States: gray spinner while loading, green dot for 'complete', amber dot for 'partial', gray dot with 'No data' label for 'none', and amber warning icon for 'error'. This component should call GET /api/stats/dates (passing the current fromDate and toDate as query params with a Firebase Auth bearer token) whenever the active date range changes. On a 401 response, attempt getIdToken(true) to refresh the Firebase token and retry once. On failure after retry, the dot shows the error state but does not block the toolbar. On 403, show 'Access Denied' inline. On 404 or empty response, show 'No data'. On 500, show the amber warning icon.

**DateRangeToolbar** — The parent component that assembles everything into a single compact row. It should be sticky — fixed below the sidebar nav on desktop so it doesn't scroll away. Left side: the PresetPills component. Right side (when preset is 'custom'): the CustomDatePicker inline. Far right: the DataAvailabilityDot and the 'Last synced' timestamp badge (reads lastSynced from the Zustand store, formatted with date-fns as a human-readable relative time like '2 minutes ago'). The whole toolbar should be a single row that doesn't wrap on desktop. Keep it compact — this sits above KPI cards and shouldn't dominate the page.
