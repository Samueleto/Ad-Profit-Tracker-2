We need several reusable UI pieces that the ThemeSettingsSection will use. Build these before the section itself so they can be imported by name.

1. ThemeSegmentedControl — A three-option toggle that lets users pick Light, Dark, or System. Each option shows an icon and a label: a sun icon for Light, a moon icon for Dark, and a monitor/computer icon for System. The active option should be visually highlighted. It takes a value prop (one of 'light', 'dark', 'system') and an onChange callback. This is the primary control for switching themes.

2. AccentColorSwatch — A single 32px circular color swatch. It takes a color hex value, a label, whether it's currently selected, and an onClick callback. When selected, show a checkmark in the center of the circle. The circle should have a subtle ring or border when active to make the selection obvious. This gets used in a row of 8 swatches.

3. AccentColorPicker — A row of 8 AccentColorSwatch components using the preset colors defined in the constants file (indigo, emerald, rose, amber, sky, violet, orange, slate). After the 8 swatches, show a 'Custom' option. When the user clicks 'Custom', reveal a hex input field below the swatches (progressive disclosure — hidden until needed). The hex input should validate that the entered value is a valid hex color (with or without the # prefix). It takes a selectedColor prop and an onChange callback that fires with the hex string.

4. ThemePreviewCard — A compact 120px-tall card that shows a miniature KPI card mockup. It should render a small card with a title like 'Revenue', a fake number like '$12,450', and a small upward trend indicator. This card should visually reflect the currently selected theme (light/dark background) and accent color (use the accent for the trend indicator or a highlight element). It takes themeMode and accentColor as props and renders accordingly using inline styles or CSS variables so it can preview independently of the actual applied theme.

All four components should be built as clean, self-contained pieces with no side effects — they just display and fire callbacks.
