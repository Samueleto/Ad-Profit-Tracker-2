We need three packages installed before we can build anything:

1. **jspdf** (version ^2.5.1) — This is the core PDF generation library. It runs server-side in Node.js and lets us create PDF documents programmatically, set fonts, draw text, and control layout.

2. **jspdf-autotable** — A plugin for jsPDF that makes it easy to render data tables inside PDFs. We'll use this heavily for the Executive Summary, Daily Trend, Geographic Breakdown, Per-Network Stats, and Activity Log sections. It needs to be imported and applied to the jsPDF instance before use.

3. **html2canvas** — This one is frontend-only. It captures a rendered HTML element (specifically the Daily Profit Trend recharts chart) as a canvas image, which we can then embed into the PDF. Install it as a frontend dependency.

Install all three: `npm install jspdf jspdf-autotable html2canvas`.

After installing, do a quick sanity check: in the PDF route file, try importing jsPDF and jspdf-autotable together and make sure TypeScript doesn't complain. The jspdf-autotable package ships its own types, but you may need to add a small type augmentation so TypeScript knows about `doc.autoTable()` — check the jspdf-autotable README for the recommended import pattern (usually `import autoTable from 'jspdf-autotable'` and calling `autoTable(doc, { ... })`).

Also confirm that `firebase-admin` and `node-cache` are already installed from the Excel export feature — if not, add them now. The PDF endpoint will use both.
