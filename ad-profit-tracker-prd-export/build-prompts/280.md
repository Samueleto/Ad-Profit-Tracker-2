We're building state management for the PdfExportTab component inside the ExportModal. The goal is to keep the tab's logic clean and testable, separate from the rendering.

Here's what state the PDF tab needs to track locally:

- selectedSections: which of the six section checkboxes are checked (default all checked)
- orientation: 'portrait' or 'landscape' (default 'portrait')
- paperSize: 'a4' or 'letter' (default 'a4')
- filename: string (default empty, optional)
- advancedOpen: boolean tracking whether the Advanced chevron is expanded
- exportStatus: one of 'idle' | 'exporting' | 'success' | 'error'
- errorMessage: string or null, for inline error display

The preview row counts (used for the badges and the empty-state check) should come from shared state that the Excel tab already manages — do not duplicate the GET /api/export/preview fetch. If the ExportModal uses a context or a shared hook for preview data, read from that. If it passes preview data as props, accept it as a prop.

The export action itself should be a handler function that:
1. Sets exportStatus to 'exporting'
2. Reads the current Firebase ID token using getIdToken() from the Firebase Auth SDK
3. Builds the request payload from the current state (selectedSections mapped to the correct API keys, orientation, paperSize, dateFrom/dateTo from the modal's shared date range, and filename if non-empty)
4. Calls POST /api/export/pdf with the Bearer token in the Authorization header
5. On a 200 response, reads the response as a Blob, creates an object URL, programmatically clicks a hidden anchor element to trigger the download, revokes the object URL, sets exportStatus to 'success', shows the success toast, and schedules modal close after 1.5 seconds
6. On any error response, sets exportStatus to 'error' and sets errorMessage based on the status code (see error handling step for the mapping)

The 'Export Now' button should be disabled when: exportStatus is 'exporting', OR all preview row counts are zero (no data for the range), OR no sections are selected.

Keep this logic in a custom hook (e.g. usePdfExport) so the PdfExportTab component stays focused on rendering. The hook should return all the state values and the handler functions the component needs.
